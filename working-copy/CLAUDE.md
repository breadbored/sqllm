Stable copy lives at `~/sqllm` once the workflow is validated here.

## Running

Interactive mode ‚Äî enter prompts one at a time:

```bash
npx ts-node src/main.ts training_data.jsonl
```

Batch mode ‚Äî reads prompts from `prompts.txt`:

```bash
npx ts-node src/main.ts training_data.jsonl prompts.txt
```

Type `done` to exit interactive mode. Output is appended to the JSONL file, so
interrupted runs are not lost.

## Key Files

**`system-prompt.txt`** ‚Äî the full SQLLM reasoning system prompt fed to Opus via
`claude -p --system-prompt-file`. Edit this to refine reasoning behavior, but
keep a copy of the previous version before changing it so generated samples
remain traceable to the prompt version that produced them.

**`prompts.txt`** ‚Äî batch input file. One prompt per line. Lines starting with
`#` and blank lines are ignored. Add new prompts here rather than retyping them
in interactive mode once the workflow is stable.

**`training_data.jsonl`** ‚Äî append-only output. Each line is a self-contained
JSON object with a `messages` array in standard chat format compatible with
Unsloth/QLoRA fine-tuning. Do not edit this file manually.

## Workflow

1. Add a prompt to `prompts.txt` or run interactive mode for one-off samples
2. Review the generated assistant response in the JSONL output
3. If the reasoning trace is wrong, note the failure mode and adjust
   `system-prompt.txt`
4. Re-run the problem after adjusting ‚Äî append to the same JSONL or use a
   separate file during prompt iteration
5. Once the reasoning trace quality is acceptable, switch to batch mode for
   volume generation

## Notes

- The script calls `claude -p` directly and requires Claude Code to be installed
  and authenticated with a Max subscription
- `--model claude-sonnet-4-6` is hardcoded in the script ‚Äî do not change to a
  cheaper model for seed data generation
- The system prompt file path is resolved relative to `src/main.ts` via
  `__dirname` ‚Äî keep `system-prompt.txt` at the project root
- Batch runs on large `prompts.txt` files will hit Max rate limits ‚Äî the script
  has a 1s delay between calls but you may need to increase this if you see
  errors
