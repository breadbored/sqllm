Show me a BigQuery query that identifies active user engagement patterns by grouping events and filtering based on engagement thresholds.

--- AGENT ---

This query identifies "active" user engagement within individual sub-sessions by grouping page events according to sub-session ID, application, and URL path. It then filters to retain only those groups where the user demonstrated significant engagement—either spending more than 60 seconds on the page OR clicking more than once.

The output tags all qualifying groups as 'active' behavior and records the earliest timestamp (MIN of created) for each group, along with the app and URL path context. Note that the query includes an inline comment acknowledging that the minimum timestamp may not be semantically ideal for representing "when" the user became active.

The structural relationship between `sub_session_id` and the other columns is preserved, and `seconds_on_page` and the `type` column (which contains at least 'click' values) appear to be event-level metrics from the source table.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  'active' as tag,
  'behavior' as class,
  e.app as app,
  e.url_path as url_path,
  MIN(e.created) as dt -- this is not an ideal timestamp for "when" they became active

FROM
  ${ref("base_buckets_events_app_speedy")} as e

GROUP BY
  e.sub_session_id,
  e.app,
  e.url_path

HAVING
  MAX(e.seconds_on_page) > 60
OR
  SUM(IF(e.type = 'click', 1, 0)) > 1
```
