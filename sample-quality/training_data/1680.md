Show me how to identify sessions with very short durations and capture their entry point for behavioral analysis.

--- AGENT ---

This query groups events from an application event stream by `sub_session_id` and `app`, then filters for groups using the `HAVING` clause where the time span between earliest and latest events is 5 seconds or less. For each group meeting this criterion, it returns the sub_session_id, a hardcoded tag 'bounced', a class label 'behavior', the app identifier, the URL path associated with the earliest event in that group (via `MIN_BY` applied to the created timestamp), and the earliest timestamp. The fixed tag and class values indicate this is part of a behavioral tagging system for marking sessions with compressed activity windows.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  'bounced' as tag,
  'behavior' as class,
  e.app as app,
  MIN_BY(e.url_path, e.created) as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_app_speedy")} as e
GROUP BY
  e.sub_session_id,
  e.app
HAVING
  DATETIME_DIFF(
    MAX(e.created),
    MIN(e.created),
    SECOND
  ) <= 5
```
