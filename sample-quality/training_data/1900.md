Show me a query that enriches course-lab-server records with the set of client IDs associated with each user, defaulting to a primary client when no external client relationship exists.

--- AGENT ---

This query denormalizes client information onto user-course-lab-server records. It builds a mapping table of users to their client IDs (including a primary client ID of 1 plus any external client ID stored in the users table), then left-joins that mapping onto the core user_course_lab_servers table.

The CTE `user_clients` creates an array for each user containing the fixed value 1 plus their external_client_id (when present). The main query returns all columns from user_course_lab_servers enriched with a client_ids array: when a user has an external client relationship, the array contains both IDs; when no relationship exists, it defaults to `[1]`.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  _utb.id,
  _utb.user_id,
  _utb.course_id,
  _utb.lab_id,
  _utb.server_id,
  _utb.lab_instance_id,
  _utb.created,
  _utb.updated,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_course_lab_servers")} as _utb
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = _utb.user_id
```
