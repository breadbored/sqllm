## Question
Show me a BigQuery query that combines user activity from multiple course tables and aggregates distinct clients by user and day.

--- AGENT ---

This query combines two sources of user-course engagement and aggregates them by user and day. The first UNION branch joins user course lesson progress records through users, courses, and client assignments. The second branch joins user course enrollment records through the same path.

The outer query groups the combined results by user_id, user creation date, and the day of the activity (timestamp truncated to day level), and collects all distinct client IDs associated with each user-day combination into a sorted array. This produces one row per user-day showing all clients that user touched through their course activities that day.

SELECT
  uclp.user_id,
  uclp.user_created,
  date_trunc(uclp.created, DAY) AS day,
  array_agg(
    DISTINCT client_id IGNORE NULLS
    ORDER BY
      client_id
  )                             AS client_id
FROM (
  SELECT
    uclp.user_id,
    u.created AS user_created,
    uclp.created,
    ccc.client_id
  FROM
    ${ref("stg_user_course_lesson_progress")} AS uclp
  INNER JOIN
    ${ref("stg_users")} AS u
    ON
      uclp.user_id = u.id
  INNER JOIN
    ${ref("stg_courses")} AS c
    ON
      uclp.course_id = c.id
  INNER JOIN
    ${ref("stg_clients_course_containers")} AS ccc
    ON
      c.container_id = ccc.course_container_id
  UNION ALL
  SELECT
    uc.user_id,
    u.created AS user_created,
    uc.created,
    ccc.client_id
  FROM
    ${ref("stg_user_courses")} AS uc
  INNER JOIN
    ${ref("stg_users")} AS u
    ON
      uc.user_id = u.id
  INNER JOIN
    ${ref("stg_courses")} AS c
    ON
      uc.course_id = c.id
  INNER JOIN
    ${ref("stg_clients_course_containers")} AS ccc
    ON
      c.container_id = ccc.course_container_id
) AS uclp
GROUP BY
  uclp.user_id,
  uclp.user_created,
  date_trunc(uclp.created, DAY)
