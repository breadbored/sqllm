--- USER ---

Extract the earliest app and URL path accessed in each browser session, labeling each record by browser type as an attribute.

--- AGENT ---

This query aggregates events into session-level attributes by grouping on `sub_session_id` and browser type. For each browser encountered in a sub-session, it extracts:
- The `app` value from the event with the earliest timestamp (`MIN_BY(es.app, es.created)`)
- The `url_path` value from the event with the earliest timestamp (`MIN_BY(es.url_path, es.created)`)
- The earliest timestamp itself (`MIN(es.created)`)

Each resulting row is labeled with a `tag` that identifies the browser (e.g., `'browser_name_Chrome'`) and marked with class `'attribute'` to indicate it represents a session attribute rather than a raw event. This is useful for building session profiles that capture which app and path were first accessed via each browser type.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  CONCAT('browser_name_', es.browser_name) as tag,
  'attribute' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref('base_buckets_events_speedy')} as es 

GROUP BY
  sub_session_id,
  tag
```
