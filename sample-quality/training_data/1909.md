## Question
Create a normalized user profile dataset that includes all user attributes in lowercase, associates each user with their assigned client IDs, and excludes test and selenium automation users.

--- AGENT ---

This query produces a denormalized user profile table with the following characteristics:

**Step 1: Client ID Mapping (CTE)**
The `user_clients` CTE creates an array of client IDs for each user. If a user has an `external_client_id`, the array contains `[1, external_client_id]`. Users without an external client ID are excluded from this CTE.

**Step 2: Main User Selection**
The primary SELECT normalizes user data by converting all text fields (name, email, title, company metadata, etc.) to lowercase using the LOWER() function. This ensures consistent casing across the dataset. Numeric IDs, authentication tokens, timestamps, and boolean flags remain unchanged.

**Step 3: Client ID Association**
A LEFT JOIN connects users to their client ID mappings. The COALESCE logic assigns:
- The client ID array from the CTE if the user has an external client ID (e.g., `[1, 12345]`)
- A default array of `[1]` for users without an external client ID

**Step 4: Filtering**
The query excludes:
- Test users (where `test_user = 0`)
- Users with email addresses matching the Selenium automation test pattern (`^seleniumtestuser[0-9]+@seleniumtestuser\.com$`)

This produces a clean, production-ready user dataset with normalized text fields and explicit client associations.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  u.id,
  LOWER(first_name)       AS first_name,
  LOWER(last_name)        AS last_name,
  LOWER(username)         AS username,
  LOWER(email)            AS email,
  LOWER(title)            AS title,
  LOWER(bio)              AS bio,
  LOWER(company)          AS company,
  LOWER(company_role)     AS company_role,
  LOWER(company_size)     AS company_size,
  LOWER(company_industry) AS company_industry,
  u.created,
  u.updated,
  u.last_online,
  u.facebook_id,
  u.google_id,
  u.amazon_id,
  u.github_id,
  u.profile_image,
  u.suspect_developer,
  u.test_user,
  u.external_id,
  u.meta_quest_id,
  u.refresh_token,
  u.refresh_token_salt,
  u.signup_location,
  u.external_client_id,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM 
    ${ref("users")} as u
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = u.id
WHERE 
    test_user = 0 AND NOT REGEXP_CONTAINS(email, r'^seleniumtestuser[0-9]+\\@seleniumtestuser\\.com$')
```
