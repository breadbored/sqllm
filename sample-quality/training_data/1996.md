--- USER ---

Give me a filtered feed of course feedback responses enriched with completion status and client ownership. Exclude feedback less than 24 hours old unless it has no response text or has been flagged as comment-only. Redact the response text for comment-only entries, and default unassigned courses to client ID 1.

--- AGENT ---

## Step 1 — Identify shapes

**Input tables and their observable columns:**

- `feedback` (f): `id, response_sentiment, response_text, video_seconds, user_id, course_id, lesson_id, course_completed, created, updated`
- `clients_course_containers` (ccc): `client_id, course_container_id`
- `courses` (c): `id, container_id`
- `stg_users` (u): `id`
- `stg_cleared_feedbacks` (cf): `feedback_id, comment_only`

**Target shape:**

```
id, response_sentiment, response_text, video_seconds, user_id, course_id, lesson_id, course_completed, created, updated, status, client_ids
```

This is a row-per-feedback output with two derived columns (`status`, `client_ids`) and one conditionally redacted column (`response_text`).

## Step 2 — Identify sources and relationships

- `feedback.course_id` → `courses.id` (indirect, via `course_clients` CTE)
- `courses.container_id` → `clients_course_containers.course_container_id` (join inside the CTE to resolve which clients own which course)
- `feedback.user_id` → `stg_users.id` (INNER JOIN — filters to valid users only)
- `feedback.id` → `stg_cleared_feedbacks.feedback_id` (LEFT JOIN — presence indicates the feedback has been reviewed/cleared; `comment_only` flag controls redaction)
- `course_clients.course_id` → `feedback.course_id` (LEFT JOIN — not every course has a client assignment)

The chain for client ownership: `feedback.course_id` → `courses.id` → `courses.container_id` = `clients_course_containers.course_container_id` → `client_id`. This is a two-hop relationship collapsed into a single CTE.

## Step 3 — Identify aggregation depth

There is one level of aggregation, and it lives entirely inside the CTE:

```
course_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

No further aggregation occurs in the main query — it is a row-per-feedback select with joins. Arrow notation:

```
course_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
-> (no further aggregation — joined back to feedback rows 1:1)
```

## Step 4 — Identify dependencies

- **Grouping driver:** `course_id` in the CTE — one row per course with its array of client IDs.
- **Derived columns in the main SELECT:**
  - `status` — derived from `f.course_completed` via CASE expression, no aggregation needed.
  - `client_ids` — comes from the CTE via LEFT JOIN, with `COALESCE(..., [1])` defaulting unmatched courses to a single-element array containing client ID 1.
  - `response_text` — conditionally redacted: if `cf.comment_only IS NULL` (meaning no cleared-feedback record exists, or it exists but `comment_only` is NULL), pass through the original text lowercased; otherwise emit empty string.
- **Filter logic (WHERE):** feedback is included if ANY of these hold:
  1. It is at least 24 hours old
  2. `response_text` is NULL
  3. `response_text` is empty string
  4. No cleared-feedback record exists (`cf.comment_only IS NULL`) OR `comment_only = true`
- **Inner join on `stg_users`:** acts as an existence filter — only feedback from known/valid users passes through.

## Step 5 — Plan CTE layers bottom-up

**`course_clients`** — Joins `courses` to `clients_course_containers` via `container_id` and aggregates to one row per `course_id` with an array of distinct client IDs. This pre-aggregation is necessary so the main query can LEFT JOIN without fan-out.

No additional CTEs are required. The main query joins `feedback` to `course_clients`, `stg_users`, and `stg_cleared_feedbacks`, applies the WHERE filter, and derives `status` and conditional `response_text`.

## Step 6 — Write each CTE with justification

**`course_clients`** — Resolves the two-hop relationship from course to client by joining through the container ID, then collapses to one row per course with an `ARRAY_AGG` of client IDs. This prevents row duplication in the main query when a course belongs to multiple clients.

## Step 7 — Compose the final query

```sql
WITH
course_clients AS (
  SELECT
    c.id as course_id,
    ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_course_containers")} as ccc
  JOIN
    ${ref("courses")} as c
  ON
    c.container_id = ccc.course_container_id
  GROUP BY
    c.id
)
SELECT
  f.id,
  f.response_sentiment,
  LOWER(
    CASE WHEN cf.comment_only IS NULL
      THEN f.response_text
      ELSE ''
    END
  ) AS response_text,
  f.video_seconds,
  f.user_id,
  f.course_id,
  f.lesson_id,
  f.course_completed,
  f.created,
  f.updated,
  CASE
    WHEN f.course_completed = 0 THEN 'in_progress'
    WHEN f.course_completed = 1 THEN 'completed'
  END AS status,
  COALESCE(cc.client_ids, [1]) AS client_ids
FROM
  ${ref("feedback")} AS f
LEFT JOIN
  course_clients as cc
ON
  cc.course_id = f.course_id
JOIN
  ${ref("stg_users")} AS u
ON
  f.user_id = u.id
LEFT JOIN
  ${ref("stg_cleared_feedbacks")} as cf
ON
  cf.feedback_id = f.id
WHERE
  DATETIME_DIFF(CURRENT_DATETIME, f.created, HOUR) >= 24 
OR 
  f.response_text IS NULL 
OR 
  f.response_text = ''
OR
  (cf.comment_only IS NULL OR cf.comment_only = true)
```
