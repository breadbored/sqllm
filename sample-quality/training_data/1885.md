This query enriches lesson resource records with arrays of associated client identifiers.

The CTE `stg_lesson_resources` performs a four-table join (clients_course_containers → courses → course_lessons → lessons) to establish which clients are associated with each lesson through course container relationships. For each lesson, it aggregates the distinct client IDs into an array.

The main query selects all columns from the lesson_resources table and left-joins with the CTE on lesson_id. The COALESCE function provides a fallback: if a lesson has no associated clients in the CTE (null result), the query defaults to an array containing a single value [1].

The name column is lowercased in the output. All other lesson_resource columns are passed through unchanged.

```sql
WITH
stg_lesson_resources as (
    SELECT
        cl.lesson_id as lesson_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("lessons")} as l
    ON
        l.id = cl.lesson_id
    GROUP BY
        cl.lesson_id
)


SELECT
  lr.id,
  LOWER(name) AS name,
  lr.url,
  lr.lock_until_over,
  lr.lesson_id,
  lr.created,
  lr.updated,
  lr.`order`,
  lr.file,
  lr.creator_admin_id,
  lr.updated_admin_id,
  COALESCE(slr.client_ids, [1]) AS client_ids
FROM
  ${ref("lesson_resources")} as lr
LEFT JOIN
    stg_lesson_resources as slr
ON
    slr.lesson_id = lr.lesson_id
```
