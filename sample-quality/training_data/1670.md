Show me how to identify returning user sub-sessions and tag their first event's app and URL path.

--- AGENT ---

This query identifies sub-sessions that are marked as "returning" and extracts the earliest event metadata for each one.

The filter `WHERE SPLIT(eas.sub_session_num, '-')[OFFSET(0)] != '0'` excludes sub-sessions whose numeric identifier begins with '0', thereby keeping only returning sessions. 

For each returning sub-session, the query groups all events and captures:
- The `app` value from the earliest event (using `MIN_BY` ordered by `created`)
- The `url_path` from the earliest event (using `MIN_BY` ordered by `created`)
- The minimum timestamp (`MIN(eas.created)`) in that group
- A static tag `"returning_session"` and class label `"behavior"` for categorization

The output has one row per returning sub-session.

```sql
SELECT
  eas.sub_session_id as sub_session_id,
  "returning_session" as tag,
  'behavior' as class,
  MIN_BY(eas.app, eas.created) as app,
  MIN_BY(eas.url_path, eas.created) as url_path,
  MIN(eas.created) as dt

FROM
  ${ref("base_buckets_events_speedy")} as eas

WHERE
  SPLIT(eas.sub_session_num, '-')[OFFSET(0)] != '0'

GROUP BY 
  sub_session_id
```
