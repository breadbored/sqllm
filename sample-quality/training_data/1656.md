--- USER ---

Generate a query that extracts geographic attributes (continent and country) for each sub-session and app, tagging them and capturing the earliest event associated with each geographic attribute.

--- AGENT ---

This query produces a row for each geographic attribute (continent or country) encountered within a sub-session and app combination, capturing the earliest occurrence of that attribute.

**First branch (continent):**
The query filters the events table for records where `geo_continent` is not null, groups by `sub_session_id`, a dynamically constructed tag prefixed with "continent_", and `app`. For each group, it extracts:
- The `url_path` from the earliest event in that group (using `MIN_BY` on the `created` timestamp)
- The minimum timestamp (the first event's timestamp)
- A hardcoded `class` value of 'attribute'

**Second branch (country):**
Identical logic, but operates on `geo_country` instead, creating tags prefixed with "country_code_".

**Combined output:**
The `UNION ALL` stacks both branches, producing a single result set where each row represents a geographic attribute tag (e.g., "continent_North America", "country_code_US") associated with a specific sub-session and app, along with the URL path and timestamp of the first event where that attribute was observed.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  CONCAT('continent_', e.geo_continent) as tag,
  'attribute' as class,
  e.app as app,
  MIN_BY(e.url_path, e.created) as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_speedy")} as e
WHERE
  e.geo_continent IS NOT NULL

GROUP BY
  sub_session_id,
  tag,
  app

UNION ALL

SELECT
  e.sub_session_id as sub_session_id,
  CONCAT('country_code_', e.geo_country) as tag,
  'attribute' as class,
  e.app as app,
  MIN_BY(e.url_path, e.created) as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_speedy")} as e
WHERE
  e.geo_country IS NOT NULL

GROUP BY
  sub_session_id,
  tag,
  app
```
