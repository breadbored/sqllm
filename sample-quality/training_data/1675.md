Show me an example of a BigQuery query that extracts the earliest event attributes from a grouped session, using MIN_BY to get chronologically first values.

--- AGENT ---

This query groups events by `sub_session_id` and extracts a single row per session capturing the earliest event data. For each session group, it:
- Uses `MIN_BY(e.app, e.created)` to retrieve the app value from the event with the earliest timestamp
- Uses `MIN_BY(e.url_path, e.created)` to retrieve the url_path value from the earliest event
- Records the minimum (earliest) created timestamp for each session with `MIN(e.created)`
- Adds two static literal columns: `tag='new_app'` and `class='attribute'`, suggesting this query is one branch of a UNION that assigns a classification label to this result set

The query filters to events created on or after October 1, 2025. This pattern is typical for session-level attribution queries where the first event in a session determines session properties.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  'new_app' as tag,
  'attribute' as class,
  MIN_BY(e.app, e.created) as app,
  MIN_BY(e.url_path, e.created) as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_app_speedy")} as e
WHERE
  e.created >= '2025-10-01'
GROUP BY
  e.sub_session_id
```
