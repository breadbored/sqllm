--- AGENT ---

This query builds a reference table from ISO 3166 country metadata where country names are cleaned through progressive string decomposition, and every row includes an array of all client IDs.

The key operation is a chain of nested SPLIT functions on the `name` column. This progressively removes delimiters and suffixes:
- First split on comma takes the primary element (removes comma-separated alternatives)
- Then splits on " of " and takes the first element (strips phrases like "Kingdom of")
- Then splits on " (" and takes the first element (removes parenthetical qualifiers)
- Finally splits on " and" and takes the first element (strips conjunctive phrases)

The result is a normalized country name in a canonical form.

The query projects the ISO 3166 codes (alpha-2, alpha-3, country-code, iso_3166-2) and regional hierarchies (region, sub-region, intermediate-region and their codes). Column names with hyphens are quoted with backticks to match the source schema.

The ARRAY subquery materializes all `id` values from the clients table as a single array, which is then repeated on every row of the output. This embeds the complete client ID list into each country record.

```sql
SELECT
  SPLIT(SPLIT(SPLIT(SPLIT(i3c.name, ',')[0], ' of ')[0], ' (')[0], ' and')[0] as name,
  i3c.`alpha-2` as alpha_2,
  i3c.`iso_3166-2` as alpha_v2_2,
  i3c.`alpha-3` as alpha_3,
  i3c.`country-code` as country_code,
  i3c.region as region,
  i3c.`sub-region` as sub_region,
  i3c.`intermediate-region` as intermediate_region,
  i3c.`region-code` as region_code,
  i3c.`sub-region-code` as sub_region_code,
  i3c.`intermediate-region-code` as intermediate_region_code,
  ARRAY(SELECT id FROM ${ref("clients")}) as client_ids,
FROM
  ${ref("iso_3166_countries")} as i3c
```
