This query analyzes user engagement by quarter and client, segmenting users based on their first appearance date to measure cohort behavior and session activity.

**What it does:**

The query builds three CTEs that establish the foundation for cohort analysis:

1. **client_transactions_min**: Identifies the earliest day each user interacted with each client by unnesting the `client_id` array, then grouping by user and expanded client to find the minimum transaction day.

2. **meta_quest_min**: A pass-through that selects from the previous CTE (user_id, client_id, and their min_day).

3. **quarterly_periods**: Reconstructs the full transaction history, unnesting `client_id` and computing the quarter start date for each transaction day using EXTRACT and DATE functions.

The final SELECT joins `quarterly_periods` with `meta_quest_min` on user_id and client_id, then groups by quarter_start and client_id to produce per-quarter, per-client metrics:
- **unique_users**: distinct users active in the quarter
- **new_users** / **returning_users**: users segmented by whether their min_day matches the transaction day
- **quarterly_new_users** / **quarterly_returning_users**: users cohorted by whether their min_day falls within the quarter
- **total_sessions** / **returning_user_sessions**: raw session counts for all and returning users
- **sessions_per_user**: average session frequency (rounded to 2 decimals)

Results are ordered chronologically by quarter_start.

```sql
WITH
  client_transactions_min AS (
  SELECT
    ct.user_id,
    client_id_expanded AS client_id,
    MIN(ct.`day`) AS min_day
  FROM
    ${ref("agg_client_transactions")} AS ct,
    UNNEST(ct.client_id) AS client_id_expanded
  GROUP BY
    ct.user_id,
    client_id_expanded ),
  meta_quest_min AS (
  SELECT
    ctm.user_id,
    ctm.client_id,
    ctm.min_day
  FROM
    client_transactions_min AS ctm ),
  quarterly_periods AS (
  SELECT
    ct.user_id,
    client_id_expanded AS client_id,
    ct.`day`,
    DATE(EXTRACT(YEAR
      FROM
        `day`), 1 + (EXTRACT(QUARTER
        FROM
          `day`) - 1) * 3, 1) AS quarter_start
  FROM
    ${ref("agg_client_transactions")} AS ct,
    UNNEST(ct.client_id) AS client_id_expanded )
SELECT
  COUNT(DISTINCT ct.user_id) AS unique_users,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day = ct.`day` THEN ct.user_id
  END
    ) AS new_users,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day != ct.`day` THEN ct.user_id
  END
    ) AS returning_users,
  ( COUNT( DISTINCT
      CASE
        WHEN mqm.min_day != ct.`day` THEN ct.user_id
    END
      ) + COUNT( DISTINCT
      CASE
        WHEN mqm.min_day = ct.`day` THEN ct.user_id
    END
      ) ) - COUNT(DISTINCT ct.user_id) AS new_users_returning,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day >= quarter_start AND mqm.min_day < DATE_ADD(quarter_start, INTERVAL 3 MONTH) THEN ct.user_id
  END
    ) AS quarterly_new_users,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day < quarter_start OR mqm.min_day >= DATE_ADD(quarter_start, INTERVAL 3 MONTH) THEN ct.user_id
  END
    ) AS quarterly_returning_users,
  COUNT(ct.user_id) AS total_sessions,
  COUNT(
    CASE
      WHEN mqm.min_day != ct.`day` THEN ct.user_id
  END
    ) AS returning_user_sessions,
  ROUND(COUNT(ct.user_id) / COUNT(DISTINCT ct.user_id), 2) AS sessions_per_user,
  quarter_start,
  ct.client_id AS client_id
FROM
  quarterly_periods AS ct
INNER JOIN
  meta_quest_min AS mqm
ON
  mqm.user_id = ct.user_id
  AND mqm.client_id = ct.client_id
GROUP BY
  quarter_start,
  ct.client_id
ORDER BY
  quarter_start
```
