Show me an example of how to tag user sub-sessions with behavioral events based on note-adding activity, capturing the app and URL where each note variant was created within the session's event window.

--- AGENT ---

This query segments note-adding behaviors into three distinct tagged variants and emits one row per sub-session per variant. The three UNION ALL branches each join event records to notes on course_id and user_id, then filter for notes created during the session's event time span (from MIN to MAX of event created timestamps). For each matching sub-session, MIN_BY extracts the earliest app and url_path values by creation timestamp, and MIN(es.created) captures the session's first event. The three branches differ only in their tag label and filtering: the first captures all notes ('note_added'), the second captures notes with a prefilled_note_id ('notes_added_custom'), and the third captures notes without one ('notes_added_give'). The HAVING clause enforces temporal alignment: the note creation time must fall within the session's observed event window.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  'note_added' as tag,
  'behavior' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref("base_buckets_events_speedy")} as es

JOIN
  ${ref("course_notes")} as cn
ON
  cn.course_id = es.course_id
AND
  cn.user_id = es.user_id

GROUP BY 
  sub_session_id,
  tag

HAVING
  MIN(cn.created) BETWEEN MIN(es.created) AND MAX(es.created)

UNION ALL

SELECT
  es.sub_session_id as sub_session_id,
  'notes_added_custom' as tag,
  'behavior' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref("base_buckets_events_speedy")} as es

JOIN
  ${ref("course_notes")} as cn
ON
  cn.course_id = es.course_id
AND
  cn.user_id = es.user_id
WHERE
  cn.prefilled_note_id IS NOT NULL

GROUP BY 
  sub_session_id,
  tag

HAVING
  MIN(cn.created) BETWEEN MIN(es.created) AND MAX(es.created)

UNION ALL

SELECT
  es.sub_session_id as sub_session_id,
  'notes_added_give' as tag,
  'behavior' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref("base_buckets_events_speedy")} as es

JOIN
  ${ref("course_notes")} as cn
ON
  cn.course_id = es.course_id
AND
  cn.user_id = es.user_id

WHERE
  cn.prefilled_note_id IS NULL

GROUP BY 
  sub_session_id,
  tag

HAVING
  MIN(cn.created) BETWEEN MIN(es.created) AND MAX(es.created)
```
