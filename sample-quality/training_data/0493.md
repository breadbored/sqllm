What happens when IFERROR is nested and the inner IFERROR catches an error?

--- AGENT ---

In nested IFERROR calls, when the innermost IFERROR successfully catches an error, it returns its catch value ('b'), and the outer IFERROR evaluates that result without error, so the outermost catch expression is not needed.

```googlesql
SELECT IFERROR(IFERROR(ERROR('a'), 'b'), 'c') AS result
```
