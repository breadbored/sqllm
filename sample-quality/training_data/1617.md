## Question
Show me a BigQuery query that joins multiple tables to get the relationship between clients and their courses.

--- AGENT ---

This query performs a three-table inner join to establish the relationship between clients and courses through a course container bridge table. It joins `stg_courses` to `stg_clients_course_containers` on the container ID, then joins that result to `stg_clients` on the client ID. The output returns the client ID and name paired with the course ID and container ID. The GROUP BY clause groups by all four selected columns, effectively deduplicating the result set to return each unique client-course-container combination once.

```sql
SELECT
  ccc.client_id as client_id,
  cl.name AS client_name,
  c.id    AS course_id,
  c.container_id as container_id
FROM
  ${ref("stg_courses")} AS c
INNER JOIN
  ${ref("stg_clients_course_containers")} AS ccc
  ON c.container_id = ccc.course_container_id
INNER JOIN
  ${ref("stg_clients")} AS cl
  ON
    ccc.client_id = cl.id
GROUP BY ccc.client_id, cl.name, c.id, c.container_id
```
