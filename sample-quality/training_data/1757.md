--- USER ---

Can you show me a BigQuery query that counts page transitions within sessions, tracking when URLs change?

--- AGENT ---

This query enriches each event row with a cumulative count of page transitions within its session. It selects all columns from the pre_vaas_events_url table and adds a new column called vaas_page_visit. This new column uses a window function partitioned by session_id and ordered chronologically by created timestamp. Within each session, it performs a running sum: each event where url_path is non-null AND differs from last_url_path contributes 1 to the count, otherwise contributes 0. The result is a session-scoped page visit sequence number that increments each time the URL changes, resetting for each new session.

```sql
SELECT
  e.*,

  SUM(
    CASE
      WHEN e.url_path IS NOT NULL AND e.url_path != e.last_url_path THEN 1
      ELSE 0
    END
  ) OVER (
    PARTITION BY
      e.session_id
    ORDER BY
      e.created ASC
  ) AS vaas_page_visit,

FROM
  ${ref("pre_vaas_events_url")} AS e
```
