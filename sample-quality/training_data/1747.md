--- USER ---

For each event in a session, determine whether cookies were accepted up to that point in time based on cookie banner interaction events.

--- AGENT ---

This query propagates cookie acceptance status forward through a session's event stream using a temporal join pattern.

**Step 1 — Extract cookie banner events**
The `accepted_cookies` CTE filters the events table to rows where `name = 'cookie_banner'`, and creates a boolean column indicating whether each banner interaction was an acceptance (`type = 'accept'`). This produces a sparse table of only cookie banner events with their session IDs and timestamps.

**Step 2 — Propagate acceptance status to all events**
The main query left-joins every event in the table back to the `accepted_cookies` CTE, matching on session ID and requiring the cookie event to have occurred no later than the current event (`ac.created <= e.created`). This creates multiple matches per event—potentially many cookie banner events from the same session that happened before it.

**Step 3 — Select the most recent acceptance status**
For each event ID, `MAX_BY(ac.accepted_cookies, e.created)` selects the acceptance boolean from whichever matched cookie event is most recent by timestamp. `COALESCE(..., false)` defaults to `false` if no prior cookie banner event exists in that session.

The result is one row per event, with a column indicating the most recent cookie acceptance status at that point in the session's timeline.

```sql
WITH
accepted_cookies AS (
  SELECT
    e.session_id as session_id,
    e.type = 'accept' as accepted_cookies,
    e.created as created
  FROM
    ${ref("events_numbered")} as e
  WHERE
    e.name = 'cookie_banner'
)
SELECT
  e.id as id,
  COALESCE(MAX_BY(ac.accepted_cookies, e.created), false) as accepted_cookies
FROM
  ${ref("events_numbered")} as e
LEFT JOIN
  accepted_cookies as ac
ON
  ac.session_id = e.session_id
AND
  ac.created <= e.created
GROUP BY
  e.id
```
