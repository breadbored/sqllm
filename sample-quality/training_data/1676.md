--- USER ---

Show me a BigQuery query that extracts the earliest event from each sub-session, capturing its app and URL path, and tags the results for attribution purposes.

--- AGENT ---

This query segments events by `sub_session_id` and extracts the earliest event's attributes from each segment. For each sub_session, it:

- Retrieves the `app` and `url_path` values associated with the earliest event (determined by `created` timestamp) using `MIN_BY`
- Records that earliest timestamp as `dt`
- Adds constant labels: the tag `'old_app'` and class `'attribute'` to identify this as an old app attribution record

The query filters to only events created before October 1, 2025, suggesting it is reconstructing historical event sequences for a specific time window. The result is a deduplicated table with one row per sub_session, containing the first event's properties.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  'old_app' as tag,
  'attribute' as class,
  MIN_BY(e.app, e.created) as app,
  MIN_BY(e.url_path, e.created) as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_app_speedy")} as e
WHERE
  e.created < '2025-10-01'
GROUP BY
  sub_session_id
```
