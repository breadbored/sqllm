## Question
Show me an example of how to denormalize cleared feedback records by attaching a complete list of all client IDs to each row as an array column.

--- AGENT ---

This query retrieves feedback records from the `cleared_feedbacks` table and enriches each row with a denormalized array containing all client IDs.

**Structure:**
- The query selects three columns directly from `cleared_feedbacks`: the feedback `id`, `feedback_id`, and `comment_only` flag
- It adds a fourth column named `client_ids` that is populated by a scalar ARRAY subquery
- The ARRAY subquery executes on every output row and extracts all `id` values from the `clients` table
- This means each feedback record will have an identical array of all client IDs attached to it

**Output shape:**
- One row per feedback record in `cleared_feedbacks`
- Each row contains the feedback's own metadata plus a denormalized array of all possible client identifiers

This pattern is useful when a downstream process needs to know which clients *could* be associated with a given feedback without requiring an explicit join table. The array is rebuilt identically for every row, making this appropriate only when the client list is small or when the redundancy is intentional for simplicity.

**The original query:**

```sql
SELECT
  cf.id,
  cf.feedback_id,
  cf.comment_only,
  ARRAY(SELECT id FROM ${ref("clients")}) as client_ids,
FROM
  ${ref("cleared_feedbacks")} AS cf
```
