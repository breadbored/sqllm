## Question
Provide an example BigQuery table function that analyzes user acquisition and engagement metrics (new vs. returning users) for a specific client on a daily and monthly basis.

--- AGENT ---

This query is a parameterized table function that performs cohort analysis by client, comparing user activity against their first transaction date. It establishes a baseline in two CTEs: first, it finds the earliest transaction date for each user within each client by unnesting an array of client IDs; second, it filters to only the specified client parameter. The main query then joins the full transaction history against this baseline and groups by month, computing:

- **Daily acquisition metrics**: new users (first transaction on that calendar day), returning users (first transaction on a different calendar day), and their session counts
- **Monthly acquisition metrics**: new and returning users by month of first transaction  
- **Engagement metrics**: total distinct users, average sessions per user, and a calculated field measuring overlap/anomalies in new vs. returning counts
- **Temporal buckets**: results grouped and sorted by month with both formatted display and date forms

All metrics are filtered to transactions containing the specified client ID in its array field.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64) AS (
  WITH
    client_transactions_min AS (
    SELECT
      ct.user_id,
      cid AS client_id,
      MIN(ct.`day`) min_day,
    FROM
      ${ref("agg_client_transactions")} AS ct,
      UNNEST(ct.client_id) AS cid
    GROUP BY
      ct.user_id,
      cid),
    client_min AS (
    SELECT
      ctm.user_id,
      ctm.client_id,
      ctm.min_day,
    FROM
      client_transactions_min AS ctm
    WHERE
      ctm.client_id = clientId)
  SELECT
    COUNT(DISTINCT ct.user_id) AS unique_users,
    COUNT(DISTINCT
      CASE
        WHEN DATE_TRUNC(cm.min_day, DAY) = DATE_TRUNC(day, DAY) THEN ct.user_id
    END
      ) AS new_users,
    COUNT(DISTINCT
      CASE
        WHEN DATE_TRUNC(cm.min_day, DAY) != DATE_TRUNC(day, DAY) THEN ct.user_id
    END
      ) AS returning_users,
    (COUNT(DISTINCT
        CASE
          WHEN DATE_TRUNC(cm.min_day, DAY) != DATE_TRUNC(day, DAY) THEN ct.user_id
      END
        ) + COUNT(DISTINCT
        CASE
          WHEN DATE_TRUNC(cm.min_day, DAY) = DATE_TRUNC(day, DAY) THEN ct.user_id
      END
        )) - COUNT(DISTINCT ct.user_id) AS new_users_returning,
    COUNT(DISTINCT
      CASE
        WHEN DATE_TRUNC(cm.min_day, MONTH) = DATE_TRUNC(day, MONTH) THEN ct.user_id
    END
      ) AS monthly_new_users,
    COUNT(DISTINCT
      CASE
        WHEN DATE_TRUNC(cm.min_day, MONTH) != DATE_TRUNC(day, MONTH) THEN ct.user_id
    END
      ) AS monthly_returning_users,
    COUNT(ct.user_id) AS total_sessions,
    COUNT(CASE
        WHEN DATE_TRUNC(cm.min_day, DAY) != DATE_TRUNC(day, DAY) THEN ct.user_id
    END
      ) AS returning_user_sessions,
    ROUND(COUNT(ct.user_id) / COUNT(DISTINCT ct.user_id), 2) AS sessions_per_user,
    FORMAT_DATE('%b %y', DATE_TRUNC(day, MONTH)) AS year_month,
    DATE_TRUNC(day, MONTH) AS month,
  FROM
    ${ref("agg_client_transactions")} AS ct
  JOIN
    client_min AS cm
  ON
    cm.user_id = ct.user_id
  WHERE
    clientId IN UNNEST(ct.client_id)
  GROUP BY
    FORMAT_DATE('%b %y', DATE_TRUNC(day, MONTH)),
    DATE_TRUNC(day, MONTH)
  ORDER BY
    month)
```
