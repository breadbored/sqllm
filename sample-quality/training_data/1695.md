Provide an example BigQuery query that identifies user sessions coinciding with account creation.

--- AGENT ---

This query identifies sub-sessions where a user's account creation timestamp falls within the event timeline of that sub-session. It performs a join between events and users on user_id, then groups by sub_session_id and the user's creation timestamp. The HAVING clause filters to only those groupings where the user's created timestamp falls between the earliest and latest event timestamps in the sub-session—effectively finding sessions that occurred at or near the time of account creation.

For each matching sub-session, the query captures the app and URL path from the earliest event (using MIN_BY to select by the created timestamp), the session ID, and the earliest event timestamp. It tags these records with a literal 'new_user' label and 'behavior' class, indicating this is part of a behavioral tagging system. This pattern is useful for detecting onboarding behavior—the specific apps and pages users first interact with when they create their account.

```sql
SELECT
  eas.sub_session_id as sub_session_id,
  'new_user' as tag,
  'behavior' as class,
  MIN_BY(eas.app, eas.created) as app,
  MIN_BY(eas.url_path, eas.created) as url_path,
  MIN(eas.created) as dt
FROM
  ${ref("base_buckets_events_app_speedy")} as eas

JOIN
  ${ref("stg_users")} as u
ON
  u.id = eas.user_id

GROUP BY 
  eas.sub_session_id,
  u.created

HAVING
  u.created BETWEEN MIN(eas.created) AND MAX(eas.created)
```
