Get all cookie banner events for each user, showing the event type, action taken, and when it occurred.

- `name` drives the filter predicate but is not selected in output — it is a filter-only column.

- No columns depend on intermediate computation.

**5. Plan CTE layers bottom-up**

No CTEs needed. The query is a single-pass filter with column projection. One `SELECT ... FROM ... WHERE` is sufficient.

**7. Compose the final query**

```sql

  user_id,
  type,
  action,
  created
FROM ${ref("stg_events_speedy")}
```
