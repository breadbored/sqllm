--- USER ---

Provide me with a BigQuery SQL query that enriches event data with lagged path and attribution information, and tracks page and attribution changes within a session.

--- AGENT ---

This query enriches event-level data with navigation and attribution context by adding three key pieces of information:

1. **Previous event state**: Using the LAG window function, it pulls the prior event's URL path, referrer, landing path, and all UTM parameters (campaign, content, medium, source, term) for each session, ordered by creation time.

2. **Page visit sequence**: A running count (`page_visit`) that increments each time the `url_path` changes within a session. This gives each distinct page in a user's journey a sequential visit number.

3. **Attribution change sequence**: A running count (`utm_visit`) that increments whenever any attribution field changes (landing path, referrer, or any UTM parameter). This is partitioned by both session and segment_id, tracking when the traffic source or campaign context shifted.

4. **Change detection flags**: A boolean `page_changed` that indicates whether the current event's URL differs from the previous event's URL in the same session.

The output preserves all original event columns (via `*`) alongside the new derived columns, making it useful for analyzing user journeys where you need to understand both the current state and recent transitions.

```sql
/**
* Last Paths
*
* This layer is just to get the path, referrer, and UTMs on the event before it.
**/
WITH
  last_path_events AS (
    SELECT
      -- Last URL path
      LAG(e.url_path) OVER (
        PARTITION BY
          e.session_id
        ORDER BY
          e.created ASC
      ) AS last_url_path,
      -- Last referrer, UTMs, landing path
      LAG(e.utm_campaign) OVER (
        PARTITION BY
          e.session_id
        ORDER BY
          e.created ASC
      ) AS last_utm_campaign,
      LAG(e.utm_content) OVER (
        PARTITION BY
          e.session_id
        ORDER BY
          e.created ASC
      ) AS last_utm_content,
      LAG(e.utm_medium) OVER (
        PARTITION BY
          e.session_id
        ORDER BY
          e.created ASC
      ) AS last_utm_medium,
      LAG(e.utm_source) OVER (
        PARTITION BY
          e.session_id
        ORDER BY
          e.created ASC
      ) AS last_utm_source,
      LAG(e.utm_term) OVER (
        PARTITION BY
          e.session_id
        ORDER BY
          e.created ASC
      ) AS last_utm_term,
      LAG(e.referrer) OVER (
        PARTITION BY
          e.session_id
        ORDER BY
          e.created ASC
      ) AS last_referrer,
      LAG(e.landing_path) OVER (
        PARTITION BY
          e.session_id
        ORDER BY
          e.created ASC
      ) AS last_landing_path,
      *
    FROM
      ${ref("events_subsession")} AS e
  )
  -- Get last URL path count
SELECT
  SUM(
    CASE
      WHEN ee.url_path != ee.last_url_path THEN 1
      ELSE 0
    END
  ) OVER (
    PARTITION BY
      ee.session_id
    ORDER BY
      ee.created ASC
  ) AS page_visit,
  ee.url_path != ee.last_url_path AS page_changed,
  SUM(
    CASE
      WHEN ee.landing_path != ee.last_landing_path
      OR ee.referrer != ee.last_referrer
      OR ee.utm_campaign != ee.last_utm_campaign
      OR ee.utm_content != ee.last_utm_content
      OR ee.utm_medium != ee.last_utm_medium
      OR ee.utm_source != ee.last_utm_source
      OR ee.utm_term != ee.last_utm_term THEN 1
      ELSE 0
    END
  ) OVER (
    PARTITION BY
      ee.session_id,
      ee.segment_id
    ORDER BY
      ee.created ASC
  ) AS utm_visit,
  *
FROM
  last_path_events AS ee
```
