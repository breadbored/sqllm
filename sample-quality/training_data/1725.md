Show me a BigQuery query that extracts the first occurrence of a specific event type within session groups, capturing the app and URL path from that earliest event.

--- AGENT ---

This query identifies the earliest `select_course` event within each sub_session_id and captures the application and URL path associated with that event.

The query filters the events table to rows where `name` equals `select_course` (note: the first two WHERE conditions excluding 'page_view' and 'open_link' are redundant given the third equality condition). It groups by sub_session_id and uses `MIN_BY()` to extract the app and url_path values from whichever row has the earliest created timestamp within each group. The `MIN(created)` also returns that earliest timestamp. Each result row is tagged with the literal strings `select_course` and `behavior` as classification fields.

SELECT
    es.sub_session_id as sub_session_id,
    'select_course' as tag,
    'behavior' as class,
    MIN_BY(es.app, es.created) as app,
    MIN_BY(es.url_path, es.created) as url_path,
    MIN(es.created) as dt

FROM
    ${ref('base_buckets_events_docs_speedy')} as es 

WHERE
    es.name != 'page_view'
AND
    es.name != 'open_link'
AND
    es.name = 'select_course'

GROUP BY
    sub_session_id,
    tag
