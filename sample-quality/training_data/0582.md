How do I parse WKB data with planar versus geodesic line semantics?

--- AGENT ---

`ST_GEOGFROMWKB` parses WKB with a `planar` parameter that controls line interpretation—when TRUE it uses planar semantics with intermediate interpolation points, when FALSE it uses geodesic great circles.

```googlesql
WITH wkb_data AS (
  SELECT '010200000002000000feffffffffffef3f000000000000f03f01000000000008400000000000000040' geo
)
SELECT
  ST_GeogFromWkb(geo, planar=>TRUE) AS from_planar,
  ST_GeogFromWkb(geo, planar=>FALSE) AS from_geodesic,
FROM wkb_data
```
