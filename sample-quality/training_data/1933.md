--- USER ---

Give me a per-user, per-course progress summary showing how far each user has progressed through a course, the total course duration, progress percentage, time spent on the course, and which clients are associated with the course. Include course metadata like name and container ID.

--- AGENT ---

## 1. Identify shapes

**Input tables and their observable columns:**
- `stg_user_courses` (uc): `user_id`, `course_id`, `created`, `updated`, `status`
- `stg_courses` (c): `id`, `container_id`, `name`
- `stg_users` (u): `id`
- `stg_user_course_lesson_progress` (uclp): `user_id`, `course_id`, `lesson_id`, `progress_seconds`, `interval_seconds`
- `stg_lessons` (l): `id`, `video_id`
- `stg_videos` (v): `id`, `duration`
- `stg_course_lessons` (cl): `course_id`, `lesson_id`
- `stg_clients_course_containers` (ccc): `course_container_id`, `client_id`
- `stg_clients` (clients): `id`

**Target shape (one row per user per course):**
`user_id, course_id, container_id, course_name, course_started, course_updated, course_status, course_progress_seconds, course_progress_minutes, course_length_minutes, course_progress_percentage, time_on_course_minutes, time_on_course, client_id[]`

## 2. Identify sources and relationships

The spine of the query is `stg_user_courses` — one row per user-course enrollment.

**Join relationships:**
- `uc.course_id = c.id` — course metadata
- `uc.user_id = u.id` — user existence check (no columns selected from `u`, so this is a filter/validation join)
- `uc.course_id = ulp.course_id AND uc.user_id = ulp.user_id` — pre-aggregated user lesson progress, joined on the composite key (user + course)
- `uc.course_id = cd.course_id` — pre-aggregated course duration
- `c.container_id = ccc.course_container_id` — bridge from course container to clients
- `clients.id = ccc.client_id` — client entity

The progress subquery (`ulp`) requires pre-aggregation because progress is recorded at the lesson level (`user_id, course_id, lesson_id`) but must be joined at the course level (`user_id, course_id`). Without pre-aggregation, the join would fan out and corrupt the aggregation.

The course duration subquery (`cd`) requires pre-aggregation because duration lives on `stg_videos`, reached through `stg_lessons` and `stg_course_lessons`, and must be summed to one row per course before joining.

The client relationship fans out (a course container can have multiple clients), which is why the outer query uses `GROUP BY` and `ARRAY_AGG(DISTINCT clients.id)` to collapse clients back into an array.

## 3. Identify aggregation depth

There are two independent pre-aggregation paths plus one outer aggregation:

**Path A — User lesson progress (two levels):**
```
user_id, course_id, lesson_id, MAX(progress_seconds) as user_total_lesson_progress, SUM(interval_seconds) as time_on_lesson
-> user_id, course_id, SUM(user_total_lesson_progress)/60 as user_total_lesson_progress, SUM(user_total_lesson_progress) as user_total_lesson_progress_seconds, SUM(time_on_lesson) as time_on_course
```

**Path B — Course duration (one level):**
```
course_id, SUM(v.duration)/60 as course_duration_minutes
```

**Outer aggregation (collapses client fan-out):**
```
user_id, course_id, container_id, course_name, ANY_VALUE(scalar_columns), ARRAY_AGG(DISTINCT clients.id) as client_id
```

## 4. Identify dependencies

- **Grouping drivers:** `user_id` and `course_id` are the primary grain of the output. `container_id` and `course_name` are functionally dependent on `course_id` but included in GROUP BY for correctness.
- **Derived — progress path:** `progress_seconds` and `interval_seconds` must first be aggregated per lesson, then summed per course. The two-level aggregation prevents averaging averages — `MAX(progress_seconds)` gets the furthest point per lesson, then `SUM` rolls those up to the course level.
- **Derived — duration path:** `v.duration` must be summed across all lessons in a course via the `stg_course_lessons` bridge table.
- **Derived — percentage:** `course_progress_percentage` depends on both `user_total_lesson_progress` (from path A) and `course_duration_minutes` (from path B), so both must be resolved before the outer SELECT.
- **Fan-out source:** The `clients` join through `stg_clients_course_containers` can produce multiple rows per course, requiring the outer GROUP BY + ARRAY_AGG.

## 5. Plan CTE layers bottom-up

This query uses inline subqueries rather than named CTEs, but the logical layers are:

1. **`lesson_progress` (innermost subquery of `ulp`)** — Aggregates raw progress events to one row per user × course × lesson, taking `MAX(progress_seconds)` and `SUM(interval_seconds)`.
2. **`ulp` (outer subquery)** — Rolls lesson-level progress up to one row per user × course by summing lesson progress and time-on-lesson.
3. **`cd` (course duration subquery)** — Joins lessons → videos → course_lessons and sums video durations to one row per course.
4. **Outer query** — Joins the spine (`uc`) to course metadata (`c`), user validation (`u`), pre-aggregated progress (`ulp`), pre-aggregated duration (`cd`), and client chain (`ccc` → `clients`). Groups by user × course to collapse the client fan-out into an array.

## 6. Write each CTE with justification

**`lesson_progress` subquery:** Exists because progress is tracked per lesson and must be deduplicated (MAX) and summed (interval_seconds) at the lesson grain before rolling up to the course grain — skipping this level would double-count intervals across lessons.

**`ulp` subquery:** Exists to roll lesson-level aggregates up to one row per user × course, so the join to `uc` is 1:1 and does not fan out the spine.

**`cd` subquery:** Exists because course duration is derived from individual video durations linked through a bridge table (`course_lessons`), and must be pre-summed to one value per course before joining.

**Outer GROUP BY:** Exists solely to collapse the fan-out from the client join chain — all other joined data is already 1:1 with the spine, which is why `ANY_VALUE` is safe for all non-client columns.

## 7. Compose the final query

```sql
SELECT
  uc.user_id,
  uc.course_id,
  c.container_id,
  c.name AS course_name,
  ANY_VALUE(uc.created) AS course_started,
  MAX(uc.updated) AS course_updated,
  ANY_VALUE(uc.status) AS course_status,
  ANY_VALUE(ulp.user_total_lesson_progress_seconds) AS course_progress_seconds,
  ANY_VALUE(ulp.user_total_lesson_progress) AS course_progress_minutes,
  ANY_VALUE(cd.course_duration_minutes) AS course_length_minutes,
  LEAST( ROUND(((ANY_VALUE(ulp.user_total_lesson_progress) / ANY_VALUE(cd.course_duration_minutes))), 2), 1 ) AS course_progress_percentage,
  ANY_VALUE(ROUND((ulp.time_on_course / 60), 3)) AS time_on_course_minutes,
  ANY_VALUE(ulp.time_on_course) AS time_on_course,
  ARRAY_AGG(DISTINCT clients.id IGNORE NULLS
  ORDER BY
    clients.id) AS client_id
FROM
  ${ref("stg_user_courses")} AS uc
INNER JOIN
  ${ref("stg_courses")} AS c
ON
  uc.course_id = c.id
INNER JOIN
  ${ref("stg_users")} AS u
ON
  uc.user_id = u.id
LEFT JOIN (
  SELECT
    lesson_progress.user_id,
    lesson_progress.course_id,
    ROUND((SUM(lesson_progress.user_total_lesson_progress) / 60), 3) AS user_total_lesson_progress,
    (SUM(lesson_progress.user_total_lesson_progress)) AS user_total_lesson_progress_seconds,
    SUM(time_on_lesson) AS time_on_course,
  FROM (
    SELECT
      user_id,
      course_id,
      MAX(progress_seconds) AS user_total_lesson_progress,
      SUM(interval_seconds) AS time_on_lesson,
    FROM
      ${ref("stg_user_course_lesson_progress")} AS uclp
    WHERE
      user_id IS NOT NULL
    GROUP BY
      user_id,
      course_id,
      lesson_id ) AS lesson_progress
  GROUP BY
    lesson_progress.user_id,
    lesson_progress.course_id) AS ulp
ON
  uc.course_id = ulp.course_id
  AND uc.user_id = ulp.user_id
LEFT JOIN (
  SELECT
    cl.course_id,
    ROUND((SUM(v.duration) / 60), 3) AS course_duration_minutes
  FROM
    ${ref("stg_lessons")} AS l
  LEFT JOIN
    ${ref("stg_videos")} AS v
  ON
    l.video_id = v.id
  LEFT JOIN
    ${ref("stg_course_lessons")} AS cl
  ON
    l.id = cl.lesson_id
  GROUP BY
    course_id
  ORDER BY
    course_id ) AS cd
ON
  uc.course_id = cd.course_id
LEFT JOIN
  ${ref("stg_clients_course_containers")} AS ccc
ON
  c.container_id = ccc.course_container_id
LEFT JOIN
  ${ref("stg_clients")} AS clients
ON
  clients.id = ccc.client_id
WHERE
  ulp.user_id IS NOT NULL
GROUP BY
  uc.user_id,
  uc.course_id,
  c.container_id,
  c.name
```
