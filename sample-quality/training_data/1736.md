--- USER ---

Generate tags for video-playing events in our analytics events table, creating both behavior-level tags (state only) and content-level tags (state plus course name), with the earliest timestamp for each session/tag/URL combination.

--- AGENT ---

This query produces two complementary rows for each unique combination of sub-session, playing state, and URL path — one behavior-level tag and one content-level tag — capturing the earliest occurrence of each.

**First branch (behavior):**
Filters events where `vaas_state_playing` is populated and creates a tag of the form `vaas_playing:<state>`. Groups by sub_session_id, this tag, and URL path, returning the minimum timestamp for each group.

**Second branch (content):**
Joins the same events table to a `courses` table on `vaas_id = vaas_url_video_id`, then creates an enriched tag of the form `vaas_playing:<state>:<course_name>`. Also groups by sub_session_id, tag, and URL path, returning the minimum timestamp for each group. This adds course context to the playing state.

Both branches apply a `class` label (`behavior` vs `content`) and an `app` label (`vaas`) to distinguish the tag types. The UNION ALL merges both branching outputs into a single result set.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  CONCAT('vaas_playing:', e.vaas_state_playing) as tag,
  'behavior' as class,
  'vaas' as app,
  e.vaas_url_url_path as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_vaas_speedy")} as e
WHERE
  e.vaas_state_playing IS NOT NULL
GROUP BY
  sub_session_id,
  tag,
  url_path

UNION ALL

SELECT
  e.sub_session_id as sub_session_id,
  CONCAT('vaas_playing:', e.vaas_state_playing, ':', c.name) as tag,
  'content' as class,
  'vaas' as app,
  e.vaas_url_url_path as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_vaas_speedy")} as e
JOIN
  ${ref("courses")} as c
ON
  c.vaas_id = e.vaas_url_video_id
WHERE
  e.vaas_state_playing IS NOT NULL
GROUP BY
  sub_session_id,
  tag,
  url_path
```
