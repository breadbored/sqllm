--- USER ---

For each knowledge check, retrieve its associated client IDs by traversing the course container hierarchy up from its learning cue, defaulting to client 1 if no clients are found.

--- AGENT ---

This query aggregates client associations for knowledge checks by tracing a table hierarchy from learning cues backward through lessons, course lessons, courses, and finally to client-course container mappings.

The CTE `learning_cue_clients` performs a multi-join chain to resolve all clients connected to each learning cue:
- Joins from `learning_cues` up through `lessons` → `course_lessons` → `courses` → `clients_course_containers`
- Aggregates all distinct client IDs for each learning cue into an array using `ARRAY_AGG(DISTINCT ... IGNORE NULLS)`

The main query then:
- Selects the id, learning_cue_id, created, and updated columns from `knowledge_checks`
- LEFT JOINs each knowledge check to its associated learning cue's client array
- Uses `COALESCE` to provide a default array `[1]` when no clients are found (i.e., when the LEFT JOIN result is NULL)

The output is each knowledge check with its resolved client IDs, or `[1]` if resolution yields no results.

```sql
WITH
learning_cue_clients AS (
    SELECT
        lc.id as learning_cue_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("lessons")} as l
    ON 
        l.id = cl.lesson_id
    JOIN
        ${ref("learning_cues")} as lc
    ON
        lc.lesson_id = l.id
    GROUP BY
        lc.id
)
SELECT
  kc.id,
  kc.learning_cue_id,
  kc.created,
  kc.updated,
  COALESCE(lcc.client_ids, [1]) AS client_ids
FROM
  ${ref("knowledge_checks")} as kc
LEFT JOIN
  learning_cue_clients as lcc
ON
  lcc.learning_cue_id = kc.learning_cue_id
```
