## Question
Give me a cleaned listing of all certifications with their associated client IDs. Text fields should be lowercased, and certifications with no client associations should default to client ID 1.

--- AGENT ---

## 1. Identify shapes

**Input tables:**
- `certifications`: `id, name, description, published, headline, thumbnail, created, updated, has_been_published, is_public, lock_order, background_image, badge_image, footer_description, for_me_description, seo`
- `clients_certifications`: `certification_id, client_id` (a many-to-many bridge table)

**Target shape:**
One row per certification with all its descriptive columns (text fields lowercased) plus a `client_ids` array column containing the distinct client IDs associated with that certification, defaulting to `[1]` when none exist.

## 2. Identify sources and relationships

- `clients_certifications.certification_id` joins to `certifications.id` — this is the natural join key.
- The relationship is one-to-many (one certification can have multiple client associations), so the client side must be pre-aggregated into an array before joining to avoid row duplication on the certification dimension.
- A LEFT JOIN is required because not every certification has client associations, and we want all certifications in the output.

## 3. Identify aggregation depth

Single-level aggregation, pre-joined:

```
certification_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

No further aggregation after the join — the main query is a flat SELECT with no GROUP BY. The pre-aggregation collapses the many-to-many bridge into one row per certification before joining, which preserves the one-row-per-certification grain of the final output.

## 4. Identify dependencies

- **Grouping driver:** `certification_id` in the bridge table drives the pre-aggregation.
- **Derived column:** `client_ids` is derived from `ARRAY_AGG(DISTINCT client_id IGNORE NULLS)`.
- **Default logic:** `COALESCE(cc.client_ids, [1])` depends on the LEFT JOIN producing NULLs for unmatched certifications — this must happen after the join, not inside the CTE.
- **Text normalization:** `LOWER()` is applied to string columns in the final SELECT. These are independent of the aggregation logic.

## 5. Plan CTE layers bottom-up

1. **`cert_clients`** — Pre-aggregates `clients_certifications` to one row per `certification_id`, collecting distinct `client_id` values into an array. This prevents row multiplication when joined to `certifications`.

That is the only CTE. The final query joins `certifications` to `cert_clients` and applies text normalization and the COALESCE default.

## 6. Write each CTE with justification

**`cert_clients`**: Collapses the many-to-many bridge table into one array per certification so the subsequent LEFT JOIN does not duplicate certification rows.

```sql
cert_clients AS (
  SELECT
    cc.certification_id as certification_id,
    ARRAY_AGG(DISTINCT cc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_certifications")} as cc
  GROUP BY
    cc.certification_id
)
```

## 7. Compose the final query

```sql
WITH
cert_clients AS (
  SELECT
    cc.certification_id as certification_id,
    ARRAY_AGG(DISTINCT cc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_certifications")} as cc
  GROUP BY
    cc.certification_id
)
SELECT
  id,
  LOWER(name)               AS name,
  LOWER(description)        AS description,
  published,
  LOWER(headline)           AS headline,
  thumbnail,
  created,
  updated,
  has_been_published,
  is_public,
  lock_order,
  background_image,
  badge_image,
  LOWER(footer_description) AS footer_description,
  LOWER(for_me_description) AS for_me_description,
  LOWER(seo)                AS seo,
  COALESCE(cc.client_ids, [1]) AS client_ids
FROM
  ${ref("certifications")} as c
LEFT JOIN
  cert_clients as cc
ON
  c.id = cc.certification_id
```

## 8. Verify output shape

Target: one row per certification with lowercased text fields and a `client_ids` array defaulting to `[1]`.

Output columns: `id, name, description, published, headline, thumbnail, created, updated, has_been_published, is_public, lock_order, background_image, badge_image, footer_description, for_me_description, seo, client_ids`.

- Grain is one row per `certifications.id` — preserved by pre-aggregating the bridge table before joining.
- Text columns (`name, description, headline, footer_description, for_me_description, seo`) are wrapped in `LOWER()`. ✓
- `client_ids` uses `COALESCE(..., [1])` to default unmatched certifications. ✓
- LEFT JOIN ensures all certifications appear regardless of client associations. ✓

Output shape matches the target.
