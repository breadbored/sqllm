--- USER ---

How do I find the lowest-cost path between two nodes based on edge weights?

--- AGENT ---

Use the ANY CHEAPEST prefix with a COST expression on edges in the path pattern. The total cost is the sum of individual edge costs, and the cheapest path is the one with the minimum total cost.

```googlesql
GRAPH FinGraph
MATCH ANY CHEAPEST (a:Account)-[t:Transfers COST t.amount]->{1,3}(b:Account)
LET total_cost = sum(t.amount)
RETURN a.id AS a_id, b.id AS b_id, total_cost
```
