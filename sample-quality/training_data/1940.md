## Question
Build a landing page acquisition report at the sub-session level. For each sub-session, show the landing page URL, extracted UTM parameters, engagement metrics (time on page, scroll depth, clicks), session-level aggregates (page views, duration, bounce/engaged flag), what the user landed on (course, badge, or channel with its name), authentication details, whether they visited docs, the first course they started that day, and the associated client names — defaulting to "Data Protocol" when no specific client is found.

--- AGENT ---

## Step 1 — Identify shapes

**Input tables (observable from the query):**

- `stg_events_app_speedy` — `session_id, sub_session_id, user_id, current_user_id, url_path, url, created, scroll_value, scroll_num, type, name, referrer, landed_from, page_visit, action, app`
- `stg_events_speedy` — `sub_session_id, app, created`
- `stg_courses` — `id, name, container_id`
- `stg_clients_course_containers` — `client_id, course_container_id`
- `stg_clients` — `id, name`
- `stg_learning_paths` — `id, name`
- `learning_path_course_containers` — `learning_path_id, course_container_id`
- `clients_course_containers` — `client_id, course_container_id`
- `stg_categories` — `link, name`
- `stg_users` — `id, created`
- `reporting_user_courses` — `user_id, course_name, container_id, course_id, course_status, course_progress_percentage, course_started`

**Target shape — one row per sub-session:**

`session_id, sub_session_id, user_id, current_user_id, landing_path, name, type, time_on_page, landing_page_click_count, auth_action, auth_type, started_at_docs, went_to_docs, page_views, session_duration, session_click_count, engaged, course_name, container_id, course_id, course_status, course_progress_percentage, course_started, utm_source, utm_medium, utm_campaign, utm_content, referrer, landed_from, created, client_id (array), client_name (array)`

## Step 2 — Identify sources and relationships

The query is built from seven independent CTEs that each pre-aggregate or reshape a dimension, then joined together in the final SELECT on the `events_acquisition` CTE as the spine:

1. **events_acquisition ↔ subsession_data** — joined on `sub_session_id` (both derived from `stg_events_app_speedy`)
2. **events_acquisition ↔ course_data** — joined on `container_id` OR `course_id IN UNNEST(c.course_id)` (extracted IDs from landing URL matched to course dimension)
3. **events_acquisition ↔ badge_data** — joined on `badge_id` (extracted from landing URL)
4. **events_acquisition ↔ channel_data** — joined on `landing_path = cd.link` (with a special-case rewrite for `/channels/meta-quest` → `/channels/meta-horizon`)
5. **events_acquisition ↔ auth_data** — joined on `sub_session_id`
6. **events_acquisition ↔ user_course_data** — joined on `user_id` + `DATE(created) = DATE(course_started)` + container overlap across course/badge/channel dimensions + `rn = 1` (first course per user per day)
7. **events_acquisition ↔ went_to_docs** — joined on `sub_session_id`
8. **Final join to stg_clients** — joined on client IDs found in any of course_data, badge_data, or channel_data, excluding client_id = 1

Key indirect relationships:
- Course → client: `stg_courses` → `stg_clients_course_containers` (on `container_id = course_container_id`) → `stg_clients`
- Badge → client: `stg_learning_paths` → `learning_path_course_containers` → `clients_course_containers` → `stg_clients`
- Channel → client: `stg_categories` → `stg_clients` (on `LOWER(name)`) → `stg_clients_course_containers`

## Step 3 — Identify aggregation depth

There are multiple independent single-level aggregations feeding a final assembly join with one more aggregation for client arrays. Arrow notation for each CTE:

**events_acquisition:**
```
session_id, sub_session_id, MAX(user_id) as user_id, MIN_BY(url_path, created) as landing_path, DATETIME_DIFF(...) as time_on_page, MAX(scroll_value) as max_scroll_depth, COUNTIF(...) as landing_page_click_count, REGEXP_EXTRACT(MIN_BY(url_path, created)) as course_id/container_id/badge_id, UTM extractions, MIN(created) as created
```

**subsession_data:**
```
sub_session_id, COUNT(DISTINCT url_path) as page_views, DATETIME_DIFF(...) as session_duration, COUNTIF(click) as session_click_count, CASE(...) as engaged
```

**went_to_docs:**
```
sub_session_id, MIN_BY(app, created) = 'docs' as started_at_docs, MIN(app) = 'docs' as went_to_docs
```

**course_data:**
```
container_id, MAX_BY(name, id) as course_name, ARRAY_AGG(id) as course_id, ARRAY_AGG(DISTINCT client_id) as client_id, ARRAY_AGG(DISTINCT client_name) as client_name
```

**badge_data:**
```
badge_id, badge_name, ARRAY_AGG(DISTINCT course_container_id) as container_ids, ARRAY_AGG(DISTINCT client_id) as client_id, ARRAY_AGG(DISTINCT client_name) as client_name
```

**channel_data:**
```
link, client_name, ARRAY_AGG(course_container_id) as channel_container_ids
```

**auth_data:**
```
session_id, sub_session_id, MAX_BY(type, created) as auth_type, CASE(...) as auth_action
```

**user_course_data (two levels):**
```
user_id, course_name, container_id, course_id, course_status, course_progress_percentage, course_started, ARRAY_AGG(DISTINCT client_id) as client_ids
-> *, ROW_NUMBER() OVER (PARTITION BY user_id, DATE(course_started) ORDER BY course_started) as rn
```

**Final assembly:**
```
[all CTEs joined on events_acquisition spine]
session_id, sub_session_id, ..., MAX(started_at_docs), MAX(went_to_docs), ARRAY_AGG(clients.id) as client_id, ARRAY_AGG(clients.name) as client_name
```

## Step 4 — Identify dependencies

**Grouping drivers:**
- `session_id, sub_session_id` — the grain of `events_acquisition` and the final output
- `sub_session_id` — the grain of `subsession_data`, `went_to_docs`, `auth_data`
- `container_id` — the grain of `course_data`
- `badge_id` — the grain of `badge_data`
- `link` — the grain of `channel_data`
- `user_id, course_started` — the grain of `user_course_data` (after rn=1 filter)

**Derived columns:**
- `landing_path` is derived in `events_acquisition` via `MIN_BY(url_path, created)` and then drives which dimension CTE (course/badge/channel) matches
- `course_id`, `container_id`, `badge_id` are regex-extracted from `landing_path` inside `events_acquisition`
- `name` and `type` in the final SELECT are derived from which dimension CTE matched (COALESCE logic)
- `client_id` and `client_name` arrays in final output are assembled from the `stg_clients` join, which depends on all three dimension CTEs

**Intermediate steps required:**
- `user_course_data` needs an inner aggregation to collect client_ids, then a window function for row numbering — hence the subquery + ROW_NUMBER pattern
- All dimension CTEs must be pre-aggregated before the final join to avoid fan-out from their source joins (courses have multiple clients, badges have multiple containers, etc.)

## Step 5 — Plan CTE layers bottom-up

1. **`events_acquisition`** — Aggregates raw events to one row per `(session_id, sub_session_id)`, computing landing page, engagement metrics, URL-extracted entity IDs, and UTM parameters.

2. **`subsession_data`** — Aggregates raw events to one row per `sub_session_id` for session-level metrics (page views, duration, click count, engaged flag).

3. **`went_to_docs`** — Aggregates the broader events table (not app-only) per `sub_session_id` to determine whether the user started at or visited docs.

4. **`course_data`** — Joins courses → client_course_containers → clients, grouped by `container_id`, to produce course name and associated client arrays.

5. **`badge_data`** — Joins learning_paths → learning_path_course_containers → clients_course_containers → clients, grouped by `badge_id`, to produce badge name, container list, and client arrays.

6. **`channel_data`** — Joins categories → clients → client_course_containers, grouped by `link`, to produce channel name and container list.

7. **`auth_data`** — Aggregates login events joined to users per `(session_id, sub_session_id)` to determine auth type and whether the action was a registration or login.

8. **`user_course_data`** — Aggregates user course records with client IDs, then applies `ROW_NUMBER` partitioned by `(user_id, DATE(course_started))` to isolate the first course started per user per day.

9. **Final SELECT** — Joins all CTEs onto the `events_acquisition` spine via LEFT JOINs, computes `name`/`type` from COALESCE across dimensions, aggregates client arrays with a default fallback to `[1] / ['Data Protocol']`.

## Step 6 — Write each CTE with justification

**`events_acquisition`** — Pre-aggregates the raw event stream to the sub-session grain so the final query operates on one row per landing page visit rather than one row per event. This is the spine of the output.

**`subsession_data`** — Computes session-level engagement metrics (page views, duration, click count, bounce detection) separately from the landing-page-focused acquisition CTE, because these metrics span all events in the sub-session, not just the first page.

**`went_to_docs`** — Uses the broader `stg_events_speedy` table (not just app events) to detect whether the sub-session touched the docs app, since docs visits may not appear in the app-specific event stream.

**`course_data`** — Resolves the many-to-many relationship between courses and clients through the `course_containers` junction table, pre-aggregating to one row per `container_id` to avoid fan-out when joined to the acquisition spine.

**`badge_data`** — Resolves the indirect badge → course_container → client relationship chain, pre-aggregating to one row per `badge_id` with container and client arrays.

**`channel_data`** — Maps channel landing paths (from `stg_categories.link`) to their associated course containers and client names, using the name-based join between categories and clients.

**`auth_data`** — Isolates login/registration events and determines whether the authentication was a login or registration (including the special-case MetaQuest timestamp proximity check against user creation time).

**`user_course_data`** — Prepares user course enrollment data with a row-number ranking so the final join can pick only the first course a user started on the same day as their landing page visit.

## Step 7 — Compose the final query

```sql
WITH
  events_acquisition AS (
  SELECT
    e.session_id,
    e.sub_session_id,
    MAX(e.user_id) AS user_id,
    MAX(e.current_user_id) AS current_user_id,
    MIN_BY(e.url_path, e.created) AS landing_path,
    DATETIME_DIFF(MAX(e.created), MIN(e.created), SECOND) AS time_on_page,
    MAX(e.scroll_value) AS max_scroll_depth,
    MAX(e.scroll_num) AS scroll_events_count,
    COUNTIF(e.type = "click"
      AND e.name != "any_click") AS landing_page_click_count,
    SAFE_CAST(REGEXP_EXTRACT( MIN_BY(e.url_path, e.created), r'/(?:courses|short-codes)/([^/]+)') AS int64) AS course_id,
    SAFE_CAST(REGEXP_EXTRACT( MIN_BY(e.url_path, e.created), r'/container/([^/]+)') AS int64) AS container_id,
    SAFE_CAST(REGEXP_EXTRACT( MIN_BY(e.url_path, e.created), r'/badges/([^/]+)') AS int64) AS badge_id,
    MIN_BY(e.referrer, e.created) AS referrer,
    LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, created), r'utm_source=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_source,
    LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, created), r'utm_medium=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_medium,
    LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, created), r'utm_campaign=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_campaign,
    LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, created), r'utm_content=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_content,
    LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, created), r'utm_term=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_term,
    MAX(e.landed_from) as landed_from,
    MIN_BY(e.page_visit, e.created) AS page_visit,
    MIN(e.created) AS created
  FROM
    ${ref("stg_events_app_speedy")} e
  GROUP BY
    e.session_id,
    e.sub_session_id),
  subsession_data AS (
  SELECT
    sub_session_id,
    COUNT(DISTINCT url_path) AS page_views,
    DATETIME_DIFF(MAX(created), MIN(created), SECOND) AS session_duration,
    COUNTIF(type = "click") AS session_click_count,
    CASE
      WHEN DATETIME_DIFF(MAX(created), MIN(created), SECOND) < 30 AND COUNT(DISTINCT url_path) = 1 AND COUNTIF(type = "click") = 0 THEN FALSE
    -- Bounce criteria
      ELSE TRUE
    -- Engaged if not a bounce
  END
    AS engaged,
  FROM
    ${ref("stg_events_app_speedy")}
  GROUP BY
    sub_session_id),
  went_to_docs AS (
  SELECT
    e.sub_session_id,
    MIN_BY(e.app, e.created) = 'docs' AS started_at_docs,
    MIN(e.app) = 'docs' AS went_to_docs
  FROM
    ${ref("stg_events_speedy")} AS e
  GROUP BY
    e.sub_session_id),
  course_data AS (
  SELECT
    MAX_BY(c.name, c.id) AS course_name,
    ARRAY_AGG(c.id) AS course_id,
    c.container_id AS container_id,
    ARRAY_AGG(DISTINCT ccc.client_id) AS client_id,
    ARRAY_AGG(DISTINCT clients.name) AS client_name
  FROM
    ${ref("stg_clients_course_containers")} ccc
  JOIN
    ${ref("stg_clients")} clients
  ON
    clients.id = ccc.client_id
  JOIN
    ${ref("stg_courses")} c
  ON
    c.container_id = ccc.course_container_id
  GROUP BY
    c.container_id),
  badge_data AS (
  SELECT
    lp.id AS badge_id,
    lp.name AS badge_name,
    ARRAY_AGG(DISTINCT lpcc.course_container_id) AS container_ids,
    ARRAY_AGG(DISTINCT ccc.client_id) AS client_id,
    ARRAY_AGG(DISTINCT clients.name) AS client_name
  FROM
    ${ref("stg_learning_paths")} lp
  JOIN
    ${ref("learning_path_course_containers")} lpcc
  ON
    lpcc.learning_path_id = lp.id
  JOIN
    ${ref("clients_course_containers")} ccc
  ON
    ccc.course_container_id = lpcc.course_container_id
  JOIN
    ${ref("stg_clients")} clients
  ON
    clients.id = ccc.client_id
  GROUP BY
    lp.id,
    lp.name),
  channel_data AS (
  SELECT
    cats.link,
    cats.name AS client_name,
    ARRAY_AGG(ccc.course_container_id) AS channel_container_ids
  FROM
    ${ref("stg_categories")} cats
  JOIN
    ${ref("stg_clients")} clients
  ON
    LOWER(clients.name) = cats.name
  JOIN
    ${ref("stg_clients_course_containers")} ccc
  ON
    ccc.client_id = clients.id
  WHERE
    link IS NOT NULL
  GROUP BY
    cats.link,
    cats.name),
  auth_data AS (
  SELECT
    e.session_id,
    e.sub_session_id,
    MAX_BY(e.type, e.created) AS auth_type,
    CASE
      WHEN MAX_BY(e.action, e.created) LIKE '%register%' THEN 'register'
      WHEN MAX_BY(e.type, e.created) = 'metaquest'
    AND ABS(TIMESTAMP_DIFF(MIN(e.created), MIN(u.created), SECOND)) <= 5 THEN 'register'
      ELSE 'login'
  END
    AS auth_action
  FROM
    ${ref("stg_events_app_speedy")} AS e
  JOIN
    ${ref("stg_users")} AS u
  ON
    u.id = e.user_id
  WHERE
    e.name = "login"
  GROUP BY
    e.session_id,
    e.sub_session_id),
  user_course_data AS (
  SELECT
    *,
    ROW_NUMBER() OVER (PARTITION BY user_id, DATE (course_started)
    ORDER BY
      course_started ASC ) AS rn
  FROM (
    SELECT
      c.user_id,
      c.course_name,
      c.container_id,
      c.course_id,
      c.course_status,
      c.course_progress_percentage,
      c.course_started,
      ARRAY_AGG(DISTINCT CCC.client_id IGNORE NULLS) AS client_ids
    FROM
      ${ref("reporting_user_courses")} c
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    GROUP BY
      user_id,
      course_name,
      container_id,
      course_id,
      course_status,
      course_progress_percentage,
      course_started))
SELECT
  e.session_id,
  e.sub_session_id,
  e.user_id,
  e.current_user_id,
  e.landing_path,
  COALESCE(c.course_name, bd.badge_name, CONCAT(INITCAP(cd.client_name), ' Channel')) AS name,
  CASE
    WHEN landing_path LIKE '/courses/%' OR landing_path LIKE '/short-codes/%' OR landing_path LIKE '/container/%' THEN 'Course'
    WHEN landing_path LIKE '/channels/%' THEN 'Channel'
    WHEN landing_path LIKE '/badges/%' THEN 'Badge'
END
  AS type,
  e.time_on_page,
  e.landing_page_click_count,
  ad.auth_action,
  ad.auth_type,
  COALESCE(MAX(wtd.started_at_docs), FALSE) AS started_at_docs,
  COALESCE(MAX(wtd.went_to_docs), FALSE) AS went_to_docs,
  sd.page_views,
  sd.session_duration,
  sd.session_click_count,
  sd.engaged,
  ucd.course_name,
  ucd.container_id,
  ucd.course_id,
  ucd.course_status,
  ucd.course_progress_percentage,
  ucd.course_started,
  e.utm_source,
  e.utm_medium,
  e.utm_campaign,
  e.utm_content,
  e.referrer,
  e.landed_from,
  e.created,
  COALESCE(ARRAY_CONCAT(ARRAY_AGG(clients.id IGNORE NULLS), [1]), [1]) AS client_id,
  COALESCE(ARRAY_CONCAT(ARRAY_AGG(clients.name IGNORE NULLS), ['Data Protocol']), ['Data Protocol']) AS client_name
FROM
  events_acquisition e
LEFT JOIN
  course_data c
ON
  e.container_id = c.container_id
  OR e.course_id IN UNNEST(c.course_id)
LEFT JOIN
  badge_data bd
ON
  bd.badge_id = e.badge_id
LEFT JOIN
  channel_data cd
ON
  cd.link =
  CASE
    WHEN e.landing_path = "/channels/meta-quest" THEN "/channels/meta-horizon"
    ELSE e.landing_path
END
LEFT JOIN
  auth_data ad
ON
  ad.sub_session_id = e.sub_session_id
LEFT JOIN
  ${ref("stg_clients")} clients
ON
  (LOWER(clients.name) = cd.client_name
    OR clients.id IN UNNEST(bd.client_id)
    OR clients.id IN UNNEST(c.client_id))
  AND NOT clients.id = 1
LEFT JOIN
  subsession_data AS sd
ON
  sd.sub_session_id = e.sub_session_id
LEFT JOIN
  user_course_data ucd
ON
  e.user_id = ucd.user_id
  AND DATE(e.created) = DATE(ucd.course_started)
  AND (c.container_id = ucd.container_id
    OR ucd.container_id IN UNNEST(bd.container_ids)
    OR ucd.container_id IN UNNEST(cd.channel_container_ids))
  AND ucd.rn = 1
  -- Ensures only the first course per user per day
LEFT JOIN
  went_to_docs AS wtd
ON
  wtd.sub_session_id = e.sub_session_id
GROUP BY
  e.session_id,
  e.sub_session_id,
  e.user_id,
  e.current_user_id,
  e.landing_path,
  e.time_on_page,
  e.landing_page_click_count,
  COALESCE(c.course_name, bd.badge_name, CONCAT(INITCAP(cd.client_name), ' Channel')),
  ad.auth_action,
  ad.auth_type,
  sd.page_views,
  sd.session_duration,
  sd.session_click_count,
  sd.engaged,
  ucd.course_name,
  ucd.container_id,
  ucd.course_id,
  ucd.course_status,
  ucd.course_progress_percentage,
  ucd.course_started,
  e.utm_source,
  e.utm_medium,
  e.utm_campaign,
  e.utm_content,
  e.referrer,
  e.landed_from,
  e.created
```
