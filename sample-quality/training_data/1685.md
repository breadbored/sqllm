## Question
Identify subsessions where users completed a lesson, and tag them with the completion timestamp—but only include subsessions where the lesson completion actually occurred during the subsession's event time window.

--- AGENT ---

This query correlates lesson completion events with app subsessions. It joins the events table (base_buckets_events_app_speedy) with the lessons completion table (stg_user_finished_course_lessons) on user_id, then groups by subsession, app, and URL path along with the lesson completion timestamp. The HAVING clause filters to keep only groups where the lesson completion time (`ufcl.created`) falls within the time range of events in that subsession—that is, between the earliest and latest event timestamp in the group. The result is a tagged behavior dataset marking which subsessions contained events correlated with lesson completion.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  'finished_a_lesson' AS tag,
  'behavior' as class,
  es.app AS app,
  es.url_path AS url_path,
  ufcl.created AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS es
  JOIN ${ref("stg_user_finished_course_lessons")} AS ufcl ON es.user_id = ufcl.user_id
GROUP BY
  es.sub_session_id,
  es.app,
  es.url_path,
  ufcl.created
HAVING
  ufcl.created BETWEEN MIN(es.created) AND MAX(es.created)
```
