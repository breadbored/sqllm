## Question
Generate a query that summarizes assessment performance metrics for a specific course container within a client account, including user counts, attempt counts, average scores, and how many users passed.

--- AGENT ---

This query is a table function that returns assessment summary statistics for a single container within a single client. It joins the assessment records to courses, users, and the client-container mapping table, then filters to the specified `clientId` and `containerId`. 

The output contains four aggregated columns:
- `users`: the count of distinct users who attempted assessments in this container
- `attempts`: the total count of assessment attempts (across all users)
- `avg_score`: the mean score rounded to the nearest integer
- `users_passed`: the count of distinct users whose `passed` field equals 1

The aggregation is grouped by `container_id`, so the function returns one row per container.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    COUNT(DISTINCT (uca.user_id)) AS users,
    COUNT(uca.id) AS attempts,
    ROUND(AVG(score)) AS avg_score,
    COUNT(DISTINCT
      CASE
        WHEN passed = 1 THEN uca.user_id
    END
      ) AS users_passed
  FROM
    ${ref("stg_user_course_assessments")} uca
  JOIN
    ${ref("stg_courses")} c
  ON
    c.id = uca.course_id
  JOIN
    ${ref("stg_users")} u
  ON
    u.id = uca.user_id
  JOIN
    ${ref("stg_clients_course_containers")} ccc
  ON
    ccc.course_container_id = c.container_id
  WHERE
    container_id = containerId
    AND client_id = clientId
  GROUP BY
    c.container_id )
```
