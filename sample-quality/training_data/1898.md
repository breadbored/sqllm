Can you provide me an example of how to enrich assessment question records with client ID information from user profiles?

--- AGENT ---

This query enriches user course assessment question records with client ID information. It builds a CTE that maps each user to an array containing client ID 1 plus their external_client_id (for users who have one). The main query then left-joins the assessment_questions table to this client mapping on user_id. For each assessment question record, it returns all original columns plus a client_ids array that either comes from the user's record (an array of [1, external_client_id]) or defaults to [1] if the user had no external_client_id or no matching user record was found.

The pattern is:
1. Create a CTE that pairs each user (where external_client_id is not null) with an array combining 1 and their external_client_id
2. Left join assessment questions to this CTE on user_id
3. Use COALESCE to default client_ids to [1] for non-matching rows

WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  _utb.id,
  _utb.user_id,
  _utb.course_id,
  _utb.assessment_id,
  _utb.question_id,
  _utb.user_course_assessment_id,
  _utb.completed,
  _utb.score,
  _utb.created,
  _utb.updated,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_course_assessment_questions")} as _utb
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = _utb.user_id
