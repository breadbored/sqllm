Show me VaaS events grouped by session segment, categorized by whether they have a valid external ID, along with the timestamp of the first event in each group.

--- AGENT ---

This query identifies and validates external identity information from VaaS event metadata, then groups events by session segment and authentication status.

The first CTE (`with_external_id`) extracts the `externalId` field from the JSON `action_meta` column and validates it against two known ID formats: integers greater than zero, or strings matching the UUID v1-5 pattern. IDs that match neither pattern are marked as NULL.

The main query then groups these events by `sub_session_id`, `app`, `url_path`, and authentication status (encoded in the `tag` column as either 'vaas_externally_authed' or 'vaas_externally_unauthed'). For each group, it captures the earliest event timestamp and assigns a fixed behavior classification.

The output shape is: one row per unique combination of session segment, authentication status, app, and URL path, with the first timestamp of events in that group.

```sql
WITH
with_external_id AS (
  SELECT
    e.*,
    CASE 
      -- INT ID
      WHEN SAFE_CAST(JSON_EXTRACT_SCALAR(e.action_meta, '$.externalId') AS INT64) > 0
        THEN
          JSON_EXTRACT_SCALAR(e.action_meta, '$.externalId')
      -- UUID
      WHEN REGEXP_CONTAINS(JSON_EXTRACT_SCALAR(e.action_meta, '$.externalId'), r'^[0-9a-f]{8}-[0-9a-f]{4}-[1-5][0-9a-f]{3}-[89abAB][0-9a-f]{3}-[0-9a-f]{12}$')
        THEN
          JSON_EXTRACT_SCALAR(e.action_meta, '$.externalId')
      -- Other ID types
      ELSE
        NULL
    END as external_id

  FROM
    ${ref("base_buckets_events_vaas_speedy")} as e
)
SELECT
  e.sub_session_id as sub_session_id,
  CONCAT('vaas_externally_', IF(e.external_id IS NOT NULL, 'authed', 'unauthed')) as tag,
  'behavior' as class,
  e.app as app,
  e.vaas_url_url_path as url_path,
  MIN(e.created) as dt

FROM
  with_external_id as e

GROUP BY
  sub_session_id,
  tag,
  app,
  url_path
```
