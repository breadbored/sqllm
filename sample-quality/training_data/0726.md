How do I match paths that pass through intermediate nodes with specific properties?

--- AGENT ---

The `MATCH` statement can specify intermediate node properties in a repeating path pattern. This example matches paths that go through one to two blocked accounts between source and destination.

```googlesql
GRAPH FinGraph
MATCH
  (src:Account)
  ((a:Account)-[:Transfers]->(b:Account {is_blocked:true}) WHERE a != b ){1,2}
  -[:Transfers]->(dst:Account)
RETURN src.id AS source_account_id, dst.id AS destination_account_id
```
