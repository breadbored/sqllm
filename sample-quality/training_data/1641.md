## Question
Show me a BigQuery example that joins events to both course-level and lesson-level resources, handling two different event schema formats.

--- AGENT ---

This query enriches event records by joining them to their associated resources (either course resources or lesson resources), with explicit handling for schema evolution.

**First branch (Course Resources):** Joins events to course_resources where the event's `related_id` matches the resource's id, the `related_entity` is either `'resources'` (backward-compatible format) or `'course_resource'` (new format from July 2024), and the event's `course_id` matches the resource's `course_id`. Returns the event_id, course_resource_id, and NULL for lesson_resource_id.

**Second branch (Lesson Resources):** Joins events to lesson_resources where the event's `related_id` matches the resource's id. For backward-compatible events, it extracts `lesson_id` from JSON stored in the `action_meta` column and casts it to int64, then matches against the lesson_resource's lesson_id. For newer events, it matches on `related_entity = 'lesson_resource'`. Returns the event_id, NULL for course_resource_id, and lesson_resource_id.

**Final query:** The joined_resources CTE (which now contains event_ids paired with their resource IDs) is joined back to the original events table on event_id, returning all event columns enriched with the course_resource_id and lesson_resource_id columns.

```sql
WITH joined_resources AS (
  -- Course Resources
  SELECT
    e.id  AS event_id,
    cr.id AS course_resource_id,
    NULL  AS lesson_resource_id
  FROM
    ${ref("stg_events_app_speedy")} AS e
  INNER JOIN
    ${ref("course_resources")} AS cr
    ON
      e.related_id = cr.id
      AND
      (
      -- Backwards compatibility
        (
          e.related_entity = 'resources'
          AND
          e.course_id = cr.course_id
        )
        OR
        -- From event changes in July 2024
        e.related_entity = 'course_resource'
      )
      AND
      e.course_id = cr.course_id

  UNION ALL

  -- Lesson Resources
  SELECT
    e.id  AS event_id,
    NULL  AS course_resource_id,
    lr.id AS lesson_resource_id
  FROM
    ${ref("stg_events_app_speedy")} AS e
  INNER JOIN
    ${ref("stg_lesson_resources")} AS lr
    ON
      e.related_id = lr.id
      AND
      (
      -- Backwards compatibility
        (
          e.related_entity = 'resources'
          AND
          lr.lesson_id = safe_cast(json_extract_scalar(e.action_meta, '$.lesson_id') AS int64)
        )
        OR
        -- From event changes in July 2024
        e.related_entity = 'lesson_resource'
      )
)

SELECT
  e.*,
  jr.course_resource_id,
  jr.lesson_resource_id
FROM
  joined_resources AS jr
INNER JOIN
  ${ref("stg_events_app_speedy")} AS e
  ON
    jr.event_id = e.id
```
