How can I identify and tag click events related to resources for behavioral analysis?

--- AGENT ---

This query identifies click events that involve resources (where `related_entity` contains the substring 'resource') and tags them for classification as 'resource_engager' behavior events. It groups these click events by sub_session_id, app, url_path, and creation timestamp, collapsing duplicate clicks on the same resource path within a session. The `HAVING COUNT(*) >= 1` clause has no filtering effect (any grouped result will have at least one row), so all qualifying click events are returned with the added classification tags.

```sql
SELECT
  es.sub_session_id,
  'resource_engager' as tag,
  'behavior' as class,
  es.app as app,
  es.url_path as url_path,
  es.created as dt
FROM
  ${ref("base_buckets_events_app_speedy")} as es
WHERE
  es.related_entity LIKE '%resource%'
AND
  es.type = 'click'
GROUP BY
  es.sub_session_id,
  es.app,
  es.url_path,
  es.created
HAVING
  COUNT(*) >= 1
```
