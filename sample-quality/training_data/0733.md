What happens if ORDER BY is not immediately followed by LIMIT or OFFSET?

--- AGENT ---

When `ORDER BY` is not immediately followed by `LIMIT` or `OFFSET`, the sort is discarded and results are returned unordered. This example shows a warning case where the sort has no effect because no limiting clause follows.

```googlesql
-- Warning: The transfer.amount values aren't sorted because the
-- LIMIT statement is missing.
GRAPH FinGraph
MATCH (src_account:Account)-[transfer:Transfers]->(dst_account:Account)
ORDER BY transfer.amount DESC
RETURN src_account.id AS account_id, transfer.amount AS transfer_amount
```
