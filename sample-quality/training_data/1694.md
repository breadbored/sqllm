--- USER ---

Identify sub-sessions that never had a logged-in event, and tag them with their entry point details.

--- AGENT ---

This query identifies and tags user sub-sessions that never reached a logged-in state. It uses a NOT IN subquery to exclude all `sub_session_id` values that have at least one event where `current_logged_in = TRUE`, then groups the remaining events by sub_session_id.

For each never-logged-in sub_session, it returns:
- The sub_session_id and constant classifiers ('never_logged_in' tag, 'behavior' class)
- The app and url_path from the earliest event in that sub_session (via MIN_BY ordered on the created timestamp)
- The timestamp of that earliest event (MIN of created)

The result is a set of behavioral tags marking sub-sessions that never authenticated, with metadata about where they first appeared.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  'never_logged_in' as tag,
  'behavior' as class,
  MIN_BY(e.app, e.created) as app,
  MIN_BY(e.url_path, e.created) as url_path,
  MIN(e.created) as dt
FROM
  ${ref('base_buckets_events_app_speedy')} as e
WHERE
  e.sub_session_id NOT IN (
    SELECT
      DISTINCT _e.sub_session_id
    FROM
      ${ref('base_buckets_events_app_speedy')} as _e
    WHERE
      _e.sub_session_id IS NOT NULL
    AND
      _e.current_logged_in = TRUE
  )
GROUP BY
  sub_session_id
```
