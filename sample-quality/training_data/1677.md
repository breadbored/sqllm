Show me how to extract the earliest event attributes grouped by session and operating system in BigQuery.

--- AGENT ---

This query extracts one record per unique sub_session_id and operating system combination, capturing the earliest event's identifying information for each group.

The logic:
1. Groups events by `sub_session_id` and a derived tag field that concatenates 'os_name_' with the operating system name
2. For each group, returns:
   - The `sub_session_id` identifier
   - The constructed `tag` string (e.g., 'os_name_iOS')
   - A constant class label 'attribute'
   - The `app` and `url_path` values from the chronologically first event, using `MIN_BY()` ordered by `created` timestamp
   - The earliest `created` timestamp in the group

The `MIN_BY()` function is the key aggregator here — it returns the value of its first argument from whichever row has the minimum value in its second argument. This extracts the app and url_path from the earliest event in each session-OS combination while preserving temporal ordering information.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  CONCAT('os_name_', es.os_name) as tag,
  'attribute' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref('base_buckets_events_app_speedy')} as es 

GROUP BY
  sub_session_id,
  tag
```
