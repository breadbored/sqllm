## Question
Show me the total watch time and course time aggregated by month for a specific client and container, using a table function.

--- AGENT ---

This query creates a table function parameterized by a `clientId` and `containerId`. It aggregates engagement metrics from the `reporting_user_courses` table by formatting the `course_started` date into month-year format (e.g., "Jan 26").

The function filters rows where:
- The parameter `clientId` matches a value in the `client_id` column (which is an array, unpacked via UNNEST)
- The `container_id` column equals the parameter `containerId`

It then aggregates two metrics by month:
- `watch_time`: the sum of `course_progress_minutes` divided by 60 to convert to hours, coalesced to 0 if null, rounded to 1 decimal place
- `course_time`: the sum of `time_on_course_minutes` divided by 60 to convert to hours, coalesced to 0 if null, rounded to 1 decimal place

The result is ordered chronologically by parsing the formatted date strings back to actual date values for proper month sequencing.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    FORMAT_DATE('%b %y', course_started) AS date,
    ROUND(IFNULL(SUM(course_progress_minutes), 0) / 60.0, 1) AS watch_time,
    ROUND(IFNULL(SUM(time_on_course_minutes), 0) / 60.0, 1) AS course_time
  FROM
    ${ref("reporting_user_courses")}
  WHERE
    clientId IN UNNEST(client_id)
    AND container_id = containerId
  GROUP BY
    date
  ORDER BY
    PARSE_DATE('%b %y', date))
```
