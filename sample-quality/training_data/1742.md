--- AGENT ---

This query enriches raw event data with human-readable descriptions by mapping specific combinations of event attributes to business-friendly text labels. The core logic is a large nested CASE statement that examines combinations of `e.name` (event name), `e.type` (event type), `e.action` (action identifier), and `e.app` (application context) to produce a `text_description` column that explains what user action occurred.

For example:
- An event with `name = 'page_view'` and `type = 'view'` becomes "User visited page"
- An event with `name = 'login'`, `type = 'google'`, and `action = 'oauth'` becomes "User signed in via Google"
- An event with `name = 'scroll'`, `type = 'page-scroll'`, and `action = 'scroll_25'` becomes "User scrolled to 25% of the page height"

The query handles dozens of event patterns across navigation, user interactions, form submissions, video playback, authentication, and system events. When an event combination doesn't match any predefined pattern, the ELSE clause returns a concatenated string of the raw attributes.

The GROUP BY clause groups by all selected columns (the computed `text_description` along with the original `id`, `name`, `type`, `action`, and `app` fields), effectively deduplicating identical combinations while retaining all source data.
