--- USER ---

Retrieve all client certification records, including a single-element array containing the client ID for each row.

--- AGENT ---

## Step 1: Identify shapes

**Input shape:**
A single table `clients_certifications` with columns: `id`, `certification_id`, `client_id`, `created`, `updated`.

**Target shape:**
`id`, `certification_id`, `client_id`, `created`, `updated`, `client_ids` (an array containing the row's own `client_id`).

## Step 2: Identify sources and relationships

There is only one table involved: `clients_certifications`. No joins are required.

The `ARRAY(SELECT client_id)` is a correlated scalar subquery that wraps the current row's `client_id` into a single-element array. This is not a join — it is a per-row transformation.

## Step 3: Identify aggregation depth

There is no aggregation in this query. It is a 1:1 row-preserving SELECT with a per-row array construction.

Single-level shape (no grouping, no reduction):

```
id, certification_id, client_id, created, updated, ARRAY(client_id) as client_ids
```

No arrows — no aggregation levels.

## Step 4: Identify dependencies

- All output columns are direct columns from the source table except `client_ids`.
- `client_ids` is derived from `client_id` via `ARRAY(SELECT client_id)`, which in BigQuery produces a single-element array containing that row's `client_id` value.
- No intermediate steps are required.

## Step 5: Plan CTE layers bottom-up

No CTEs are needed. The query is a single pass over one table with no aggregation and no joins.

## Step 6: Write each CTE with justification

No CTEs to write — the transformation is expressible in a single SELECT.

## Step 7: Compose the final query

```sql
SELECT
  id,
  certification_id,
  client_id,
  created,
  updated,
  ARRAY(SELECT client_id) as client_ids,
FROM
  ${ref("clients_certifications")}
```

## Step 8: Verify output shape

Target shape: `id`, `certification_id`, `client_id`, `created`, `updated`, `client_ids`.

Final SELECT produces: `id`, `certification_id`, `client_id`, `created`, `updated`, `client_ids`.

Output shape matches. Each row in the source table produces exactly one row in the output, with `client_ids` as a single-element `ARRAY<INT64>` (or whatever the type of `client_id` is) containing that row's `client_id`.
