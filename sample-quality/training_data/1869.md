--- USER ---

Show me a query that enriches a documents table with an array of associated client IDs, handling cases where documents may have no clients.

--- AGENT ---

This query retrieves all documents along with an array of client IDs associated with each document.

The CTE `doc_clients` performs a many-to-one aggregation: it groups the `documents_clients` table by `doc_id` and collects all distinct non-null client IDs for each document into an array using `ARRAY_AGG(DISTINCT ... IGNORE NULLS)`.

The main query then left-joins the documents table to this aggregated structure. Every row from documents is preserved (via LEFT JOIN). For documents that have associated clients in the join result, the `client_ids` column contains the aggregated array. For documents with no matches in `doc_clients`, the `COALESCE` expression defaults the array to `[1]`.

The output is a flattened document record set with all original document columns plus the enriched `client_ids` array.

```sql
WITH
doc_clients AS (
  SELECT
    dc.doc_id as doc_id,
    ARRAY_AGG(DISTINCT dc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("documents_clients")} as dc
  GROUP BY
    dc.doc_id
)
SELECT
  d.id,
  d.name,
  d.path,
  d.type,
  d.description,
  d.image,
  d.created,
  d.updated,
  d.minutes_to_read,
  d.hidden,
  d.category_id,
  d.client_id,
  COALESCE(dc.client_ids, [1]) AS client_ids
FROM
  ${ref("documents")} as d
LEFT JOIN
  doc_clients as dc
ON
  dc.doc_id = d.id
```
