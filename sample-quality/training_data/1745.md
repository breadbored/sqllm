Enrich event data with subsession identifiers, event sequence numbers, and cumulative time measurements within each page visit.

--- AGENT ---

This query transforms raw events by adding three computed columns:

1. **`sub_session_id`**: Combines the session ID and segment ID into a single identifier for each subsession.

2. **`seconds_since_url_path`**: A running sum that accumulates the seconds elapsed within a given page visit. It adds `seconds_from_last` only when the current event's URL path matches the previous event's URL path AND `page_changed` is false; otherwise it contributes 0 to the sum. The window partitions by session, page_visit, and segment, ordered chronologically.

3. **`event_num`**: Assigns a sequential number to each event within its session, ordered by creation timestamp.

All other columns from the source table are retained with the `*` selector.

The query essentially enriches session event streams with subsession grouping, event ordering, and time-on-page calculations within URL boundaries.

```sql
/**
* Numbered Events
*
* This layer is used to number the events in a session, to give the event a subsession id, and to get the number of seconds since the last qualified event in the subsession.
**/
SELECT
  -- Subsession ID is a combination of the session ID and the segment ID
  CONCAT(e.session_id, '-', e.segment_id) AS sub_session_id,
  -- Page visit is a number that increments when the URL path changes
  SUM(
    CASE
      WHEN e.url_path = e.last_url_path
      AND NOT page_changed THEN e.seconds_from_last
      ELSE 0
    END
  ) OVER (
    PARTITION BY
      e.session_id,
      e.page_visit,
      e.segment_id
    ORDER BY
      e.created ASC
  ) AS seconds_since_url_path,
  -- Number the events in a session
  ROW_NUMBER() OVER (
    PARTITION BY
      e.session_id
    ORDER BY
      created ASC
  ) AS event_num,
  -- Everything else
  *
FROM
  ${ref("events_paths")} AS e
```
