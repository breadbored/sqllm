What happens if I use ERROR in a WHERE clause without controlling the evaluation order?

--- AGENT ---

This demonstrates incorrect usage of ERROR in a WHERE clause. Because GoogleSQL does not guarantee the evaluation order of AND conditions, the ERROR function may execute even when the first condition is false, making the result unpredictable.

```googlesql
SELECT *
FROM (SELECT -1 AS x)
WHERE x > 0 AND ERROR('Example error');
```
