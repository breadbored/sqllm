Enrich user knowledge check completion records with their associated client IDs, including both internal and external client identifiers.

--- AGENT ---

This query retrieves all records from the `user_finished_knowledge_checks` table and enriches each record with an array of client IDs associated to the user who completed the check.

The logic works as follows:

1. **CTE `user_clients`**: For each user with a non-null `external_client_id`, it creates an array containing two elements: the integer 1 (representing an internal/default client) and the user's external client ID.

2. **Main query**: It selects all columns from `user_finished_knowledge_checks` and left-joins to the `user_clients` CTE on `user_id`. 

3. **Client ID resolution**: The `COALESCE` expression returns the user's client ID array from the join if available, or defaults to `[1]` (internal client only) for users with no external client ID or those not found in the join.

The result is a flattened record set where each knowledge check completion row now carries the client IDs that should be associated with it, enabling multi-tenancy or client-aware filtering downstream.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  _utb.id,
  _utb.user_id,
  _utb.course_id,
  _utb.lesson_id,
  _utb.knowledge_check_id,
  _utb.created,
  _utb.updated,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_finished_knowledge_checks")} as _utb
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = _utb.user_id
```
