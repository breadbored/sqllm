## Question
Provide me with a BigQuery SQL query that aggregates event data by date, country, and application, while attributing events to courses or documents and enriching the output with ISO country code metadata.

--- AGENT ---

This query performs multi-step event attribution and aggregation:

**Event Data CTE (first level):**
The query starts by joining `all_events` to three dimension tables. It attempts to match events to courses through three independent strategies: a direct `derived_course_id` field, URL pattern extraction for course/short-code paths, and VaaS video ID matching via JSON metadata. It also joins to a documents table by matching a URL prefix pattern. The result is filtered to include only events that matched a course OR document, with additional business logic filters (course events must be from a "/learn" path with a valid client_id, OR the event must not be from the 'app' application). The CTE groups by date (day granularity), country, app, and client_id, counting distinct sessions and users at this grain.

**Main query (second level):**
The aggregation level groups across the client_id dimension — collapsing multiple client associations per country/date/app combination into an array while preserving the session and user counts from the first level. It enriches the output with ISO 3166-1 country codes and names by joining to a reference table, returning flags for whether the country code is recognized, the alpha-2 country code, the country name, the application, the date, the session count, the user count, and an array of all client_ids associated with that country/date/app combination.

**Original SQL:**

```sql
WITH
event_data AS (
  SELECT
    e.app,
    COALESCE(ccc.client_id, d.client_id) as client_id,
    DATE_TRUNC(DATE(e.created), DAY) AS min_date,
    UPPER(e.geo_country) AS country_code,
    COUNT(DISTINCT e.sub_session_id) AS sessions,
    COUNT(DISTINCT e.user_id) AS users
  FROM
    ${ref("all_events")} as e
  /**
   * APP & VAAS
   **/
  LEFT JOIN
    ${ref("stg_courses")} as c
  ON
    -- derived course_id can be for any implicitly known course attached to an event
    c.id = e.derived_course_id
  OR
    -- In App, you can get the course ID after `/(?:courses|short-codes)/`
    SAFE_CAST(REGEXP_EXTRACT(e.url_path, r'^/(?:courses|short-codes)/([0-9]+)', 1) AS INT64) = c.id
  OR
    -- In VaaS, you can get the vaas_id from the URL
    JSON_EXTRACT_SCALAR(e.action_meta, '$.video-id') = c.vaas_id
  LEFT JOIN
    ${ref("stg_clients_course_containers")} as ccc
  ON
    ccc.course_container_id = c.container_id
  /**
   * DOCS
   **/
  LEFT JOIN
    ${ref("stg_documents")} as d
  ON
    d.path = REGEXP_EXTRACT(e.url_path, r'^(/[a-zA-Z0-9\-]+/[a-zA-Z0-9\-]+)', 1)
  WHERE
    (c.id IS NOT NULL OR d.id IS NOT NULL)
  AND
    ((e.url_path LIKE '%/learn%'  AND ccc.client_id IS NOT NULL) OR e.app != 'app')
  GROUP BY
    DATE_TRUNC(DATE(e.created), DAY),
    e.geo_country,
    e.app,
    COALESCE(ccc.client_id, d.client_id)
)
SELECT
  COALESCE(country_codes.name) IS NOT NULL as valid_country,
  COALESCE(country_codes.alpha_2, '??') as country_code,
  COALESCE(country_codes.name, 'Unknown') as country_name,
  e.app,
  e.min_date AS first_date,
  e.sessions,
  e.users,
  ARRAY_AGG(e.client_id IGNORE NULLS) as client_ids
FROM
  event_data e
LEFT JOIN
  ${ref("stg_iso_3166_countries")} as country_codes
ON
  country_codes.alpha_2 = e.country_code
GROUP BY
  valid_country,
  country_code,
  country_name,
  e.app,
  e.min_date,
  e.sessions,
  e.users
```
