--- USER ---

For a given client and container, count the total number of sentiment question answers and the distinct number of users who answered them.

--- AGENT ---

This table function takes a client ID and container ID as parameters and returns aggregated sentiment question answer metrics for that scope. It joins the answer records with the courses table (to access container information), the users table (for user references), and the client-container mapping table (to ensure the container belongs to the specified client). The function filters to the specified client and container, then returns a single row containing the total count of answers and the count of distinct users who provided those answers.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    COUNT(ucsqa.id) AS answers,
    COUNT(DISTINCT(ucsqa.user_id)) AS users
  FROM
    ${ref("stg_user_course_sentiment_question_answers")} AS ucsqa
  JOIN
    ${ref("stg_courses")} c
  ON
    c.id = ucsqa.course_id
  JOIN
    ${ref("stg_users")} u
  ON
    u.id = ucsqa.user_id
  JOIN
    ${ref("stg_clients_course_containers")} ccc
  ON
    ccc.course_container_id = c.container_id
  WHERE
    container_id = containerId
    AND client_id = clientId
  GROUP BY
    c.container_id)
```
