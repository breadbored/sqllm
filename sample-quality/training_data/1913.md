## Question
For each landing page on the Horizon product, show daily conversion rates from the MH docs app to the docs/app platform, broken down by whether the user played a video during the session. Fill in any missing days in the date range so every page has a continuous daily time series.

--- AGENT ---

## 1. Identify shapes

**Input shape (single source table):**
`stg_events_speedy`: `sub_session_id, created, app, action_meta (JSON with $.visible, $.url_path, $.playing)`

**Target shape:**
`day, start_at_page, started_visible, base_conversion_perc, played_conversion_perc, not_played_conversion_perc, started_at_mh_docs__denominator, visited_both__base_numerator, visited_both_played__played_numerator, visited_both_not_played__not_played_numerator`

One row per day per landing page, with no gaps in the date series.

## 2. Identify sources and relationships

There is a single source table: `stg_events_speedy`. All columns are derived from it. The query constructs a date spine (`range_days` / `grouped_range_days`) from the data itself and left-joins aggregated metrics back onto it, ensuring every page × day combination appears even when there is no activity.

The join relationship between the date spine and the aggregated metrics is on `(day, start_at_page)`.

## 3. Identify aggregation depth

```
sub_session_id, day, MIN_BY(app) as start_at, MIN_BY(url_path) as start_at_page, ARRAY_AGG(app) as apps_at, ARRAY_AGG(playing) as play_states
-> day, start_at_page, COUNT(DISTINCT CASE ... sub_session_id) as [various conditional counts]
```

Plus a parallel date-spine construction:

```
start_at_page, MIN(day), MAX(day) -> GENERATE_DATE_ARRAY -> UNNEST -> start_at_page, day (one row per page per day)
```

Then a final join (no further aggregation — just a left join of the spine to the counts, with computed ratio columns).

## 4. Identify dependencies

**Grouping drivers:**
- `sub_session_id` + `day` define the grain of the `test` CTE (one row per sub-session per day)
- `day` + `start_at_page` define the grain of the `sums` CTE
- `start_at_page` + `day` define the grain of the date spine (`grouped_range_days`)

**Derived columns:**
- `start_at` (first app in the session by time) — drives "started at" conditional counts
- `start_at_page` (first URL path) — drives grouping and filtering
- `started_visible` (visibility state of first event) — carried through as metadata
- `apps_at` (array of all apps visited) — drives "visited" conditional counts via `IN UNNEST`
- `play_states` (array of playing states) — drives played/not-played segmentation via `IN UNNEST`
- All `sums` columns are conditional distinct counts of `sub_session_id`
- Final conversion percentages are ratios of those counts

**Intermediate steps required:**
- Session-level feature extraction must happen before day-page aggregation (can't conditionally count sub-sessions without first knowing each session's start app, pages visited, and play state)
- Date spine must be generated from the global min/max day range, crossed with all distinct pages
- The spine and the counts are independent and joined at the end

## 5. Plan CTE layers bottom-up

1. **`test`** — Collapse raw events to one row per sub-session per day, extracting session-level features: starting app, starting page, visibility, array of all apps visited, array of play states. Filter to Horizon pages only, excluding staging.

2. **`sums`** — Aggregate `test` to one row per day per landing page, computing conditional distinct counts of sub-sessions across multiple dimensions: starting app, visited apps, video play state, and cross-products of those.

3. **`range_days`** — Cross the global date range (min to max day across all sessions) with every distinct `(start_at_page, started_visible)` pair, producing a `GENERATE_DATE_ARRAY` per combination.

4. **`grouped_range_days`** — Unnest the date arrays from `range_days` to produce one row per page per day, carrying `started_visible` forward.

5. **Final SELECT** — Left join the date spine to `sums` on `(day, start_at_page)`, compute conversion percentages as ratios, and order by day.

## 6. Write each CTE with justification

### `test`
**Why:** Raw events are at the event grain — many rows per sub-session. We need session-level features (first app, first page, all apps visited, all play states) to classify each session before counting. The `HAVING` clause filters to sessions that started on a non-null Horizon page (URL path starts with `/horizon/` and excludes staging).

### `sums`
**Why:** Once sessions are characterized, we aggregate to the day × page grain, counting distinct sub-sessions under various conditional slices. This produces the numerators and denominators for conversion metrics. The conditions form a matrix: {started_at, visited} × {mh_docs, dp, both} × {base, played, not_played}.

### `range_days`
**Why:** Some pages may have no sessions on certain days. To produce a gapless time series, we generate the full date range from the global min/max day, crossed with every page. The `started_visible` attribute is resolved per page by taking the value from the earliest day (`MIN_BY(started_visible, day)`). The `ON 1 = 1` cross join ensures every page gets the full date range.

### `grouped_range_days`
**Why:** `range_days` contains arrays of dates. This CTE unnests them into individual rows — one per page per day — forming the left side of the final join. The `GROUP BY` with `MIN_BY` on `started_visible` deduplicates in case the cross join produced multiple rows per page-day.

## 7. Compose the final query

```sql
WITH
test AS (
  SELECT
    e.sub_session_id,
    DATETIME_TRUNC(e.created, DAY) AS day,
    MIN_BY(e.app, e.created) AS start_at,
    MIN_BY(JSON_EXTRACT_SCALAR(e.action_meta, '$.visible'), e.created) as started_visible,
    MIN_BY(JSON_EXTRACT_SCALAR(e.action_meta, '$.url_path'), e.created) AS start_at_page,
    (ARRAY_AGG(e.app)) AS apps_at,
    (ARRAY_AGG(JSON_EXTRACT_SCALAR(e.action_meta, '$.playing') IGNORE NULLS)) play_states
  FROM
    ${ref("stg_events_speedy")} AS e
  WHERE
    e.created >= '2025-05-16'
  GROUP BY
    e.sub_session_id,
    DATETIME_TRUNC(e.created, DAY)
  HAVING
    MIN_BY(JSON_EXTRACT_SCALAR(e.action_meta, '$.url_path'), e.created) IS NOT NULL
  AND
    MIN_BY(JSON_EXTRACT_SCALAR(e.action_meta, '$.url_path'), e.created) LIKE '/horizon/%'
  AND
    MIN_BY(JSON_EXTRACT_SCALAR(e.action_meta, '$.url_path'), e.created) NOT LIKE '%staging%'
),
sums AS (
  SELECT
    t.day AS day,
    t.start_at_page,
    COUNT(DISTINCT t.sub_session_id) AS subsessions,
    -- base
    COUNT(DISTINCT
      CASE
        WHEN t.start_at = 'vaas' THEN t.sub_session_id
        ELSE NULL
    END
      ) AS started_at_mh_docs,
    COUNT(DISTINCT
      CASE
        WHEN 'vaas' IN UNNEST(t.apps_at) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_mh_docs,
    COUNT(DISTINCT
      CASE
        WHEN t.start_at = 'docs' OR t.start_at = 'app' THEN t.sub_session_id
        ELSE NULL
    END
      ) AS started_at_dp,
    COUNT(DISTINCT
      CASE
        WHEN 'docs' IN UNNEST(t.apps_at) OR 'app' IN UNNEST(t.apps_at) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_dp,
    COUNT(DISTINCT
      CASE
        WHEN 'vaas' IN UNNEST(t.apps_at) AND ('docs' IN UNNEST(t.apps_at) OR 'app' IN UNNEST(t.apps_at)) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_both,
    COUNT(DISTINCT
      CASE
        WHEN (t.start_at = 'docs' OR t.start_at = 'app') AND 'vaas' IN UNNEST(t.apps_at) AND ('docs' IN UNNEST(t.apps_at) OR 'app' IN UNNEST(t.apps_at)) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_both_dp_start,
    COUNT(DISTINCT
      CASE
        WHEN (t.start_at = 'vaas') AND 'vaas' IN UNNEST(t.apps_at) AND ('docs' IN UNNEST(t.apps_at) OR 'app' IN UNNEST(t.apps_at)) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_both_mh_start,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' IN UNNEST(t.play_states) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS played_video,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' NOT IN UNNEST(t.play_states) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS didnt_play_video,
    -- played
    COUNT(DISTINCT
      CASE
        WHEN 'playing' IN UNNEST(t.play_states) AND t.start_at = 'vaas' THEN t.sub_session_id
        ELSE NULL
    END
      ) AS started_at_mh_docs_played,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' IN UNNEST(t.play_states) AND 'vaas' IN UNNEST(t.apps_at) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_mh_docs_played,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' IN UNNEST(t.play_states) AND t.start_at = 'docs' OR t.start_at = 'app' THEN t.sub_session_id
        ELSE NULL
    END
      ) AS started_at_dp_played,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' IN UNNEST(t.play_states) AND 'docs' IN UNNEST(t.apps_at) OR 'app' IN UNNEST(t.apps_at) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_dp_played,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' IN UNNEST(t.play_states) AND 'vaas' IN UNNEST(t.apps_at) AND ('docs' IN UNNEST(t.apps_at) OR 'app' IN UNNEST(t.apps_at)) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_both_played,
    -- didn't play
    COUNT(DISTINCT
      CASE
        WHEN 'playing' NOT IN UNNEST(t.play_states) AND t.start_at = 'vaas' THEN t.sub_session_id
        ELSE NULL
    END
      ) AS started_at_mh_docs_not_played,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' NOT IN UNNEST(t.play_states) AND 'vaas' IN UNNEST(t.apps_at) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_mh_docs_not_played,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' NOT IN UNNEST(t.play_states) AND t.start_at = 'docs' OR t.start_at = 'app' THEN t.sub_session_id
        ELSE NULL
    END
      ) AS started_at_dp_not_played,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' NOT IN UNNEST(t.play_states) AND 'docs' IN UNNEST(t.apps_at) OR 'app' IN UNNEST(t.apps_at) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_dp_not_played,
    COUNT(DISTINCT
      CASE
        WHEN 'playing' NOT IN UNNEST(t.play_states) AND 'vaas' IN UNNEST(t.apps_at) AND ('docs' IN UNNEST(t.apps_at) OR 'app' IN UNNEST(t.apps_at)) THEN t.sub_session_id
        ELSE NULL
    END
      ) AS visited_both_not_played,
  FROM
    test AS t
  GROUP BY
    t.day,
    t.start_at_page
), range_days AS (
    SELECT
        sp.start_at_page,
        sp.started_visible,
        GENERATE_DATE_ARRAY(MIN(DATE(t.day)), MAX(DATE(t.day))) as ranged
    FROM
        test as t
    JOIN
        (
            SELECT
                t.start_at_page,
                MIN_BY(t.started_visible, day) as started_visible,
            FROM
                test as t
            GROUP BY
                t.start_at_page
        ) as sp
    ON
        1 = 1
    GROUP BY
        sp.start_at_page,
        sp.started_visible
), grouped_range_days AS (
    SELECT
        rd.start_at_page,
        MIN_BY(rd.started_visible, r) as started_visible,
        r as day
    FROM
        range_days as rd, UNNEST(rd.ranged) as r
    GROUP BY
        rd.start_at_page,
        r
)
SELECT
  rd.day as day,
  rd.start_at_page,
  rd.started_visible,
  (s.visited_both / s.started_at_mh_docs) AS base_conversion_perc,
  (s.visited_both_played / s.started_at_mh_docs) AS played_conversion_perc,
  (s.visited_both_not_played / s.started_at_mh_docs) AS not_played_conversion_perc,
  s.started_at_mh_docs AS started_at_mh_docs__denominator,
  s.visited_both AS visited_both__base_numerator,
  s.visited_both_played AS visited_both_played__played_numerator,
  s.visited_both_not_played AS visited_both_not_played__not_played_numerator,
FROM
  grouped_range_days as rd
LEFT JOIN
  sums AS s
ON
  DATE(s.day) = DATE(rd.day)
AND
  s.start_at_page = rd.start_at_page
ORDER BY
  rd.day ASC
```
