Show me a SQL query that counts user knowledge check answers by month for a specific client and course container, including a running cumulative total.

--- AGENT ---

This is a table-valued function that takes a client ID and container ID as parameters. It counts the number of knowledge check answers submitted per month (formatted as 'Mon YY') within the given client and container scope, then adds a running cumulative sum of those monthly counts ordered chronologically.

The inner subquery joins four tables—user knowledge check answers with their associated users, courses, and client-to-container mappings—filters to the specified client and container via a WHERE clause, and groups by the formatted date to count answers per month. The outer query applies a window function (SUM OVER) to compute the cumulative total of monthly answers, preserving the monthly granularity in the final output. Results are ordered by date in ascending order.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    date,
    monthly_answers,
    SUM(monthly_answers) OVER (ORDER BY PARSE_DATE('%b %y', date) ASC) AS cumulative_answers
  FROM (
    SELECT
      FORMAT_DATE('%b %y', ukca.created) AS date,
      COUNT(ukca.id) AS monthly_answers
    FROM
      ${ref("stg_user_knowledge_check_answers")} ukca
    JOIN
      ${ref("stg_users")} u
    ON
      u.id = ukca.user_id
    JOIN
      ${ref("stg_courses")} c
    ON
      c.id = ukca.course_id
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    WHERE
      ccc.client_id = clientId
      AND container_id = containerId
    GROUP BY
      date) AS subquery
  ORDER BY
    PARSE_DATE('%b %y', date))
```
