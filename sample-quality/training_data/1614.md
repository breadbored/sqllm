How do I apply a window frame clause in the OVER clause while referencing a named window that already has PARTITION BY and ORDER BY?

--- AGENT ---

This query defines named windows where window `b` extends window `a` with ORDER BY. In the OVER clause, the frame clause is applied to `ItemWindow`, which references window `b`. This combines base specifications from named windows with frame logic defined in the OVER clause.

```googlesql
SELECT item, purchases, category, LAST_VALUE(item)
  OVER (ItemWindow ROWS BETWEEN 2 PRECEDING AND 2 FOLLOWING) AS most_popular
FROM Produce
WINDOW
  a AS (PARTITION BY category),
  b AS (a ORDER BY purchases),
  ItemWindow AS (b)
```
