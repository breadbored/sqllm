This is a BigQuery table function that generates a progress distribution report. It analyzes users' course progress across percentage buckets and produces three key metrics for each bucket:

1. **Bucket generation**: The query creates progress ranges (0%, 5%, 10%, ..., 95%, 100%) using `GENERATE_ARRAY` and UNNEST.

2. **Matching logic**: For each bucket, it counts how many users have reached that progress threshold by joining the generated buckets against the `reporting_user_courses` table. There is special handling for progress above 90%, where the threshold caps at 95%.

3. **Metrics calculated**:
   - `name`: A human-readable label ("Started" for 0%, or "{bucket}%" for others)
   - `value`: The count of users whose `course_progress_percentage` meets or exceeds that bucket's threshold
   - `minutes`: An estimated course duration in minutes, calculated as the maximum course length multiplied by the bucket percentage (e.g., if max course is 100 minutes and bucket is 50%, the estimate is 50 minutes)

The function filters results by a specific `clientId` and `containerId` passed as parameters, groups by bucket, and sorts ascending by bucket value.

```sql
CREATE OR REPLACE TABLE FUNCTION ${self()} (clientId INT64, containerId INT64) AS (
  SELECT
    CASE
      WHEN range_hack.bucket = 0 THEN 'Started'
      ELSE CONCAT(range_hack.bucket, '%')
    END AS name,
    COUNT(
      CASE
        WHEN uc.course_progress_percentage >= (
          (
            CASE
              WHEN range_hack.bucket > 90 THEN 95
              ELSE range_hack.bucket
            END
          ) / 100
        ) THEN 1
      END
    ) AS value,
    ROUND(
      MAX(uc.course_length_minutes) * (range_hack.bucket / 100),
      1
    ) AS minutes
  FROM
    (
      SELECT
        h AS bucket
      FROM
        UNNEST (GENERATE_ARRAY(0, 100, 5)) AS h
    ) AS range_hack
    JOIN ${ref("reporting_user_courses")} uc ON uc.course_progress_percentage >= (
      (
        CASE
          WHEN range_hack.bucket > 90 THEN 95
          ELSE range_hack.bucket
        END
      ) / 100
    )
  WHERE
    clientId IN (
      SELECT
        *
      FROM
        uc.client_id
    )
    AND container_id = containerId
  GROUP BY
    range_hack.bucket
  ORDER BY
    range_hack.bucket ASC
)
```
