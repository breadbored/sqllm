--- USER ---

Give me all lesson resources with their associated client IDs. A lesson resource belongs to whichever clients own the course container that contains the course that contains the lesson. If a lesson resource has no client association, default to client ID 1.

--- AGENT ---

## 1. Identify Shapes

**Input tables and columns (observable from query):**
- `clients_course_containers` (ccc): `client_id`, `course_container_id`
- `courses` (c): `id`, `container_id`
- `course_lessons` (cl): `course_id`, `lesson_id`
- `lessons` (l): `id`
- `lesson_resources` (lr): `id`, `name`, `url`, `lock_until_over`, `lesson_id`, `created`, `updated`, `order`, `file`, `creator_admin_id`, `updated_admin_id`

**Target shape:**
```
id, name, url, lock_until_over, lesson_id, created, updated, order, file, creator_admin_id, updated_admin_id, client_ids
```
One row per lesson resource, with a `client_ids` array attached.

## 2. Identify Sources and Relationships

The core entity is `lesson_resources`. The goal is to attach `client_ids` to each lesson resource.

There is no direct relationship between `lesson_resources` and `clients_course_containers`. The join chain resolves this indirectly:

```
clients_course_containers.course_container_id = courses.container_id
courses.id = course_lessons.course_id
course_lessons.lesson_id = lessons.id
```

This traces: **client → course container → course → course_lesson → lesson**.

The final join to `lesson_resources` is on `lesson_id`:
```
stg_lesson_resources.lesson_id = lesson_resources.lesson_id
```

The `lessons` table is joined in the CTE (`l.id = cl.lesson_id`) but no columns from it are selected. It serves as a validation join — confirming that the `lesson_id` in `course_lessons` actually exists in the `lessons` table. This filters out orphaned `course_lessons` rows.

The outer join is a `LEFT JOIN`, meaning lesson resources with no client association still appear, with `client_ids` defaulting to `[1]` via `COALESCE`.

## 3. Identify Aggregation Depth

Single-level aggregation in the CTE — collecting distinct client IDs per lesson:

```
lesson_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

Arrow notation:

```
lesson_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

No further aggregation after this. The final SELECT is a 1:1 join (one `client_ids` array per `lesson_id`) onto the `lesson_resources` detail rows.

## 4. Identify Dependencies

- **Grouping column:** `cl.lesson_id` — this is the grain of the CTE.
- **Derived column:** `client_ids` — aggregated from `ccc.client_id` across the full join chain.
- **Intermediate step required:** The four-table join chain must be resolved before the aggregation can be expressed, because `client_id` and `lesson_id` live four tables apart.
- **Default logic:** `COALESCE(slr.client_ids, [1])` depends on the LEFT JOIN producing NULLs for unmatched lesson resources.

## 5. Plan CTE Layers Bottom-Up

**CTE 1: `stg_lesson_resources`** — Joins the chain from clients through course containers, courses, and course_lessons down to lessons, then aggregates distinct client IDs per lesson_id.

**Final SELECT** — Joins lesson_resources to the CTE on lesson_id, selecting all resource columns plus the client_ids array (defaulting to `[1]` when no clients are found).

## 6. Write Each CTE with Justification

**`stg_lesson_resources`:** This CTE exists because the relationship between lessons and clients is indirect (spanning four tables), and the client IDs must be pre-aggregated into an array per lesson before joining to the detail-level `lesson_resources` table. Without this pre-aggregation, joining directly would fan out lesson_resources rows — one per client per course container path.

## 7. Compose the Final Query

```sql
WITH
stg_lesson_resources as (
    SELECT
        cl.lesson_id as lesson_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("lessons")} as l
    ON
        l.id = cl.lesson_id
    GROUP BY
        cl.lesson_id
)


SELECT
  lr.id,
  LOWER(name) AS name,
  lr.url,
  lr.lock_until_over,
  lr.lesson_id,
  lr.created,
  lr.updated,
  lr.`order`,
  lr.file,
  lr.creator_admin_id,
  lr.updated_admin_id,
  COALESCE(slr.client_ids, [1]) AS client_ids
FROM
  ${ref("lesson_resources")} as lr
LEFT JOIN
    stg_lesson_resources as slr
ON
    slr.lesson_id = lr.lesson_id
```
