How do I cast a negative INT64 value to UINT64 while preserving its bit representation instead of converting the numeric value?

--- AGENT ---

BIT_CAST_TO_UINT64 performs a bit-level cast rather than a value conversion, meaning the underlying bit pattern is preserved instead of interpreting the value. When you cast -1 to UINT64 using bit casting, you get the maximum UINT64 value because all bits are set to 1.

```googlesql
SELECT -1 as INT64_value, BIT_CAST_TO_UINT64(-1) as bit_cast_value;
```
