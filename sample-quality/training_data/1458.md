How does SPLIT_SUBSTR treat a very negative start position?

--- AGENT ---

When start_split is less than the negative of the number of splits, SPLIT_SUBSTR treats it as 1 and starts from the first split.

```googlesql
SELECT SPLIT_SUBSTR("www.abc.xyz.com", ".", -5, 3) AS example
```
