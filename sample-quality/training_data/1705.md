--- AGENT ---

This query identifies user sub-sessions in which an in-progress assessment was created. It joins the app events table with the assessments table by user_id and filters to assessments with status `'in_progress'`. The HAVING clause enforces a temporal constraint: the assessment's creation timestamp must fall within the time window spanned by that sub-session's events (between the earliest and latest event timestamp in the group). The output tags these instances as `'started_an_assessment'` behavioral events and includes the sub-session ID, app, URL path, and the assessment creation timestamp.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  'started_an_assessment' AS tag,
  'behavior' as class,
  es.app AS app,
  es.url_path AS url_path,
  uca.created AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS es
  JOIN ${ref("user_course_assessments")} AS uca ON uca.user_id = es.user_id
WHERE
  uca.status = 'in_progress'
GROUP BY
  es.sub_session_id,
  es.app,
  es.url_path,
  uca.created
HAVING
  uca.created BETWEEN MIN(es.created) AND MAX(es.created)
```
