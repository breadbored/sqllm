How can I eliminate duplicate rows from a recursive CTE result instead of keeping all variations?

--- AGENT ---

Using `UNION DISTINCT` in a recursive CTE instead of `UNION ALL` removes duplicate rows from the CTE result set. The `DISTINCT` variant ensures only one copy of each unique row is retained as the recursion builds, producing a clean set of distinct values without repetition:

```googlesql
WITH RECURSIVE
  CTE_1 AS (
    (SELECT 1 AS iteration UNION ALL SELECT 1 AS iteration)
    UNION DISTINCT
    SELECT iteration + 1 AS iteration FROM CTE_1 WHERE iteration < 3
  )
SELECT iteration FROM CTE_1
ORDER BY 1 ASC
```
