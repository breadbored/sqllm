How are columns matched when using INTERSECT without the BY NAME modifier?

--- AGENT ---

Without the `BY NAME` modifier, `INTERSECT` matches columns by position instead of by name. Columns with the same name in different order are treated as different columns.

```googlesql
WITH
  NumbersTable AS (
    SELECT 1 AS one_digit, 10 AS two_digit
    UNION ALL
    SELECT 2, 20
    UNION ALL
    SELECT 3, 30
  )
SELECT one_digit, two_digit FROM NumbersTable
|> INTERSECT ALL
    (SELECT 10 AS two_digit, 1 AS one_digit);
```
