## Question
Enrich each question set option with an array of client IDs that can access the underlying question through the course hierarchy. If a question isn't found in the course path, default to client [1].

--- AGENT ---

This query builds a client access list for each question by traversing the course hierarchy (clients → course containers → courses → lessons → learning cues → knowledge checks → questions), then enriches the `question_set_options` table with that access array.

The CTE `stg_question_set_option` walks the join path from `clients_course_containers` down to `question_sets`, aggregating distinct client IDs for each question. This produces one row per question_id with an array of all clients that have access to it through the course structure.

The main query then left-joins `question_set_options` to this CTE, using `question_set_id` from the options table to match against `question_id` in the CTE. For each row in `question_set_options`, it preserves all existing columns (id, question_set_id, question_option_id, created, updated) and adds a `client_ids` column that contains either the aggregated client IDs from the course hierarchy, or [1] as a fallback if no matching course path exists.

```sql
WITH
stg_question_set_option AS (
    SELECT
        qs.question_id as question_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("learning_cues")} as lc
    ON
        lc.lesson_id = cl.lesson_id
    JOIN
        ${ref("knowledge_checks")} as kc
    ON
        kc.learning_cue_id = lc.id
    JOIN
        ${ref("knowledge_check_questions")} as kcq
    ON
        kcq.knowledge_check_id = kc.id
    JOIN
        ${ref("question_sets")} as qs
    ON
        qs.question_id = kcq.question_id
    GROUP BY
        qs.question_id
)


SELECT
    qso.id,
    qso.question_set_id,
    qso.question_option_id,
    qso.created,
    qso.updated,
    COALESCE(sqso.client_ids, [1]) AS client_ids
FROM
    ${ref("question_set_options")} as qso 
LEFT JOIN  
    stg_question_set_option as sqso
ON
    sqso.question_id = qso.question_set_id
```
