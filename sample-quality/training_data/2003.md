--- USER ---

For each lesson container, return its metadata (creator, timestamps) along with the list of client IDs whose course containers include that lesson. If a lesson container has no associated clients, default to client ID 1.

--- AGENT ---

## 1. Identify shapes

**Input tables and columns (observable from the query):**
- `lesson_container`: `id`, `creator_admin_id`, `created`, `updated`
- `clients_course_containers` (ccc): `client_id`, `course_container_id`
- `courses` (c): `id`, `container_id`
- `course_lessons` (cl): `course_id`, `lesson_id`
- `lessons` (l): `id`
- `lesson_resources` (lr): `lesson_id`

**Target shape:**
`id, creator_admin_id, created, updated, client_ids` — one row per lesson container, with `client_ids` as an array.

## 2. Identify sources and relationships

The core question is: which clients are associated with a given lesson container? There is no direct relationship — it requires a multi-hop chain:

```
lesson_container.id
  → lessons.id (via lesson_container.id = ?)  — actually joined in final SELECT via slc.lesson_id = lc.id
  → lesson_resources.lesson_id (existence filter — ensures the lesson has resources)
  → course_lessons.lesson_id → course_lessons.course_id
  → courses.id → courses.container_id
  → clients_course_containers.course_container_id → clients_course_containers.client_id
```

The join chain in the CTE resolves this indirect relationship:
- `clients_course_containers.course_container_id = courses.container_id` — links clients to course containers
- `courses.id = course_lessons.course_id` — links courses to their lessons
- `lessons.id = course_lessons.lesson_id` — links lessons into the chain
- `lessons.id = lesson_resources.lesson_id` — filters to lessons that have at least one resource

The final query joins `lesson_container` to the CTE on `slc.lesson_id = lc.id`. This implies `lesson_container.id` corresponds to `lessons.id` — they represent the same entity or share the same key space.

The LEFT JOIN ensures lesson containers with no matching clients still appear, with `COALESCE(..., [1])` providing a default array.

## 3. Identify aggregation depth

Single-level aggregation — collapse the many-to-many client-to-lesson relationship into one array per lesson:

```
lesson_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

No further aggregation is needed. The final SELECT is a 1:1 join (lesson_container to the pre-aggregated CTE), so no second level exists.

Arrow notation:

```
lesson_id, ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
-> (final SELECT: 1:1 join to lesson_container, no further aggregation)
```

## 4. Identify dependencies

- **Grouping driver:** `l.id` (lesson_id) — the grain of the CTE output.
- **Derived column:** `client_ids` — aggregated from `ccc.client_id` across the full join chain.
- **Intermediate step required:** The 4-table join chain (`clients_course_containers → courses → course_lessons → lessons → lesson_resources`) must be resolved and aggregated *before* joining to `lesson_container`, because the relationship is many-to-many and would fan out the final result if not pre-aggregated.
- **Existence filter:** The join to `lesson_resources` acts as an inner-join filter — only lessons that have at least one resource entry will appear in the CTE. Lessons without resources get no CTE row, and thus fall through to the `COALESCE` default.

## 5. Plan CTE layers bottom-up

**`stg_lesson_container`** — Joins the full chain from clients through course containers, courses, and course-lessons down to lessons (filtered by resource existence), then aggregates distinct client IDs per lesson.

**Final SELECT** — Left-joins `lesson_container` to the CTE to pick up client arrays, defaulting to `[1]` for unmatched rows.

## 6. Write each CTE with justification

**`stg_lesson_container`:** This CTE exists because the relationship between lessons and clients is indirect (4 hops) and many-to-many. Pre-aggregating into one row per lesson with an array of client IDs prevents row fan-out when joining to `lesson_container`.

## 7. Compose the final query

```sql
WITH
stg_lesson_container as (
    SELECT
        l.id as lesson_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("lessons")} as l
    ON
        l.id = cl.lesson_id
    JOIN
        ${ref("lesson_resources")} as lr
    ON
        l.id = lr.lesson_id
    GROUP BY
        l.id
)


SELECT
    lc.id,
    lc.creator_admin_id,
    lc.created,
    lc.updated,
    COALESCE(slc.client_ids, [1]) AS client_ids
FROM
    ${ref("lesson_container")} as lc
LEFT JOIN
    stg_lesson_container as slc
ON
    slc.lesson_id = lc.id
```
