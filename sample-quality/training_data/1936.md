## Step 2: Identify sources and relationships

**Core entity:** `agg_user_course_progress` at grain `(user_id, course_id)` — this is the spine.

**First/second lesson identification:**
- `stg_course_lessons` maps `course_id` to ordered lesson positions via its `id` column (surrogate key that determines lesson order within a course).
- First lesson: `MIN(id)` per `course_id` in `stg_course_lessons`.
- Second lesson: `ROW_NUMBER() OVER (PARTITION BY course_id ORDER BY id) = 2` in `stg_course_lessons`.

**Lesson duration resolution chain:**
- `stg_course_lessons.lesson_id` → `stg_lessons.id` → `stg_lessons.video_id` → `stg_videos.id` → `stg_videos.duration`
- This is a 3-hop chain: course_lessons → lessons → videos to get duration.

**User progress on a specific lesson:**
- `stg_user_course_lesson_progress` joined to the identified first/second lesson on `(course_id, lesson_id)`, then aggregated per `(user_id, course_id)` using `MAX(progress_seconds)`.

**Completed lessons subquery:**

This subquery is joined into both the first-lesson and second-lesson blocks but its output (`completed_lessons`) is never selected — it appears to be vestigial (possibly used in a WHERE clause in an earlier version of the query, or included for filtering that was later removed). It does not affect the output shape.

**Join to spine:**
- First lesson progress → INNER JOIN to `ucp` on `(user_id, course_id)` — only users with first-lesson progress appear.
- Second lesson progress → LEFT JOIN to `ucp` on `(user_id, course_id)` — users without second-lesson progress still appear with NULLs.

## Step 3: Identify aggregation depth

There are two parallel single-level aggregations (first lesson and second lesson), each following the same pattern:

**First lesson progress:**
```
user_id, course_id, lesson_id, first_lesson_length, MAX(progress_seconds) as user_first_lesson_progress, LEAST(ROUND(MAX(progress_seconds)/first_lesson_length, 2), 1) as first_lesson_progress_percentage
```

**Second lesson progress:**
```
user_id, course_id, lesson_id, second_lesson_length, MAX(progress_seconds) as user_second_lesson_progress, LEAST(ROUND(MAX(progress_seconds)/second_lesson_length, 2), 1) as second_lesson_progress_percentage
```

These are both single-level aggregations — no arrow (`->`) needed because there is no second rollup. The `lesson_id` is fixed per course (it's always the first or second lesson), so the GROUP BY on `(user_id, course_id, lesson_id, lesson_length)` effectively produces one row per `(user_id, course_id)`.

**Pre-aggregation for lesson identification:**
```
course_id, MIN(id) as id  [first lesson]
course_id, ROW_NUMBER() filtered to rn=2  [second lesson]
```

These are zero-level aggregations — they identify which lesson to filter on before the progress aggregation happens.

## Step 4: Identify dependencies

- **Grouping drivers:** `user_id` and `course_id` are the primary keys of the output, inherited from `agg_user_course_progress`.
- **Derived — lesson identity:** First and second lesson IDs are derived from `stg_course_lessons` via MIN(id) and ROW_NUMBER()=2, respectively.
- **Derived — lesson duration:** Requires joining `stg_course_lessons` → `stg_lessons` → `stg_videos` to resolve `duration`.
- **Derived — progress metrics:** `MAX(progress_seconds)` requires filtering `stg_user_course_lesson_progress` to only rows matching the identified first/second lesson, then aggregating per user-course.
- **Derived — percentage:** Computed inline from `MAX(progress_seconds) / duration`, capped at 1.

**Dependency order:**
1. Identify first/second lesson per course
2. Resolve lesson duration via lessons → videos chain
3. Filter user progress to that specific lesson and aggregate
4. Join aggregated progress back to spine

## Step 5: Plan CTE layers bottom-up

The query uses two named CTEs and inline subqueries. Here is the logical layering:

1. **`rankedlessons`** — assigns a row number to each course-lesson by `id` order within each course, used to identify the second lesson.
2. **`secondlesson`** — filters `rankedlessons` to `rn = 2`, yielding the second lesson's `id` per course.
3. **First lesson detail (inline)** — identifies the first lesson per course via `MIN(id)`, then joins through `stg_lessons` → `stg_videos` to get its duration.
4. **First lesson progress (inline)** — joins `stg_user_course_lesson_progress` to the first lesson detail on `(course_id, lesson_id)`, aggregates `MAX(progress_seconds)` per `(user_id, course_id)`.
5. **Second lesson detail (inline)** — joins `stg_course_lessons` to `secondlesson` CTE, then through `stg_lessons` → `stg_videos` to get its duration.
6. **Second lesson progress (inline)** — same pattern as #4 but for the second lesson.
7. **Final SELECT** — joins `agg_user_course_progress` to first lesson progress (INNER) and second lesson progress (LEFT).

## Step 6: Write each CTE with justification

**`rankedlessons`:** Assigns positional order to lessons within each course so the second lesson can be identified by filtering on `rn = 2`. Uses `ROW_NUMBER() OVER (PARTITION BY course_id ORDER BY id)`.

**`secondlesson`:** Filters to only the second-position lesson per course, providing a clean join target for the second-lesson progress subquery.

**First lesson identification (inline subquery `fl`):** Uses `MIN(id) GROUP BY course_id` to find the first lesson's row in `stg_course_lessons` — an alternative to the ROW_NUMBER approach used for the second lesson.

**First/second lesson detail (inline subqueries `fld`/`sld`):** Joins the identified lesson row through `stg_course_lessons` → `stg_lessons` → `stg_videos` to resolve `lesson_id` and `duration` for the target lesson.

**First/second lesson progress (inline subqueries `uflp`/`uslp`):** Filters `stg_user_course_lesson_progress` to only the target lesson via INNER JOIN to the lesson detail subquery, then aggregates `MAX(progress_seconds)` per `(user_id, course_id)` and computes the capped percentage.

**Completed lessons subquery (inline `ucl`):** Counts distinct completed lessons per user-course. This subquery is LEFT JOINed into both progress blocks but its output is never referenced in the SELECT — it is structurally present but functionally inert in this version of the query.

## Step 7: Compose the final query

```sql
WITH
rankedlessons AS (
  SELECT
    course_id,
    id,
    row_number() OVER (PARTITION BY course_id ORDER BY id) AS rn
  FROM
    ${ref("stg_course_lessons")}
),

secondlesson AS (
  SELECT
    rl.course_id,
    rl.id
  FROM
    rankedlessons AS rl
  WHERE
    rl.rn = 2
)

SELECT
  ucp.*,
  uflp.first_lesson_length,
  uflp.user_first_lesson_progress,
  uflp.first_lesson_progress_percentage,
  uslp.second_lesson_length,
  uslp.user_second_lesson_progress,
  uslp.second_lesson_progress_percentage
FROM
  ${ref("agg_user_course_progress")} AS ucp
  -- first lesson
INNER JOIN (
  SELECT
    uclp.user_id,
    uclp.course_id,
    uclp.lesson_id,
    max(uclp.progress_seconds)                                                   AS user_first_lesson_progress,
    fld.first_lesson_length,
    least(round(((max(uclp.progress_seconds) / fld.first_lesson_length)), 2), 1) AS first_lesson_progress_percentage
  FROM
    ${ref("stg_user_course_lesson_progress")} AS uclp
  INNER JOIN (
    SELECT
      cl.course_id,
      cl.lesson_id AS first_lesson_id,
      v.duration   AS first_lesson_length
    FROM
      ${ref("stg_course_lessons")} AS cl
    INNER JOIN (
      SELECT
        course_id,
        min(id) AS id
      FROM
        ${ref("stg_course_lessons")}
      GROUP BY
        course_id
    ) AS fl
      ON
        cl.id = fl.id
        AND cl.course_id = fl.course_id
    INNER JOIN
      ${ref("stg_lessons")} AS l
      ON
        cl.lesson_id = l.id
    INNER JOIN
      ${ref("stg_videos")} AS v
      ON
        l.video_id = v.id
  ) AS fld
    ON
      uclp.course_id = fld.course_id
      AND uclp.lesson_id = fld.first_lesson_id
  LEFT JOIN (
    SELECT
      ucp.user_id,
      ucp.course_id,
      count(DISTINCT ufcl.lesson_id) AS completed_lessons
    FROM
      ${ref("agg_user_course_progress")} AS ucp

    LEFT JOIN
      ${ref("stg_user_finished_course_lessons")} AS ufcl
      ON
        ucp.course_id = ufcl.course_id
        AND ucp.user_id = ufcl.user_id
    GROUP BY
      ucp.user_id,
      ucp.course_id
  ) AS ucl
    ON
      uclp.user_id = ucl.user_id
      AND uclp.course_id = ucl.course_id
  GROUP BY
    uclp.user_id,
    uclp.course_id,
    lesson_id,
    fld.first_lesson_length
) AS uflp
  ON
    ucp.user_id = uflp.user_id
    AND ucp.course_id = uflp.course_id
-- second lesson
LEFT JOIN (
  SELECT
    uclp.user_id,
    uclp.course_id,
    uclp.lesson_id,
    max(uclp.progress_seconds)                                                    AS user_second_lesson_progress,
    sld.second_lesson_length,
    least(round(((max(uclp.progress_seconds) / sld.second_lesson_length)), 2), 1) AS second_lesson_progress_percentage
  FROM
    ${ref("stg_user_course_lesson_progress")} AS uclp
  INNER JOIN (
    SELECT
      cl.course_id,
      cl.lesson_id AS second_lesson_id,
      v.duration   AS second_lesson_length
    FROM
      ${ref("stg_course_lessons")} AS cl
      -- get second lesson
    INNER JOIN
      secondlesson AS sl
      ON
        cl.id = sl.id
        AND cl.course_id = sl.course_id
    INNER JOIN
      ${ref("stg_lessons")} AS l
      ON
        cl.lesson_id = l.id
    INNER JOIN
      ${ref("stg_videos")} AS v
      ON
        l.video_id = v.id
  ) AS sld
    ON
      uclp.course_id = sld.course_id
      AND uclp.lesson_id = sld.second_lesson_id
  LEFT JOIN (
    SELECT
      ucp.user_id,
      ucp.course_id,
      count(DISTINCT ufcl.lesson_id) AS completed_lessons
    FROM
      ${ref("agg_user_course_progress")} AS ucp

    LEFT JOIN
      ${ref("stg_user_finished_course_lessons")} AS ufcl
      ON
        ucp.course_id = ufcl.course_id
        AND ucp.user_id = ufcl.user_id
    GROUP BY
      ucp.user_id,
      ucp.course_id
  ) AS ucl
    ON
      uclp.user_id = ucl.user_id
      AND uclp.course_id = ucl.course_id
  GROUP BY
    uclp.user_id,
    uclp.course_id,
    lesson_id,
    sld.second_lesson_length
) AS uslp
  ON
    ucp.user_id = uslp.user_id
    AND ucp.course_id = uslp.course_id
```
