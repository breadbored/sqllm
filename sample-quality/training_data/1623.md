## Question
Show me a BigQuery query that segments daily active users into returning and new user cohorts, using account creation date as the distinguishing factor.

--- AGENT ---

This query produces a daily breakdown of users transacting on the platform, segmented by whether their accounts were created on the same day or a prior day.

The main query counts all distinct users active on each transaction date, joined with user records. It then LEFT JOINs a subquery that counts the subset of active users whose account creation date differs from the transaction date—these are the "returning" users. The difference between total active users and returning users yields the count of new users (those whose accounts were created on the same date as their first transaction).

The logic relies on comparing `date(u.created)` (account creation) against `date(t.created)` (transaction date). Users appearing in the returning subquery have these dates unequal, meaning they transacted after their account creation. The absence of a user from the returning subquery implies their account was created on the same day as their transaction, classifying them as new.

```sql
SELECT
  date(t.created)                                                 AS date,
  count(DISTINCT t.user_id)                                       AS active_users,
  coalesce(returning.active_users, 0)                             AS returning_users,
  count(DISTINCT t.user_id) - coalesce(returning.active_users, 0) AS new_users
FROM ${ref("agg_transactions")} AS t
INNER JOIN
  ${ref("stg_users")} AS u
  ON t.user_id = u.id
LEFT JOIN
  (
    SELECT
      date(t.created)           AS date,
      count(DISTINCT t.user_id) AS active_users
    FROM ${ref("agg_transactions")} AS t
    INNER JOIN
      ${ref("stg_users")} AS u
      ON t.user_id = u.id
    WHERE date(u.created) != date(t.created)
    GROUP BY date(t.created)
  ) AS returning
  ON returning.date = date(t.created)
GROUP BY
  date(t.created), returning.active_users
ORDER BY date
```
