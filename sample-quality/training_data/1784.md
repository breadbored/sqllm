## Question
Show me how to create a table function that calculates cumulative monthly course engagement metrics for a specific client and container, displaying both progress and time spent in hours.

--- AGENT ---

This BigQuery table function accepts a `clientId` and `containerId` as parameters and returns a time-series of cumulative course engagement metrics grouped by month.

The query operates in two stages:

**Stage 1 (MonthlyData CTE)**: Aggregates raw activity from the `reporting_user_courses` table by formatted month-year. For the specified client and container, it sums two metrics—`course_progress_minutes` and `time_on_course_minutes`—grouped by month and ordered chronologically.

**Stage 2 (CumulativeData CTE)**: Transforms monthly totals into running cumulative sums using window functions. Each row contains the year-to-date total for both metrics, with the window ordered by parsed date to ensure proper month-by-month accumulation.

**Output**: The final result returns three columns per month: the formatted date, `watch_time` (cumulative course progress minutes divided by 60 and rounded), and `course_time` (cumulative time on course minutes divided by 60 and rounded). The effect is a cumulative view of engagement over time, where each month's values represent the total accumulated from the beginning of the period.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  WITH
    MonthlyData AS (
    SELECT
      FORMAT_DATE('%b %y', course_started) AS date,
      SUM(course_progress_minutes) AS total_course_progress_minutes,
      SUM(time_on_course_minutes) AS total_time_on_course_minutes
    FROM
      ${ref("reporting_user_courses")}
    WHERE
      clientId IN UNNEST(client_id)
      AND container_id = containerId
    GROUP BY
      date
    ORDER BY
      PARSE_DATE('%b %y', date) ),
    CumulativeData AS (
    SELECT
      date,
      SUM(total_course_progress_minutes) OVER (ORDER BY PARSE_DATE('%b %y', date)) AS cumulative_course_progress_minutes,
      SUM(total_time_on_course_minutes) OVER (ORDER BY PARSE_DATE('%b %y', date)) AS cumulative_time_on_course_minutes
    FROM
      MonthlyData )
  SELECT
    date,
    ROUND(cumulative_course_progress_minutes / 60) AS watch_time,
    ROUND(cumulative_time_on_course_minutes / 60) AS course_time
  FROM
    CumulativeData)
```
