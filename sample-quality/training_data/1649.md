Can you provide a query that returns each user's profile information along with a count of their completed and in-progress courses?

--- AGENT ---

This query retrieves a summary of each user's account information paired with course enrollment statistics. It joins the users table to their course enrollments, then groups by user ID to aggregate enrollment counts.

The logic works as follows:
- The query selects all user profile columns (id, name, email, creation and update timestamps, and external identity references) using MAX(), which in this grouped context returns the single value associated with each user
- It left joins to the user_courses table to capture course relationships, and includes a left join to the courses table (though the courses table is not referenced in the final select, suggesting it may be used for filtering or future extension)
- It then counts courses by status: a CASE statement conditional on `uc.status = 'completed'` counts completed courses, and another conditional on `uc.status = 'in_progress'` counts courses in progress
- The WHERE clause excludes test users (`test_user = 0`)
- Results are grouped by user ID and ordered by user ID for consistent ordering

```sql
SELECT
  max(u.id)            AS user_id,
  max(u.first_name)    AS first_name,
  max(u.last_name)     AS last_name,
  max(u.email)         AS email,
  max(u.created)       AS user_created,
  max(u.updated)       AS user_updated,
  max(u.last_online)   AS last_online,
  max(u.meta_quest_id) AS meta_quest_id,
  max(u.facebook_id)   AS facebook_id,
  max(u.google_id)     AS google_id,
  max(u.amazon_id)     AS amazon_id,
  max(u.test_user)     AS test_user,
  sum(
    CASE
      WHEN uc.status = 'completed' THEN 1
      ELSE
        0
    END
  )                    AS completed_courses,
  sum(
    CASE
      WHEN uc.status = 'in_progress' THEN 1
      ELSE
        0
    END
  )                    AS in_progress_courses
FROM
  ${ref("stg_users")} AS u
LEFT JOIN
  ${ref("stg_user_courses")} AS uc
  ON
    u.id = uc.user_id
LEFT JOIN
  ${ref("stg_courses")} AS c
  ON
    uc.course_id = c.id
WHERE
  u.test_user = 0
GROUP BY
  u.id
ORDER BY
  u.id
```
