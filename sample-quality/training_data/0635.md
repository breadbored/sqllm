--- USER ---

How does the concatenation operator handle duplicate nodes when two paths share a common intermediate node?

--- AGENT ---

When concatenating graph paths that share a common node at the boundary, the first node in each subsequent path is removed from the concatenated path, preserving the nodes in order without duplication.

```googlesql
-- The concatenated path called `full_path` contains these elements:
-- src, t1, mid, t2, dst.
MATCH
  p=(src:Account)-[t1:Transfers]->(mid:Account),
  q=(mid)-[t2:Transfers]->(dst:Account)
LET full_path = p || q
```
