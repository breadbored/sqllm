--- AGENT ---

This query consolidates event tags from 76 separate bucket views into a unified summary. 

The query uses a Dataform JavaScript function to dynamically generate a UNION ALL statement across all bucket views. Each view contains the same schema: a sub_session_id (session identifier), tag (the event or behavior label), class, app, and url_path (contextual fields), and dt (a timestamp).

By UNIONing all 76 views, the query merges rows from every event tracking table into a single result set. It then groups by the key dimensions—sub_session_id, tag, class, app, and url_path—to consolidate duplicate or related rows. For each group, it counts the total instances using `COUNT(*)`, and captures the earliest and latest timestamp with `MIN(dt)` and `MAX(dt)`.

The output is a deduplicated summary where each row represents a unique combination of session, tag, class, app, and URL path, along with how many times that combination occurred and the time range over which it was observed.
