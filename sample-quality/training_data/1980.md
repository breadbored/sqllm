--- USER ---

Give me all course notes with their associated client IDs. Each note should include the user, course, lesson, note content (lowercased), timing, and timestamps. If a course has no client association, default to client ID 1.

--- AGENT ---

## 1. Identify shapes

**Input shape:**
- `course_notes`: `id, user_id, course_id, lesson_id, note, note_seconds, created, updated, prefilled_note, prefilled_note_id, prefilled_note_content_id`
- `courses`: `id, container_id`
- `clients_course_containers`: `client_id, course_container_id`

**Target shape:**
- `id, user_id, course_id, lesson_id, note, note_seconds, created, updated, prefilled_note, prefilled_note_id, prefilled_note_content_id, client_ids`

One row per course note, with a `client_ids` array attached from the client-course relationship.

## 2. Identify sources and relationships

- `course_notes.course_id` → `courses.id` (indirect, through the CTE)
- `courses.container_id` → `clients_course_containers.course_container_id` (the link between courses and their client ownership)
- The relationship from `course_notes` to clients is **two degrees removed**: `course_notes.course_id` → `courses.id` → `courses.container_id = clients_course_containers.course_container_id` → `clients_course_containers.client_id`

The CTE collapses the `courses` ↔ `clients_course_containers` chain into a single lookup keyed on `course_id`, producing an array of distinct client IDs per course.

The final join is `course_notes.course_id = course_clients.course_id` — a LEFT JOIN because not every course may have client associations, and those orphaned notes should still appear with a default `[1]`.

## 3. Identify aggregation depth

Single-level aggregation in the CTE only:

```
course_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

No further aggregation after this — the final SELECT is a row-level pass-through join. Arrow notation:

```
course_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids -> (no further aggregation; joined 1:1 to course_notes)
```

## 4. Identify dependencies

- **Grouping driver:** `courses.id` (aliased `course_id`) in the CTE — one row per course with its collected client IDs.
- **Derived column:** `client_ids` — aggregated from `clients_course_containers.client_id` through the `courses.container_id` join.
- **Intermediate step required:** The two-table join (`courses` ↔ `clients_course_containers`) must be resolved and aggregated to `course_id` grain *before* joining to `course_notes`, otherwise client multiplicity would duplicate note rows.
- **Default handling:** `COALESCE(cc.client_ids, [1])` fills in the default client array when the LEFT JOIN produces NULL (no client association for that course).

## 5. Plan CTE layers bottom-up

1. **`course_clients`** — Joins `courses` to `clients_course_containers` via `container_id = course_container_id`, then aggregates to one row per `course_id` with a distinct array of `client_id` values.

Final SELECT joins `course_notes` LEFT JOIN `course_clients` on `course_id`, selects all note fields (lowercasing text columns), and applies the `COALESCE` default for missing client associations.

## 6. Write each CTE with justification

**`course_clients`:** This CTE exists because the relationship from courses to clients is indirect (through `container_id` / `course_container_id`) and many-to-many. Pre-aggregating to an array per `course_id` prevents row duplication when joining to `course_notes`.

## 7. Compose the final query

```sql
WITH
course_clients AS (
  SELECT
    c.id as course_id,
    ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_course_containers")} as ccc
  JOIN
    ${ref("courses")} as c
  ON
    c.container_id = ccc.course_container_id
  GROUP BY
    c.id
)
SELECT
  cn.id,
  cn.user_id,
  cn.course_id,
  cn.lesson_id,
  LOWER(cn.note)           AS note,
  cn.note_seconds,
  cn.created,
  cn.updated,
  LOWER(cn.prefilled_note) AS prefilled_note,
  cn.prefilled_note_id,
  cn.prefilled_note_content_id,
  COALESCE(cc.client_ids, [1]) AS client_ids
FROM
  ${ref("course_notes")} as cn
LEFT JOIN
  course_clients as cc
ON
  cc.course_id = cn.course_id
```

## 8. Verify output shape

Target: `id, user_id, course_id, lesson_id, note, note_seconds, created, updated, prefilled_note, prefilled_note_id, prefilled_note_content_id, client_ids`

Final SELECT produces exactly these 12 columns. Grain is one row per course note (preserved by the LEFT JOIN to the pre-aggregated `course_clients` CTE). Text fields `note` and `prefilled_note` are lowercased. `client_ids` is an array, defaulting to `[1]` when no client association exists. ✓
