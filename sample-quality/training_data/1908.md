--- USER ---

I need to enrich our lab usage records with client IDs for each user. Can you provide a query that joins lab usage data with user client information, defaulting to client ID 1 when a user has no external client assignment?

--- AGENT ---

This query enriches user lab usage records with client ID information. It first builds a lookup table via CTE that maps each user to an array containing client ID 1 plus their external client ID (where one exists). It then left-joins this lookup to the user_lab_usage table, returning all lab usage columns along with a client_ids array that either contains the user's external client assignments or defaults to [1] for users without an external client ID.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  ulu.id,
  ulu.interval_seconds,
  ulu.user_id,
  ulu.course_id,
  ulu.lab_id,
  ulu.server_id,
  ulu.created,
  ulu.updated,
  COALESCE(uc.client_ids, [1]) as client_ids
FROM
  ${ref("user_lab_usage")} as ulu
LEFT JOIN
    user_clients as uc
ON
    uc.user_id = ulu.user_id
```
