--- USER ---

Show me a BigQuery query that extracts the earliest app and URL path attributes from ping events, grouped by sub-session.

--- AGENT ---

This query filters event records for 'ping' events and groups them by sub_session_id and creation timestamp. For each group, it uses `MIN_BY()` to extract the app and url_path values from the row with the earliest timestamp within that group. The query tags all results with the literal string 'ping' and a class identifier 'attribute', effectively creating a flattened attribute record per sub-session/timestamp combination.

The filtering conditions on `name` are redundant (the first two negations have no effect given the final equality check), but the query's intent is to isolate ping events and extract their earliest attribute values.

```sql
SELECT
    es.sub_session_id as sub_session_id,
    'ping' as tag,
    'attribute' as class,
    MIN_BY(es.app, es.created) as app,
    MIN_BY(es.url_path, es.created) as url_path,
    es.created as dt

FROM
    ${ref('base_buckets_events_docs_speedy')} as es 

WHERE
    es.name != 'page_view'
AND
    es.name != 'open_link'
AND
    es.name = 'ping'

GROUP BY
    sub_session_id,
    tag,
    dt
```
