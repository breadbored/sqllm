--- USER ---

Show me an example of how to segment mobile events by a timestamp cutoff and extract the earliest app and URL information from each segment.

--- AGENT ---

This query partitions mobile device events across two time periods relative to a January 26, 2026 cutoff. The first UNION ALL branch tags events on or after that cutoff as 'post_mobile_learn_buttons', while the second branch tags events from the preceding window (after September 30, 2025) as 'pre_mobile_learn_buttons'.

For each sub_session_id within each time segment, the query aggregates by extracting:
- The app value from the earliest event (MIN_BY on creation timestamp)
- The url_path value from the earliest event (MIN_BY on creation timestamp)  
- The minimum creation timestamp across the group

Each row represents one sub_session tagged by its temporal cohort, with the earliest observed app and url_path values preserved for that segment. The class field is hardcoded to 'attribute' to classify these as attribute-type records.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  'post_mobile_learn_buttons' as tag,
  'attribute' as class,
  MIN_BY(e.app, e.created) as app,
  MIN_BY(e.url_path, e.created) as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_app_speedy")} as e
WHERE
  e.device_category = 'mobile'
AND
  e.created >= DATETIME('2026-01-26T12:00:00', 'America/New_York')
GROUP BY
  sub_session_id

UNION ALL

SELECT
  e.sub_session_id as sub_session_id,
  'pre_mobile_learn_buttons' as tag,
  'attribute' as class,
  MIN_BY(e.app, e.created) as app,
  MIN_BY(e.url_path, e.created) as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_app_speedy")} as e
WHERE
  e.device_category = 'mobile'
AND
  e.created < DATETIME('2026-01-26T12:00:00', 'America/New_York')
AND
  e.created > DATETIME('2025-09-30T10:00:00', 'America/New_York')
GROUP BY
  sub_session_id
```
