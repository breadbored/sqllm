Show me an example of how to identify user sessions where specific user behavior occurs, like when they take notes, alongside their app activity during those sessions.

--- AGENT ---

This query identifies sub-sessions where a user exhibited note-taking behavior within their app activity window. It joins app event records with course note records for the same user, then filters to sub-sessions containing at least 3 events where a user-created note (not prefilled) was authored during the event time window. The result tags these sub-sessions with a 'note_taker' behavioral label, capturing the app context (app name and URL path) and the exact moment the note was created.

```sql
SELECT
  es.sub_session_id,
  'note_taker' AS tag,
  'behavior' as class,
  es.app AS app,
  es.url_path AS url_path,
  cn.created AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS es
  JOIN ${ref("stg_course_notes")} AS cn ON es.user_id = cn.user_id
  AND cn.prefilled_note_id IS NULL
GROUP BY
  es.sub_session_id,
  es.app,
  es.url_path,
  cn.created
HAVING
  COUNT(*) >= 3
  AND cn.created BETWEEN MIN(es.created) AND MAX(es.created)
```
