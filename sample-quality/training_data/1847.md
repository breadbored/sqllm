## Question
Provide me a BigQuery query that retrieves all assessments with their metadata and segments them by the clients associated with each assessment.

--- AGENT ---

This query retrieves all assessments with their metadata while segmenting them by the distinct clients associated with each assessment through course relationships.

The logic operates in two stages:

1. **`asessment_clients` CTE**: Joins the `clients_course_containers` table with the `courses` table to establish the relationship between clients and assessments. For each assessment, it aggregates all distinct client IDs into an array using `ARRAY_AGG(DISTINCT ... IGNORE NULLS)`.

2. **Main query**: Selects all fields from the `assessments` table and performs a left join with the CTE. This ensures all assessments are returned even if no clients are associated with them. When no clients are found for an assessment, the `COALESCE` function defaults the `client_ids` array to `[1]`. The `type` field is converted to lowercase.

The result is a flattened view of assessments with their associated client segments, useful for understanding which clients interact with which assessments.

```sql

WITH
asessment_clients AS (
    SELECT
        c.assessment_id as assessment_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    GROUP BY
        c.assessment_id
)
SELECT
  a.id,
  a.pass_score,
  a.question_count,
  a.created,
  a.updated,
  LOWER(a.type) AS type,
  COALESCE(ac.client_ids, [1]) AS client_ids
FROM
  ${ref("assessments")} as a
LEFT JOIN
    asessment_clients as ac
ON
    ac.assessment_id = a.id
```
