## Question
Show me a BigQuery query that retrieves certification records with their associated client information.

This query retrieves all records from the clients_certifications table, which appears to be a join table mapping clients to certifications. For each record, it outputs:

- **id**: the record identifier
- **certification_id**: a foreign key to a certification entity
- **client_id**: a foreign key to a client entity
- **created** and **updated**: timestamp columns tracking record lifecycle
- **client_ids**: a computed array column that wraps each row's client_id value into a single-element array

The ARRAY subquery is unconditional and has no WHERE clause, so it simply creates a one-element array containing the scalar client_id from each row, making it functionally redundant with the directly selected client_id column.

```sql
SELECT
  id,
  certification_id,
  client_id,
  created,
  updated,
  ARRAY(SELECT client_id) as client_ids,
FROM
  ${ref("clients_certifications")}
```
