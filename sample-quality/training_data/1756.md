Show me an example of how to flatten event data while adding navigation context from a window function and filtering based on visibility metadata in BigQuery.

--- AGENT ---

This query flattens event data for the 'vaas' app while adding session-level navigation context.

The first CTE, `backfill_visible`, identifies the chronologically first value of a `visible` flag (extracted from JSON metadata) for each sub-session per day.

The main query:
1. Selects all columns from the `all_events` table except a large exclusion list (URL variants, UTM parameters, referrer tracking, scroll metrics, and various ID columns)
2. Adds three derived columns:
   - `first_visible`: a boolean flag indicating whether the backfilled visibility status was 'true'
   - `url_path`: extracted from the JSON `action_meta` field
   - `last_url_path`: the previous event's `url_path` within the same session, computed via a LAG window function ordered by event creation time
3. Left-joins to the `backfill_visible` CTE on sub-session ID and event date
4. Filters to include only events where `app = 'vaas'`

The result is a denormalized event stream with navigation history (current and prior page per session) and visibility metadata, with many tracking columns removed.

```sql
WITH
backfill_visible AS (
    SELECT
        e.sub_session_id,
        DATE(e.created) as date_saw,
        MIN_BY(JSON_EXTRACT_SCALAR(e.action_meta, '$.visible'), e.created) as first_visible
    FROM
        ${ref("all_events")} AS e
    GROUP BY
        e.sub_session_id,
        date_saw
)
SELECT
  e.* EXCEPT(
    url, 
    url_path, 
    session_start_url_path, 
    subsession_start_url_path,
    referrer,
    derived_utm_campaign,
    derived_utm_medium,
    derived_utm_term,
    derived_utm_content,
    derived_utm_source,
    derived_referrer,
    subsession_start_utm_campaign,
    subsession_start_utm_medium,
    subsession_start_utm_term,
    subsession_start_utm_content,
    subsession_start_utm_source,
    subsession_start_referrer,
    scroll_direction,
    scroll_num,
    scroll_value,
    scroll_changed,
    accepted_cookies,
    landed_from,
    course_id, -- replaced by vaas_id
    derived_course_id,
    doc_id,
    derived_doc_id,
    client_ids,
    related_entity,
    related_id
  ),
  -- Add app specific columns below here from the metadata

  COALESCE(bv.first_visible = 'true', false) as first_visible,
  JSON_EXTRACT_SCALAR(e.action_meta, '$.url_path') as url_path,
  LAG(JSON_EXTRACT_SCALAR(e.action_meta, '$.url_path')) OVER (
    PARTITION BY
      e.session_id
    ORDER BY
      e.created
  ) as last_url_path,

FROM
  ${ref("all_events")} AS e

LEFT JOIN
  backfill_visible AS bv
ON
    bv.sub_session_id = e.sub_session_id
AND
    bv.date_saw = DATE(e.created)

WHERE 
  e.app = 'vaas'
```
