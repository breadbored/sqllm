--- USER ---

Show me a function that calculates the monthly count of sentiment question answers for a specific client, along with a running total of those answers over time.

--- AGENT ---

This query is a table-valued function that takes a client ID as input and returns a time series of sentiment survey responses for that client.

The inner subquery counts the number of question answer records (from `stg_user_course_sentiment_question_answers`) grouped by month, joined through users and courses to the client's course containers. Only records belonging to courses owned by the specified client are included. The count is labeled `monthly_answers`.

The outer query wraps this monthly aggregation with a window function that computes a running cumulative sum of the monthly counts, ordered chronologically. This produces three columns:
- `date`: the month formatted as "Mon YY" (e.g., "Jan 25")
- `monthly_answers`: the count of answers recorded in that month
- `cumulative_answers`: the total number of answers from the beginning of the time series through that month

The result is ordered by calendar date.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64) AS (
  SELECT
    date,
    monthly_answers,
    SUM(monthly_answers) OVER (ORDER BY PARSE_DATE('%b %y', date) ASC) AS cumulative_answers
  FROM (
    SELECT
      FORMAT_DATE('%b %y', ucsqa.created) AS date,
      COUNT(ucsqa.id) AS monthly_answers
    FROM
      ${ref("stg_user_course_sentiment_question_answers")} ucsqa
    JOIN
      ${ref("stg_users")} u
    ON
      u.id = ucsqa.user_id
    JOIN
      ${ref("stg_courses")} c
    ON
      c.id = ucsqa.course_id
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    WHERE
      ccc.client_id = clientId
    GROUP BY
      date) AS subquery
  ORDER BY
    PARSE_DATE('%b %y', date))
```
