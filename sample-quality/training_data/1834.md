--- USER ---

Can you retrieve all user knowledge check responses along with the complete list of clients associated with each question's course container?

--- AGENT ---

This query retrieves knowledge check response records from `agg_user_knowledge_checks` and enriches each record with aggregated client information. It performs two LEFT JOINs:

1. First, it joins to `stg_clients_course_containers` on the container ID to identify which clients are associated with each course container
2. Then it joins to `stg_clients` to resolve client names from client IDs

The aggregation step groups by all the individual knowledge check attributes (user ID, course, lesson, question, answer details, etc.) and collects two distinct arrays for each group:
- `client_id`: an array of distinct client IDs associated with that container
- `client_name`: an array of distinct client names associated with that container

The result is one row per unique knowledge check record, with arrays capturing all clients linked to that record's container. NULL client IDs are excluded from the first array.

```sql
SELECT
  user_id,
  course_id,
  course_name,
  container_id,
  lesson_id,
  question_id,
  question,
  answer_option_id,
  user_answer,
  correct,
  date_answered,
  correct_option_id,
  correct_answer,
  ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) AS client_id,
  ARRAY_AGG(DISTINCT clients.name) AS client_name
FROM
  ${ref("agg_user_knowledge_checks")} ukc
LEFT JOIN
  ${ref("stg_clients_course_containers")} ccc
ON
  ccc.course_container_id = ukc.container_id
LEFT JOIN
  ${ref("stg_clients")} clients
ON
  clients.id = ccc.client_id
GROUP BY
  user_id,
  course_id,
  course_name,
  container_id,
  lesson_id,
  question_id,
  question,
  answer_option_id,
  user_answer,
  correct,
  date_answered,
  correct_option_id,
  correct_answer
```
