--- USER ---

Give me a list of all course resources with their metadata, enriched with the client IDs that have access to each resource's course. If a course has no client associations, default to client ID 1.

--- AGENT ---

## 1. Identify shapes

**Input tables:**
- `course_resources` (cr): `id, name, url, lock_until_over, course_id, created, updated, order, file`
- `courses` (c): `id, container_id`
- `clients_course_containers` (ccc): `client_id, course_container_id`

**Target shape:**
`id, name, url, lock_until_over, course_id, created, updated, order, file, client_ids`

One row per course resource, with a `client_ids` array column appended.

## 2. Identify sources and relationships

- `course_resources.course_id` → `courses.id` — each resource belongs to a course
- `courses.container_id` → `clients_course_containers.course_container_id` — courses are grouped into containers, and client access is granted at the container level
- The relationship between `course_resources` and client IDs is indirect: `course_resources` → `courses` → `clients_course_containers`
- The join from `course_resources` to the client list is a LEFT JOIN — resources without any client association still appear, defaulting to `[1]` via COALESCE

## 3. Identify aggregation depth

There is one level of aggregation: collapsing the many client IDs per course into a single array, grouped by course.

```
course_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

No further aggregation follows — the main query is a flat join with no GROUP BY.

Arrow notation:

```
course_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
-> (joined to course_resources, no further aggregation)
```

## 4. Identify dependencies

- **Grouping driver:** `course_id` — one array of client IDs per course
- **Derived column:** `client_ids` — aggregated from `clients_course_containers.client_id` via the `courses` join
- **Intermediate step required:** The client-to-course mapping must be resolved and aggregated into an array *before* joining to `course_resources`, otherwise the join would fan out `course_resources` rows by the number of client associations
- **Default behavior:** `COALESCE(cc.client_ids, [1])` handles courses with no client container mapping by assigning the default client ID array `[1]`

## 5. Plan CTE layers bottom-up

1. **`course_clients`** — Joins `courses` to `clients_course_containers` via the container ID, then aggregates all distinct client IDs into an array per course. This pre-aggregation prevents row fanout in the final join.

2. **Final SELECT** — Left-joins `course_resources` to `course_clients` on `course_id`, picks up metadata columns, lowercases the name, and applies COALESCE for the default client array.

## 6. Write each CTE with justification

**`course_clients`:** This CTE resolves the indirect relationship between courses and clients (through the container layer) and collapses it to one row per `course_id` with an `ARRAY_AGG` of distinct client IDs. Without this pre-aggregation, joining `course_resources` directly to the container-client table would duplicate resource rows for each client.

```sql
course_clients AS (
  SELECT
    c.id as course_id,
    ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
  FROM clients_course_containers as ccc
  JOIN courses as c
    ON c.container_id = ccc.course_container_id
  GROUP BY c.id
)
```

**Final SELECT:** Joins each course resource to its pre-aggregated client list. LEFT JOIN ensures resources on courses with no client-container mapping are retained. `COALESCE(..., [1])` provides the default. `LOWER(cr.name)` normalizes the resource name to lowercase.

## 7. Compose the final query

```sql
WITH
course_clients AS (
  SELECT
    c.id as course_id,
    ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_course_containers")} as ccc
  JOIN
    ${ref("courses")} as c
  ON
    c.container_id = ccc.course_container_id
  GROUP BY
    c.id
)
SELECT
  cr.id,
  LOWER(cr.name) AS name,
  cr.url,
  cr.lock_until_over,
  cr.course_id,
  cr.created,
  cr.updated,
  cr.`order`,
  cr.file,
  COALESCE(cc.client_ids, [1]) AS client_ids
FROM
  ${ref("course_resources")} as cr
LEFT JOIN
  course_clients as cc
ON
  cc.course_id = cr.course_id
```
