## Question
Enrich watchlist records with associated client IDs from the users table, defaulting to a single client if the user has no external client ID.

--- AGENT ---

This query enriches watchlist records by joining them with user client information and providing a sensible default.

The CTE `user_clients` extracts from the users table a mapping of each user ID to an array of client IDs. The array always contains the value `1` in the first position, followed by the user's `external_client_id` in the second position — but only for users where `external_client_id IS NOT NULL`.

The main query then selects all columns from the watchlist table and performs a LEFT JOIN against this CTE on user ID. The key column is `client_ids`, which uses COALESCE to handle unmatched users: if a watchlist entry's user ID has no match in `user_clients` (meaning either the user record is missing or their external_client_id was NULL), the query defaults to an array containing only `[1]`. Otherwise, it returns the two-element array `[1, external_client_id]` from the joined user record.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  w.id,
  w.user_id,
  w.course_container_id,
  w.created,
  w.updated,
  COALESCE(uc.client_ids, [1]) as client_ids
FROM
  ${ref("watchlist")} as w
LEFT JOIN
  user_clients as uc
ON
  uc.user_id = w.user_id
```
