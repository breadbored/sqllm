## Question
List all resource-opened events enriched with the resource name and URL, the user's course progress status and percentage, and the client context. Each row should represent one event where a user opened a resource within a course.

--- AGENT ---

## 1. Identify shapes

**Input shape (all tables joined):**
`e.user_id, e.course_id, e.action, e.related_id, e.name, e.created, lr.id, lr.name, lr.url, ucp.container_id, ucp.course_name, ucp.course_status, ucp.course_progress_percentage, ucp.course_id, ucp.user_id, ucp.client_id`

**Target shape:**
`user_id, course_id, container_id, course_name, action, related_id, name, url, course_status, course_progress_percentage, created, client_id`

This is a flat detail query — one row per resource-opened event, enriched with dimension attributes from two lookup tables. No aggregation, no grain change.

## 2. Identify sources and relationships

Three tables:

- **`stg_events_app_speedy` (e)** — the fact table. Contains event-level rows with `user_id`, `course_id`, `related_id`, `action`, `name`, `created`.
- **`lesson_resources` (lr)** — a dimension table for resources. Joined on `e.related_id = lr.id`. This tells us `related_id` is a foreign key to the resource entity, and the join enriches each event with the resource's `name` and `url`.
- **`agg_user_course_progress` (ucp)** — a pre-aggregated dimension table for user-course progress. Joined on `e.course_id = ucp.course_id AND e.user_id = ucp.user_id`. This is a composite key join — one progress record per user per course. Enriches each event with `container_id`, `course_name`, `course_status`, `course_progress_percentage`, and `client_id`.

Both joins are INNER, meaning:
- Events without a matching resource in `lesson_resources` are excluded (orphaned `related_id` values drop out).
- Events for user-course combinations not present in `agg_user_course_progress` are excluded.

The `WHERE e.name = "opened_resource"` filter restricts to a single event type before enrichment.

## 3. Identify aggregation depth

No aggregation. This is a pure join-and-filter query. The grain of the output equals the grain of the fact table (one row per event), narrowed by the filter and the inner join exclusions.

Arrow notation for the single-level shape (no aggregation):

```
e.user_id, e.course_id, e.action, e.related_id, e.created, lr.name, lr.url, ucp.container_id, ucp.course_name, ucp.course_status, ucp.course_progress_percentage, ucp.client_id
(no aggregation — passthrough)
```

## 4. Identify dependencies

- **Filter dependency:** `e.name = "opened_resource"` must be applied to the events table to narrow the fact set.
- **Join dependency (lr):** `e.related_id` must exist and match `lr.id`. The resource's `name` and `url` are passthrough columns from this dimension.
- **Join dependency (ucp):** `e.user_id` and `e.course_id` must both match in `ucp`. The progress columns (`course_status`, `course_progress_percentage`) and context columns (`container_id`, `course_name`, `client_id`) are passthrough from this dimension.
- No derived columns, no intermediate calculations, no window functions.

## 5. Plan CTE layers bottom-up

No CTEs required. This is a single-pass query: filter the events table, join to two dimension tables, select the output columns.

## 6. Write each CTE with justification

No CTEs — the query is a direct three-table join with a filter. No intermediate steps are needed because:
- There is no aggregation requiring pre-processing.
- Both dimension tables join cleanly to the fact table on direct key relationships.
- The filter is a simple equality predicate on the fact table.

## 7. Compose the final query

```sql
SELECT
  e.user_id,
  e.course_id,
  ucp.container_id,
  ucp.course_name,
  e.action,
  e.related_id,
  lr.name,
  lr.url,
  ucp.course_status,
  ucp.course_progress_percentage,
  e.created,
  ucp.client_id
FROM ${ref("stg_events_app_speedy")} AS e
INNER JOIN ${ref("lesson_resources")} AS lr ON e.related_id = lr.id
INNER JOIN ${ref("agg_user_course_progress")} AS ucp ON e.course_id = ucp.course_id AND e.user_id = ucp.user_id
WHERE e.name = "opened_resource"
```
