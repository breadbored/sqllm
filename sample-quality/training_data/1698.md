Show me an example of how to correlate app session events with completed assessments, tagging sessions where a user has passed an assessment.

--- AGENT ---

This query identifies and tags app sessions during which a user passed a course assessment. It joins session events with completed assessments on `user_id`, filters for passing assessments (`uca.passed = 1`), and groups by session metadata (sub_session_id, app, url_path) and assessment update time. The HAVING clause filters to keep only groups where the assessment update timestamp falls within the time window of events in that session (between the minimum and maximum event creation times). The output is one tagged behavioral event per matching session, marking it as "passed_an_assessment" along with app and URL context.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  'passed_an_assessment' AS tag,
  'behavior' as class,
  es.app AS app,
  es.url_path AS url_path,
  uca.updated AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS es
JOIN
  ${ref("stg_user_course_assessments")} AS uca
ON
  uca.user_id = es.user_id
WHERE
  uca.status = 'completed'
AND
  uca.passed = 1
GROUP BY
  es.sub_session_id,
  es.app,
  es.url_path,
  uca.updated
HAVING
  uca.updated BETWEEN MIN(es.created) AND MAX(es.created)
```
