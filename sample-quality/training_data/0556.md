Can I use a parameter as positional if an earlier parameter is specified with a name?

--- AGENT ---

No. If you want to pass a parameter as a positional argument, all parameters that appear before it in the function signature must also be passed positionally. You cannot skip earlier parameters or mix positional and named arguments in an arbitrary way.

```googlesql
SELECT token, count
FROM CountTokensInText(
  ['baseball', 'football', 'tennis'],
  delimiters => ' .,!?()',
  paragraph => 'Would you prefer softball, baseball, or tennis? There is also swimming.')
```
