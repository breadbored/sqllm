--- USER ---

Show me user statistics by month, broken down by new vs returning users, including session metrics for transactions involving a specific client.

--- AGENT ---

This query analyzes user activity patterns by month, distinguishing between new and returning users based on when their first transaction with client_id=1 occurred.

The logic flows in three stages:

1. **client_transactions_min CTE**: Unnests the client_id array from the agg_client_transactions table and finds the earliest transaction day (min_day) for each user-client pairing.

2. **meta_quest_min CTE**: Filters the previous result to only client_id=1, creating a lookup table of the first transaction date for each user who transacted with that client.

3. **Main query**: Joins transactions back to this lookup table on user_id and groups by month. It counts:
   - Unique users and total sessions
   - New users vs returning users using CASE logic that compares the first transaction date (mqm.min_day) against some comparison value
   - Sessions per user (total sessions ÷ unique users)
   - Separate monthly cohorts of new and returning users

The WHERE clause `1 IN unnest(ct.client_id)` restricts to transactions where client_id array contains 1, and results are grouped and ordered by month.

**Note**: The date comparison logic in the CASE statements references `date_trunc(DAY, DAY)` and `date_trunc(DAY, MONTH)`, where the second parameter appears to be a keyword rather than a column reference—this may indicate an error in the original query, as these don't reference an actual transaction date column.

```sql
WITH
client_transactions_min AS (
  SELECT
    ct.user_id,
    cid           AS client_id,
    min(ct.`day`) AS min_day
  FROM
     ${ref("agg_client_transactions")} AS ct,
    unnest(ct.client_id) AS cid
  GROUP BY
    ct.user_id,
    cid
),

meta_quest_min AS (
  SELECT
    ctm.user_id,
    ctm.client_id,
    ctm.min_day
  FROM
    client_transactions_min AS ctm
  WHERE
    ctm.client_id = 1
)

SELECT
  count(DISTINCT ct.user_id)                               AS unique_users,
  count(
    DISTINCT
    CASE
      WHEN date_trunc(mqm.min_day, DAY) = date_trunc(DAY, DAY) THEN ct.user_id
    END
  )                                                        AS new_users,
  count(
    DISTINCT
    CASE
      WHEN date_trunc(mqm.min_day, DAY) != date_trunc(DAY, DAY) THEN ct.user_id
    END
  )                                                        AS returning_users,
  (count(
    DISTINCT
    CASE
      WHEN date_trunc(mqm.min_day, DAY) != date_trunc(DAY, DAY) THEN ct.user_id
    END
  ) + count(
    DISTINCT
    CASE
      WHEN date_trunc(mqm.min_day, DAY) = date_trunc(DAY, DAY) THEN ct.user_id
    END
  )) - count(DISTINCT ct.user_id)                          AS new_users_returning,
  count(
    DISTINCT
    CASE
      WHEN date_trunc(mqm.min_day, MONTH) = date_trunc(DAY, MONTH) THEN ct.user_id
    END
  )                                                        AS monthly_new_users,
  count(
    DISTINCT
    CASE
      WHEN date_trunc(mqm.min_day, MONTH) != date_trunc(DAY, MONTH) THEN ct.user_id
    END
  )                                                        AS monthly_returning_users,
  count(ct.user_id)                                        AS total_sessions,
  count(
    CASE
      WHEN date_trunc(mqm.min_day, DAY) != date_trunc(DAY, DAY) THEN ct.user_id
    END
  )                                                        AS returning_user_sessions,
  round(count(ct.user_id) / count(DISTINCT ct.user_id), 2) AS sessions_per_user,
  format_date('%b %y', date_trunc(DAY, MONTH))             AS year_month,
  date_trunc(DAY, MONTH)                                   AS month
FROM
   ${ref("agg_client_transactions")} AS ct
INNER JOIN
  meta_quest_min AS mqm
  ON
    ct.user_id = mqm.user_id
WHERE
  1 IN unnest(ct.client_id)
GROUP BY
  format_date('%b %y', date_trunc(DAY, MONTH)),
  date_trunc(DAY, MONTH)
ORDER BY
  month
```
