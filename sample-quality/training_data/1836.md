Can you show me an example BigQuery SQL query that analyzes user engagement metrics—specifically new vs. returning users—aggregated by month and client?

--- AGENT ---

This query attempts to analyze user transaction patterns by identifying when each user first transacted (on a per-client basis), then categorizing users as "new" or "returning" based on whether their first transaction occurred in the current period. It groups results by month and client to produce engagement metrics.

The query structure works as follows:

1. **`client_transactions_min` CTE**: Unnests the `client_id` array field and finds the earliest transaction date (`min_day`) for each user-client pair.

2. **`meta_quest_min` CTE**: A pass-through that preserves the minimum dates from the first CTE.

3. **Main query**: Joins transactions back to minimum dates and groups by calendar month and client. It calculates:
   - Total unique users
   - User counts broken down by whether their earliest transaction date falls in the current month (new) vs. earlier (returning)
   - Session counts and sessions-per-user ratio
   - Month labels for grouping

**Note**: The query contains structural ambiguities in its CASE conditions. Expressions like `DATE_TRUNC(DAY, DAY)` and `DATE_TRUNC(DAY, MONTH)` reference an undefined column `DAY` rather than the transaction date from the joined table. The logic intent is observable (comparing first-transaction-date against current-period boundaries), but the exact reference is unclear from the SQL as written.

```sql
WITH
  client_transactions_min AS (
  SELECT
    ct.user_id,
    cid AS client_id,
    MIN(ct.`day`) AS min_day
  FROM
    ${ref("agg_client_transactions")} AS ct,
    UNNEST(ct.client_id) AS cid
  GROUP BY
    ct.user_id,
    cid ),
  meta_quest_min AS (
  SELECT
    ctm.user_id,
    ctm.client_id,
    ctm.min_day
  FROM
    client_transactions_min AS ctm )
SELECT
  COUNT(DISTINCT ct.user_id) AS unique_users,
  COUNT( DISTINCT
    CASE
      WHEN DATE_TRUNC(mqm.min_day, DAY) = DATE_TRUNC(DAY, DAY) THEN ct.user_id
  END
    ) AS new_users,
  COUNT( DISTINCT
    CASE
      WHEN DATE_TRUNC(mqm.min_day, DAY) != DATE_TRUNC(DAY, DAY) THEN ct.user_id
  END
    ) AS returning_users,
  ( COUNT( DISTINCT
      CASE
        WHEN DATE_TRUNC(mqm.min_day, DAY) != DATE_TRUNC(DAY, DAY) THEN ct.user_id
    END
      ) + COUNT( DISTINCT
      CASE
        WHEN DATE_TRUNC(mqm.min_day, DAY) = DATE_TRUNC(DAY, DAY) THEN ct.user_id
    END
      ) ) - COUNT(DISTINCT ct.user_id) AS new_users_returning,
  COUNT( DISTINCT
    CASE
      WHEN DATE_TRUNC(mqm.min_day, MONTH) = DATE_TRUNC(DAY, MONTH) THEN ct.user_id
  END
    ) AS monthly_new_users,
  COUNT( DISTINCT
    CASE
      WHEN DATE_TRUNC(mqm.min_day, MONTH) != DATE_TRUNC(DAY, MONTH) THEN ct.user_id
  END
    ) AS monthly_returning_users,
  COUNT(ct.user_id) AS total_sessions,
  COUNT(
    CASE
      WHEN DATE_TRUNC(mqm.min_day, DAY) != DATE_TRUNC(DAY, DAY) THEN ct.user_id
  END
    ) AS returning_user_sessions,
  ROUND(COUNT(ct.user_id) / COUNT(DISTINCT ct.user_id), 2) AS sessions_per_user,
  FORMAT_DATE('%b %y', DATE_TRUNC(DAY, MONTH)) AS year_month,
  DATE_TRUNC(DAY, MONTH) AS month,
  cl AS client_id
FROM
  ${ref("agg_client_transactions")} AS ct,
  ct.client_id AS cl
INNER JOIN
  meta_quest_min AS mqm
ON
  mqm.user_id = ct.user_id
  AND mqm.client_id = cl
GROUP BY
  FORMAT_DATE('%b %y', DATE_TRUNC(DAY, MONTH)),
  DATE_TRUNC(DAY, MONTH),
  cl
ORDER BY
  month
```
