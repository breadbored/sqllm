Show me an example of a BigQuery query that joins an event stream with aggregated course progress data and lesson resource metadata to enrich resource-opening events.

--- AGENT ---

This query enriches "opened_resource" events with course progress metrics and lesson metadata. It performs two inner joins: first to `lesson_resources` on the resource identifier (matching event `related_id` to resource `id`), and second to a pre-aggregated `agg_user_course_progress` table on the user-course pair. The result is a wide row containing the event timestamp and action, the resource name and URL, and the user's current course status and progress percentage at the time of the event. The WHERE clause filters to only resource-open events, making this suitable for analyzing which lessons users engage with during their course progression.

```sql
SELECT
  e.user_id,
  e.course_id,
  ucp.container_id,
  ucp.course_name,
  e.action,
  e.related_id,
  lr.name,
  lr.url,
  ucp.course_status,
  ucp.course_progress_percentage,
  e.created,
  ucp.client_id
FROM ${ref("stg_events_app_speedy")} AS e
INNER JOIN ${ref("lesson_resources")} AS lr ON e.related_id = lr.id
INNER JOIN ${ref("agg_user_course_progress")} AS ucp ON e.course_id = ucp.course_id AND e.user_id = ucp.user_id
WHERE e.name = "opened_resource"
```
