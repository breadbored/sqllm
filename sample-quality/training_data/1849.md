## Question
Show me an example of a query that retrieves all category records with normalized text fields. I need to understand how to apply case normalization in BigQuery when selecting from a table.

--- AGENT ---

This query retrieves all columns from the `categories` table and applies lowercase normalization to three text fields: `name`, `description`, and `headline`. All other columns (IDs, timestamps, images, statuses, and configuration fields) pass through unchanged. The output maintains the same row count as the source table with the same granularity — one row per category.

```sql
SELECT
  id,
  LOWER(name)        AS name,
  created,
  updated,
  LOWER(description) AS description,
  image,
  LOWER(headline)    AS headline,
  type,
  `order`,
  card_image,
  partner_status,
  internal_message,
  link,
  published_state,
  channel_details,
  hex_inner,
  hex_outer,
  image_color,
  seo
FROM
  ${ref("categories")}
```
