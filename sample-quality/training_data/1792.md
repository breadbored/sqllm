--- USER ---

For a given sentiment question, retrieve the question with all its available answer options and show the count of user responses for each option within a specific client and course container.

--- AGENT ---

This table-valued function retrieves a sentiment question along with its associated answer options and response counts. It works as follows:

1. **Inner subquery logic**: Aggregates sentiment question options by joining them with sentiment questions (to obtain the question name), then left joins with user responses (filtered to specific courses and users). The response counts are further scoped to a specific client and course container via a left join to the clients_course_containers table. The subquery groups by option name and question name, returning the maximum option ID, maximum sentiment_question_id, and the count of matching user responses.

2. **Outer query logic**: Joins the aggregated options back to the sentiment_questions table using the question name, filters to the input sentiment question ID, and groups by question name. All option records are aggregated into a single ARRAY_AGG column called `answers`.

3. **Result structure**: Returns the question name paired with an array of option objects. Each option object contains its ID, name, the parent question name, sentiment_question_id, and a response count—all scoped to the input client ID and course container ID.

The function accepts three parameters: `clientId`, `containerId`, and `sentimentQuestionId`, which are used to filter the response counts and to select the target sentiment question.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64,
    sentimentQuestionId INT64) AS (
  SELECT
    sq.name,
    ARRAY_AGG(sqo) AS `answers`
  FROM
    ${ref("stg_sentiment_questions")} sq
  JOIN (
    SELECT
      MAX(sqo.id) AS id,
      sqo.name,
      _sq.name AS question_name,
      MAX(sqo.sentiment_question_id) AS sentiment_question_id,
      COUNT(ucsqa.id) AS count
    FROM
      ${ref("stg_sentiment_question_options")} AS sqo
    JOIN
      ${ref("stg_sentiment_questions")} AS _sq
    ON
      _sq.id = sqo.sentiment_question_id
    LEFT JOIN (
      SELECT
        ucsqa.id,
        ucsqa.sentiment_question_option_id,
        ucsqa.course_id
      FROM
        ${ref("stg_user_course_sentiment_question_answers")} AS ucsqa
      JOIN
        ${ref("stg_users")} AS u
      ON
        ucsqa.user_id = u.id ) AS ucsqa
    ON
      ucsqa.sentiment_question_option_id = sqo.id
    LEFT JOIN
      ${ref("stg_courses")} c
    ON
      c.id = ucsqa.course_id
    LEFT JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
      AND container_id = containerId
      AND ccc.client_id = clientId
    GROUP BY
      sqo.name,
      _sq.name) AS sqo
  ON
    sqo.question_name = sq.name
  WHERE
    sq.id = sentimentQuestionId
  GROUP BY
    sq.name)
```
