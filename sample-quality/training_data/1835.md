Show me a BigQuery table function that analyzes behavioral correlations for users who performed a specific set of actions, tracking which other behaviors preceded a target conversion event and comparing against baseline conversion rates.

--- AGENT ---

This query is a table function that performs cohort-based behavioral correlation analysis for conversion funnel analysis.

**What it does:**

The function accepts a `qualifier` array (a set of prerequisite tags/behaviors) and a `thing` (a target conversion event). It filters to sessions containing ALL behaviors in the qualifier set, then analyzes every OTHER behavior observed in those sessions to understand conversion patterns.

**The reasoning flow:**

1. **first_utm** captures the initial page view event per session, extracting the earliest utm_campaign and session start time
2. **thing_doer** identifies sessions that performed ALL tags in the qualifier array (using a HAVING clause to enforce exact match count)
3. **first_goal** locates the first occurrence of the target `thing` event per session (if it occurred at all)
4. **baseline_sentiment** computes summary statistics: total sessions in the cohort and how many performed the target `thing`
5. **tag_sentiment** is the main analysis layer that:
   - Cross-joins session records with first_utm (session identifiers and UTM metadata)
   - Joins with thing_doer to restrict to the qualifying cohort
   - Left joins with first_goal to determine if/when conversion happened
   - Left joins inline views to flag whether each session contains the target `thing` or not
   - Filters to behaviors that are NOT in the qualifier array and NOT the target thing itself
   - Only includes behaviors that occurred before conversion, or in non-converting sessions
   - Buckets behavior frequency into four categories (1, 2-4, 5-10, 10+ times)
   - Counts sessions with and without the target behavior per tag/frequency combination
6. **Final SELECT** outputs per-session analysis: whether positive/negative behaviors occurred, plus baseline totals for comparison

The output is one row per session-tag-frequency combination, enabling downstream analysis of which behaviors predict or correlate with the target conversion.

```sql
CREATE OR REPLACE TABLE FUNCTION ${self()} (qualifier ARRAY<STRING>, thing STRING) AS (
WITH 
first_utm AS (
  SELECT
    eas.sub_session_id as sub_session_id,
    MIN_BY(eas.utm_campaign, eas.created) as utm_campaign,
    MIN(eas.created) as session_started
  FROM 
    ${ref("base_buckets_events_app_speedy")} as eas
  WHERE
    eas.name = 'page_view'
  GROUP BY
    sub_session_id
),
thing_doer AS (
  SELECT 
    aabt.sub_session_id as sub_session_id
  FROM 
    ${ref("agg_app_bucket_tags")} as aabt, UNNEST(qualifier) as _t
  WHERE
    aabt.tag = _t
  GROUP BY
    sub_session_id
  HAVING
    COUNT(DISTINCT aabt.tag) = ARRAY_LENGTH(qualifier)
),
first_goal AS (
  SELECT
    sub_session_id,
    MAX(max_dt) as first_goal_dt  -- or whatever timestamp field exists
  FROM
    ${ref("agg_app_bucket_tags")}
  WHERE
    tag = thing
  GROUP BY
    sub_session_id
),
baseline_sentiment AS (
  SELECT 
    COUNT(
      DISTINCT 
      CASE 
        WHEN aabt.tag = thing
          THEN sub_session_id 
      END
    ) as did_thing,
    COUNT(
      DISTINCT 
      aabt.sub_session_id
    ) as total_did_thing
  FROM
    ${ref("agg_app_bucket_tags")} as aabt
  WHERE 
    sub_session_id IN (
      SELECT
        ds.sub_session_id as sub_session_id
      FROM
        thing_doer as ds
    )
  AND
    aabt.tag = thing
),
tag_sentiment AS (
  SELECT 
    t.tag,
    t.app,
    fu.sub_session_id,
    fu.utm_campaign,
    fu.session_started,
    -- Add frequency bucket
    CASE 
      WHEN t.instances = 1 THEN '1 time'
      WHEN t.instances BETWEEN 2 AND 4 THEN '2-4 times'
      WHEN t.instances BETWEEN 5 AND 10 THEN '5-10 times'
      ELSE '10+ times'
    END as frequency_bucket,
    COUNT(
      DISTINCT
      CASE
        WHEN p.sub_session_id IS NOT NULL
          THEN ds.sub_session_id
      END
    ) as did_thing,
    COUNT(
      DISTINCT
      CASE
        WHEN n.sub_session_id IS NOT NULL
          THEN ds.sub_session_id
      END
    ) as didnt_did_thing,
  FROM
    ${ref("agg_app_bucket_tags")} as t
  JOIN
    first_utm as fu
  ON
    fu.sub_session_id = t.sub_session_id
  JOIN
    thing_doer as ds
  ON
    t.sub_session_id = ds.sub_session_id
  LEFT JOIN 
    first_goal as fg
  ON
    t.sub_session_id = fg.sub_session_id
  LEFT JOIN 
    (
      SELECT
        DISTINCT
        _aabt.sub_session_id as sub_session_id 
      FROM
        ${ref("agg_app_bucket_tags")} as _aabt
      WHERE
        _aabt.tag = thing
    ) as p
  ON
    ds.sub_session_id = p.sub_session_id
  LEFT JOIN 
    (
      SELECT
        DISTINCT
        _aabt.sub_session_id as sub_session_id 
      FROM
        ${ref("agg_app_bucket_tags")} as _aabt
      WHERE
        _aabt.tag != thing
    ) as n
  ON
    ds.sub_session_id = n.sub_session_id
  WHERE
    t.tag NOT IN UNNEST(qualifier)
  AND
    t.tag != thing
  AND
    (
      fg.first_goal_dt IS NULL          -- user never converted
    OR
      t.min_dt < fg.first_goal_dt    -- behavior happened before goal
    )
  GROUP BY
    t.tag,
    t.app,
    fu.sub_session_id,
    fu.utm_campaign,
    fu.session_started,
    frequency_bucket
)
SELECT 
  -- per visit, per tag, per frequency, per etc...
  ts.sub_session_id as sub_session_id,
  ts.app as app,
  ts.utm_campaign as utm_campaign,
  ts.session_started as session_started,
  ts.tag as tag,
  ts.frequency_bucket as frequency_bucket,
  -- ts.feedback_givers_with_tag as feedback_givers_with_tag,
  -- ts.positive_count as positive_count,
  -- ts.negative_count as negative_count,
  ts.did_thing > 0 as has_positive,
  ts.didnt_did_thing > 0 as has_negative,
  
  -- TOTALS
  b.total_did_thing as total_did_thing,
  b.did_thing as did_thing,
FROM
  tag_sentiment as ts
CROSS JOIN 
  baseline_sentiment as b

ORDER BY
  has_positive DESC
)
```
