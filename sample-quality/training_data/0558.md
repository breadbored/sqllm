How does the SAFE prefix change function behavior when an error occurs?

--- AGENT ---

The SAFE prefix causes a function to return NULL instead of raising an error when the function encounters an invalid input value. If the function completes successfully, SAFE has no effect on the result.

```googlesql
SELECT SAFE.SUBSTR('foo', 0, -2) AS safe_output UNION ALL
SELECT SAFE.SUBSTR('bar', 0, 2) AS safe_output;

/*-------------+
 | safe_output |
 +-------------+
 | NULL        |
 | ba          |
 +-------------*/
```
