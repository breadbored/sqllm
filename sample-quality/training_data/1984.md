## Question
Create a view of Datadog session events for our application domains (app.dataprotocol.com and docs.dataprotocol.com), normalizing all string fields to lowercase, extracting UTM parameters, course IDs from URL paths, and including a cross-reference array of all client IDs.

## Step 2: Identify sources and relationships

This is fundamentally a **single-table projection** from `dd_events` with column renaming and transformation. There is no join in the traditional sense.

The `client_ids` column is produced by a **scalar subquery** against `dp-dwh.dev_sources.clients` — it selects all `id` values from that table and wraps them in an `ARRAY()`. This is effectively a cross-reference — every row in the output gets the same array of all client IDs. There is no join key; the subquery is independent of the outer row.

The `WHERE` clause filters to two specific hostnames: `app.dataprotocol.com` and `docs.dataprotocol.com`.

## Step 3: Identify aggregation depth

There is **no aggregation** in this query. It is a 1:1 row projection — each input row from `dd_events` (post-filter) maps to exactly one output row. The only structural transformation is column aliasing, `LOWER()` normalization, `SUBSTR()` extraction, `REGEXP_EXTRACT()` for course ID parsing, and the scalar array subquery.

## Step 4: Identify dependencies

- All output columns except `course_id`, `environment`, `client_ids`, and `hostname` are direct `LOWER()` renames of source columns.
- `environment` depends on `attributes_tags_2` — it strips the first 4 characters (likely a prefix like `env:`) via `SUBSTR(..., 5)` and lowercases the result.
- `course_id` depends on `attributes_view_url` — a regex extracts a numeric ID from URL paths matching `/courses/<digits>` or `/short-codes/<digits>`.
- `hostname` is `attributes_view_url_host` passed through **without** `LOWER()` — notably the only string column not normalized, and a duplicate of `host` (which is the lowered version). This appears intentional or a minor inconsistency.
- `client_ids` is an independent scalar subquery — no dependency on outer row data.
- The `WHERE` clause depends on `attributes_view_url_host` (raw, not lowered).

```sql
SELECT
  LOWER(attributes_session_id) AS session_id,
  LOWER(attributes_view_url_query_utm_id) AS utm_id,
  LOWER(attributes_view_url_query_utm_campaign) AS utm_campaign,
  LOWER(attributes_view_url_query_utm_medium) AS utm_medium,
  LOWER(attributes_view_url_query_utm_source) AS utm_source,
  LOWER(attributes_view_url_query_utm_term) AS utm_term,
  attributes_usr_id AS user_id,
  LOWER(attributes_usr_authed) AS logged_in,
  LOWER(attributes_view_url_host) AS host,
  LOWER(attributes_service) AS service,
  LOWER(SUBSTR(attributes_tags_2, 5)) AS environment,
  LOWER(attributes_session_useragent) AS user_agent,
  LOWER(attributes_view_url_path) AS url_path,
  LOWER(attributes_resource_url_path) AS resource_url_path,
  LOWER(attributes_view_url) AS view_url,
  LOWER(attributes_view_loading_type) AS view_loading_type,
  attributes_timestamp AS timestamp,
  LOWER(attributes_geo_city) AS city,
  LOWER(attributes_geo_country) AS country,
  LOWER(attributes_geo_country_iso_code) AS country_code,
  attributes_device_type AS device_type,
  LOWER(attributes_view_referrer) AS referrer,
  LOWER(attributes_view_referrer_url_url_path) AS referrer_url,
  attributes_action_name AS action_name,
  attributes_resource_url_query_en AS event,
  REGEXP_EXTRACT(attributes_view_url, r'/(?:courses|short-codes)/(\d+)') AS course_id,
  attributes_view_url_host AS hostname,
  ARRAY(SELECT id FROM `dp-dwh.dev_sources.clients`) as client_ids,
FROM
  ${ref("dd_events")}
WHERE
  attributes_view_url_host = 'app.dataprotocol.com'
  OR attributes_view_url_host = 'docs.dataprotocol.com'
```
