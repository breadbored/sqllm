## Question
I need to enrich the lessons table with a list of clients that have access to each lesson. Can you provide an example BigQuery query?

--- AGENT ---

This query enriches the lessons table with information about which clients have access to each lesson through a two-stage approach.

First, a staging CTE (`stg_lessons`) builds a mapping of lesson IDs to client IDs. It joins through three tables — clients_course_containers, courses, and course_lessons — to trace the relationship from lessons back to clients. For each lesson, it aggregates distinct client IDs into an array using ARRAY_AGG, ignoring null values.

Then, the main query selects all columns from the lessons table (with description converted to lowercase) and performs a LEFT JOIN with the staging CTE on lesson ID. For lessons with no matching clients in the CTE, the COALESCE function provides a default fallback of `[1]`.

```sql
WITH
stg_lessons as (
    SELECT
        cl.lesson_id as lesson_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    GROUP BY
        cl.lesson_id
)


SELECT
    l.id,
    l.name,
    l.created,
    l.updated,
    l.container_id,
    l.updated_admin_id,
    l.creator_admin_id,
    l.version,
    l.published,
    l.mobile_friendly,
    LOWER(description) AS description,
    l.video_id,
    l.thumbnail,
    l.has_been_published,
    l.archived,
    l.theater_view,
    l.show_lab,
    l.skippable,
    COALESCE(sl.client_ids, [1]) AS client_ids
FROM
    ${ref("lessons")} as l
LEFT JOIN
    stg_lessons as sl
ON
    sl.lesson_id = l.id
```
