## Question
Generate a BigQuery query that creates temporal attribute tags for events across multiple time granularities, grouped by sub-session and context.

--- AGENT ---

This query generates temporal attribute tags by creating one row for each combination of time granularity (day_of_week, month, hour, day_period, year) and event context (sub_session_id, app, url_path). For each time granularity, it formats the event timestamp according to a specific pattern—for example, 'day_of_week' produces the format "date:day_of_week:1:Monday", while 'month' produces "date:month:03:March". The timestamp is converted to the timezone specified in the `signals_timezone` column, or UTC if null. Results are grouped by sub_session_id, app, url_path, and the generated tag, with each group's earliest created timestamp recorded as dt. The output includes a constant class value of 'attribute' for all rows.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  CONCAT(
    'date:',
    t,
    ':',
    FORMAT_DATE(
      CASE t
        WHEN 'day_of_week' THEN '%u:%A'
        WHEN 'month' THEN '%m:%B'
        WHEN 'hour' THEN '%k'
        WHEN 'day_period' THEN '%p'
        ELSE '%Y' -- Year by default
      END,
      DATETIME(TIMESTAMP(es.created), IF(es.signals_timezone IS NOT NULL, es.signals_timezone, 'UTC'))
    )
  ) as tag,
  'attribute' as class,
  es.app as app,
  es.url_path as url_path,
  MIN(es.created) as dt

FROM
  ${ref('base_buckets_events_speedy')} as es ,
  UNNEST([
    'day_of_week',
    'month',
    'hour',
    'day_period',
    'year'
  ]) as t

GROUP BY
  sub_session_id,
  tag,
  app,
  url_path
```
