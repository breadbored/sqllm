## Question
For each user-course enrollment, show the course's container, the user's note counts (user-written and prefilled), the total resource count for the course (combining lesson-level and course-level resources), the enrollment date, and the list of client IDs associated with that course container.

--- AGENT ---

## Step 1: Identify shapes

**Input columns (across all tables):**
- `stg_user_courses`: `user_id`, `course_id`, `created`
- `stg_users`: `id`
- `stg_courses`: `id`, `container_id`
- `stg_clients_course_containers`: `course_container_id`, `client_id`
- `stg_course_notes`: `user_id`, `course_id`, `note`, `prefilled_note`
- `stg_lesson_resources`: `id`, `lesson_id`
- `stg_course_lessons`: `course_id`, `lesson_id`
- `stg_course_resources`: `id`, `course_id`

**Target shape:**
`user_id, course_id, container_id, user_note_count, prefilled_note_count, resource_count, course_started, client_id[]`

One row per user-course enrollment, with an array of distinct client IDs.

## Step 2: Identify sources and relationships

- `stg_user_courses.user_id` → `stg_users.id` (INNER JOIN — filters to valid users)
- `stg_user_courses.course_id` → `stg_courses.id` (INNER JOIN — gets `container_id`)
- `stg_courses.container_id` → `stg_clients_course_containers.course_container_id` (INNER JOIN — fans out by client, producing multiple `client_id` values per course container)
- `stg_course_notes` joins to `stg_user_courses` on both `user_id` AND `course_id` (LEFT JOIN — notes are optional)
- `stg_lesson_resources.lesson_id` → `stg_course_lessons.lesson_id` (INNER JOIN — maps lesson resources to courses)
- `stg_course_resources.course_id` → `stg_course_lessons.course_id` (LEFT JOIN — course-level resources are optional)
- The resource subquery joins to `stg_user_courses` on `course_id` (LEFT JOIN — resources are optional)

The `stg_clients_course_containers` join is the fan-out source: a single course container can belong to multiple clients. This is why the outer query needs a `GROUP BY` with `ARRAY_AGG(DISTINCT client_id IGNORE NULLS)` — to collapse the client fan-out back to one row per user-course.

## Step 3: Identify aggregation depth

There are three aggregation levels, two of which are pre-aggregated in subqueries before joining to the main spine:

**Resource count subquery (nested two levels):**
```
course_id (via course_resources), COUNT(cr.id) as course_resource_count
-> course_id (via lesson_resources + course_lessons), COUNT(lr.id) as lesson_resource_count, ANY_VALUE(course_resource_count) as course_resource_count, (lesson_resource_count + IFNULL(course_resource_count, 0)) as resource_count
```

**Note count subquery (one level):**
```
user_id, course_id, COUNTIF(note != '') as user_note_count, COUNT(prefilled_note) as prefilled_note_count
```

**Outer query (one level — collapsing client fan-out):**
```
user_id, course_id, container_id, user_note_count, prefilled_note_count, resource_count, created, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_id
```

## Step 4: Identify dependencies

- **Grouping drivers:** `user_id` and `course_id` are the primary grain of the output.
- **Derived columns:**
  - `user_note_count`, `prefilled_note_count` — derived from `stg_course_notes`, must be pre-aggregated to `(user_id, course_id)` grain before joining to avoid row multiplication.
  - `resource_count` — derived from two sources (`stg_lesson_resources` via `stg_course_lessons`, and `stg_course_resources`), must be pre-aggregated to `course_id` grain before joining.
  - `client_id[]` — derived from the fan-out join to `stg_clients_course_containers`, collapsed via `ARRAY_AGG(DISTINCT ... IGNORE NULLS)` in the outer GROUP BY.
- **Intermediate steps:**
  - `course_resource_count` must be aggregated per `course_id` before it can be combined with `lesson_resource_count` via `ANY_VALUE` in the resource subquery.
- `container_id` is a pass-through from `stg_courses`.
- `created` is a pass-through from `stg_user_courses`, aliased as `course_started`.

## Step 5: Plan CTE layers bottom-up

This query uses inline subqueries rather than named CTEs, but the logical layers are:

1. **`cr` (course resources subquery):** Counts course-level resources per `course_id`.
2. **`r` (resource count subquery):** Joins lesson resources to course_lessons to get per-course lesson resource counts, then LEFT JOINs the `cr` subquery to combine both resource types into a single `resource_count` per `course_id`.
3. **`cn` (course notes subquery):** Counts user-written notes (`COUNTIF(note != '')`) and prefilled notes (`COUNT(prefilled_note)`) per `(user_id, course_id)`.
4. **Outer query:** Builds the user-course spine from `stg_user_courses`, joins `stg_users` (validation), `stg_courses` (container_id), `stg_clients_course_containers` (client fan-out), then LEFT JOINs the pre-aggregated `cn` and `r` subqueries. Groups by the enrollment grain to collapse the client fan-out via `ARRAY_AGG`.

## Step 6: Write each CTE with justification

**`cr` subquery:** Pre-aggregates `stg_course_resources` to one row per `course_id` with a count, so it can be joined without multiplying rows in the resource subquery.

**`r` subquery:** Joins `stg_lesson_resources` to `stg_course_lessons` to map lesson-level resources to courses, then LEFT JOINs the pre-aggregated `cr` to combine both resource types. Uses `ANY_VALUE(cr.course_resource_count)` because `cr` is already at `course_id` grain and will have the same value for every row within a `course_id` group. Produces one row per `course_id` with the total `resource_count`.

**`cn` subquery:** Pre-aggregates `stg_course_notes` to `(user_id, course_id)` grain. `COUNTIF(note != '')` counts only notes where the user actually wrote something. `COUNT(prefilled_note)` counts all rows that have a non-null prefilled note (standard COUNT excludes nulls).

**Outer query:** The spine is `stg_user_courses`. The INNER JOIN to `stg_users` filters to valid users. The INNER JOIN to `stg_courses` retrieves `container_id`. The INNER JOIN to `stg_clients_course_containers` fans out by client — this is intentional, as the output needs `ARRAY_AGG(DISTINCT client_id)`. The LEFT JOINs to `cn` and `r` attach pre-aggregated metrics without row multiplication. The GROUP BY collapses the client fan-out.

## Step 7: Compose the final query

```sql
SELECT
  uc.user_id,
  uc.course_id,
  c.container_id,
  cn.user_note_count,
  cn.prefilled_note_count,
  r.resource_count,
  uc.created AS course_started,
  ARRAY_AGG(DISTINCT client_id IGNORE NULLS) AS client_id
FROM
  ${ref("stg_user_courses")} uc
JOIN
  ${ref("stg_users")} u
ON
  u.id = uc.user_id
JOIN
  ${ref("stg_courses")} c
ON
  c.id = uc.course_id
JOIN
  ${ref("stg_clients_course_containers")} ccc
ON
  ccc.course_container_id = c.container_id
LEFT JOIN (
  SELECT
    cn.user_id,
    cn.course_id,
    COUNTIF(note != '') AS user_note_count,
    COUNT(prefilled_note) AS prefilled_note_count
  FROM
    ${ref("stg_course_notes")} cn
  GROUP BY
    user_id,
    course_id) cn
ON
  cn.user_id = uc.user_id
  AND cn.course_id = uc.course_id
LEFT JOIN (
  SELECT
    cl.course_id,
    COUNT(lr.id) AS lesson_resouce_count,
    ANY_VALUE(cr.course_resource_count) AS course_resouce_count,
    (COUNT(lr.id) + IFNULL(ANY_VALUE(cr.course_resource_count), 0)) AS resource_count
  FROM
    ${ref("stg_lesson_resources")} lr
  JOIN
    ${ref("stg_course_lessons")} cl
  ON
    cl.lesson_id = lr.lesson_id
  LEFT JOIN (
    SELECT
      cr.course_id,
      COUNT(cr.id) AS course_resource_count
    FROM
      ${ref("stg_course_resources")} cr
    GROUP BY
      cr.course_id) cr
  ON
    cr.course_id = cl.course_id
  GROUP BY
    cl.course_id) r
ON
  r.course_id = uc.course_id
GROUP BY
  uc.user_id,
  uc.course_id,
  c.container_id,
  cn.user_note_count,
  cn.prefilled_note_count,
  r.resource_count,
  uc.created
```
