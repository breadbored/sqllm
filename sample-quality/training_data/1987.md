## Question
Give me all document certifications with their associated client IDs. Each certification should include an array of distinct client IDs linked to its document, defaulting to [1] if no clients are associated.

--- AGENT ---

## Step 1: Identify Shapes

**Input shape (joined):**
`documents_clients.doc_id, documents_clients.client_id, document_certifications.id, document_certifications.document_id, document_certifications.certification_id, document_certifications.order, document_certifications.created, document_certifications.updated`

**Target shape:**
`id, document_id, certification_id, order, created, updated, client_ids`

One row per document certification, with a nested array column `client_ids` containing the distinct clients associated with that document.

## Step 2: Identify Sources and Relationships

Two tables:

- `document_certifications` — the base entity, one row per certification-document relationship
- `documents_clients` — maps documents to clients, many-to-one relationship (multiple clients per document)

**Join relationship:** `documents_clients.doc_id = document_certifications.document_id`

This is a LEFT JOIN because not every document certification necessarily has client associations — the query must preserve all certification rows regardless.

The join is indirect: `documents_clients` relates to `document_certifications` through the shared document identifier (`doc_id` ↔ `document_id`).

**Fan-out risk:** Since `documents_clients` has multiple rows per `doc_id`, joining it directly to `document_certifications` would duplicate certification rows. This must be resolved by pre-aggregating `documents_clients` to one row per document before joining.

## Step 3: Identify Aggregation Depth

Single-level aggregation in a pre-join CTE:

```
doc_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

No further aggregation after the join — the main SELECT is a simple 1:1 join between `document_certifications` and the pre-aggregated `doc_clients` CTE.

Arrow notation:

```
doc_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
-> (joined to document_certifications) id, document_id, certification_id, order, created, updated, COALESCE(client_ids, [1]) as client_ids
```

## Step 4: Identify Dependencies

- **Grouping driver:** `doc_id` in `documents_clients` drives the pre-aggregation
- **Derived column:** `client_ids` — the array of distinct client IDs per document
- **Intermediate step required:** The `documents_clients` table must be collapsed to one row per `doc_id` before joining, to prevent row duplication in the final output
- **Default value:** `COALESCE(..., [1])` handles the LEFT JOIN null case — documents with no client associations get a default array containing client ID 1 (likely a system/default client)

## Step 5: Plan CTE Layers Bottom-Up

1. **`doc_clients`** — Pre-aggregates `documents_clients` to one row per document, collecting distinct client IDs into an array. This eliminates the fan-out risk before joining to certifications.

The final SELECT joins `document_certifications` to `doc_clients` and applies the COALESCE default.

## Step 6: Write Each CTE with Justification

**`doc_clients`:** Collapses the many-to-one `documents_clients` table into one row per `doc_id` with an array of distinct `client_id` values, so the subsequent LEFT JOIN produces exactly one output row per document certification without duplication. `IGNORE NULLS` ensures null client IDs are excluded from the array.

## Step 7: Compose the Final Query

```sql
WITH
doc_clients AS (
  SELECT
    dc.doc_id as doc_id,
    ARRAY_AGG(DISTINCT dc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("documents_clients")} as dc
  GROUP BY
    dc.doc_id
)
SELECT
  dcert.id,
  dcert.document_id,
  dcert.certification_id,
  dcert.`order`,
  dcert.created,
  dcert.updated,
  COALESCE(dc.client_ids, [1]) AS client_ids
FROM
  ${ref("document_certifications")} as dcert
LEFT JOIN
  doc_clients as dc
ON
  dc.doc_id = dcert.document_id
```

## Step 8: Verify Output Shape

**Target shape:** `id, document_id, certification_id, order, created, updated, client_ids`

**Final SELECT columns:** `dcert.id, dcert.document_id, dcert.certification_id, dcert.order, dcert.created, dcert.updated, COALESCE(dc.client_ids, [1]) AS client_ids`

✓ Seven columns, matching the target shape exactly. One row per document certification. The LEFT JOIN preserves all certification rows, and the pre-aggregation CTE ensures no fan-out. The COALESCE guarantees every row has a non-null `client_ids` array.
