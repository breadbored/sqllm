Create a query that extracts device-type-based tags from event data, capturing the earliest application, earliest URL path, and earliest timestamp for each device type within a sub-session. Classify these as attributes.

--- AGENT ---

This query partitions event data by sub-session and device type, then extracts temporal identity information for each partition.

The query:
1. Filters the events table to rows where `user_agent_json` contains a parseable `device_type` field
2. Groups by `sub_session_id` and a computed tag field that concatenates the string literal 'device_type_' with the extracted device type value
3. For each group, captures:
   - The `app` value associated with the earliest event (by `created` timestamp)
   - The `url_path` value associated with the earliest event
   - The minimum `created` timestamp in the group
4. Assigns a literal class label 'attribute' to all output rows

The `MIN_BY` function ensures that both `app` and `url_path` are selected from the same earliest event, preserving structural coherence. The output structure suggests this is an intermediate step in a tagging or feature engineering pipeline where device types are being classified as categorical attributes at the sub-session level.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  CONCAT('device_type_', JSON_EXTRACT_SCALAR(es.user_agent_json, '$.device_type')) as tag,
  'attribute' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref('base_buckets_events_app_speedy')} as es 

WHERE
  JSON_EXTRACT_SCALAR(es.user_agent_json, '$.device_type') IS NOT NULL

GROUP BY
    sub_session_id,
    tag
```
