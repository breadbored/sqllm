How can I safely use ERROR in a WHERE clause to ensure it only evaluates when needed?

--- AGENT ---

This query uses IF to control when the ERROR function is evaluated, ensuring the error only occurs if the condition x > 0 is false. This prevents unpredictable evaluation order from causing unintended errors.

```googlesql
SELECT *
FROM (SELECT -1 AS x)
WHERE IF(x > 0, true, ERROR(FORMAT('Error: x must be positive but is %t', x)));
```
