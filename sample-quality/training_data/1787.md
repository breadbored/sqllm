## Question
I need a query that shows monthly counts of knowledge check answers along with a running total, filtered to a specific client, container, and question. Can you provide an example?

--- AGENT ---

This table-valued function aggregates user knowledge check answers by month and calculates a running cumulative total. The query performs a multi-table join chain starting from `stg_user_knowledge_check_answers`, filtering to non-test users, and tracing through question options, questions (via a self-join that matches on name), courses, and client-container mappings to ensure only answers in the specified client and container are included.

The inner query groups answers by month (formatted as 'Mon YY') and counts them per month. The outer query applies a window function that calculates the cumulative sum of monthly answer counts, ordered chronologically from earliest to latest month. The result contains three columns: the formatted month, the count of answers in that month, and the running total of all answers up to and including that month.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64,
    knowledgeCheckQuestionId INT64) AS (
  SELECT
    date,
    monthly_answers,
    SUM(monthly_answers) OVER (ORDER BY PARSE_DATE('%b %y', date) ASC) AS cumulative_answers
  FROM (
    SELECT
      FORMAT_DATE('%b %y', ukca.created) AS date,
      COUNT(ukca.id) AS monthly_answers
    FROM
      ${ref("stg_user_knowledge_check_answers")} ukca
    JOIN
      ${ref("stg_users")} u
    ON
      u.id = ukca.user_id
    JOIN
      ${ref("stg_question_options")} qo
    ON
      qo.id = ukca.question_id
      AND qo.id = ukca.question_id
    JOIN (
      SELECT
        _q2.*
      FROM
        ${ref("stg_questions")} AS _q
      JOIN
        ${ref("stg_questions")} AS _q2
      ON
        _q.name = _q2.name
      WHERE
        _q.id = knowledgeCheckQuestionId ) AS kc
    ON
      kc.id = qo.id
    JOIN
      ${ref("stg_courses")} c
    ON
      c.id = ukca.course_id
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    WHERE
      ccc.client_id = clientId
      AND container_id = containerId
      AND u.test_user = 0
    GROUP BY
      date,
      kc.name ) AS subquery
  ORDER BY
    PARSE_DATE('%b %y', date))
```
