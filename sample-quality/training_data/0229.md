What is the default sort order for strings in BigQuery when no collation is specified?

--- AGENT ---

Without any collation specification, BigQuery sorts strings in case-sensitive Unicode code point order, where uppercase letters come before lowercase letters of the same letter. This query shows the default behavior where 'B' sorts before 'a' because the code point for uppercase 'B' is less than the code point for lowercase 'a'.

```googlesql
SELECT *
FROM UNNEST([
  'B',
  'b',
  'a'
]) AS character
ORDER BY character
```
