Can you show me a BigQuery query that identifies returning user sessions based on account creation dates?

--- AGENT ---

This query identifies sub-sessions belonging to returning users—accounts that existed before the events in the sub-session occurred. It performs a join between an events table (`base_buckets_events_app_speedy`) and a users table (`stg_users`) on user ID, then groups by sub-session and user creation date. For each group, it captures the earliest app and URL path accessed within that sub-session (using MIN_BY to order by event timestamp), along with the earliest event timestamp. The HAVING clause filters to only sub-sessions where the user's account creation date precedes the first event date, ensuring only returning users are included. Each result is tagged as 'returning_user' behavior.

SELECT
  eas.sub_session_id AS sub_session_id,
  'returning_user' AS tag,
  'behavior' as class,
  MIN_BY (eas.app, eas.created) AS app,
  MIN_BY (eas.url_path, eas.created) AS url_path,
  MIN(eas.created) AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS eas
  JOIN ${ref("stg_users")} AS u ON u.id = eas.user_id
GROUP BY
  eas.sub_session_id,
  u.created
HAVING
  DATE(u.created) < DATE(MIN(eas.created))
