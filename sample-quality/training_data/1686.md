Show me a BigQuery SQL example that identifies when users gave feedback during their app sessions, tagging those events for behavioral analysis.

--- AGENT ---

This query performs a temporal join between app events and feedback submissions to create behavioral tags. It joins events from `base_buckets_events_app_speedy` with feedback records from `stg_feedback` on matching user IDs. The GROUP BY with HAVING clause filters to retain only feedback events that occurred within the time window of events in each sub_session/app/url_path combination—that is, feedback timestamps that fall between the earliest and latest event timestamp in that group. The query outputs one row per qualifying feedback event, enriched with the sub_session ID, app, and URL path context in which the feedback occurred, and applies a fixed 'gave_feedback' tag and 'behavior' classification to all rows. This is a common pattern for generating labeled training data that correlates user feedback with their concurrent app behavior.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  'gave_feedback' AS tag,
  'behavior' as class,
  es.app AS app,
  es.url_path AS url_path,
  f.created AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS es
JOIN
  ${ref("stg_feedback")} AS f
ON
  es.user_id = f.user_id
GROUP BY
  es.sub_session_id,
  es.app,
  es.url_path,
  f.created
HAVING
  f.created BETWEEN MIN(es.created) AND MAX(es.created)
```
