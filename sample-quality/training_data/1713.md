--- USER ---

Show me how to identify user sessions that contain both app events and lab usage activity occurring within the same time window, and tag them for downstream analysis.

--- AGENT ---

This query correlates app session activity with lab feature usage by establishing session time boundaries and matching them against overlapping lab events.

The CTE `session_bounds` first groups app events by session ID, user, app name, and URL path, computing the earliest and latest event timestamps for each session. This creates a denormalized session-level view with temporal bounds.

The main query then performs a temporal join: it matches records from the lab usage table (`stg_user_lab_usage`) to sessions based on two conditions: (1) the same user ID, and (2) the lab usage timestamp falling within the session's start and end times. For each matching session, it adds static classification columns (`tag='lab_user'` and `class='content'`), groups by session and route identifiers, and returns the earliest lab usage timestamp that occurred within that session window.

The result is a tagged dataset of sessions with correlated lab activity, useful for cohort analysis or behavioral segmentation.

```sql
WITH session_bounds AS (
  SELECT
    sub_session_id,
    user_id,
    e.app as app,
    e.url_path as url_path,
    MIN(created) as session_start,
    MAX(created) as session_end
  FROM
    ${ref("base_buckets_events_app_speedy")} as e
  GROUP BY
    sub_session_id, e.app, e.url_path, user_id
)
SELECT
  sb.sub_session_id,
  'lab_user' as tag,
  'content' as class,
  sb.app as app,
  sb.url_path as url_path,
  MIN(ulu.created) as dt
FROM 
  session_bounds as sb
JOIN
  ${ref("stg_user_lab_usage")} as ulu
ON
  sb.user_id = ulu.user_id
AND
  ulu.created BETWEEN sb.session_start AND sb.session_end
GROUP BY
  sb.sub_session_id,
  sb.app,
  sb.url_path
```
