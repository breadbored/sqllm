## Question
Show me an example of a BigQuery table-valued function that counts distinct users enrolled in courses for a specific client and container.

--- AGENT ---

This is a BigQuery table-valued function parameterized by `clientId` and `containerId`. It returns a single-row result set containing a count of distinct users.

The function workflow:

1. **Identify courses**: Joins `agg_client_courses` (client-to-course mappings) with `stg_courses` (course metadata) on `container_id`, filtering for the supplied parameters. Groups by client ID and course ID.

2. **Enroll users**: Left joins the course set from step 1 to `stg_user_courses` (user enrollments) on course ID, preserving all courses even if they have no users.

3. **Validate users**: Inner joins to `stg_users` (user records) on user ID, effectively filtering to only users that exist in the user table.

4. **Count**: Counts distinct user IDs across this joined set.

Output: a single column `unique_users` containing the total number of distinct users enrolled in courses matching the input parameters.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    COUNT(DISTINCT uc.user_id) AS unique_users
  FROM (
    SELECT
      cc.client_id,
      c.id AS course_id
    FROM
      ${ref("agg_client_courses")} cc
    JOIN
      ${ref("stg_courses")} c
    ON
      c.container_id = cc.container_id
    WHERE
      cc.client_id = clientId
      AND c.container_id = containerId
    GROUP BY
      cc.client_id,
      c.id ) AS ccc
  LEFT JOIN
    ${ref("stg_user_courses")} uc
  ON
    uc.course_id = ccc.course_id
  JOIN
    ${ref("stg_users")} u
  ON
    u.id = uc.user_id)
```
