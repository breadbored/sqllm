--- USER ---

Show me a BigQuery query that identifies sub-sessions where users are returning from inactivity and extracts the entry point information (first app and URL path) for each.

--- AGENT ---

This query filters the `events_speedy` table to only events marked as `back_from_inactive = true`, then collapses each `sub_session_id` into a single row. For each group, it uses `MIN_BY` to extract the app and url_path from the chronologically earliest event (by `created` timestamp), retrieves the session's minimum timestamp, and tags the row with a class label "behavior" and a tag constructed from the literal string 'back_from_active_' concatenated with the boolean value of `back_from_inactive`. Since the WHERE clause constrains all rows to `back_from_inactive = true`, the tag will always be 'back_from_active_true'.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  CONCAT('back_from_active_', es.back_from_inactive) as tag,
  'behavior' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  `dev_sources.events_speedy` as es

WHERE
  es.back_from_inactive = true

GROUP BY 
  sub_session_id,
  tag
```
