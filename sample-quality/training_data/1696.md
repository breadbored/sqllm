Show me an example BigQuery SQL query that identifies events from users created on the same day as their first activity, with a timestamp inconsistency that warrants investigation.

--- AGENT ---

This query identifies sub-sessions from users whose account creation date matches the calendar date of their earliest event in that session, but whose creation timestamp falls outside the time window spanned by the events (either before the first event or after the last one—a structural inconsistency that may indicate timezone mismatch or data quality issues).

The query:
1. Joins events (from `base_buckets_events_app_speedy`) with users (from `stg_users`) on user ID
2. Groups by sub-session ID and user creation time
3. Tags matching records as "newish_user" with class "behavior"
4. Uses `MIN_BY` to extract the app and URL path from the chronologically earliest event in each sub-session
5. Captures the timestamp of that earliest event
6. Filters via HAVING to keep only sessions where:
   - The user was created on the same calendar day as their earliest event (`DATE(u.created) = DATE(MIN(eas.created))`)
   - AND the user creation timestamp does not fall between the min and max event times in that session (`u.created NOT BETWEEN MIN(eas.created) AND MAX(eas.created)`)

The result is a tagged dataset of anomalous new-user sessions suitable for downstream investigation.

```sql
SELECT
  eas.sub_session_id AS sub_session_id,
  'newish_user' AS tag,
  'behavior' as class,
  MIN_BY(eas.app, eas.created) AS app,
  MIN_BY(eas.url_path, eas.created) AS url_path,
  MIN(eas.created) AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS eas
JOIN
  ${ref("stg_users")} AS u
ON
  u.id = eas.user_id
GROUP BY
  eas.sub_session_id,
  u.created
HAVING
  DATE(u.created) = DATE(MIN(eas.created))
AND
  u.created NOT BETWEEN MIN(eas.created) AND MAX(eas.created)
```
