--- USER ---

Provide me with an example BigQuery SQL query that joins course sentiment data with client information and aggregates clients into arrays at the record level.

--- AGENT ---

This query enriches course sentiment records with associated client information by performing left joins to a client course container mapping table and a client master table. For each distinct combination of user, course, container, and question-answer pair (the grain preserved by the GROUP BY), it collects unique client IDs and client names into separate array columns. The ARRAY_AGG with DISTINCT ensures duplicate clients are collapsed into single array entries, while IGNORE NULLS excludes null client references that may result from the outer joins where no client association exists.

```sql
SELECT
  user_id,
  course_id,
  container_id,
  account_created,
  last_active,
  course_name,
  version,
  course_status,
  course_start_date,
  question_id,
  answer_id,
  question_value,
  answer_value,
  ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) AS client_id,
  ARRAY_AGG(DISTINCT clients.name IGNORE NULLS) AS client_name
FROM
  ${ref("agg_course_sentiment")} cs
LEFT JOIN
  ${ref("stg_clients_course_containers")} ccc
ON
  ccc.course_container_id = cs.container_id
LEFT JOIN
  ${ref("stg_clients")} clients
ON
  clients.id = ccc.client_id
GROUP BY
  user_id,
  course_id,
  container_id,
  account_created,
  last_active,
  course_name,
  version,
  course_status,
  course_start_date,
  question_id,
  answer_id,
  question_value,
  answer_value
```
