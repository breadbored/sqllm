Show me an example of how to identify completed learning paths that occurred during user sessions on specific applications.

--- AGENT ---

This query identifies badge completions that temporally co-occur with user application events, producing a tagged event stream for behavior classification. It joins three tables—badge completion records, user records, and application event logs—on user identity. The WHERE clause filters to completed badges only. The GROUP BY aggregates across sub-session boundaries while the HAVING clause applies a temporal constraint: the completion date must fall within the min and max event creation timestamps for that grouping, effectively filtering to completions that happened during an active event window. The output includes the session identifier, app context, URL path, and completion timestamp, all tagged as a 'learning_path_completed' behavior event.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  'learning_path_completed' AS tag,
  'behavior' as class,
  es.app AS app,
  es.url_path AS url_path,
  bc.completed_date AS dt
FROM
  ${ref('agg_user_badge_completion')} AS bc
  JOIN ${ref('users')} AS u ON u.id = bc.user_id
  JOIN ${ref('base_buckets_events_app_speedy')} AS es ON es.user_id = u.id
WHERE
  bc.did_complete = TRUE
GROUP BY
  sub_session_id,
  bc.completed_date,
  es.app,
  es.url_path
HAVING
  bc.completed_date BETWEEN MIN(es.created) AND MAX(es.created)
```
