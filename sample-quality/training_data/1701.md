Show me a BigQuery query that tags user events with learning path completion information, filtering for events that occurred within the timeframe of a badge completion.

--- AGENT ---

This query creates behavioral tags by joining user badge completions with user events, selecting only the events that fall temporally within each user's event session. 

The logic works as follows:
- It joins three tables: badge completions (with learning path names), users, and user events (with session identifiers and timestamps)
- For each combination of sub_session_id, learning path, app, and URL path, it assigns a tag prefixed with 'learning_path_' using the learning path name
- The HAVING clause filters to retain only groups where the completion date falls between the earliest and latest event timestamps for that user — this ensures the completion happened during an active event session
- The output includes the sub_session, the tag, a static 'behavior' class label, the app and URL path where events occurred, and the completion date

This pattern is useful for correlating user behavioral events with learning milestone achievements.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  CONCAT('learning_path_', bc.learning_path_name) AS tag,
  'behavior' as class,
  es.app AS app,
  es.url_path AS url_path,
  bc.completed_date AS dt
FROM
  ${ref('agg_user_badge_completion')} AS bc
  JOIN ${ref('users')} AS u ON u.id = bc.user_id
  JOIN ${ref('base_buckets_events_app_speedy')} AS es ON es.user_id = u.id
GROUP BY
  sub_session_id,
  bc.learning_path_name,
  bc.completed_date,
  es.app,
  es.url_path
HAVING
  bc.completed_date BETWEEN MIN(es.created) AND MAX(es.created)
```
