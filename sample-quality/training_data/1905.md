## Question
Show me a BigQuery query that enriches course lesson records with client information from the users table.

--- AGENT ---

This query enriches the `user_finished_course_lessons` table by left-joining each record with a denormalized array of client identifiers derived from the `users` table.

The `user_clients` CTE constructs an array for each user that contains the constant value `1` followed by that user's `external_client_id` (if present). The main query then performs a left join on this CTE, selecting all original columns from `user_finished_course_lessons` and populating a `client_ids` array column. For rows where the user has no external client ID (or doesn't match in the CTE), the `client_ids` column defaults to `[1]` via the COALESCE function.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  _utb.id,
  _utb.user_id,
  _utb.course_id,
  _utb.lesson_id,
  _utb.created,
  _utb.updated,
  _utb.skipped,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_finished_course_lessons")} as _utb
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = _utb.user_id
```
