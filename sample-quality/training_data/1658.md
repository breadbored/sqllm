--- USER ---

Generate screen attribute tags for each sub-session by capturing device category, orientation, and pixel dimensions at multiple levels of granularity. What different ways can we characterize a screen from the available event signals?

--- AGENT ---

This query produces four sets of screen attribute tags, each capturing device characteristics at different levels of detail:

**Branch 1** creates tags combining device category and orientation: `screen_{device_category}_{landscape|portrait}`. For each sub_session_id and tag, it records the earliest app, earliest url_path (by creation timestamp), and the minimum timestamp in that group.

**Branch 2** creates orientation-only tags: `screen_{landscape|portrait}`, removing the device category dimension to capture orientation behavior in isolation.

**Branch 3** creates detailed tags combining device category and exact pixel dimensions: `screen_{device_category}_{width}x{height}`, capturing the full device profile with category and resolution.

**Branch 4** creates pixel-only tags: `screen_{width}x{height}`, stripping device category to focus purely on screen resolution characteristics.

All branches filter to events where screen dimensions are available, aggregate by sub_session_id and their respective tag variant, and use MIN_BY to identify the earliest event for each dimension by the `created` timestamp. The result is a multi-granular set of screen attributes — each row tagged as an 'attribute' — that allow analysis of the same sub_session at different levels of screen characteristic aggregation.

```sql
SELECT
    es.sub_session_id as sub_session_id,
    CONCAT(
        'screen_', es.device_category, '_', 
        CASE 
        WHEN 
            es.signals_screen_avail_width > es.signals_screen_avail_height
        THEN 
            'landscape' 
        ELSE 
            'portrait' 
        END 
        ) as tag,
    'attribute' as class,
    MIN_BY(es.app, es.created) as app,
    MIN_BY(es.url_path, es.created) as url_path,
    MIN(es.created) as dt

FROM
    ${ref("base_buckets_events_speedy")} as es

WHERE
    es.signals_screen_avail_height IS NOT NULL
AND
    es.signals_screen_avail_width IS NOT NULL

GROUP BY
    sub_session_id,
    tag

UNION ALL
```
