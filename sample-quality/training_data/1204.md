How do I check if an optional field has been explicitly set in a protocol buffer?

--- AGENT ---

This query uses a virtual field `has_country` to detect whether the optional field `country` has been explicitly set in a nested protocol buffer message. The `has_` prefix is a special syntax that returns a boolean indicating whether the field has a value.

```googlesql
SELECT
  c.Orders.shipping_address.has_country
FROM
  Customer AS c;
```
