Show me a query that stages events data for a specific application, filtering based on app name and client, while standardizing the app column to a consistent value.

--- AGENT ---

This query creates a staging table of events filtered to a specific application. It selects all columns from the `all_events` table except the `app` column, which it replaces with a hardcoded value of `'devalliance'`.

The WHERE clause applies two filtering paths:
1. Events where the `app` column already equals `'devalliance'` are included directly
2. Events where `app` equals `'app'` AND `client` equals `'devalliance'` AND the event's created date is on or before 2026-02-13 are also included

This effectively consolidates events from two sources — those already tagged as devalliance, and recent client-specific events tagged with a generic app value — into a single staging table with normalized app naming.

```sql
/**
* Staging Events for Clients before Speedy Build
*
* Uses the event layers to create a staging table of events.
**/
SELECT
  e.* EXCEPT(app),
  'devalliance' as app,
  -- Add app specific columns below here from the metadata
FROM
  ${ref("all_events")} AS e
WHERE
  e.app = 'devalliance'
OR
  (
    e.app = 'app'
  AND
    e.client = 'devalliance'
  AND
    DATE(e.created) <= '2026-02-13'
  )
```
