## Question
Show me how to enrich an assessments_questions table with a list of client IDs for each assessment, pulling those client IDs from a course-containers relationship and defaulting to a fallback value when no clients exist.

--- AGENT ---

This query enriches the `assessments_questions` table by attaching to each assessment question record an array of distinct client IDs that are associated with that assessment.

The logic works in two steps:

**Step 1 (CTE `asessment_clients`):** The query joins `clients_course_containers` with `courses` on the container relationship, then groups by `assessment_id` to build an array of distinct client IDs for each assessment. This array captures all unique clients connected to an assessment through its course containers.

**Step 2 (Main query):** The query selects all columns from `assessments_questions` and left joins the aggregated client mapping. The `COALESCE` function provides a fallback: if an assessment has no associated clients (resulting in NULL from the left join), it defaults `client_ids` to `[1]`.

The result is a row per assessment question with its metadata plus the list of client IDs that can access or are associated with that assessment, or a default client ID if none are found.

```sql
WITH
asessment_clients AS (
    SELECT
        c.assessment_id as assessment_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    GROUP BY
        c.assessment_id
)
SELECT
  aq.id,
  aq.assessment_id,
  aq.question_id,
  aq.created,
  aq.updated,
  COALESCE(ac.client_ids, [1]) AS client_ids
FROM
  ${ref("assessments_questions")} as aq
LEFT JOIN
    asessment_clients as ac
ON
    ac.assessment_id = aq.assessment_id
```
