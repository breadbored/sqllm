--- USER ---

Identify the first scroll event within each user sub-session and extract the app and URL path where that scroll occurred.

--- AGENT ---

This query extracts the earliest scroll event for each sub-session from an events table. It filters the raw event stream for records where the event type is 'scroll' (the WHERE clause explicitly filters to name = 'scroll'), then groups by sub_session_id. For each sub-session, it uses MIN_BY to retrieve the app and url_path values from the scroll event with the earliest timestamp, and captures that earliest timestamp with MIN(created). The output includes one row per sub-session with the sub_session_id, a static tag of 'scroll', a class of 'behavior', the app and URL path from the first scroll event, and the creation timestamp of that event.

```sql
SELECT 
    es.sub_session_id as sub_session_id,
    'scroll' as tag,
    'behavior' as class,
    MIN_BY(es.app, es.created) as app,
    MIN_BY(es.url_path, es.created) as url_path,
    MIN(es.created) as dt
FROM
    ${ref("base_buckets_events_docs_speedy")} as es
WHERE
    es.name != 'page_view'
AND
    es.name != 'open_link'
AND
    es.name = 'scroll'

GROUP BY
    sub_session_id,
    tag
```
