## Question
Can you show me a BigQuery query that tracks user progress through courses on a daily basis, including cumulative totals and progress percentages?

--- AGENT ---

This query produces a daily-granular course progress report for each user, tracking how much progress they made, time spent, and percentage of course completed.

The query works through four main stages:

1. **pre_lesson_progress**: Aggregates raw lesson progress events by day, capturing the maximum progress reached in each lesson per day and total time spent, joined with the users table.

2. **lesson_progress**: Calculates daily deltas using a LAG window function. Since progress_seconds can remain flat or jump backward, this isolation extracts only the new progress made each day by subtracting the previous day's maximum from today's.

3. **course_progress** (the outer CTE): Aggregates lesson-level deltas to the daily course level, then layers on running cumulative totals using window functions ordered by date. This produces daily_progress_seconds, current_progress_seconds (cumulative to that day), daily_time_on_course, and total_time_on_course.

4. **course_length**: Pre-calculates the total duration of all videos in each course by joining lessons to videos.

The final SELECT joins these two branches, calculates progress percentages (daily and cumulative divided by course length), and caps all percentages at 1.0 using LEAST to prevent overflow when a user exceeds course length. Numeric formatting is applied using FORMAT() and CAST for output consistency.

```sql
WITH
  course_progress AS (
  WITH
    pre_lesson_progress AS (
    SELECT
      user_id,
      course_id,
      lesson_id,
      MAX(progress_seconds) AS user_total_lesson_progress,
      SUM(interval_seconds) AS time_on_lesson,
      DATE_TRUNC(uclp.created, DAY) AS progress_date,
    FROM
      ${ref("stg_user_course_lesson_progress")} AS uclp
    JOIN
      ${ref("stg_users")} u
    ON
      u.id = uclp.user_id
    GROUP BY
      user_id,
      course_id,
      lesson_id,
      progress_date
    ORDER BY
      user_id,
      course_id,
      lesson_id,
      progress_date),
    lesson_progress AS (
    SELECT
      lp.user_id,
      lp.course_id,
      lp.lesson_id,
      lp.user_total_lesson_progress - (GREATEST( COALESCE( LAG( lp.user_total_lesson_progress) OVER (PARTITION BY lp.user_id, lp.course_id, lp.lesson_id ORDER BY lp.progress_date ASC), 0), 0)) AS user_total_lesson_progress,
      lp.user_total_lesson_progress AS old_user_total_lesson_progress,
      lp.time_on_lesson,
      lp.progress_date,
    FROM
      pre_lesson_progress AS lp)
  SELECT
    lp.user_id AS user_id,
    lp.course_id AS course_id,
    lp.progress_date AS progress_date,
    SUM(lp.user_total_lesson_progress) AS daily_progress_seconds,
    SUM( GREATEST(SUM(lp.user_total_lesson_progress), 0)) OVER (PARTITION BY lp.user_id, lp.course_id ORDER BY lp.progress_date ASC) AS total_progress_seconds,
    SUM( SUM(lp.user_total_lesson_progress)) OVER (PARTITION BY lp.user_id, lp.course_id ORDER BY lp.progress_date ASC) AS current_progress_seconds,
    SUM(time_on_lesson) AS daily_time_on_course,
    SUM(SUM(time_on_lesson)) OVER (PARTITION BY user_id, course_id ORDER BY lp.progress_date) AS total_time_on_course
  FROM
    lesson_progress AS lp
  GROUP BY
    lp.user_id,
    lp.course_id,
    lp.progress_date
  ORDER BY
    lp.user_id,
    lp.course_id,
    lp.progress_date),
  course_length AS (
  SELECT
    cl.course_id,
    ROUND(SUM(v.duration), 3) AS course_length
  FROM
    ${ref("stg_lessons")} AS l
  LEFT JOIN
    ${ref("stg_videos")} AS v
  ON
    l.video_id = v.id
  LEFT JOIN
    ${ref("stg_course_lessons")} AS cl
  ON
    l.id = cl.lesson_id
  GROUP BY
    course_id
  ORDER BY
    course_id)
SELECT
  cp.user_id,
  cp.course_id,
  cp.progress_date,
  cp.daily_progress_seconds,
  cp.current_progress_seconds,
  cp.total_progress_seconds,
  cp.daily_time_on_course,
  cp.total_time_on_course,
  CAST(FORMAT('%g', cd.course_length) AS FLOAT64) AS course_length,
  CAST(FORMAT('%g', LEAST(ROUND((cp.daily_progress_seconds / cd.course_length), 2), 1)) AS FLOAT64) AS progress_percentage_made,
  CAST(FORMAT('%g', LEAST(ROUND((cp.total_progress_seconds / cd.course_length), 2), 1)) AS FLOAT64) AS total_course_progress_percentage,
  CAST(FORMAT('%g', LEAST(ROUND((GREATEST(cp.current_progress_seconds, 0) / cd.course_length), 2), 1)) AS FLOAT64) AS current_course_progress_percentage
FROM
  course_progress cp
JOIN
  course_length cd
ON
  cd.course_id = cp.course_id
```
