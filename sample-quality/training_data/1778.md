--- USER ---

Show me the monthly assessment counts for a specific client and course container, along with a running total of attempts over time. Exclude test users from the calculation.

--- AGENT ---

This is a BigQuery table function that produces monthly assessment activity with a cumulative total. 

The query joins four tables: user course assessments, users, courses, and a mapping table linking courses to client containers. It filters to a specific client and container, excluding test users.

The inner query counts assessments per month (formatted as 'Mon YY'). The outer query then applies a window function to create a running sum of those monthly counts, ordered chronologically. The result is three columns: the month, the number of assessment attempts that month, and the cumulative attempts up to that point.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    date,
    monthly_attempts,
    SUM(monthly_attempts) OVER (ORDER BY PARSE_DATE('%b %y', date) ASC) AS cumulative_attempts
  FROM (
    SELECT
      FORMAT_DATE('%b %y', uca.created) AS date,
      COUNT(uca.id) AS monthly_attempts
    FROM
      ${ref("stg_user_course_assessments")} uca
    JOIN
      ${ref("stg_users")} u
    ON
      u.id = uca.user_id
    JOIN
      ${ref("stg_courses")} c
    ON
      c.id = uca.course_id
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    WHERE
      ccc.client_id = clientId
      AND container_id = containerId
      AND u.test_user = 0
    GROUP BY
      date) AS subquery
  ORDER BY
    PARSE_DATE('%b %y', date))
```
