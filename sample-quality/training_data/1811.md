## Question
Show me a BigQuery table function that calculates engagement metrics for a specific client's courses by counting distinct users and their survey responses.

--- AGENT ---

This query defines a BigQuery table function that accepts a client ID as a parameter and returns two aggregate metrics:

1. **total_users** — the count of distinct users who answered sentiment or quality questions
2. **questions_answered** — the total count of question answer records

The query joins four tables:
- `stg_user_course_sentiment_question_answers` as the base fact table containing individual question responses
- `stg_users` to resolve user identities
- `stg_courses` to access course information
- `stg_clients_course_containers` to filter by the client's course container hierarchy

The WHERE clause restricts results to courses belonging to the specified `clientId` parameter. The function returns a single row summarizing engagement activity for that client.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64) AS (
  SELECT
    COUNT(DISTINCT user_id) AS total_users,
    COUNT(ucsqa.id) AS questions_answered
  FROM
    ${ref("stg_user_course_sentiment_question_answers")} ucsqa
  JOIN
    ${ref("stg_users")} u
  ON
    u.id = ucsqa.user_id
  JOIN
    ${ref("stg_courses")} c
  ON
    c.id = ucsqa.course_id
  JOIN
    ${ref("stg_clients_course_containers")} ccc
  ON
    ccc.course_container_id = c.container_id
  WHERE
    ccc.client_id = clientId)
```
