## Question
Can you help me understand a query that identifies suspicious user behavior patterns in lesson completion and tags those users' app session events?

--- AGENT ---

This query identifies users whose lesson progress increases by more than 30 seconds at a time (compared to the previous recorded maximum) within non-skippable, published lessons, then associates those suspicious behavior flags with the corresponding app session events.

The logic flows through four stages:

1. **Stage 1 — Window maximum tracking**: For each user-course-lesson combination, calculate a running maximum of progress_seconds ordered by creation time. This establishes what the highest progress was at each point in the history.

2. **Stage 2 — Lag previous maximum**: Using LAG, capture the previous row's running maximum. This allows detection of jumps: when current progress minus the previous maximum exceeds 30 seconds.

3. **Stage 3 — Aggregate cheating signals**: Filter the lesson progression table to only non-skippable lessons that have been published, then group by user-course-lesson. For each group, flag whether a suspicious jump occurred (cheated = true/false), and capture both the magnitude of the largest suspicious jump (cheated_by) and the timestamp when it happened (cheated_at).

4. **Stage 4 — Join to session events**: Cross-reference the cheaters list with app event records, matching on user_id and the date of the suspicious activity. The join also attempts multiple course_id match paths (direct course_id, derived_course_id, or course_id embedded in the URL path). Group by session identifiers and filter to events that bracket the cheating timestamp, then tag these sessions with the behavior class 'cheater'.

The final output is a denormalized list of app sessions associated with detected suspicious behavior.

```sql
WITH
user_course_lesson_progress_with_max AS (
  SELECT
    uclp.user_id as user_id,
    uclp.course_id as course_id,
    uclp.lesson_id as lesson_id,
    uclp.progress_seconds as progress_seconds,
    uclp.created as created,
    MAX(uclp.progress_seconds) OVER (
      PARTITION BY
        uclp.user_id,
        uclp.course_id,
        uclp.lesson_id
      ORDER BY
        uclp.created
    ) as max_progress_up_until_this_point
  FROM
    `dp-dwh.dev_sources.user_course_lesson_progress` as uclp
),
user_course_lesson_progress_with_last_max AS (
  SELECT
    uclpwm.user_id as user_id,
    uclpwm.course_id as course_id,
    uclpwm.lesson_id as lesson_id,
    uclpwm.progress_seconds as progress_seconds,
    uclpwm.max_progress_up_until_this_point as max_progress_up_until_this_point,
    LAG(uclpwm.max_progress_up_until_this_point) OVER (
      PARTITION BY
        uclpwm.user_id,
        uclpwm.course_id,
        uclpwm.lesson_id
      ORDER BY
        uclpwm.created
    ) as last_max_progress,
    uclpwm.created as created,
  FROM
    user_course_lesson_progress_with_max as uclpwm
),
cheaters AS (
  SELECT
    uclpwlm.user_id,
    uclpwlm.course_id,
    uclpwlm.lesson_id,
    MAX((uclpwlm.progress_seconds - uclpwlm.last_max_progress) > 30) as cheated,
    MAX_BY((uclpwlm.progress_seconds - uclpwlm.last_max_progress), (uclpwlm.progress_seconds - uclpwlm.last_max_progress) > 30) as cheated_by,
    MAX_BY(uclpwlm.created, (uclpwlm.progress_seconds - uclpwlm.last_max_progress) > 30) as cheated_at,
  FROM
    user_course_lesson_progress_with_last_max uclpwlm
  JOIN
    `dp-dwh.dev_sources.lessons` as l
  ON
    l.id = uclpwlm.lesson_id
  WHERE
    l.skippable = 0
  AND
    l.has_been_published = 1
  GROUP BY
    uclpwlm.user_id,
    uclpwlm.course_id,
    uclpwlm.lesson_id
  ORDER BY
    cheated DESC,
    cheated_by DESC
)

SELECT
  eas.sub_session_id AS sub_session_id,
  'cheater' AS tag,
  'behavior' as class,
  eas.app AS app,
  eas.url_path AS url_path,
  MIN(ch.cheated_at) AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS eas
JOIN
  cheaters as ch
ON
  ch.user_id = eas.user_id
AND
  DATE(ch.cheated_at) = DATE(eas.created)
AND
    (
        ch.course_id = eas.course_id
    OR
        ch.course_id = eas.derived_course_id
    OR
        SAFE_CAST(REGEXP_EXTRACT(eas.url_path, r's/(\d+)$', 1, 1) AS INT64) = ch.course_id
    )
GROUP BY
  sub_session_id,
  eas.app,
  eas.url_path
HAVING
  MIN(ch.cheated_at) BETWEEN MIN(eas.created) AND MAX(eas.created)
```
