--- USER ---

Build a video engagement summary per session segment. For each combination of session, sub-session, segment, page path, and video, aggregate playback metrics (time played, progress, completion), device/browser metadata, UTM attribution, geo location, visibility state, and whether the video had been played in a prior segment of the same session. Also include a count of distinct prior visit sessions to the same page (excluding visits from the same hour).

## 2. Identify sources and relationships

There is a single source table `vaas_events`. The query self-joins through several CTEs, all derived from the same table. The relationships are:

- **`pages_with_plays`** → filters to only (video_id, url_path) pairs where at least one `playing` event exists. Joined to `vaas_events` on `vaas_url_video_id` and normalized `vaas_url_url_path` to produce `events_with_played_pages_filtered`.
- **`visits`** → a two-level aggregation of `vaas_events` that counts distinct sub-sessions per (session_id, url_path). Joined to the main events on `session_id` and normalized `url_path`, with an additional filter excluding visits from the same hour (`DATE_TRUNC(..., HOUR) != DATE_TRUNC(..., HOUR)`).
- **`previously_played` chain** (`previously_played_base` → `previously_played_lag` → `previously_played`) → determines whether a video was played in any *prior* segment within the same session. Joined on `session_id, sub_session_id, vaas_url_video_id, vaas_url_url_path`.

All joins use normalized URL paths via `RTRIM(..., '/')`.

## 3. Identify aggregation depth

Multiple independent aggregation chains feed into the final aggregation:

**Chain A — page-play filter (pages_with_plays):**
```
vaas_url_video_id, vaas_url_url_path, [WHERE playing] -> GROUP BY vaas_url_video_id, vaas_url_url_path (existence filter, no agg expression needed)
```

**Chain B — visit counting (visits):**
```
session_id, sub_session_id, vaas_url_url_path, MIN(created) as first_visited
-> session_id, vaas_url_url_path, COUNT(DISTINCT sub_session_id) as sessions, MIN(first_visited) as first_visited
```

**Chain C — previously-played detection (3 layers):**
```
session_id, sub_session_id, segment_id, sub_session_num, vaas_url_video_id, vaas_url_url_path, MAX(playing) as played
-> [window: LAG(played), MAX(played) OVER session_id ORDER BY segment_id] -> session_id, sub_session_id, segment_id, vaas_url_video_id, vaas_url_url_path, played, last_played
-> [window: MAX(last_played) OVER session_id ORDER BY segment_id] -> session_id, sub_session_id, segment_id, vaas_url_video_id, vaas_url_url_path, played, last_played, ever_played
```

**Final aggregation:**
```
[events_with_played_pages_filtered LEFT JOIN previously_played ON session+sub_session+video+path, LEFT JOIN visits ON session+path WHERE different hour]
session_id, sub_session_id, segment_id, RTRIM(vaas_url_url_path), vaas_url_video_id,
  SUM(time_played), MAX(progress), MAX(completed), MAX(ever_played), COALESCE(MAX(v.sessions), 0), ... (many ANY_VALUE/MAX/MIN aggregates)
```

## 4. Identify dependencies

- **Grouping drivers:** `session_id`, `sub_session_id`, `segment_id` (derived from `sub_session_num`), `vaas_url_url_path` (normalized), `vaas_url_video_id`.
- **Derived — `segment_id`:** `SAFE_CAST(REPLACE(sub_session_num, '-', '.') AS FLOAT64)` — a numeric sort key created from a hyphenated string like `"1-2"` → `1.2`. This enables ordered window functions over segments.
- **Derived — `played` (boolean per segment):** `MAX(vaas_state_playing = 'playing')` — collapses event-level rows to one boolean per segment.
- **Derived — `ever_played`:** requires two sequential window function layers — first `LAG(played)` to get the previous segment's play state, then a running `MAX(last_played)` to propagate "ever previously played" forward.
- **Derived — `sessions` (visit count):** requires pre-aggregation in `visits` before it can be joined, because it counts distinct sub-sessions at a coarser grain than the final output.
- **Intermediate dependency:** `events_with_played_pages_filtered` must exist before both the `previously_played` chain and the final SELECT can run, since it applies the page-play filter and adds `segment_id`.

## 5. Plan CTE layers bottom-up

1. **`pages_with_plays`** — identifies which (video_id, url_path) combinations have at least one play event, acting as an inclusion filter.

2. **`visits` (with nested `first_visited`)** — computes the number of distinct sub-sessions that visited each page within a session, along with the earliest visit time, to count prior visits.

3. **`events_with_played_pages_filtered`** — inner-joins raw events to `pages_with_plays` to keep only events on pages where a video was actually played, and adds the computed `segment_id`.

4. **`previously_played_base`** — aggregates filtered events to one row per (session, sub_session, segment, video, path) with a boolean `played` flag.

5. **`previously_played_lag`** — adds `LAG(played)` and running `MAX(played)` window functions partitioned by session, ordered by segment, to detect if the video was played in any earlier segment.

6. **`previously_played`** — corrects the `ever_played` column by taking `MAX(last_played)` instead of `MAX(played)` — this ensures `ever_played` reflects whether a *prior* segment had play activity, not the current one.

7. **Final SELECT** — joins filtered events with `previously_played` and `visits`, then aggregates to the target grain with all metric and metadata columns.

## 6. Write each CTE with justification

### CTE 1: `pages_with_plays`
**Why:** We need to restrict the entire analysis to only video+page combinations where at least one `playing` event occurred. Without this filter, segments with zero play activity on non-video pages would pollute the results.

### CTE 2: `visits` (with nested CTE `first_visited`)
**Why:** The final output needs a count of *other* sessions' visits to the same page (`sessions` column). This requires two levels: first deduplicate to one timestamp per (session, sub_session, path), then count distinct sub_sessions per (session, path). The nested CTE `first_visited` handles the deduplication, and the outer `visits` CTE handles the counting. The `REGEXP_REPLACE` strips trailing slashes for consistent path matching.

### CTE 3: `events_with_played_pages_filtered`
**Why:** This is the main working set — all raw events, filtered to only pages with plays, and enriched with `segment_id` (derived from `sub_session_num` by replacing hyphens with dots and casting to float). The `SELECT e.*` carries forward all event columns needed downstream.

### CTE 4: `previously_played_base`
**Why:** Collapses the event-level grain down to one row per segment (session × sub_session × segment × video × path) with a boolean `played` flag. This is the prerequisite grain for the window functions that detect prior-segment play activity.

### CTE 5: `previously_played_lag`
**Why:** Adds two window functions: `LAG(played)` returns the previous segment's play status, and `MAX(played) OVER (ORDER BY segment_id)` computes a running maximum. The `LAG` value (`last_played`) is the key input for the next layer — it represents whether the *immediately preceding* segment had play activity.

### CTE 6: `previously_played`
**Why:** Computes `MAX(last_played) OVER (ORDER BY segment_id)` — a running maximum over `last_played`, not over `played`. This subtle distinction is critical: `ever_played` becomes true only when a *prior* segment had play activity (via `last_played`), not when the current segment itself is playing. This gives us the "was this video previously played in this session?" flag.

### Final SELECT
**Why:** Joins the filtered event set with the previously-played flags and visit counts, then aggregates everything to the target grain. The `LEFT JOIN` to `visits` includes a `DATE_TRUNC(..., HOUR) != DATE_TRUNC(..., HOUR)` condition that excludes visits from the same hour — effectively counting only *prior* or *other-time* visits to the page, not the current session's own page view.

## 7. Compose the final query

```sql
WITH
pages_with_plays AS (
  SELECT
    e.vaas_url_video_id as vaas_url_video_id,
    e.vaas_url_url_path as vaas_url_url_path
  FROM
    ${ref("vaas_events")} AS e
  WHERE
    e.vaas_state_playing = 'playing'
  GROUP BY
    vaas_url_video_id,
    vaas_url_url_path
),
visits AS (
    WITH
    first_visited AS (
        SELECT
            e.session_id,
            e.sub_session_id,
            REGEXP_REPLACE(e.vaas_url_url_path, r'/$', '') AS vaas_url_url_path,
            MIN(e.created) as first_visited
        FROM
            ${ref("vaas_events")} as e
        GROUP BY
            e.session_id,
            e.sub_session_id,
            e.vaas_url_url_path
    )
    SELECT
        fv.session_id,
        fv.vaas_url_url_path,
        COUNT(DISTINCT fv.sub_session_id) as sessions,
        MIN(fv.first_visited) as first_visited
    FROM
        first_visited as fv
    GROUP BY
        fv.session_id,
        fv.vaas_url_url_path
),
events_with_played_pages_filtered AS (
  SELECT
    e.*,
    SAFE_CAST(REPLACE(e.sub_session_num, '-', '.') AS FLOAT64) as segment_id
  FROM
    ${ref("vaas_events")} AS e
  JOIN
    pages_with_plays as pwp
  ON
    RTRIM(pwp.vaas_url_url_path, '/') = RTRIM(e.vaas_url_url_path, '/')
  AND
    pwp.vaas_url_video_id = e.vaas_url_video_id
),
previously_played_base AS (
  SELECT
    e.session_id as session_id,
    e.sub_session_id as sub_session_id,
    e.segment_id as segment_id,
    e.vaas_url_video_id as vaas_url_video_id,
    e.vaas_url_url_path as vaas_url_url_path,
    MAX(e.vaas_state_playing = 'playing') as played
  FROM
    events_with_played_pages_filtered AS e
  GROUP BY
    e.session_id,
    e.sub_session_id,
    e.segment_id,
    e.sub_session_num,
    e.vaas_url_video_id,
    e.vaas_url_url_path
  ORDER BY
    e.session_id ASC,
    segment_id ASC
),
previously_played_lag AS (
  SELECT
    e.session_id as session_id,
    e.sub_session_id as sub_session_id,
    e.segment_id as segment_id,
    e.vaas_url_video_id as vaas_url_video_id,
    e.vaas_url_url_path as vaas_url_url_path,
    e.played as played,
    LAG(e.played) OVER (
      PARTITION BY
        e.session_id
      ORDER BY
        e.segment_id ASC
    ) as last_played,
    MAX(e.played) OVER (
      PARTITION BY
        e.session_id
      ORDER BY
        e.segment_id ASC
    ) as ever_played
  FROM
    previously_played_base as e
),
previously_played AS (
  SELECT
    e.session_id as session_id,
    e.sub_session_id as sub_session_id,
    e.segment_id as segment_id,
    e.vaas_url_video_id as vaas_url_video_id,
    e.vaas_url_url_path as vaas_url_url_path,
    e.played as played,
    e.last_played as last_played,
    MAX(e.last_played) OVER (
      PARTITION BY
        e.session_id
      ORDER BY
        e.segment_id ASC
    ) as ever_played
  FROM
    previously_played_lag as e
)
SELECT
  e.session_id,
  e.sub_session_id,
  e.segment_id,
  COALESCE(MAX(v.sessions), 0) as sessions,
  ANY_VALUE(e.browser_name) as browser_name,
  ANY_VALUE(e.browser_version) as browser_version,
  ANY_VALUE(e.device_model) as device_model,
  ANY_VALUE(e.device_vendor) as device_vendor,
  ANY_VALUE(e.os_name) as os_name,
  ANY_VALUE(e.os_version) as os_version,
  ANY_VALUE(e.device_category) as device_category,
  SUM(e.time_played) as time_played_seconds,
  RTRIM(ANY_VALUE(vaas_url_url), '/') AS vaas_url_url,
  RTRIM(e.vaas_url_url_path, '/') AS vaas_url_url_path,
  e.vaas_url_video_id AS video_id,
  'playing' IN UNNEST (ARRAY_AGG(vaas_state_playing))
  OR MAX(CAST(vaas_state_progress AS FLOAT64)) > 0 AS played_video,
  -- additional information
  MAX(user_id) AS user_id,
  MAX(current_user_id) AS current_user_id,
  ANY_VALUE(NULLIF(external_user_id, 'null')) AS external_user_id,
  MAX(CAST(vaas_state_progress AS FLOAT64)) AS progress_seconds,
  MAX(vaas_state_completed = 'true') AS completed,
  ANY_VALUE(vaas_url_domain) AS vaas_url_domain,
  ANY_VALUE(vaas_url_hash) AS vaas_url_hash,
  ANY_VALUE(vaas_url_query) AS vaas_url_query,
  ANY_VALUE(vaas_url_unique_id) AS vaas_url_unique_id,
  ANY_VALUE(vaas_url_utm_campaign) as vaas_url_utm_campaign,
  ANY_VALUE(vaas_url_utm_medium) as vaas_url_utm_medium,
  ANY_VALUE(vaas_url_utm_term) as vaas_url_utm_term,
  ANY_VALUE(vaas_url_utm_content) as vaas_url_utm_content,
  ANY_VALUE(vaas_url_utm_source) as vaas_url_utm_source,
  ANY_VALUE(utm_campaign) as utm_campaign,
  ANY_VALUE(utm_medium) as utm_medium,
  ANY_VALUE(utm_term) as utm_term,
  ANY_VALUE(utm_content) as utm_content,
  ANY_VALUE(utm_source) as utm_source,
  MAX(vaas_settings_captions != 'off') AS used_captions,
  MAX(vaas_settings_fullscreen = 'fullscreen') AS used_fullscreen,
  MAX(vaas_settings_muted = 'muted') AS muted_video,
  MAX(
    vaas_settings_picture_in_picture = 'pip'
  ) AS used_picture_in_picture,
  MAX(CAST(vaas_settings_playback_speed AS FLOAT64)) AS max_playback_speed,
  MAX(vaas_settings_quality) AS max_quality,
  MIN_BY (geo_city, created) AS geo_city,
  MIN_BY (geo_country, created) AS geo_country,
  MIN(created) AS session_start,
  MAX(created) AS session_end,
  TIMESTAMP_DIFF(MAX(created), MIN(created), SECOND) AS session_duration,
  MAX(SAFE_CAST(e.vaas_state_duration AS FLOAT64)) as video_duration,
  MIN_BY(e.vaas_state_visible = 'true', e.created) as started_visible,
  MAX(e.vaas_state_visible = 'true') as ever_visible,
  MAX((e.vaas_state_visible = 'false' AND e.vaas_state_playing = 'playing')) as ever_not_visible_while_playing,
  COALESCE(MAX(pp.ever_played), false) as ever_played
FROM
  events_with_played_pages_filtered AS e
LEFT JOIN
  previously_played as pp
ON
  pp.session_id = e.session_id
AND
  pp.sub_session_id = e.sub_session_id
AND
  pp.vaas_url_video_id = e.vaas_url_video_id
AND
  RTRIM(pp.vaas_url_url_path, '/') = RTRIM(e.vaas_url_url_path, '/')
LEFT JOIN
    visits as v
ON
    v.session_id = e.session_id
AND
    RTRIM(v.vaas_url_url_path, '/') = RTRIM(e.vaas_url_url_path, '/')
AND
    DATE_TRUNC(v.first_visited, HOUR) != DATE_TRUNC(e.created, HOUR)
GROUP BY
  e.session_id,
  e.sub_session_id,
  e.segment_id,
  RTRIM(e.vaas_url_url_path, '/'),
  e.vaas_url_video_id
```
