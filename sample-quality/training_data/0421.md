Can I compare a UUID column with a string literal without an explicit CAST?

--- AGENT ---

Yes, string literals can be implicitly coerced to UUID when used in a comparison context. The comparison follows the same lexicographical ordering rules as the explicit cast version.

```googlesql
SELECT UUID_id >= "00000000-0000-0000-0000-000000000000" FROM T1;
```
