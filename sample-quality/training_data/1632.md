--- USER ---

Show me an example BigQuery query that tracks which search result a user is currently viewing as they navigate through paginated search results, enriched with document names.

--- AGENT ---

This query reconstructs the user's search result pagination behavior. It starts by filtering for search events from the raw events table, extracting the search term and result count from nested JSON metadata. Using a window function, it computes the cumulative pagination offset for each search session—counting +1 when the user clicks 'next' and -1 when they click 'prev'—to derive which result position they are viewing at each event.

The main query joins the enriched events with a documents table to add the document name, then applies a modulo formula to safely normalize the result position back into the valid range [1, result_count], handling cases where pagination wraps or result counts are zero. The final output is a row-per-event stream showing session context, search metadata, the current result position, and document name, ordered by event timestamp.

```sql
WITH search_events AS (
  SELECT
    e.session_id,
    e.sub_session_id,
    e.name,
    e.type,
    e.action,
    e.url_path,
    e.doc_id,
    JSON_EXTRACT_SCALAR(e.action_meta, '$.search_term') as search_term,
    SAFE_CAST(JSON_EXTRACT_SCALAR(e.action_meta, '$.result_count') AS INT64) as result_count,
    CASE WHEN SAFE_CAST(JSON_EXTRACT_SCALAR(e.action_meta, '$.result_count') AS INT64) = 0 THEN 0
    ELSE
      SUM(
        CASE 
          WHEN action = 'next' THEN 1
          WHEN action = 'prev' THEN -1
          ELSE 0
        END
      ) OVER (
        PARTITION BY 
          e.session_id,
          e.sub_session_id,
          e.name,
          e.type,
          -- e.action
          JSON_EXTRACT_SCALAR(e.action_meta, '$.search_term'),
          e.url_path,
          e.doc_id
        ORDER BY
          e.created ASC
      ) + 1
    END AS result_num,
    e.created
  FROM
    ${ref('stg_events_docs_speedy')} as e
  WHERE
    e.name = 'search'
  ORDER BY
    e.session_id ASC,
    e.sub_session_id ASC,
    e.created ASC
)
SELECT
  se.session_id,
  se.sub_session_id,
  se.name,
  se.type,
  se.action,
  se.url_path,
  d.name as doc_name,
  se.doc_id,
  se.search_term,
  se.result_count,
  CASE WHEN se.result_count <= 0 
    THEN 0
    ELSE
        MOD(
          CASE WHEN se.result_num < 0 THEN se.result_count - se.result_num ELSE se.result_num END,
          se.result_count
        ) + 1
  END AS result_num,
  se.created
FROM
  search_events as se
JOIN
  ${ref('stg_documents')} as d
ON
  se.doc_id = d.id
ORDER BY
  se.created ASC
```
