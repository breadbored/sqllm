--- USER ---

How many knowledge check answers and unique users are there for a specific client and course container, excluding test responses?

--- AGENT ---

This query is a BigQuery table function that aggregates knowledge check answer activity at the client-container level. It counts the total number of answers submitted and the number of distinct users who submitted them, filtered to a specific client ID and container ID.

The query joins the answers table (`stg_user_knowledge_check_answers`) with three related tables: `stg_courses` (to access the container association), `stg_users`, and `stg_clients_course_containers` (to enforce the client-container relationship). The WHERE clause filters to non-zero `test_user` values (structurally: records where a test_user flag matches the provided parameter), a specific container via the courses table, and a specific client via the client-container mapping table. The GROUP BY on `container_id` applies after these filters are applied.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    COUNT(ukca.id) AS answers,
    COUNT(DISTINCT(ukca.user_id)) AS users
  FROM
    ${ref("stg_user_knowledge_check_answers")} ukca
  JOIN
    ${ref("stg_courses")} c
  ON
    c.id = ukca.course_id
  JOIN
    ${ref("stg_users")} u
  ON
    u.id = ukca.user_id
  JOIN
    ${ref("stg_clients_course_containers")} ccc
  ON
    ccc.course_container_id = c.container_id
  WHERE
    container_id = containerId
    AND client_id = clientId
    AND test_user = 0
  GROUP BY
    c.container_id)
```
