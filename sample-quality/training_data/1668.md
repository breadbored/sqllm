--- USER ---

Write a BigQuery query that identifies and tags menu-related events by session and click target, capturing the earliest event details for each group.

--- AGENT ---

This query filters a speedy events table for menu-related events, then groups them by sub_session_id and a constructed tag that combines "opened:_" with the click_target value. For each group, it extracts the first event's app and URL path using MIN_BY (which returns the value associated with the earliest timestamp), along with the minimum created timestamp. All results are tagged with a static classification label of 'behavior'.

The query performs a single-level aggregation with no pre-aggregation CTEs required, since all grouped columns come from the events table directly:

```sql
SELECT
  es.sub_session_id as sub_session_id,
  CONCAT('opened:_', es.click_target) as tag,
  'behavior' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref("base_buckets_events_speedy")} as es

WHERE
  es.name LIKE "%menu%"

GROUP BY 
  sub_session_id,
  tag
```
