How do I correctly escape reserved keywords in a protocol buffer extension path using backticks?

--- AGENT ---

This query shows the correct way to use backticks in extension paths: backticks must be applied to individual components of the path separately, not around multiple components or the entire path. Each potentially reserved keyword (`some`, `package`, `bar`) receives its own backticks.

```googlesql
SELECT
  foo_field.(`some`.`package`.`bar`).value = 5
FROM
  Test;
```
