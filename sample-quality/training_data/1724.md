--- USER ---

Show me how to extract the first search event from each sub-session, including its associated app and URL path.

--- AGENT ---

This query extracts the first occurrence of a 'search' event within each sub-session. It filters the events table to include only records where the event name is 'search', then groups by sub_session_id and applies a literal 'search' tag and 'behavior' class to each group. For each group, it uses MIN_BY to retrieve the app and url_path values from the chronologically earliest event (based on the created timestamp), and captures that earliest timestamp as dt. The result is a deduplicated view of sub-sessions containing search events, with the first search event's metadata preserved.

```sql
SELECT
    es.sub_session_id as sub_session_id,
    'search' as tag,
    'behavior' as class,
    MIN_BY(es.app, es.created) as app,
    MIN_BY(es.url_path, es.created) as url_path,
    MIN(es.created) as dt
FROM
    ${ref("base_buckets_events_docs_speedy")} as es
WHERE
    es.name != 'page_view'
AND
    es.name != 'open_link'
AND
    es.name = 'search'

GROUP BY
    sub_session_id,
    tag
```
