This query enriches course container records with their associated client IDs.

The query uses a two-step approach:

**Step 1 — Aggregate clients by course container:** The CTE `course_clients` groups the `clients_course_containers` table by `course_container_id` and creates an array of distinct `client_id` values (excluding nulls) for each container.

**Step 2 — Left join and apply defaults:** The main query selects all fields from the `course_container` table and left joins it with the aggregated `course_clients` CTE. For each course container, it retrieves the list of associated client IDs. Where no matching clients exist (resulting in a NULL), it defaults to the array `[1]` using COALESCE.

The output is a row for each course container with its metadata (id, creator_admin_id, created, updated) and an array of client IDs — either the distinct clients associated with that container, or [1] if none exist.

```sql
WITH
course_clients AS (
  SELECT
    ccc.course_container_id as container_id,
    ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_course_containers")} as ccc
  GROUP BY
    ccc.course_container_id
)
SELECT
  id,
  creator_admin_id,
  created,
  updated,
  COALESCE(_c.client_ids, [1]) AS client_ids
FROM
  ${ref("course_container")} as cc
LEFT JOIN
  course_clients as _c
ON
  _c.container_id = cc.id
```
