Show me an example of how to aggregate course engagement metrics by month for a specific client in BigQuery.

This query is a table-valued function that accepts a `clientId` parameter and returns monthly aggregated course engagement metrics.

The query:
1. Filters the `reporting_user_courses` table to include only records where the input `clientId` matches the unnested `client_id` field
2. Extracts the month and year from `course_started` and formats it as abbreviated month + 2-digit year (e.g., "Jan 26")
3. Aggregates two metrics across all matching courses for each month:
   - `watch_time`: total `course_progress_minutes` converted to hours, rounded to 1 decimal place
   - `course_time`: total `time_on_course_minutes` converted to hours, rounded to 1 decimal place
4. Returns results grouped by month/year and ordered chronologically

The `IFNULL(..., 0)` handles cases where either metric is null by treating them as zero before division.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64) AS (
  SELECT
    FORMAT_DATE('%b %y', course_started) AS date,
    ROUND(IFNULL(SUM(course_progress_minutes), 0) / 60.0, 1) AS watch_time,
    ROUND(IFNULL(SUM(time_on_course_minutes), 0) / 60.0, 1) AS course_time
  FROM
    ${ref("reporting_user_courses")}
  WHERE
    clientId IN UNNEST(client_id)
  GROUP BY
    date
  ORDER BY
    PARSE_DATE('%b %y', date))
```
