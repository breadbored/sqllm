## Question
Show me a BigQuery SQL query that segments user activity by year and client, classifying users as new or returning based on when they first transacted, and calculating session metrics for each cohort.

--- AGENT ---

This query performs cohort analysis on user transactions across clients and years. It works in three stages:

**Stage 1: Find first-occurrence dates**
The `client_transactions_min` CTE unnests the `client_id` array and finds the earliest transaction date (`min_day`) for each user-client pair. This establishes the cohort boundary: whether a user is "new" or "returning" in a given year depends on whether their first transaction falls within that year.

**Stage 2: Establish year boundaries**
The `year_periods` CTE unnests `client_id` again and computes the year start date (January 1st) for every transaction, creating a temporal frame for grouping.

**Stage 3: Join and classify**
The final query joins all transactions back to their cohort metadata (first-transaction dates), then groups by year and client. For each cohort, it counts:
- Distinct users and sessions overall
- Users and sessions where the current transaction is their first (`new_users`, `new_user_sessions`)
- Users and sessions where it's not their first (`returning_users`, `returning_user_sessions`)
- Users whose first transaction fell within the current year vs. before it (`year_new_users` vs. `year_returning_users`)
- Sessions per user as an engagement metric

The `meta_quest_min` CTE appears to be a structural intermediate that carries the cohort data through the join.

```sql
WITH
  client_transactions_min AS (
  SELECT
    ct.user_id,
    client_id_expanded AS client_id,
    MIN(ct.`day`) AS min_day
  FROM
    ${ref("agg_client_transactions")} AS ct,
    UNNEST(ct.client_id) AS client_id_expanded
  GROUP BY
    ct.user_id,
    client_id_expanded),
  meta_quest_min AS (
  SELECT
    ctm.user_id,
    ctm.client_id,
    ctm.min_day
  FROM
    client_transactions_min AS ctm),
  year_periods AS (
  SELECT
    ct.user_id,
    client_id_expanded AS client_id,
    ct.`day`,
    DATE(EXTRACT(YEAR
      FROM
        `day`), 1, 1) AS year_start
  FROM
    ${ref("agg_client_transactions")} AS ct,
    UNNEST(ct.client_id) AS client_id_expanded)
SELECT
  COUNT(DISTINCT ct.user_id) AS unique_users,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day = ct.`day` THEN ct.user_id
  END
    ) AS new_users,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day != ct.`day` THEN ct.user_id
  END
    ) AS returning_users,
  ( COUNT( DISTINCT
      CASE
        WHEN mqm.min_day != ct.`day` THEN ct.user_id
    END
      ) + COUNT( DISTINCT
      CASE
        WHEN mqm.min_day = ct.`day` THEN ct.user_id
    END
      ) ) - COUNT(DISTINCT ct.user_id) AS new_users_returning,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day >= year_start AND mqm.min_day < DATE_ADD(year_start, INTERVAL 1 YEAR) THEN ct.user_id
  END
    ) AS year_new_users,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day < year_start OR mqm.min_day >= DATE_ADD(year_start, INTERVAL 1 YEAR) THEN ct.user_id
  END
    ) AS year_returning_users,
  COUNT(ct.user_id) AS total_sessions,
  COUNT(
    CASE
      WHEN mqm.min_day != ct.`day` THEN ct.user_id
  END
    ) AS returning_user_sessions,
  ROUND(COUNT(ct.user_id) / COUNT(DISTINCT ct.user_id), 2) AS sessions_per_user,
  FORMAT_DATE('%Y', year_start) AS year,
  ct.client_id AS client_id
FROM
  year_periods AS ct
INNER JOIN
  meta_quest_min AS mqm
ON
  mqm.user_id = ct.user_id
  AND mqm.client_id = ct.client_id
GROUP BY
  FORMAT_DATE('%Y', year_start),
  year_start,
  ct.client_id
ORDER BY
  year_start
```
