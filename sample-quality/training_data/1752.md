--- AGENT ---

This query backfills missing user IDs in event records by looking them up from earlier events in the same session subsegment.

The CTE `event_min_subsession_user_ids` creates a reference table of (session_id, segment_id, user_id) combinations and records the latest event number where each user_id appeared in that subsegment. 

The main query then performs a LEFT JOIN to match each event against this reference, filtering for rows where the event's event_num comes at or before the reference row's maximum event_num. For each event, it coalesces the event's own user_id (if present) with the earliest user_id (by event_num) found in the matching reference rows—effectively backfilling the user_id backward through the session/segment from the first event where a user_id was recorded. Results are grouped by event ID and the resolved user_id.

```sql
WITH
  event_min_subsession_user_ids AS (
    SELECT
      ne.session_id,
      -- Get the highest event number in the subsession before a user ID changes, if set
      MAX(ne.event_num) AS event_num,
      ne.segment_id,
      -- Get the event number where the user ID changed from before, if set, in the subsession
      MIN_BY (ne.user_id, ne.event_num) AS user_id
    FROM
      ${ref("events_numbered")} AS ne
    WHERE
      ne.user_id IS NOT NULL
    GROUP BY
      ne.session_id,
      ne.segment_id,
      ne.user_id
  )
SELECT
  ne.id,
  -- Backfilled user ID back to the start of the subsession, or just after a user ID change
  COALESCE(ne.user_id, MIN_BY (ne2.user_id, ne2.event_num)) AS user_id
FROM
  ${ref("events_numbered")} AS ne
  LEFT JOIN event_min_subsession_user_ids AS ne2 ON ne.session_id = ne2.session_id
  AND ne.event_num <= ne2.event_num
  AND ne.segment_id = ne2.segment_id
  AND ne2.user_id IS NOT NULL
GROUP BY
  ne.id,
  ne.user_id
```
