## Question
Show me a BigQuery query that enriches the questions table with the list of clients that have access to each question through their course hierarchy.

--- AGENT ---

This query enriches the `questions` table by tracing backward through the course learning architecture to determine which clients can access each question.

The first CTE (`stg_questions`) follows a join chain starting from `clients_course_containers`, through `courses`, `course_lessons`, `learning_cues`, `knowledge_checks`, and finally to `knowledge_check_questions`. For each question ID, it aggregates all distinct client IDs into an array (ignoring nulls).

The main SELECT retrieves all columns from the `questions` table, applies LOWER() to three text columns for normalization, and left-joins the CTE to attach the client_ids array. When a question has no entry in the CTE—meaning no traceable access path through the course hierarchy—the COALESCE expression defaults to [1].

The output is a denormalized `questions` table with a client_ids column representing the set of clients with access to each question.

```sql
WITH
stg_questions AS (
    SELECT
        kcq.id as question_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("learning_cues")} as lc
    ON
        lc.lesson_id = cl.lesson_id
    JOIN
        ${ref("knowledge_checks")} as kc
    ON
        kc.learning_cue_id = lc.id
    JOIN
        ${ref("knowledge_check_questions")} as kcq
    ON
        kcq.knowledge_check_id = kc.id
    GROUP BY
        kcq.id
)


SELECT
  q.id,
  q.name,
  LOWER(type)        AS type,
  LOWER(explanation) AS explanation,
  q.attempts,
  q.created,
  q.updated,
  q.image,
  LOWER(image_title) AS image_title,
  COALESCE(sq.client_ids, [1]) AS client_ids
FROM 
  ${ref("questions")} as q
LEFT JOIN
  stg_questions as sq
ON
  sq.question_id = q.id
```
