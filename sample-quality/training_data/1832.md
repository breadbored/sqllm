--- USER ---

Provide an example BigQuery query that performs cohort analysis on user activity, identifying new versus returning users within half-yearly periods and comparing their engagement metrics by client.

--- AGENT ---

This query performs cohort analysis on user transactions, segmented by client and half-year periods. Here's what it does step by step:

**CTE 1 — client_transactions_min**: Establishes the cohort baseline by finding the earliest transaction day (min_day) for each user within each client. The query unnests the client_id array so that users with multiple clients are tracked separately across each one.

**CTE 2 — meta_quest_min**: A pass-through that references the cohort baseline for use in the final join.

**CTE 3 — half_year_periods**: Maps each transaction to a half-year period by extracting a half_start date (either January 1 or July 1 of the transaction year, depending on whether the month falls in H1 or H2). This creates a 6-month time bucket for each transaction.

**Final SELECT**: Joins the half-year periods against the cohort baseline on user_id and client_id, then groups by half_start and client_id to compute engagement metrics:
- **unique_users**: Distinct user count in the period
- **new_users**: Users whose first transaction occurred on the current day
- **returning_users**: Users whose first transaction predates the current day
- **half_year_new_users**: Users whose first transaction falls within the current half-year period
- **half_year_returning_users**: Users whose first transaction predates the current half-year period
- **total_sessions** and **returning_user_sessions**: Raw transaction counts
- **sessions_per_user**: Average sessions per distinct user

Results are ordered chronologically by half_start, enabling analysis of cohort behavior trends over time.

```sql
WITH
  client_transactions_min AS (
  SELECT
    ct.user_id,
    client_id_expanded AS client_id,
    MIN(ct.`day`) AS min_day
  FROM
    ${ref("agg_client_transactions")} AS ct,
    UNNEST(ct.client_id) AS client_id_expanded
  GROUP BY
    ct.user_id,
    client_id_expanded ),
  meta_quest_min AS (
  SELECT
    ctm.user_id,
    ctm.client_id,
    ctm.min_day
  FROM
    client_transactions_min AS ctm ),
  half_year_periods AS (
  SELECT
    ct.user_id,
    client_id_expanded AS client_id,
    ct.`day`,
    CASE
      WHEN EXTRACT(MONTH FROM `day`) <= 6 THEN DATE(EXTRACT(YEAR FROM `day`), 1, 1)
      ELSE DATE(EXTRACT(YEAR
      FROM
        `day`), 7, 1)
  END
    AS half_start
  FROM
    ${ref("agg_client_transactions")} AS ct,
    UNNEST(ct.client_id) AS client_id_expanded )
SELECT
  COUNT(DISTINCT ct.user_id) AS unique_users,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day = ct.`day` THEN ct.user_id
  END
    ) AS new_users,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day != ct.`day` THEN ct.user_id
  END
    ) AS returning_users,
  ( COUNT( DISTINCT
      CASE
        WHEN mqm.min_day != ct.`day` THEN ct.user_id
    END
      ) + COUNT( DISTINCT
      CASE
        WHEN mqm.min_day = ct.`day` THEN ct.user_id
    END
      ) ) - COUNT(DISTINCT ct.user_id) AS new_users_returning,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day >= half_start AND mqm.min_day < DATE_ADD(half_start, INTERVAL 6 MONTH) THEN ct.user_id
  END
    ) AS half_year_new_users,
  COUNT( DISTINCT
    CASE
      WHEN mqm.min_day < half_start OR mqm.min_day >= DATE_ADD(half_start, INTERVAL 6 MONTH) THEN ct.user_id
  END
    ) AS half_year_returning_users,
  COUNT(ct.user_id) AS total_sessions,
  COUNT(
    CASE
      WHEN mqm.min_day != ct.`day` THEN ct.user_id
  END
    ) AS returning_user_sessions,
  ROUND(COUNT(ct.user_id) / COUNT(DISTINCT ct.user_id), 2) AS sessions_per_user,
  --   FORMAT_DATE('%b %Y', half_start) AS half_year,
  half_start,
  ct.client_id AS client_id
FROM
  half_year_periods AS ct
INNER JOIN
  meta_quest_min AS mqm
ON
  mqm.user_id = ct.user_id
  AND mqm.client_id = ct.client_id
GROUP BY
  half_start,
  ct.client_id
ORDER BY
  half_start
```
