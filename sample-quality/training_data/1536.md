What error occurs when I try to parse a timestamp with format elements in a different order than the string?

--- AGENT ---

The `PARSE_TIMESTAMP` function produces an error if the positions of format elements in the format string do not match their positions in the input string. For example, putting the year element in a different location than where it appears in the timestamp string causes a parsing failure.

```googlesql
SELECT PARSE_TIMESTAMP("%a %b %e %Y %I:%M:%S", "Thu Dec 25 07:30:00 2008");
```
