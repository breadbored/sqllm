--- USER ---

Extract all 'any_click' events, group them by sub-session, and identify the earliest app and URL path associated with each sub-session along with the first event timestamp.

--- AGENT ---

This query filters the events table for rows where the event name is 'any_click', groups them by sub-session ID, and extracts structural information about the earliest event in each group.

For each sub-session, it captures:
- The app value from whichever row has the earliest timestamp (using MIN_BY)
- The URL path value from whichever row has the earliest timestamp (using MIN_BY)
- The minimum created timestamp across the group
- A fixed tag label 'any_click' and class label 'behavior'

The redundant conditions (name != 'page_view' and name != 'open_link') are logically subsumed by the name = 'any_click' filter but are preserved as written. The output has one row per sub-session, representing the entry point and timing of the first click event in that session.

```sql
SELECT
    es.sub_session_id as sub_session_id,
    'any_click' as tag,
    'behavior' as class,
    MIN_BY(es.app, es.created) as app,
    MIN_BY(es.url_path, es.created) as url_path,
    MIN(es.created) as dt

FROM
    ${ref('base_buckets_events_docs_speedy')} as es 

WHERE
    es.name != 'page_view'
AND
    es.name != 'open_link'
AND
    es.name = 'any_click'

GROUP BY
    sub_session_id,
    tag
```
