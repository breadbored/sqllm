How do I use `OPTIONAL CALL` to include rows with NULL values when a subquery produces no output?

--- AGENT ---

The `OPTIONAL CALL` clause ensures all rows from the outer query are retained even if the subquery produces no results for some rows, with `NULL` substituted for missing values. This example finds each person's most recent transfers but includes people with no transfer records.

```googlesql
GRAPH FinGraph
MATCH (p:Person)
OPTIONAL CALL (p) {
  MATCH (p)-[:Owns]->(a:Account)-[t:Transfers]->()
  RETURN a.Id AS account_id, t.amount AS transfer_amount, DATE(t.create_time) AS transfer_date
  ORDER BY transfer_date DESC
  LIMIT 2
}
RETURN p.name, account_id, transfer_amount, transfer_date
ORDER BY p.name, transfer_date DESC;
```
