Can you write a query that retrieves user download records and associates each with a set of client IDs, including both a default client and any external client mapping the user may have?

--- AGENT ---

This query retrieves user download records and enriches them with client ID assignments. 

The CTE `user_clients` builds a mapping of users to their client IDs by creating an array containing the literal value 1 (a default client) and the user's `external_client_id` for all users where an external mapping exists.

The main query then selects all columns from the `user_data_downloads` table and performs a LEFT JOIN against this mapping on `user_id`. For the `client_ids` column, it uses COALESCE to return the array from the user mapping if one exists, otherwise defaults to `[1]` (the default client only). This ensures every download record has a non-null client ID array, whether the user has an explicit external client assignment or not.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  _utb.id,
  LOWER(_utb.key) AS key,
  _utb.user_id,
  _utb.removed_from_s3,
  _utb.expiration,
  _utb.created,
  _utb.updated,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_data_downloads")} as _utb
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = _utb.user_id
```
