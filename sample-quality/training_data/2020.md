--- USER ---

Get all user course lesson progress records enriched with a per-user `client_ids` array that identifies which clients the user belongs to, defaulting to `[1]` for users without an external client association. Records representing more than 10 minutes of idle time at the same playback position — where progress has stalled but interval rows kept accumulating — should be excluded for data recorded after January 6th, 2024.

## Step 1 — Identify Shapes

**Input tables and their observable columns:**

- `users`: `id, external_client_id`
- `user_course_lesson_progress`: `id, progress_seconds, user_id, course_id, lesson_id, created, updated, interval_seconds`

**Conceptual joined column set (as if already joined):**
`user_course_lesson_progress.id, user_course_lesson_progress.progress_seconds, user_course_lesson_progress.user_id, user_course_lesson_progress.course_id, user_course_lesson_progress.lesson_id, user_course_lesson_progress.created, user_course_lesson_progress.updated, user_course_lesson_progress.interval_seconds, users.id, users.external_client_id`

**Target output shape:**
`id, progress_seconds, user_id, course_id, lesson_id, created, updated, interval_seconds, client_ids`

The only derived column is `client_ids` — everything else passes through from `user_course_lesson_progress` unchanged.

## Step 2 — Identify Sources and Relationships

The join relationship is:
`user_course_lesson_progress.user_id → users.id`

This is a natural, single-column FK relationship. The join is **LEFT JOIN** because not every progress record belongs to a user with an `external_client_id` — users without one still have progress records and must be retained, with `client_ids` defaulting to `[1]` via COALESCE.

The `users` table is not joined directly — it is pre-processed into the `user_clients` CTE first, which restricts to users with a non-null `external_client_id` and constructs the array `[1, external_client_id]`. This means the LEFT JOIN on `user_clients` will naturally produce NULL for users without an external client, which COALESCE then converts to `[1]`.

A second self-reference to `user_course_lesson_progress` appears inside the `NOT IN` subquery. This is not a join — it is a row-elimination filter that uses `ROW_NUMBER()` + `QUALIFY` to identify and exclude idle records.

---

## Step 3 — Identify Aggregation Depth

This query contains **no GROUP BY aggregation**. It is a filtering and enrichment query. The `ROW_NUMBER()` window function is used for row elimination, not reduction. Step 3 expresses the full transformation plan in arrow notation anyway, per convention:

**CTE layer — `user_clients` (row-level transform, no aggregation):**
```
user_id, [1, external_client_id] as client_ids
  (WHERE external_client_id IS NOT NULL)
```

**Inline filter subquery (anti-semi-join via NOT IN):**
```
[exclude _utb.id where:]
user_id, course_id, lesson_id, progress_seconds,
  ROW_NUMBER() OVER (PARTITION BY user_id, course_id, lesson_id, progress_seconds) as rn
  QUALIFY rn >= 10 * 60 / interval_seconds
  (WHERE interval_seconds > 0 AND created > '2024-01-06')
```

**Main query (single level, enrichment join):**
```
id, progress_seconds, user_id, course_id, lesson_id, created, updated, interval_seconds,
  COALESCE(ucl.client_ids, [1]) as client_ids
  (LEFT JOIN user_clients on user_id, NOT IN idle_ids)
```

There is no `->` reducing rows across aggregation levels. The arrow is absent because no rollup occurs — row count is determined by `user_course_lesson_progress` minus the excluded idle rows.

## Step 4 — Identify Dependencies

- `client_ids` depends on the `user_clients` CTE, which depends on `users.external_client_id`.
- The idle-row exclusion depends on `interval_seconds` being greater than zero (to avoid division by zero in `10 * 60 / interval_seconds`) and `created > '2024-01-06'` (the retroactive data correction cutoff — note the comment says "Jan 6th, 2025" but the literal in the query is `'2024-01-06'`, a discrepancy that is observable but cannot be resolved from the SQL alone).
- The QUALIFY threshold `10 * 60 / interval_seconds` is adaptive: it scales the 10-minute window to the sampling rate of each individual record, making the idle detection robust to variable interval sizes.
- The partition key of the window function includes `progress_seconds` — this is the core of the idle detection logic. A user idle at the same playback position accumulates multiple rows with identical `(user_id, course_id, lesson_id, progress_seconds)`. The row number within that group counts how many interval-length ticks have elapsed at that position. Rows beyond the threshold are idle rows.

## Step 5 — Plan CTE Layers Bottom-Up

**Layer 1 — `user_clients`:**
Produces one row per user who has a non-null `external_client_id`, with a `client_ids` array of `[1, external_client_id]`. This is the lookup table for the LEFT JOIN in the main query.

**No additional CTE for the idle filter:**
The idle-exclusion logic is expressed as an inline `NOT IN` subquery using `QUALIFY`. It does not need its own named CTE because it produces only a set of `id` values to exclude and is referenced once.

---

## Step 6 — Write Each CTE with Justification

**`user_clients`** — exists to pre-compute the `client_ids` array for all users with an external client association exactly once, so the main query can perform a clean LEFT JOIN with a COALESCE fallback rather than scattering conditional array-construction logic across the SELECT list.

The `NOT IN` subquery is inline rather than a named CTE — it reads as a self-filter on `user_course_lesson_progress` and is structurally tighter as a WHERE clause predicate since it produces only a key set (`id`) with no columns needed in the outer query.

## Step 7 — Compose the Final Query

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  _utb.id,
  _utb.progress_seconds,
  _utb.user_id,
  _utb.course_id,
  _utb.lesson_id,
  _utb.created,
  _utb.updated,
  _utb.interval_seconds,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_course_lesson_progress")} as _utb
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = _utb.user_id
WHERE
  -- Cutoff any >10 minute idles after Jan 6th, 2025
  _utb.id NOT IN (
    SELECT
      _uclp.id as id
    FROM
      ${ref("user_course_lesson_progress")} as _uclp
    WHERE
      _uclp.interval_seconds > 0
    AND
      _uclp.created > '2024-01-06'
    QUALIFY
      ROW_NUMBER() OVER (PARTITION BY _uclp.user_id, _uclp.course_id, _uclp.lesson_id, _uclp.progress_seconds ORDER BY _uclp.user_id, _uclp.course_id, _uclp.lesson_id, _uclp.progress_seconds) >= 10 * 60 / _uclp.interval_seconds -- 10 minutes, div by interval
  )
```
