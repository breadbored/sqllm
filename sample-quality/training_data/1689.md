--- USER ---

Show me a BigQuery query that tags sub-sessions by their last app and extracts the earliest event details for each tag.

--- AGENT ---

This query segments events by `sub_session_id` and creates a tag based on the `last_event_app` value (prefixed with `'last_app_'`). It filters out records where `last_event_app` equals `'app'`. For each `sub_session_id` and tag combination, it extracts:

- The `app` and `url_path` values associated with the earliest timestamp (using `MIN_BY` to find the value in that column corresponding to the minimum `created` timestamp)
- The earliest `created` timestamp for that group
- A `class` label hardcoded as `'behavior'`

The result is a denormalized behavior classification table where each row represents the earliest event details in a tagged sub-session.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  CONCAT('last_app_', e.last_event_app) as tag,
  'behavior' as class,
  MIN_BY(e.app, e.created) as app,
  MIN_BY(e.url_path, e.created) as url_path,
  MIN(e.created) as dt

FROM
  ${ref("base_buckets_events_app_speedy")} as e

WHERE
  e.last_event_app != 'app'

GROUP BY
  sub_session_id,
  tag
```
