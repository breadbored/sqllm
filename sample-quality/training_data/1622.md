--- USER ---

Show me an example of a BigQuery query that combines sentiment survey responses with course and user context, including user account details and enrollment information.

--- AGENT ---

This query constructs a denormalized fact table of sentiment survey responses with full context. It starts from sentiment questions and navigates through a course hierarchy (learning_cues → lessons → course_lessons → courses) to associate each question with a course. The query then joins with sentiment answer options and actual user responses. For each unique combination of question, answer option, and responding user, it aggregates the associated course data (course_id, name, version, enrollment status) and user data (account creation date, last active date). Test users are excluded and the query only includes responses to questions that are associated with a course.

```sql
SELECT
  ucsqa.user_id AS user_id,
  MAX(c.course_id) AS course_id,
  -- Get the latest course ID
  MAX(c.container_id) AS container_id,
  MAX(u.created) AS account_created,
  MAX(u.last_online) AS last_active,
  MAX(c.name) AS course_name,
  MAX(c.version) AS version,
  MAX(uc.status) AS course_status,
  MAX(uc.created) AS course_start_date,
  sq.id AS question_id,
  sqo.id AS answer_id,
  sq.name AS question_value,
  sqo.name AS answer_value
FROM
  ${ref("stg_sentiment_questions")} AS sq
  -- Start from sentiment_questions to narrow down applicable questions with JOINs
LEFT JOIN
  -- Straight JOIN because any lesson that isn't associated with a course is not needed
  -- Subquery to glue together course_id and learning_cue_id
  (
  SELECT
    lc.id AS learning_cue_id,
    _c.id AS course_id,
    _c.name AS course_name,
    _c.container_id AS container_id,
    _c.name,
    _c.version
  FROM
    ${ref("stg_learning_cues")} AS lc
  JOIN
    ${ref("stg_lessons")} AS l
  ON
    lc.lesson_id = l.id
  JOIN
    ${ref("stg_course_lessons")} AS cl
  ON
    cl.lesson_id = l.id
  JOIN
    ${ref("stg_courses")} AS _c
  ON
    _c.id = cl.course_id) AS c
ON
  c.learning_cue_id = sq.learning_cue_id
JOIN
  ${ref("stg_sentiment_question_options")} AS sqo
ON
  sq.id = sqo.sentiment_question_id
LEFT JOIN
  -- LEFT JOIN because some answers will have 0 people responding and we don't want to throw those out
  ${ref("stg_user_course_sentiment_question_answers")} AS ucsqa
ON
  ucsqa.sentiment_question_option_id = sqo.id
LEFT JOIN
  ${ref("stg_users")} u
ON
  u.id = ucsqa.user_id
LEFT JOIN
  ${ref("stg_user_courses")} uc
ON
  uc.course_id = ucsqa.course_id
  AND uc.user_id = ucsqa.user_id
WHERE
  c.course_id IS NOT NULL
  AND u.test_user = 0
  -- Remove questions that are not / no longer associated with a course
GROUP BY
  -- Important GROUP BYs
  sq.id,
  sqo.id,
  ucsqa.user_id,
  -- Unimportant GROUP BYs, the values won't change
  sq.name,
  sqo.name
```
