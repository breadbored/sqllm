--- USER ---

For each user badge, produce all badge fields along with an array of client IDs that the badge should be visible to. If the badge owner has an external client association, the badge belongs to both the default client (ID 1) and the external client; otherwise it belongs only to the default client.

--- AGENT ---

**Step 1 — Identify shapes**

Input tables and their observable columns:

- `users` (u): `u.id`, `u.external_client_id`
- `user_badges` (ub): `ub.id`, `ub.name`, `ub.image`, `ub.user_id`, `ub.learning_path_id`, `ub.created`, `ub.updated`, `ub.accredible_id`, `ub.credly_id`

Target shape (one row per badge):

```
id, name, image, user_id, learning_path_id, created, updated, accredible_id, credly_id, client_ids (ARRAY)
```

The only derived column is `client_ids` — an array literal synthesized from `external_client_id`. Every other output column is a direct passthrough from `user_badges`.

---

**Step 2 — Identify sources and relationships**

The join key between `user_badges` and `users` is `user_badges.user_id = users.id`. The CTE `user_clients` aliases `u.id` as `user_id` explicitly, and the main query joins on `ucl.user_id = ub.user_id`, confirming this natural key.

The join is a `LEFT JOIN`, meaning every badge row is preserved regardless of whether its owner exists in `user_clients`. This is the mechanism that allows `COALESCE(..., [1])` to supply the default for unmatched badges.

The filter `u.external_client_id IS NOT NULL` in `user_clients` means only users with an external association appear in the CTE at all — users without one are intentionally absent, producing a `NULL` on the left join that `COALESCE` then replaces with `[1]`.

---

**Step 3 — Identify aggregation depth**

There is no `GROUP BY` in this query. This is a join-and-reshape problem, not an aggregation problem. The single-level shape in arrow notation:

```
ub.id, ub.name, ub.image, ub.user_id, ub.learning_path_id, ub.created, ub.updated,
ub.accredible_id, ub.credly_id, COALESCE(ucl.client_ids, [1]) as client_ids
```

One arrow level. No rollup required.

---

**Step 4 — Identify dependencies**

- `client_ids` depends on `users.external_client_id`, accessed via a `user_id` join to `user_badges`.
- The array literal `[1, u.external_client_id]` is constructed inside the CTE so it exists as a single, nullable array value that `COALESCE` can test atomically in the outer query.
- If the array were constructed in the outer query directly (e.g. `[1, u.external_client_id]`), the `COALESCE` fallback to `[1]` would still need the same pre-filter logic to avoid including `NULL` as a second array element for users without an external client — so the CTE isolation is load-bearing.
- The constant `1` appearing in both the array literal and the `COALESCE` default signals that client ID 1 is a universal default that every badge always belongs to.

---

**Step 5 — Plan CTE layers bottom-up**

One CTE is required:

**`user_clients`** — For each user with a non-null `external_client_id`, construct the two-element array `[1, external_client_id]` keyed by `user_id`. This pre-builds the array as a clean scalar value so the outer join can use `COALESCE` on it without needing to handle partial nulls inside the array itself.

---

**Step 6 — Write each CTE with justification**

**`user_clients`**: This layer exists to isolate the array construction and the `IS NOT NULL` filter so that the outer query's `COALESCE` can treat the entire `client_ids` array as either present-and-complete or absent, with no risk of emitting `[1, NULL]` for users who have no external client.

---

**Step 7 — Compose the final query**

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  ub.id,
  ub.name,
  ub.image,
  ub.user_id,
  ub.learning_path_id,
  ub.created,
  ub.updated,
  ub.accredible_id,
  ub.credly_id,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_badges")} as ub
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = ub.user_id
```
