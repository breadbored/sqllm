--- USER ---

How do I execute multiple graph traversals sequentially, using the results from one query as input to the next?

--- AGENT ---

This query chains two linear query statements using the NEXT keyword, making the output of the first statement the input working table for the second. The first statement retrieves blocked accounts or accounts owned by person 2 using UNION ALL. The second statement takes those accounts, finds all accounts that receive transfers from them using a traverse pattern, applies WITH DISTINCT to deduplicate, and returns the receiving account nicknames.

```googlesql
GRAPH FinGraph

MATCH (a:Account {is_blocked: TRUE})
RETURN a
UNION ALL
MATCH (a:Account)<-[:Owns]-(p:Person {id: 2})
RETURN a

NEXT

MATCH (a:Account)-[t:Transfers]->(oa:Account)
WITH DISTINCT oa
RETURN oa.nick_name
```
