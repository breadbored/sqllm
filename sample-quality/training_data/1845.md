## Question
Show me a BigQuery example that demonstrates joining an enrollment table with multiple enrichment tables, including nested aggregations for resource counts and client relationships.

--- AGENT ---

This query constructs a denormalized fact table for user-course enrollments, enriched with metadata about notes, resources, and associated clients.

**Main join path:** The query starts with the `stg_user_courses` enrollment table and validates the user and course existence via inner joins to `stg_users` and `stg_courses`. It then joins to `stg_clients_course_containers` to establish the relationship between clients and course delivery containers.

**Left join 1 (notes):** A subquery aggregates the course notes table by user and course, counting non-empty notes (`user_note_count`) and prefilled note occurrences (`prefilled_note_count`). This is left-joined to preserve enrollments without notes.

**Left join 2 (resources):** A nested subquery calculates the total resource count per course by combining lesson-level resources (joined through the course-lesson bridge table) with course-level resources. It uses `ANY_VALUE()` to extract the course resource count (which is the same for all rows in the subquery's GROUP BY) and sums both sources. This is left-joined to handle courses without resources.

**Final aggregation:** The result groups by the enrollment identity (user, course, container) plus the enriched dimensions (note counts, resource count, start date), and aggregates the client IDs into a distinct array per enrollment.

The output is one row per user-course combination, showing when the enrollment was created, which container it belongs to, note and resource metrics, and all client IDs associated with that enrollment.

```sql
SELECT
  uc.user_id,
  uc.course_id,
  c.container_id,
  cn.user_note_count,
  cn.prefilled_note_count,
  r.resource_count,
  uc.created AS course_started,
  ARRAY_AGG(DISTINCT client_id IGNORE NULLS) AS client_id
FROM
  ${ref("stg_user_courses")} uc
JOIN
  ${ref("stg_users")} u
ON
  u.id = uc.user_id
JOIN
  ${ref("stg_courses")} c
ON
  c.id = uc.course_id
JOIN
  ${ref("stg_clients_course_containers")} ccc
ON
  ccc.course_container_id = c.container_id
LEFT JOIN (
  SELECT
    cn.user_id,
    cn.course_id,
    COUNTIF(note != '') AS user_note_count,
    COUNT(prefilled_note) AS prefilled_note_count
  FROM
    ${ref("stg_course_notes")} cn
  GROUP BY
    user_id,
    course_id) cn
ON
  cn.user_id = uc.user_id
  AND cn.course_id = uc.course_id
LEFT JOIN (
  SELECT
    cl.course_id,
    COUNT(lr.id) AS lesson_resouce_count,
    ANY_VALUE(cr.course_resource_count) AS course_resouce_count,
    (COUNT(lr.id) + IFNULL(ANY_VALUE(cr.course_resource_count), 0)) AS resource_count
  FROM
    ${ref("stg_lesson_resources")} lr
  JOIN
    ${ref("stg_course_lessons")} cl
  ON
    cl.lesson_id = lr.lesson_id
  LEFT JOIN (
    SELECT
      cr.course_id,
      COUNT(cr.id) AS course_resource_count
    FROM
      ${ref("stg_course_resources")} cr
    GROUP BY
      cr.course_id) cr
  ON
    cr.course_id = cl.course_id
  GROUP BY
    cl.course_id) r
ON
  r.course_id = uc.course_id
GROUP BY
  uc.user_id,
  uc.course_id,
  c.container_id,
  cn.user_note_count,
  cn.prefilled_note_count,
  r.resource_count,
  uc.created
```
