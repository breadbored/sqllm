--- USER ---

Extract and normalize event tracking data from DataDog events, standardizing string fields to lowercase and extracting key dimensions like session IDs, UTM parameters, user authentication status, geographic information, and course context from URL patterns.

--- AGENT ---

This query transforms raw event tracking data from a DataDog events source into a normalized, analytics-ready format. It performs the following operations:

**Field Normalization:** The query converts most string attributes to lowercase (session IDs, UTM parameters, authentication status, hostnames, paths, geographic data, referrers, etc.) to ensure consistent casing for deduplication and joins.

**Dimension Extraction:** It extracts specific business dimensions:
- UTM tracking parameters (id, campaign, medium, source, term) from query strings
- User authentication status and user IDs
- Geographic dimensions (city, country, country code)
- Device type information
- URL structure (host, path, referrer)
- Page loading type and resource paths

**Derived Fields:** The query extracts the `environment` by taking a substring of a tags field (skipping the first 4 characters), and uses a regex pattern to extract `course_id` from URLs containing `/courses/` or `/short-codes/` followed by digits.

**Data Enrichment:** A cross join appends an array of all client IDs from the `dev_sources.clients` table to every row (this creates a denormalized array structure attached to each event).

**Filtering:** Results are filtered to only events from two specific hosts: 'app.dataprotocol.com' and 'docs.dataprotocol.com'.

The query preserves the original timestamp and action name in their original case, while normalizing all other text fields for consistency.

```sql
SELECT
  LOWER(attributes_session_id) AS session_id,
  LOWER(attributes_view_url_query_utm_id) AS utm_id,
  LOWER(attributes_view_url_query_utm_campaign) AS utm_campaign,
  LOWER(attributes_view_url_query_utm_medium) AS utm_medium,
  LOWER(attributes_view_url_query_utm_source) AS utm_source,
  LOWER(attributes_view_url_query_utm_term) AS utm_term,
  attributes_usr_id AS user_id,
  LOWER(attributes_usr_authed) AS logged_in,
  LOWER(attributes_view_url_host) AS host,
  LOWER(attributes_service) AS service,
  LOWER(SUBSTR(attributes_tags_2, 5)) AS environment,
  LOWER(attributes_session_useragent) AS user_agent,
  LOWER(attributes_view_url_path) AS url_path,
  LOWER(attributes_resource_url_path) AS resource_url_path,
  LOWER(attributes_view_url) AS view_url,
  LOWER(attributes_view_loading_type) AS view_loading_type,
  attributes_timestamp AS timestamp,
  LOWER(attributes_geo_city) AS city,
  LOWER(attributes_geo_country) AS country,
  LOWER(attributes_geo_country_iso_code) AS country_code,
  attributes_device_type AS device_type,
  LOWER(attributes_view_referrer) AS referrer,
  LOWER(attributes_view_referrer_url_url_path) AS referrer_url,
  attributes_action_name AS action_name,
  attributes_resource_url_query_en AS event,
  REGEXP_EXTRACT(attributes_view_url, r'/(?:courses|short-codes)/(\d+)') AS course_id,
  attributes_view_url_host AS hostname,
  ARRAY(SELECT id FROM `dp-dwh.dev_sources.clients`) as client_ids,
FROM
  ${ref("dd_events")}
WHERE
  attributes_view_url_host = 'app.dataprotocol.com'
  OR attributes_view_url_host = 'docs.dataprotocol.com'
```
