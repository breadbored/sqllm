Show me an example BigQuery query that tracks when a user switches courses within a session and enriches the event records with segment-level identifiers.

--- AGENT ---

This query enriches event records by tracking course changes within user sessions. 

The query operates in two stages:

**Stage 1 — Course Segment Identification:**
A window function counts each time `derived_course_id` differs from the previous row's `last_derived_course_id` within a session (ordered chronologically). This running count becomes `course_segment_id`, partitioning each session into segments by course.

**Stage 2 — Hierarchical ID Construction:**
The query rebuilds two ID fields by concatenating the original `sub_session_id` and `sub_session_num` with the new `course_segment_id`, creating hierarchical identifiers that nest segment information into the existing session hierarchy. All other event columns pass through unchanged.

The result is an event stream where each row knows which course segment it belongs to within its session, enabling downstream analysis of user course transitions.

```sql
WITH
base_events_with_count AS (
  SELECT
    e.*,
    SUM(
      CASE
        WHEN e.last_derived_course_id IS NOT NULL AND e.derived_course_id != e.last_derived_course_id THEN 1
        ELSE 0
      END
    ) OVER (
      PARTITION BY
        e.session_id
      ORDER BY
        e.created ASC
    ) AS course_segment_id
  FROM
    ${ref("all_events")} as e
)
SELECT
  e.* EXCEPT(
    sub_session_id,
    sub_session_num
  ),
  CONCAT(e.sub_session_id, '-', e.course_segment_id) as sub_session_id,
  CONCAT(e.sub_session_num, '-', e.course_segment_id) as sub_session_num,
FROM
  base_events_with_count as e
```
