Show me an example of a BigQuery query that correlates app events with user course progress to identify video-playing behavior within sessions.

--- AGENT ---

This query identifies instances of video-playing behavior by correlating app events with user course lesson progress records. It groups events by session, app, path, and lesson progress timestamp, then filters to keep only groups where the lesson progress timestamp falls within the timespan of the events in that group (using a HAVING clause that checks if the progress timestamp is between the minimum and maximum event timestamps). The output tags each matched instance as a 'played_video' behavior event with the session ID, app, URL path, and progress creation timestamp.

```sql
SELECT
  eas.sub_session_id AS sub_session_id,
  'played_video' AS tag,
  'behavior' as class,
  eas.app AS app,
  eas.url_path AS url_path,
  uclp.created AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS eas
  JOIN ${ref("stg_user_course_lesson_progress")} AS uclp ON uclp.user_id = eas.user_id
GROUP BY
  sub_session_id,
  eas.app,
  eas.url_path,
  uclp.created
HAVING
  uclp.created BETWEEN MIN(eas.created) AND MAX(eas.created)
```
