--- USER ---

Given a knowledge check question ID along with client and container context, retrieve the question name and all possible answer options with their correctness status and user selection counts.

--- AGENT ---

This is a BigQuery table-valued function that retrieves a knowledge check question along with its associated answer options and aggregated user interaction statistics.

The function accepts three parameters (clientId, containerId, and knowledgeCheckQuestionId) and returns a table with the question name and an array of answer option records.

**Inner layer (answers subquery)**:
- Joins question set options with their option details and parent question records
- Left joins user knowledge check answers (filtered to exclude test users via a user flag check) to capture which users selected each answer
- Left joins course and client-course container tables to scope the answer data to the specified client and container
- Groups by option name, question name, and a correctness flag, producing: the max option ID, option name, parent question name, max question ID, a count of users who selected this answer, and the correctness indicator
- Filters to the specified knowledge check question ID

**Outer layer**:
- Joins the aggregated answers back to the questions table on question name
- Packages all answer option records into a single ARRAY column called "answers"
- Filters where the question ID matches the parameter
- Groups by question name to produce one row per question

The result is a denormalized record containing a single question name paired with an array of answer objects, where each object includes the option name, correctness flag, and the count of users who selected that option.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64,
    knowledgeCheckQuestionId INT64) AS (
  SELECT
    q.name,
    ARRAY_AGG(answers) AS `answers`
  FROM
    ${ref("stg_questions")} q
  JOIN (
    SELECT
      MAX(qso.id) AS id,
      qo.name,
      _q.name AS question_name,
      MAX(qso.question_set_id) AS question_id,
      COUNT(ukca.id) AS count,
      ukca.correct
    FROM
      ${ref("stg_question_set_options")} qso
    JOIN
      ${ref("stg_question_options")} qo
    ON
      qo.id = qso.question_option_id
    JOIN
      ${ref("stg_questions")} _q
    ON
      _q.id = qso.question_set_id
    LEFT JOIN (
      SELECT
        ukca.id,
        ukca.answer_option_id,
        ukca.course_id,
        ukca.correct
      FROM
        ${ref("stg_user_knowledge_check_answers")} ukca
      JOIN
        ${ref("stg_users")} AS u
      ON
        ukca.user_id = u.id
      WHERE
        u.test_user = 0) AS ukca
    ON
      ukca.answer_option_id = qso.question_option_id
    LEFT JOIN
      ${ref("stg_courses")} c
    ON
      c.id = ukca.course_id
    LEFT JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
      AND container_id = containerId
      AND ccc.client_id = clientId
    WHERE
      _q.id = knowledgeCheckQuestionId
    GROUP BY
      qo.name,
      _q.name,
      correct) AS answers
  ON
    answers.question_name = q.name
  WHERE
    q.id = knowledgeCheckQuestionId
  GROUP BY
    q.name)
```
