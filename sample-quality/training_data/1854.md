## Question
Show me an example of a query that retrieves all client configuration and metadata, normalizing text fields to lowercase for consistency.

--- AGENT ---

This query retrieves all columns from the `clients` table and normalizes a set of string-based configuration columns to lowercase. 

The query selects the raw integer and timestamp columns (`id`, `created`, `updated`) and structured/nested columns (`user_stats`, `time_to_proficiency`, `kc_and_final`, `sentiment_evaluation`, `cohort_course_analysis`, `persona_insight`, `dashboard_visible`, `user_responses`, `stats_users_only`, `use_client_courses_for_stats_endpoint`, `include_questions_answer_client_stats`) unchanged. 

It applies the `LOWER()` function to normalize ten string columns: `brand_color`, `logo`, `secret_saml_key`, `saml_idp_name`, `dashboard_description`, `dashboard_video`, `dashboard_image`, `domain`, `certification_logo`, and `stats_endpoint_key`. Each normalized column is aliased with the same name to replace the original.

The query also includes a column expression `ARRAY(SELECT id) as client_ids`, which appears intended to create an array of client identifiers, though the subquery lacks a FROM clause in the provided SQL.

```sql
SELECT
  id,
  name,
  created,
  updated,
  LOWER(brand_color) AS brand_color,
  LOWER(logo) AS logo,
  user_stats,
  time_to_proficiency,
  kc_and_final,
  sentiment_evaluation,
  cohort_course_analysis,
  persona_insight,
  LOWER(secret_saml_key) AS secret_saml_key,
  LOWER(saml_idp_name) AS saml_idp_name,
  LOWER(dashboard_description) AS dashboard_description,
  LOWER(dashboard_video) AS dashboard_video,
  LOWER(dashboard_image) AS dashboard_image,
  LOWER(domain) AS domain,
  LOWER(certification_logo) AS certification_logo,
  dashboard_visible,
  user_responses,
  stats_users_only,
  use_client_courses_for_stats_endpoint,
  LOWER(stats_endpoint_key) AS stats_endpoint_key,
  include_questions_answer_client_stats,
  ARRAY(SELECT id) as client_ids,
FROM
  ${ref("clients")}
```
