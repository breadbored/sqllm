--- USER ---

Give me a enriched lesson dimension table that includes all lesson attributes along with the list of client IDs associated with each lesson through its course and course container relationships. Lessons not associated with any client should default to client ID 1.

--- AGENT ---

## 1. Identify Shapes

**Input shape (joined):**
- `clients_course_containers`: `client_id`, `course_container_id`
- `courses`: `id`, `container_id`
- `course_lessons`: `course_id`, `lesson_id`
- `lessons`: `id`, `name`, `created`, `updated`, `container_id`, `updated_admin_id`, `creator_admin_id`, `version`, `published`, `mobile_friendly`, `description`, `video_id`, `thumbnail`, `has_been_published`, `archived`, `theater_view`, `show_lab`, `skippable`

**Target shape:**
One row per lesson with all lesson attributes plus a `client_ids` array column containing the distinct client IDs associated with that lesson (defaulting to `[1]` when no clients are found).

## 2. Identify Sources and Relationships

The relationship from lessons to clients is indirect and requires a three-hop chain:

- `lessons.id` → `course_lessons.lesson_id` (lesson to course mapping)
- `course_lessons.course_id` → `courses.id` (course identity)
- `courses.container_id` → `clients_course_containers.course_container_id` (course container to client mapping)

This chain resolves which clients are associated with a given lesson. The final join back to `lessons` is on `stg_lessons.lesson_id = lessons.id`, and it is a LEFT JOIN because not every lesson may be associated with a client through this chain.

## 3. Identify Aggregation Depth

Single-level aggregation — the client IDs are collected at the lesson grain:

```
lesson_id, ARRAY_AGG(DISTINCT client_id) as client_ids
```

No further rollup is needed. The CTE pre-aggregates to the lesson grain so the final join is 1:1 with the `lessons` table.

## 4. Identify Dependencies

- **Grouping column:** `course_lessons.lesson_id` — this is the grain of the CTE and the join key to the `lessons` table.
- **Derived column:** `client_ids` — aggregated from `clients_course_containers.client_id` after traversing the three-hop join chain.
- **Intermediate joins:** `courses` acts as a bridge between `clients_course_containers` (via `container_id = course_container_id`) and `course_lessons` (via `id = course_id`). These must be resolved inside the CTE before aggregation.
- **Default value:** `COALESCE(sl.client_ids, [1])` applies in the final SELECT, depending on the LEFT JOIN producing NULLs for unmatched lessons.

## 5. Plan CTE Layers Bottom-Up

**`stg_lessons`** — Joins the client-course-container chain (`clients_course_containers` → `courses` → `course_lessons`) and aggregates distinct client IDs per lesson, producing one row per `lesson_id` with a `client_ids` array.

**Final SELECT** — LEFT JOINs `lessons` to `stg_lessons` on `lesson_id`, selects all lesson attributes, and applies `COALESCE` to default `client_ids` to `[1]` for lessons with no client association.

## 6. Write Each CTE with Justification

**`stg_lessons`:** Pre-aggregates the multi-hop client-to-lesson relationship into a single row per lesson with a `client_ids` array. This is necessary because joining `lessons` directly to the three-table chain would fan out lesson rows across multiple clients — aggregating first ensures a clean 1:1 join in the final query.

## 7. Compose the Final Query

```sql
WITH
stg_lessons as (
    SELECT
        cl.lesson_id as lesson_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    GROUP BY
        cl.lesson_id
)


SELECT
    l.id,
    l.name,
    l.created,
    l.updated,
    l.container_id,
    l.updated_admin_id,
    l.creator_admin_id,
    l.version,
    l.published,
    l.mobile_friendly,
    LOWER(description) AS description,
    l.video_id,
    l.thumbnail,
    l.has_been_published,
    l.archived,
    l.theater_view,
    l.show_lab,
    l.skippable,
    COALESCE(sl.client_ids, [1]) AS client_ids
FROM
    ${ref("lessons")} as l
LEFT JOIN
    stg_lessons as sl
ON
    sl.lesson_id = l.id
```

## 8. Verify Output Shape

**Target:** One row per lesson with all lesson attributes + `client_ids` array.

**Actual output columns:** `id`, `name`, `created`, `updated`, `container_id`, `updated_admin_id`, `creator_admin_id`, `version`, `published`, `mobile_friendly`, `description`, `video_id`, `thumbnail`, `has_been_published`, `archived`, `theater_view`, `show_lab`, `skippable`, `client_ids`.

**Grain check:** The `stg_lessons` CTE groups by `lesson_id`, producing at most one row per lesson. The LEFT JOIN to `lessons` on `lesson_id = id` is therefore 1:1, preserving the lesson grain. Every lesson appears exactly once. ✓

**Default check:** `COALESCE(sl.client_ids, [1])` handles lessons with no matching client chain by providing a default array of `[1]`. ✓

**Notable detail:** `LOWER(description)` is applied without a table alias, which means it references `l.description` by default resolution. This lowercases the lesson description in the output. ✓
