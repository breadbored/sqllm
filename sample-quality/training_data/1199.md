What is another incorrect way to use backticks when accessing protocol buffer extensions?

--- AGENT ---

This query shows another incorrect backtick pattern: wrapping the entire extension path in a single set of backticks. Backticks must be applied individually to each component of the path, not around the complete dotted path.

```googlesql {.bad}
SELECT
  foo_field.(`some.package.bar`).value = 5
FROM
  Test;
```
