## Question
Give me a user-level summary of all non-test users, including their profile fields and counts of how many courses they have completed versus in progress.

--- AGENT ---

## Step 1: Identify shapes

**Input shape (joined):**
`u.id, u.first_name, u.last_name, u.email, u.created, u.updated, u.last_online, u.meta_quest_id, u.facebook_id, u.google_id, u.amazon_id, u.test_user, uc.user_id, uc.course_id, uc.status, c.id`

**Target shape:**
`user_id, first_name, last_name, email, user_created, user_updated, last_online, meta_quest_id, facebook_id, google_id, amazon_id, test_user, completed_courses, in_progress_courses`

One row per user, with profile columns carried through and two derived counts from the user-courses relationship.

## Step 2: Identify sources and relationships

Three tables:
- `stg_users` (aliased `u`) — the anchor table, one row per user
- `stg_user_courses` (aliased `uc`) — junction/enrollment table linking users to courses, with a `status` column
- `stg_courses` (aliased `c`) — course dimension table

Join relationships:
- `u.id = uc.user_id` — direct relationship between users and their course enrollments
- `uc.course_id = c.id` — direct relationship between enrollments and courses

Both joins are LEFT JOINs, so users with no course enrollments are preserved.

Note: `stg_courses` is joined but no columns from it are selected or used in any filter/aggregation. It is structurally present but functionally inert in this query — possibly a remnant from an earlier version that used course-level attributes, or present to ensure only enrollments with valid course references are counted (though as a LEFT JOIN, it wouldn't filter anything). It has no effect on the output.

## Step 3: Identify aggregation depth

Single-level aggregation. The query groups directly at the user level and counts course statuses:

```
u.id, MAX(u.first_name) as first_name, MAX(u.last_name) as last_name, ..., SUM(CASE WHEN uc.status='completed' THEN 1 ELSE 0 END) as completed_courses, SUM(CASE WHEN uc.status='in_progress' THEN 1 ELSE 0 END) as in_progress_courses
```

No multi-level aggregation is needed because the enrollment table (`uc`) is already at user×course granularity — each row represents one enrollment. Counting statuses via conditional SUM is a single aggregation pass.

## Step 4: Identify dependencies

- **Grouping column:** `u.id` — the user's primary key drives the GROUP BY
- **Carried columns:** all other `u.*` columns are functionally dependent on `u.id` (one value per user), so `MAX()` is used as a safe aggregate wrapper to satisfy GROUP BY semantics without altering the values
- **Derived columns:** `completed_courses` and `in_progress_courses` are conditional counts over `uc.status`, derived from the joined enrollment rows
- **Filter:** `u.test_user = 0` excludes test accounts before aggregation

No intermediate steps are required — the join granularity (user × course) is the right level for the aggregation.

## Step 5: Plan CTE layers bottom-up

No CTEs are needed. This is a single-pass query:
- Join users to enrollments (and courses), filter test users, group by user ID, and compute conditional counts.

## Step 6: Write each CTE with justification

No CTEs required — the query is a single SELECT with GROUP BY.

## Step 7: Compose the final query

```sql
SELECT
  max(u.id)            AS user_id,
  max(u.first_name)    AS first_name,
  max(u.last_name)     AS last_name,
  max(u.email)         AS email,
  max(u.created)       AS user_created,
  max(u.updated)       AS user_updated,
  max(u.last_online)   AS last_online,
  max(u.meta_quest_id) AS meta_quest_id,
  max(u.facebook_id)   AS facebook_id,
  max(u.google_id)     AS google_id,
  max(u.amazon_id)     AS amazon_id,
  max(u.test_user)     AS test_user,
  sum(
    CASE
      WHEN uc.status = 'completed' THEN 1
      ELSE
        0
    END
  )                    AS completed_courses,
  sum(
    CASE
      WHEN uc.status = 'in_progress' THEN 1
      ELSE
        0
    END
  )                    AS in_progress_courses
FROM
  ${ref("stg_users")} AS u
LEFT JOIN
  ${ref("stg_user_courses")} AS uc
  ON
    u.id = uc.user_id
LEFT JOIN
  ${ref("stg_courses")} AS c
  ON
    uc.course_id = c.id
WHERE
  u.test_user = 0
GROUP BY
  u.id
ORDER BY
  u.id
```
