How do I access a nested extension defined within another protocol buffer message?

--- AGENT ---

This query accesses a nested extension using extended syntax that includes the message name where the extension is declared. The expression `foo_field.(some.package.Baz.foo_ext)` specifies both the package and the message (`Baz`) that contains the extension declaration.

```googlesql
SELECT
  foo_field.(some.package.Baz.foo_ext)
FROM
  Test;
```
