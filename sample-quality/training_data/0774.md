--- USER ---

How can I find the number of distinct customers in each country who have invoices, using an approximate cardinality estimate?

--- AGENT ---

This query creates an HLL++ sketch for each country's customer IDs to estimate cardinality, then extracts the final distinct count from each sketch. Using HLL++ sketches allows for approximate distinct counting with memory efficiency, especially useful for large datasets.

```googlesql
SELECT
  country,
  HLL_COUNT.EXTRACT(HLL_sketch) AS distinct_customers_with_open_invoice
FROM
  (
    SELECT
      country,
      HLL_COUNT.INIT(customer_id) AS hll_sketch
    FROM
      UNNEST(
        ARRAY<STRUCT<country STRING, customer_id STRING, invoice_id STRING>>[
          ('UA', 'customer_id_1', 'invoice_id_11'),
          ('BR', 'customer_id_3', 'invoice_id_31'),
          ('CZ', 'customer_id_2', 'invoice_id_22'),
          ('CZ', 'customer_id_2', 'invoice_id_23'),
          ('BR', 'customer_id_3', 'invoice_id_31'),
          ('UA', 'customer_id_2', 'invoice_id_24')])
    GROUP BY country
  );
```
