How do I construct a TIMESTAMP from a string in a specific time zone when the string does not include a zone?

--- AGENT ---

The `TIMESTAMP` function accepts a string and an optional time zone parameter. When the string does not include a time zone, you can specify one explicitly to establish what civil time the string represents.

```googlesql
SELECT TIMESTAMP("2008-12-25 15:30:00", "America/Los_Angeles") AS timestamp_str;
```
