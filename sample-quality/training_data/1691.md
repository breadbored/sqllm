Show me a BigQuery query that extracts login events from an events table and creates a behavior tag by combining type and action fields.

--- AGENT ---

This query filters an events table to login records and reshapes the output by selecting specific columns and creating a new computed tag. The tag is constructed by concatenating the `type` and `action` columns with an underscore separator. A constant `class` column with value 'behavior' is added to all output rows. The `created` timestamp is renamed to `dt` for output. The query returns one row per login event with its sub_session_id, app context, url_path, and timestamp.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  CONCAT(es.type, '_', es.action) as tag,
  'behavior' as class,
  es.app as app,
  es.url_path as url_path,
  es.created as dt
FROM
  ${ref('base_buckets_events_app_speedy')} as es

WHERE 
    es.name = 'login'
```
