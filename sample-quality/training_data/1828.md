--- USER ---

Show me a BigQuery example that counts new and returning sessions for each client and day.

--- AGENT ---

This query identifies, for each client and day, how many distinct sessions are new (first appearance) versus returning (repeat visits from sessions that appeared on earlier days).

The CTE `docs_sessions` performs the setup work: it joins an events table to a documents aggregation table on `doc_id`, extracts the day from each event's timestamp, and uses a window function to compute each session's first-ever appearance date. The UNNEST on `client_id` expands the aggregated client_id array so each client is a separate row.

The main query then groups by day and client_id and counts:
- `total_sessions`: all distinct sessions observed for that day/client combination
- `new_sessions`: sessions where the current day matches their first appearance date
- `returning_sessions`: sessions where the current day is after their first appearance date

Results are ordered by day and client_id.

```sql
WITH
  docs_sessions AS (
  SELECT
    e.session_id,
    DATETIME_TRUNC(e.created, DAY) AS day,
    MIN(DATETIME_TRUNC(e.created, DAY)) OVER (PARTITION BY e.session_id) AS first_date,
    unnested_client_id
  FROM
    ${ref("stg_events_app_speedy")} e
  JOIN
    ${ref("agg_client_documents")} cd
  ON
    e.doc_id = cd.doc_id,
    UNNEST(cd.client_id) AS unnested_client_id)
SELECT
  COUNT(DISTINCT session_id) AS total_sessions,
  COUNT(DISTINCT
    CASE
      WHEN day = first_date THEN session_id
  END
    ) AS new_sessions,
  COUNT(DISTINCT
    CASE
      WHEN day != first_date THEN session_id
  END
    ) AS returning_sessions,
  day,
  unnested_client_id AS client_id
FROM
  docs_sessions
GROUP BY
  day,
  client_id
ORDER BY
  day,
  client_id
```
