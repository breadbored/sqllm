This query enriches document records by joining them to their associated clients, courses, learning paths, and certifications. It produces one row per unique combination of a document and its optional course/learning path/certification relationships.

**Key structure:**
- **Inner joins** to `stg_documents_clients` and `stg_clients` ensure documents are filtered to only those with at least one client
- **Left joins** to courses, badges, and certifications preserve documents that may lack these relationships
- **ARRAY_AGG(DISTINCT ...)** on client columns collects all unique client IDs and names into arrays per document
- **GROUP BY** includes the document ID, its text attributes, and the foreign key columns from the optional relationships (course_container_id, learning_path_id, certification_id)

The GROUP BY structure means if a document has multiple courses or certifications, the query produces multiple rows—one per unique combination of document + course + learning path + certification. The client arrays repeat identically across these rows.

**The SQL:**

```sql
SELECT
  d.id AS doc_id,
  d.name,
  d.path,
  d.type,
  ARRAY_AGG(DISTINCT c.id) AS client_id,
  ARRAY_AGG(DISTINCT c.name) AS client_name,
  dc.course_container_id AS container_id,
  db.learning_path_id AS learning_path,
  dcerts.certification_id,
  d.minutes_to_read
FROM
  ${ref("stg_documents")} AS d
INNER JOIN
  ${ref("stg_documents_clients")} AS dclients
ON
  d.id = dclients.doc_id
INNER JOIN
  ${ref("stg_clients")} AS c
ON
  dclients.client_id = c.id
LEFT JOIN
  ${ref("stg_documents_courses")} AS dc
ON
  d.id = dc.document_id
LEFT JOIN
  ${ref("stg_document_badges")} AS db
ON
  d.id = db.document_id
LEFT JOIN
  ${ref("stg_document_certifications")} AS dcerts
ON
  d.id = dcerts.document_id
GROUP BY
  d.id,
  d.name,
  d.path,
  d.type,
  dc.course_container_id,
  db.learning_path_id,
  dcerts.certification_id,
  d.minutes_to_read
```
