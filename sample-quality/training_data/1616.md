--- USER ---

For each learning path (badge), show me all associated course containers and the clients that have those containers.

--- AGENT ---

This query performs a multi-table join to flatten and aggregate learning path data into a single denormalized row per badge.

The query starts from `stg_learning_paths` and progressively joins:
1. To `stg_learning_path_course_containers` to find which course containers belong to each learning path
2. To `stg_clients_course_containers` to identify which clients are associated with those course containers
3. Left joins to `stg_clients` to retrieve client names

It then groups by learning path (badge) and uses `ARRAY_AGG(DISTINCT ...)` to collect all associated course container IDs, client IDs, and client names into separate arrays. The query filters to only non-archived learning paths.

The output is one row per badge, with denormalized arrays containing all related course containers and clients without duplicates.

```sql
SELECT
  lp.id AS badge_id,
  lp.name AS badge_name,
  ARRAY_AGG(DISTINCT ccc.course_container_id) AS badge_course_containers,
  ARRAY_AGG(DISTINCT ccc.client_id) AS client_id,
  ARRAY_AGG(DISTINCT clients.name) AS client_name
FROM
  ${ref("stg_learning_paths")} lp
JOIN
  ${ref("stg_learning_path_course_containers")} lpcc
ON
  lpcc.learning_path_id = lp.id
JOIN
  ${ref("stg_clients_course_containers")} ccc
ON
  ccc.course_container_id = lpcc.course_container_id
LEFT JOIN
  ${ref("stg_clients")} clients
ON
  clients.id = ccc.client_id
WHERE
  lp.archived = 0
GROUP BY
  lp.id,
  lp.name
```
