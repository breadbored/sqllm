Can you show me a BigQuery example that identifies when a user first viewed a course or short-code page within a session, tagging it as a behavioral event?

--- AGENT ---

This query identifies page view events within course-related URLs and tags them as behavioral signals. It filters for `page_view` events whose URL paths match either `/courses/%` or `/short-codes/%` patterns, and validates that a numeric ID extracted from the end of the URL (via regex) is positive. For each unique combination of sub-session, app, and URL path, it captures the earliest timestamp and labels the event with a constant tag (`'viewed_a_course_page'`) and class (`'behavior'`) for downstream classification.

The regex extraction `REGEXP_EXTRACT(es.url_path, r's/(\d+)$', 1, 1)` pulls a numeric identifier from the URL suffix, ensuring only valid resource references are included. The `MIN(es.created)` aggregation within the GROUP BY captures the first occurrence of each course/page view per sub-session.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  'viewed_a_course_page' as tag,
  'behavior' as class,
  es.app as app,
  es.url_path as url_path,
  MIN(es.created) as dt
FROM
  ${ref("base_buckets_events_app_speedy")} as es

WHERE
  es.name = 'page_view'
AND
  (
    es.url_path LIKE '/courses/%'
    OR
    es.url_path LIKE '/short-codes/%'
  )
AND
  SAFE_CAST(REGEXP_EXTRACT(es.url_path, r's/(\d+)$', 1, 1) AS INT64) > 0

GROUP BY
  sub_session_id,
  es.app,
  es.url_path
```
