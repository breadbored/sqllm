How do I undo changes if an error occurs during a transaction?

--- AGENT ---

Use `ROLLBACK TRANSACTION` inside an exception handler to undo all changes made within the transaction if an error is caught. This restores the database to its state before the transaction began.

```googlesql
BEGIN

  BEGIN TRANSACTION;
  INSERT INTO myschema.NewArrivals
    VALUES ('top load washer', 100, 'warehouse #1');
  -- Trigger an error.
  SELECT 1/0;
  COMMIT TRANSACTION;

EXCEPTION WHEN ERROR THEN
  -- Roll back the transaction inside the exception handler.
  SELECT @@error.message;
  ROLLBACK TRANSACTION;
END;
```
