--- USER ---

Generate a BigQuery query that tracks which learning cues (knowledge checks and sentiment questions) users reach and answer during their lessons, and produces behavioral event tags that capture different dimensions of that interaction—from full context down to simple flags.

--- AGENT ---

This query produces a behavioral event stream tracking user interactions with learning cues (knowledge checks and sentiment questions) at different granularities and dimensions.

The ETL flow:

**Stage 1: Sentiment Scoring.** The `sentiment_preprocessed` CTE normalizes sentiment question option text (e.g., "strongly agree", "very confident") by removing whitespace and special characters, then maps each to a numeric scale (4 = strongly positive, -4 = strongly negative, 0 = neutral). The `sentiment_processed` CTE converts these numeric scores to text labels ("super positive", "very negative", etc.).

**Stage 2: Session Context.** The `page_visit_sessions` CTE groups raw app events by session, user, URL, and app, capturing the min/max timestamp for each logical page visit. The `user_course_lesson_max_progress` CTE joins this with lesson progress data to find the maximum progress each user reached in each lesson during each page visit.

**Stage 3: Cue Resolution.** The `reached_cues` CTE is the core join that, for each learning cue (knowledge check or sentiment question) that a user reached during their progress, captures:
- Whether the cue was answered
- The cue type (knowledge_check or sentiment_question)
- The cue name
- Whether the answer was correct (knowledge checks only)
- The sentiment value and raw answer text (sentiment questions only)
- Session, user, lesson, and page context

**Stage 4: Multi-Dimensional Tagging.** The final 10 UNION ALL branches use the same `reached_cues` data to generate behavioral tags at different levels of detail:
- Full context: answered/unanswered + type + correct/incorrect + cue name
- Minimal context: just reached + type + cue name  
- Without correctness: answered/unanswered + type + cue name (knowledge checks only)
- Sentiment answer text: answered/unanswered + type + cue name + answer text
- Sentiment value: answered + type + sentiment score label
- Has sentiment: answered + type + sentiment presence flag
- Type + answered status
- Type + reached flag
- Answered status only
- Reached flag only
- Correct/incorrect only (knowledge checks only)

Each branch groups by sub_session_id, tag, app, and url_path, capturing the earliest timestamp each tag was encountered in that session. All branches produce output with identical structure: sub_session_id, tag, class ('behavior'), app, url_path, and dt (timestamp).

WITH
sentiment_preprocessed AS (
  SELECT
    sqo.id as sentiment_question_option_id,
    CASE REGEXP_REPLACE(LOWER(TRIM(sqo.name)), r'[^a-z0-9]', '')

      WHEN 'stronglyagree'                              THEN 4
      WHEN 'agree'                                      THEN 3
      WHEN 'neitheragreenordisagree'                    THEN 0
      WHEN 'neitheragreeordisagree'                     THEN 0
      WHEN 'disagree'                                   THEN -3
      WHEN 'stronglydisagree'                           THEN -4

      WHEN 'veryconfident'                              THEN 4
      WHEN 'confident'                                  THEN 3
      WHEN 'somewhatconfident'                          THEN 2
      WHEN 'notconfident'                               THEN -3
      WHEN 'notconfidentatall'                          THEN -4
      WHEN 'notatallconfident'                          THEN -4

      WHEN 'verycomfortable'                            THEN 4
      WHEN 'comfortable'                                THEN 3
      WHEN 'somewhatcomfortable'                        THEN 2
      WHEN 'notcomfortableatall'                        THEN -4

      WHEN 'veryfamiliar'                               THEN 4
      WHEN 'familiar'                                   THEN 3
      WHEN 'somewhatfamiliar'                           THEN 2
      WHEN 'notfamiliar'                                THEN -3
      WHEN 'notfamiliaratall'                           THEN -4

      WHEN 'veryeasy'                                   THEN 4
      WHEN 'easy'                                       THEN 3
      WHEN 'neithereasynordifficult'                    THEN 0
      WHEN 'difficult'                                  THEN -3
      WHEN 'verydifficult'                              THEN -4

      WHEN 'veryimportant'                              THEN 4
      WHEN 'somewhatimportant'                          THEN 2
      WHEN 'notveryimportant'                           THEN -2
      WHEN 'notatallimportant'                          THEN -4

      WHEN 'veryoften'                                  THEN 4
      WHEN 'occasionally'                               THEN 2
      WHEN 'rarely'                                     THEN -3
      WHEN 'never'                                      THEN -4

      WHEN 'very'                                       THEN 4
      WHEN 'somewhat'                                   THEN 2
      WHEN 'notatall'                                   THEN -4

      WHEN 'yes'                                        THEN 3
      WHEN 'no'                                         THEN -3
      WHEN 'true'                                       THEN 3
      WHEN 'false'                                      THEN -3

      WHEN 'metahasprovidedexceptionalresources'        THEN 4
      WHEN 'theresourcesmetahasprovidedareadequate'     THEN 2
      WHEN 'metahasnotgivenmewhatineedtobesuccessful'   THEN -4

      WHEN 'ultimatelymakemyproductbetter'              THEN 3
      WHEN 'ultimatelysavemetime'                       THEN 3
      WHEN 'slowdowndevelopmentwithoutcreatingvalue'    THEN -3
      WHEN 'arearoadblocktoinnovation'                  THEN -3

      WHEN 'naimenjoyingitasis'                         THEN 3
      WHEN 'nayouprovideenoughsupport'                  THEN 3
      WHEN 'naifeelcomfortablemanagingperformance'      THEN 3
      WHEN 'ifeelpreparedforalloftheabove'              THEN 3

      WHEN 'unsure'                                     THEN 0
      WHEN 'imnotsure'                                  THEN 0

      ELSE NULL

    END as value
  FROM
    ${ref("sentiment_question_options")} as sqo, 
    UNNEST(SPLIT(LOWER(REGEXP_REPLACE(sqo.name, r'[^a-zA-Z0-9 ]', '')), ' ')) as _word
  LEFT JOIN
    `dp-dwh.events_sources.sentiment_words` as sw
  ON
    sw.word = _word
  GROUP BY
    sentiment_question_option_id,
    value
),
sentiment_processed AS (
  SELECT
    sp.sentiment_question_option_id as sentiment_question_option_id,
    CASE sp.value
      WHEN 4  THEN 'super positive'
      WHEN 3  THEN 'very positive'
      WHEN 2  THEN 'positive'
      WHEN 1  THEN 'somewhat positive'
      WHEN -4 THEN 'super negative'
      WHEN -3 THEN 'very negative'
      WHEN -2 THEN 'negative'
      WHEN -1 THEN 'somewhat negative'
      WHEN 0  THEN 'neutral'
    END as value
  FROM
    sentiment_preprocessed as sp
  WHERE
    value IS NOT NULL
  GROUP BY
    sentiment_question_option_id,
    value
),
page_visit_sessions AS (
  SELECT
    e.sub_session_id as sub_session_id,
    e.user_id as user_id,
    e.url_path as url_path,
    e.app as app,
    e.page_visit as page_visit,
    MIN(e.created) as min_dt,
    MAX(e.created) as max_dt
  FROM
    ${ref("base_buckets_events_app_speedy")} as e
  GROUP BY
    sub_session_id,
    user_id,
    url_path,
    app,
    page_visit
),
user_course_lesson_max_progress AS (
  SELECT
    pvs.sub_session_id as sub_session_id,
    uclp.user_id as user_id,
    uclp.course_id as course_id,
    uclp.lesson_id as lesson_id,
    pvs.url_path as url_path,
    pvs.app as app,
    MAX(uclp.progress_seconds) as max_progress,
    MIN(GREATEST(uclp.created, pvs.min_dt)) as min_dt,
    MAX(LEAST(uclp.created, pvs.max_dt)) as max_dt
  FROM
    ${ref("user_course_lesson_progress")} as uclp
  JOIN
    page_visit_sessions as pvs
  ON
    pvs.user_id = uclp.user_id
  AND
    uclp.created BETWEEN pvs.min_dt AND pvs.max_dt
  GROUP BY
    sub_session_id,
    user_id,
    course_id,
    lesson_id,
    url_path,
    app
),
reached_cues AS (
  SELECT
    uclmp.sub_session_id as sub_session_id,
    uclmp.user_id as user_id,
    uclmp.course_id as course_id,
    uclmp.lesson_id as lesson_id,
    uclmp.max_progress as max_progress,
    lc.id as learning_cue_id,
    IF(kc.id IS NOT NULL, 'knowledge_check', 'sentiment_question') as cue_type,
    IF(q.id IS NOT NULL, q.name, sq.name) as cue_name,
    IF(kcq.id IS NOT NULL, ukca.answer_option_id IS NOT NULL, ucsqa.sentiment_question_option_id IS NOT NULL) as cue_answered,
    IF(kcq.id IS NOT NULL, NULL, sqo.name) as cue_answered_text,
    IF(kcq.id IS NOT NULL, NULL, sp.value) as cue_answered_sentiment,
    IF(kcq.id IS NOT NULL, ukca.correct = 1, NULL) as cue_correct,
    lc.start_seconds as at_seconds,
    uclmp.url_path as url_path,
    uclmp.app as app,
    uclmp.min_dt as min_dt,
    uclmp.max_dt as max_dt
  FROM
    user_course_lesson_max_progress as uclmp

  -- Learning Cues (that the user reached)
  JOIN
    ${ref("learning_cues")} as lc
  ON
    lc.lesson_id = uclmp.lesson_id
  AND
    lc.start_seconds <= uclmp.max_progress + 5

  -- Knowledge Checks
  LEFT JOIN
    ${ref("knowledge_checks")} as kc
  ON
    kc.learning_cue_id = lc.id
  LEFT JOIN
    ${ref("knowledge_check_questions")} as kcq
  ON
    kcq.knowledge_check_id = kc.id
  LEFT JOIN
    ${ref("questions")} as q
  ON
    q.id = kcq.question_id
  -- and if the user answered it
  LEFT JOIN
    ${ref("user_knowledge_check_answers")} as ukca
  ON
    ukca.course_id = uclmp.course_id
  AND
    ukca.lesson_id = uclmp.lesson_id
  AND
    ukca.user_id = uclmp.user_id
  AND
    ukca.knowledge_check_id = kc.id
  AND
    ukca.question_id = q.id

  -- Sentiment Questions
  LEFT JOIN
    ${ref("sentiment_questions")} as sq
  ON
    sq.learning_cue_id = lc.id
  LEFT JOIN
    ${ref("user_course_sentiment_question_answers")} as ucsqa
  ON
    ucsqa.course_id = uclmp.course_id
  AND
    ucsqa.lesson_id = uclmp.lesson_id
  AND
    ucsqa.user_id = uclmp.user_id
  AND
    ucsqa.sentiment_question_id = sq.id
  LEFT JOIN
    ${ref("sentiment_question_options")} as sqo
  ON
    sqo.id = ucsqa.sentiment_question_option_id
  LEFT JOIN
    sentiment_processed as sp
  ON
    sp.sentiment_question_option_id = sqo.id

  WHERE
    kc.id IS NOT NULL
  OR
    sq.id IS NOT NULL
)
