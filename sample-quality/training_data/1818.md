--- USER ---

Show me an example BigQuery SQL query that analyzes monthly user growth trends for a specific client, with month-over-month percentage changes over a rolling 3-month window.

--- AGENT ---

This table-valued function analyzes user activity trends for a given client by breaking down distinct user counts across three rolling monthly periods: the current 28 days, the preceding 28 days, and the 28 days before that. It then computes two month-over-month percentage changes to show growth acceleration or deceleration.

The function uses three CTEs to isolate each monthly cohort based on the `course_started` timestamp, then combines them in a single SELECT that projects:
- The total distinct users across all time for that client
- The three monthly user counts
- Two derived percentage change metrics using the formula: `((current - previous) / previous) * 100`

The percentage changes answer "how did growth rate change from month to month?"

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64) AS (
  WITH
    current_month AS (
    SELECT
      COUNT(DISTINCT user_id) AS this_month_users
    FROM
      ${ref("reporting_user_courses")}
    WHERE
      clientId IN UNNEST(client_id)
      AND TIMESTAMP(course_started) >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 28 DAY)),
    last_month AS (
    SELECT
      COUNT(DISTINCT user_id) AS last_month_users
    FROM
      ${ref("reporting_user_courses")}
    WHERE
      clientId IN UNNEST(client_id)
      AND TIMESTAMP(course_started) >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 56 DAY)
      AND TIMESTAMP(course_started) < TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 28 DAY)),
    previous_to_last_month AS (
    SELECT
      COUNT(DISTINCT user_id) AS previous_to_last_month_users
    FROM
      ${ref("reporting_user_courses")}
    WHERE
      clientId IN UNNEST(client_id)
      AND TIMESTAMP(course_started) >= TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 84 DAY)
      AND TIMESTAMP(course_started) < TIMESTAMP_SUB(CURRENT_TIMESTAMP(), INTERVAL 56 DAY))
  SELECT
    (
    SELECT
      COUNT(DISTINCT user_id)
    FROM
      ${ref("reporting_user_courses")}
    WHERE
      clientId IN UNNEST(client_id)) AS total_users,
    cm.this_month_users,
    lm.last_month_users,
    ptlm.previous_to_last_month_users,
    ROUND(((cm.this_month_users - lm.last_month_users) / lm.last_month_users) * 100, 1) AS percentage_change_this_month,
    ROUND(((lm.last_month_users - ptlm.previous_to_last_month_users) / ptlm.previous_to_last_month_users) * 100, 1) AS percentage_change_last_month
  FROM
    current_month cm,
    last_month lm,
    previous_to_last_month ptlm)
```
