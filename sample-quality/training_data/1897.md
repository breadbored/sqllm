## Question
Provide an example BigQuery SQL query that demonstrates how to enrich user assessment answer records with associated client identifiers, including a default client ID when external mappings are unavailable.

--- AGENT ---

This query enriches the `user_course_assessment_answers` table by attaching a `client_ids` array to each record. 

The CTE `user_clients` extracts users who have an external client mapping, creating an array that pairs a default client ID of `1` with the user's `external_client_id`. The main query left-joins this CTE back to the assessment answers table by `user_id`. For rows where a user has an external client ID, the `client_ids` column receives the paired array `[1, external_id]`. For rows where the user lacks an external mapping—or where the join finds no match—the `COALESCE` function defaults `client_ids` to `[1]` alone.

The result is a fully populated assessment answer dataset with explicit client ID assignments for downstream filtering or aggregation.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  _utb.id,
  _utb.user_id,
  _utb.course_id,
  _utb.assessment_id,
  _utb.question_id,
  _utb.user_course_assessment_id,
  _utb.user_course_assessment_question_id,
  _utb.question_set_id,
  _utb.answer_option_id,
  _utb.correct,
  _utb.created,
  _utb.updated,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_course_assessment_answers")} as _utb
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = _utb.user_id
```
