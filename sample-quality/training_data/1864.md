Show me an example of how to extract and enrich event-level data from a DataDog tracking table with session identifiers and UTM parameters, while filtering for a specific production domain.

--- AGENT ---

This query extracts event records from a DataDog events table, preserving individual events (grouped by session_id, timestamp, URL path, and other event attributes) while enriching them with aggregated UTM parameters from the session.

The aggregation logic uses `MAX(COALESCE(...))` to extract the non-null UTM values, preferring the current view's UTM parameter if present, otherwise falling back to the session's initial view's UTM parameter. This pattern recovers session-level attribution data (utm_source, utm_id, utm_campaign, utm_medium, utm_term) that may be available only on the initial pageview.

Filtering is applied to:
- A specific service tag (`service:docs.dataprotocol.com`)
- A specific host (`docs.dataprotocol.com`)
- Production environment (tag substring check)
- Non-bot traffic
- Geographic exclusions (specific cities: North Charleston, Charleston, Isle of Palms, Columbia, Irmo)

The query also embeds all client IDs as an array in every output row. The GROUP BY preserves the event's timestamp, so this remains event-level data rather than aggregating to true session granularity.

---

```sql
SELECT
  attributes_session_id                                                                                                         AS session_id,
  LOWER(attributes_view_url_path)                                                                                               AS url_path,
  attributes_timestamp                                                                                                          AS timestamp,
  attributes_geo_country                                                                                                        AS country,
  attributes_geo_city                                                                                                           AS city,
  attributes_device_type                                                                                                        AS device_type,
  attributes_view_referrer_url_url_host                                                                                         AS referrer_host,
  attributes_action_name                                                                                                        AS action,
  attributes_resource_url_query_en                                                                                              AS event,
  MAX(COALESCE(attributes_view_url_query_utm_source, COALESCE(attributes_session_initial_view_url_query_utm_source, NULL)))     AS utm_source,
  MAX(COALESCE(attributes_view_url_query_utm_id, COALESCE(attributes_session_initial_view_url_query_utm_id, NULL)))             AS utm_id,
  MAX(COALESCE(attributes_view_url_query_utm_campaign, COALESCE(attributes_session_initial_view_url_query_utm_campaign, NULL))) AS utm_campaign,
  MAX(COALESCE(attributes_view_url_query_utm_medium, COALESCE(attributes_session_initial_view_url_query_utm_medium, NULL)))     AS utm_medium,
  MAX(COALESCE(attributes_view_url_query_utm_term, COALESCE(attributes_session_initial_view_url_query_utm_term, NULL)))         AS utm_term,
  attributes_session_useragent                                                                                                  AS user_agent,
  attributes_session_ip                                                                                                         AS ip,
  ARRAY(SELECT id FROM ${ref("clients")}) as client_ids,
FROM
  ${ref("dd_events")}
WHERE
  attributes_tags_4 = 'service:docs.dataprotocol.com'
  AND attributes_geo_city != 'North Charleston'
  AND attributes_geo_city != 'Charleston'
  AND attributes_geo_city != 'Isle of Palms'
  AND attributes_geo_city != 'Columbia'
  AND attributes_geo_city != 'Irmo'
  AND SUBSTR(attributes_tags_2, 5) = 'production'
  AND attributes_view_url_host = 'docs.dataprotocol.com'
  AND attributes_device_type != 'Bot'
GROUP BY
  attributes_session_id,
  LOWER(attributes_view_url_path),
  attributes_timestamp,
  attributes_geo_country,
  attributes_geo_city,
  attributes_device_type,
  attributes_view_referrer_url_url_host,
  attributes_action_name,
  attributes_resource_url_query_en,
  attributes_session_useragent,
  attributes_session_ip
```
