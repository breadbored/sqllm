## Question

## 7. Compose the final query

```sql
SELECT
  SPLIT(SPLIT(SPLIT(SPLIT(i3c.name, ',')[0], ' of ')[0], ' (')[0], ' and')[0] as name,
  i3c.`alpha-2` as alpha_2,
  i3c.`iso_3166-2` as alpha_v2_2,
  i3c.`alpha-3` as alpha_3,
  i3c.`country-code` as country_code,
  i3c.region as region,
  i3c.`sub-region` as sub_region,
  i3c.`intermediate-region` as intermediate_region,
  i3c.`region-code` as region_code,
  i3c.`sub-region-code` as sub_region_code,
  i3c.`intermediate-region-code` as intermediate_region_code,
  ARRAY(SELECT id FROM ${ref("clients")}) as client_ids,
FROM
  ${ref("iso_3166_countries")} as i3c
```

**Name-cleaning logic breakdown** (innermost to outermost):
1. `SPLIT(i3c.name, ',')[0]` — strips everything after the first comma (e.g. "Korea, Republic of" → "Korea")
2. `SPLIT(..., ' of ')[0]` — strips " of ..." qualifiers (e.g. "Republic of the Congo" → "Republic")
3. `SPLIT(..., ' (')[0]` — strips parenthetical notes (e.g. "Iran (Islamic Republic of)" → "Iran")
4. `SPLIT(..., ' and')[0]` — strips " and..." continuations (e.g. "Trinidad and Tobago" → "Trinidad")

The splits are applied in this order so that earlier splits reduce the string before later ones operate on it. The comma split runs first because comma-separated forms like "Korea, Republic of" should yield the primary name token before the "of" split would otherwise incorrectly truncate a different part of the name.
