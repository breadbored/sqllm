I need to extract survey response data from event logs, capturing each question asked, the answers provided, and relevant metadata. Can you show me how to structure this query?

--- AGENT ---

This query extracts individual survey response records from an event log table, specifically filtering for events named `meta_horizon_survey`.

The query:
- Extracts the question, answers, and metadata (client_id, other text) from JSON fields within each event's `action_meta` column
- Unnests the `answers` array so each answer appears as a separate row (using LEFT JOIN with UNNEST)
- Joins with a users table to associate each response with a user record
- Groups by session, sub_session, user, question, answer, and metadata fields
- For each unique combination, retains the earliest event timestamp via MIN(created)
- Filters the results to only include rows where a question was actually captured

The output is one row per (session, sub_session, user, question, answer) combination, with de-duplicated metadata and the earliest timestamp for that group.

```sql
SELECT
  e.session_id as session_id,
  e.sub_session_id as sub_session_id,
  e.user_id as user_id,
  JSON_EXTRACT_SCALAR(e.action_meta, '$.question') as question,
  COALESCE(JSON_EXTRACT_SCALAR(answer), 'other') as answer,
  JSON_EXTRACT_SCALAR(e.action_meta, '$.other') as other_text,
  JSON_EXTRACT_SCALAR(e.action_meta, '$.client_id') as client_id,
  MIN(e.created) as created
FROM
  ${ref("app_events")} as e
LEFT JOIN
  UNNEST(JSON_EXTRACT_ARRAY(e.action_meta, '$.answers')) as answer
JOIN
  ${ref("stg_users")} as u
ON
  u.id = e.user_id
WHERE
  e.name = 'meta_horizon_survey'
GROUP BY
  session_id,
  sub_session_id,
  user_id,
  question,
  answer,
  other_text,
  client_id
HAVING
  question IS NOT NULL
```
