How do I write a recursive query using subpipelines instead of nested FROM clauses?

--- AGENT ---

The `RECURSIVE UNION` operator accepts a subpipeline where the recursive input table is passed in automatically as the input to the subpipeline. This eliminates the need to explicitly reference the table in a `FROM` clause.

```googlesql
FROM Employees
|> WHERE employee_id = 123456
|> RECURSIVE UNION ALL
   (
     |> JOIN Employees e ON e.manager_id = AllReportees.employee_id
     |> SELECT e.*
   ) AS AllReportees
|> WHERE State = 'AK'
|> AGGREGATE COUNT(*) AS num_employees;
```
