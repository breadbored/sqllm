--- AGENT ---

This query enriches course notes with a list of associated client IDs. It works in two steps:

**Step 1 — Build the lookup (course_clients CTE):**
The CTE joins the courses table with clients_course_containers to collect all distinct client IDs for each course. For each unique course ID, it creates an array of the associated client_ids using ARRAY_AGG(DISTINCT).

**Step 2 — Enrich and output:**
The main query selects all columns from course_notes (applying LOWER() to two note columns) and LEFT JOINs to the course_clients CTE on course_id. The COALESCE function provides a fallback default of `[1]` for any course note whose course has no associated clients in the lookup.

The result is a flattened output where each row of course notes is decorated with the array of client IDs relevant to that course.

```sql
WITH
course_clients AS (
  SELECT
    c.id as course_id,
    ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_course_containers")} as ccc
  JOIN
    ${ref("courses")} as c
  ON
    c.container_id = ccc.course_container_id
  GROUP BY
    c.id
)
SELECT
  cn.id,
  cn.user_id,
  cn.course_id,
  cn.lesson_id,
  LOWER(cn.note)           AS note,
  cn.note_seconds,
  cn.created,
  cn.updated,
  LOWER(cn.prefilled_note) AS prefilled_note,
  cn.prefilled_note_id,
  cn.prefilled_note_content_id,
  COALESCE(cc.client_ids, [1]) AS client_ids
FROM
  ${ref("course_notes")} as cn
LEFT JOIN
  course_clients as cc
ON
  cc.course_id = cn.course_id
```
