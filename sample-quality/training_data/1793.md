--- USER ---

Show me monthly sentiment question answer counts with cumulative totals for a specific question within a client's course container.

--- AGENT ---

This is a BigQuery TVF that analyzes sentiment question response patterns over time. Here's what it does:

1. **Filters and aggregates** — Starting from sentiment question answers (`stg_user_course_sentiment_question_answers`), it joins through users, sentiment question options, courses, and client-course containers to filter to a specific client, container, and sentiment question. The self-join on `stg_sentiment_questions` (matching by name) retrieves all questions sharing the same name as the input `sentimentQuestionId`.

2. **Monthly grouping** — Answers are grouped by month (formatted as 'Mon YY') and question name, with a count of answers per month.

3. **Cumulative window function** — A running sum is applied over the monthly counts, ordered chronologically, to produce a cumulative total of answers from the first month forward.

4. **Output** — Returns three columns: the month, the answer count for that month, and the cumulative answer count.

The self-join pattern suggests the schema tracks question variants or versions identified by name rather than ID alone.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64,
    sentimentQuestionId INT64) AS (
  SELECT
    date,
    monthly_answers,
    SUM(monthly_answers) OVER (ORDER BY PARSE_DATE('%b %y', date) ASC) AS cumulative_answers
  FROM (
    SELECT
      sq.name,
      FORMAT_DATE('%b %y', ucsqa.created) AS date,
      COUNT(ucsqa.id) AS monthly_answers
    FROM
      ${ref("stg_user_course_sentiment_question_answers")} ucsqa
    JOIN
      ${ref("stg_users")} u
    ON
      u.id = ucsqa.user_id
    JOIN
      ${ref("stg_sentiment_question_options")} sqo
    ON
      sqo.sentiment_question_id = ucsqa.sentiment_question_id
      AND sqo.id = ucsqa.sentiment_question_option_id
    JOIN (
      SELECT
        _sq2.*
      FROM
        ${ref("stg_sentiment_questions")} AS _sq
      JOIN
        ${ref("stg_sentiment_questions")} AS _sq2
      ON
        _sq.name = _sq2.name
      WHERE
        _sq.id = sentimentQuestionId ) AS sq
    ON
      sq.id = sqo.sentiment_question_id
    JOIN
      ${ref("stg_courses")} c
    ON
      c.id = ucsqa.course_id
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    WHERE
      ccc.client_id = clientId
      AND container_id = containerId
    GROUP BY
      date,
      sq.name ) AS subquery
  ORDER BY
    PARSE_DATE('%b %y', date) )
```
