How do I define multiple node types in a property graph, with different labels and properties for each node?

--- AGENT ---

The NODE TABLES clause defines the node types in a property graph, each with optional labels, properties, and aliases. You can specify multiple nodes with different configurations—some with explicit labels and property lists, some with all columns as properties, and some with aliases if the same table is used for multiple node types.

```googlesql
NODE TABLES (
  Account,
  Customer
    LABEL Client
    PROPERTIES (cid, name),
  Location AS GeoLocation
    DEFAULT LABEL
    PROPERTIES ALL COLUMNS
)
```
