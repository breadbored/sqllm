--- USER ---

How do I enrich event tracking data with referrer information by matching landing URLs to a backlinks lookup table, applying utm parameter filtering when available?

--- AGENT ---

This query enriches event session data with referrer source information by left-joining a backlinks lookup table.

**Logic:**
1. Joins `events_filter` to `backlinks` on matching landing/destination URLs
2. The join includes conditional utm parameter matching: for each utm field (campaign, content, medium, source, term), the condition passes if the event's utm field is NULL, empty, or if the backlink's destination_url contains that utm value
3. Groups results by session_id and landing_url
4. Returns the minimum source_url from matching backlinks records for each session/landing combination

The soft matching on utm parameters means utm fields act as optional filters—missing or empty utm values don't constrain the match, while present utm values require the backlink destination to contain them.

**Query:**

```sql
SELECT
  e.session_id AS session_id,
  e.landing_url AS landing_url,
  MIN(b.source_url) AS landed_from
FROM
  ${ref("events_filter")} AS e
LEFT JOIN
  ${ref("backlinks")} AS b
ON
  e.landing_url = b.destination_url
AND
  (e.utm_campaign IS NULL OR e.utm_campaign = '' OR REGEXP_CONTAINS(b.destination_url, e.utm_campaign))
AND
  (e.utm_content IS NULL OR e.utm_content = '' OR REGEXP_CONTAINS(b.destination_url, e.utm_content))
AND
  (e.utm_medium IS NULL OR e.utm_medium = '' OR REGEXP_CONTAINS(b.destination_url, e.utm_medium))
AND
  (e.utm_source IS NULL OR e.utm_source = '' OR REGEXP_CONTAINS(b.destination_url, e.utm_source))
AND
  (e.utm_term IS NULL OR e.utm_term = '' OR REGEXP_CONTAINS(b.destination_url, e.utm_term))
GROUP BY
  e.session_id,
  e.landing_url
```
