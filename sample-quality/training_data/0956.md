How do I use implicit timestamp coercion with a timezone name instead of just an offset?

--- AGENT ---

A string with a timestamp in canonical format followed by a named timezone is automatically coerced to a TIMESTAMP literal when used in a comparison context. This allows you to include timezone information using named zones rather than offsets.

```googlesql
SELECT * FROM foo
WHERE timestamp_col = "2014-09-27 12:30:00.45 America/Los_Angeles"
```
