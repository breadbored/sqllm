--- USER ---

Generate a behavioral tagging dataset that identifies when users make progress on course lessons or reach course progress milestones within their sub-sessions, tagged by progress level.

--- AGENT ---

This query produces a behavioral tagging dataset by combining two progress signals via UNION ALL, both rooted in the concept of marking user advancement within a sub-session.

**First branch:** Identifies lesson-level progress by joining application events with user course lesson progress records where actual progress was recorded (progress_seconds > 0). It matches users across the same date using three different course ID matching strategies: direct match on course_id, match on derived_course_id, or extraction of a course ID from the URL path via regex. All matches are tagged as 'made_progress'. The query groups by sub_session_id, app, and url_path, capturing the earliest progress timestamp for each combination. The HAVING clause constrains results to progress events that fall within the event time window for that sub-session.

**Second branch:** Identifies course-level progress by joining application events with course reporting data. It creates fine-grained progress tags by bucketing course_progress_percentage into 5% intervals, ranging from 95% down to <10% (tagged as 'progress_dropped'). The query groups by sub_session_id, course start time, app, and a standardized url_path constructed as "/courses/{course_id}". The HAVING clause ensures course start times fall within the event window.

Both branches produce records with identical structure: sub_session_id, tag, class ('behavior'), app, url_path, and timestamp. The combined output creates a comprehensive behavioral tag set for analyzing course progress across sub-sessions.

```sql
SELECT
  eas.sub_session_id AS sub_session_id,
  'made_progress' AS tag,
  'behavior' as class,
  eas.app AS app,
  eas.url_path AS url_path,
  MIN(uclp.created) AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS eas
JOIN
  ${ref("stg_user_course_lesson_progress")} AS uclp
ON
  uclp.user_id = eas.user_id
AND
  DATE(uclp.created) = DATE(eas.created)
AND
    (
        uclp.course_id = eas.course_id
    OR
        uclp.course_id = eas.derived_course_id
    OR
        SAFE_CAST(REGEXP_EXTRACT(eas.url_path, r's/(\d+)$', 1, 1) AS INT64) = uclp.course_id
    )
WHERE
  uclp.progress_seconds > 0
GROUP BY
  sub_session_id,
  eas.app,
  eas.url_path
HAVING
  MIN(uclp.created) BETWEEN MIN(eas.created) AND MAX(eas.created)

UNION ALL

SELECT
  es.sub_session_id AS sub_session_id,
  CASE
    WHEN uc.course_progress_percentage >= 0.95 THEN 'progress_95%'
    WHEN uc.course_progress_percentage >= 0.90 THEN 'progress_90%'
    WHEN uc.course_progress_percentage >= 0.85 THEN 'progress_85%'
    WHEN uc.course_progress_percentage >= 0.80 THEN 'progress_80%'
    WHEN uc.course_progress_percentage >= 0.75 THEN 'progress_75%'
    WHEN uc.course_progress_percentage >= 0.70 THEN 'progress_70%'
    WHEN uc.course_progress_percentage >= 0.65 THEN 'progress_65%'
    WHEN uc.course_progress_percentage >= 0.60 THEN 'progress_60%'
    WHEN uc.course_progress_percentage >= 0.55 THEN 'progress_55%'
    WHEN uc.course_progress_percentage >= 0.50 THEN 'progress_50%'
    WHEN uc.course_progress_percentage >= 0.45 THEN 'progress_45%'
    WHEN uc.course_progress_percentage >= 0.40 THEN 'progress_40%'
    WHEN uc.course_progress_percentage >= 0.35 THEN 'progress_35%'
    WHEN uc.course_progress_percentage >= 0.30 THEN 'progress_30%'
    WHEN uc.course_progress_percentage >= 0.25 THEN 'progress_25%'
    WHEN uc.course_progress_percentage >= 0.20 THEN 'progress_20%'
    WHEN uc.course_progress_percentage >= 0.15 THEN 'progress_15%'
    WHEN uc.course_progress_percentage >= 0.10 THEN 'progress_10%'
    WHEN uc.course_progress_percentage < 0.10 THEN 'progress_dropped'
  END AS tag,
  'behavior' as class,
  es.app AS app,
  CONCAT("/courses/", uc.course_id) AS url_path,
  uc.course_started AS dt,
FROM
  ${ref('base_buckets_events_app_speedy')} AS es
JOIN
  ${ref('reporting_user_courses')} AS uc ON uc.user_id = es.user_id
GROUP BY
  sub_session_id,
  uc.course_started,
  es.app,
  url_path,
  tag
HAVING
  uc.course_started BETWEEN MIN(es.created) AND MAX(es.created)
```
