--- USER ---

Build a parameterized table function that performs behavioral lift analysis. Given a set of qualifying behavioral tags and a goal tag, find all sessions that match the qualifier, then for each other behavioral tag those sessions exhibited (before goal conversion), output per-session rows with frequency buckets, UTM attribution, and baseline conversion rates for comparison.

--- AGENT ---

## Step 1 — Identify shapes

**Input tables:**

- `base_buckets_events_app_speedy` (aliased `eas`): `sub_session_id`, `utm_campaign`, `created`, `name` — raw event-level data with UTM attribution and timestamps
- `agg_app_bucket_tags` (aliased `aabt`/`t`): `sub_session_id`, `tag`, `app`, `instances`, `min_dt`, `max_dt` — pre-aggregated behavioral tag assignments per session, with instance counts and time bounds

**Parameters:** `qualifier ARRAY<STRING>` (tags that define the qualifying population), `thing STRING` (the goal tag)

**Target shape (one row per qualifying session × non-qualifier/non-goal tag × frequency bucket):**

`sub_session_id, app, utm_campaign, session_started, tag, frequency_bucket, has_positive, has_negative, total_did_thing, did_thing`

## Step 2 — Identify sources and relationships

- `base_buckets_events_app_speedy` → `agg_app_bucket_tags`: joined on `sub_session_id` (session identity)
- `agg_app_bucket_tags` is the central table — it is self-joined multiple times for different filtered views:
  - Once to identify qualifying sessions (all qualifier tags present)
  - Once to find goal conversion timestamps
  - Once as inline subqueries to determine positive/negative outcome per session
  - Once as the driving table for candidate behavior tags
- `first_utm` provides UTM attribution, joined to tag rows on `sub_session_id`
- `thing_doer` is a filter CTE — restricts the population to sessions that have ALL qualifier tags (set intersection via `COUNT(DISTINCT tag) = ARRAY_LENGTH(qualifier)`)
- `first_goal` provides the conversion timestamp, LEFT JOINed — sessions may never have converted
- `p` and `n` are inline distinct-session subqueries checking whether a session did or didn't do the goal `thing`
- `baseline_sentiment` is a single-row aggregate (no join key) — CROSS JOINed to every output row

## Step 3 — Identify aggregation depth

There are multiple aggregation concerns but they operate on **different slices** of data rather than forming a strict hierarchical chain. The aggregation plan:

**Branch A — Session attribution (first_utm):**
```
sub_session_id, MIN_BY(utm_campaign, created) as utm_campaign, MIN(created) as session_started
```
Single level. Groups raw page_view events to one row per session.

**Branch B — Qualifying population (thing_doer):**
```
sub_session_id, COUNT(DISTINCT tag) as tag_count [HAVING tag_count = ARRAY_LENGTH(qualifier)]
```
Single level. Filters to sessions possessing all qualifier tags. Output is a distinct list of `sub_session_id`.

**Branch C — Goal timestamp (first_goal):**
```
sub_session_id, MAX(max_dt) as first_goal_dt [WHERE tag = thing]
```
Single level. One row per session with the goal tag's latest timestamp.

**Branch D — Baseline conversion (baseline_sentiment):**
```
[filtered to thing_doer sessions]
COUNT(DISTINCT CASE WHEN tag = thing THEN sub_session_id END) as did_thing, COUNT(DISTINCT sub_session_id) as total_did_thing
```
Single level, scalar output (one row total). Counts how many qualifying sessions converted.

**Branch E — Tag-level sentiment (tag_sentiment):**
```
tag, app, sub_session_id, utm_campaign, session_started, frequency_bucket,
  COUNT(DISTINCT CASE positive) as did_thing, COUNT(DISTINCT CASE negative) as didnt_did_thing
```
Single level, but depends on branches A–C being pre-materialized. Groups by session × tag × frequency bucket.

**Final output:**
```
tag_sentiment columns, boolean-cast did_thing/didnt_did_thing, CROSS JOIN baseline scalars
```
No further aggregation — just column projection and a cross join.

## Step 4 — Identify dependencies

- `thing_doer` depends only on `agg_app_bucket_tags` + the `qualifier` parameter
- `first_utm` depends only on `base_buckets_events_app_speedy`
- `first_goal` depends only on `agg_app_bucket_tags` + the `thing` parameter
- `baseline_sentiment` depends on `thing_doer` (to filter the population) and `agg_app_bucket_tags`
- `tag_sentiment` depends on all four preceding CTEs:
  - `first_utm` (for UTM attribution)
  - `thing_doer` (to restrict to qualifying sessions)
  - `first_goal` (to enforce "behavior before conversion" temporal filter)
  - Inline subqueries `p` and `n` against `agg_app_bucket_tags` (for positive/negative flags)
- The final SELECT depends on `tag_sentiment` and `baseline_sentiment`

## Step 5 — Plan CTE layers bottom-up

1. **`first_utm`** — Extracts the first UTM campaign and session start time per session from page_view events.
2. **`thing_doer`** — Identifies sessions that possess every tag in the qualifier array (set intersection via HAVING COUNT DISTINCT = ARRAY_LENGTH).
3. **`first_goal`** — Finds the conversion timestamp per session for the goal tag.
4. **`baseline_sentiment`** — Computes a single-row baseline: how many qualifying sessions converted vs total qualifying sessions.
5. **`tag_sentiment`** — For each qualifying session and each non-qualifier, non-goal tag observed before conversion, computes frequency bucket and positive/negative outcome flags at the session × tag × frequency granularity.
6. **Final SELECT** — Projects tag_sentiment with boolean-cast flags, cross-joined to the baseline scalar row.

## Step 6 — Write each CTE with justification

**`first_utm`** — Pre-aggregates raw events to one row per session so that UTM campaign attribution and session start time can be joined efficiently downstream. Uses `MIN_BY` to pick the campaign from the earliest page_view event.

**`thing_doer`** — Implements set-intersection logic: UNNESTing the qualifier parameter, filtering `agg_app_bucket_tags` to rows matching any qualifier tag, then requiring the distinct count of matched tags equals the full qualifier array length. This ensures a session must have ALL qualifying tags, not just one.

**`first_goal`** — Isolates the goal tag's timestamp per session so that downstream joins can enforce temporal ordering (behavior must precede conversion). LEFT JOINed downstream because not all sessions will have converted.

**`baseline_sentiment`** — Produces a single-row denominator/numerator pair for the overall qualifying population. Scoped to `thing_doer` sessions via a subquery filter. Uses conditional COUNT DISTINCT to split sessions that did vs didn't do the goal thing.

**`tag_sentiment`** — The core analysis CTE. Joins the tag table to all supporting CTEs, excludes qualifier and goal tags from the candidate set, enforces temporal ordering via `t.min_dt < fg.first_goal_dt` (or allows rows where the session never converted), and groups by session × tag × app × frequency bucket. The `p` and `n` inline subqueries flag whether each session did or didn't perform the goal action, producing `did_thing` and `didnt_did_thing` counts per group.

## Step 7 — Compose the final query

```sql
CREATE OR REPLACE TABLE FUNCTION ${self()} (qualifier ARRAY<STRING>, thing STRING) AS (
WITH 
first_utm AS (
  SELECT
    eas.sub_session_id as sub_session_id,
    MIN_BY(eas.utm_campaign, eas.created) as utm_campaign,
    MIN(eas.created) as session_started
  FROM 
    ${ref("base_buckets_events_app_speedy")} as eas
  WHERE
    eas.name = 'page_view'
  GROUP BY
    sub_session_id
),
thing_doer AS (
  SELECT 
    aabt.sub_session_id as sub_session_id
  FROM 
    ${ref("agg_app_bucket_tags")} as aabt, UNNEST(qualifier) as _t
  WHERE
    aabt.tag = _t
  GROUP BY
    sub_session_id
  HAVING
    COUNT(DISTINCT aabt.tag) = ARRAY_LENGTH(qualifier)
),
first_goal AS (
  SELECT
    sub_session_id,
    MAX(max_dt) as first_goal_dt  -- or whatever timestamp field exists
  FROM
    ${ref("agg_app_bucket_tags")}
  WHERE
    tag = thing
  GROUP BY
    sub_session_id
),
baseline_sentiment AS (
  SELECT 
    COUNT(
      DISTINCT 
      CASE 
        WHEN aabt.tag = thing
          THEN sub_session_id 
      END
    ) as did_thing,
    COUNT(
      DISTINCT 
      aabt.sub_session_id
    ) as total_did_thing
  FROM
    ${ref("agg_app_bucket_tags")} as aabt
  WHERE 
    sub_session_id IN (
      SELECT
        ds.sub_session_id as sub_session_id
      FROM
        thing_doer as ds
    )
  AND
    aabt.tag = thing
),
tag_sentiment AS (
  SELECT 
    t.tag,
    t.app,
    fu.sub_session_id,
    fu.utm_campaign,
    fu.session_started,
    -- Add frequency bucket
    CASE 
      WHEN t.instances = 1 THEN '1 time'
      WHEN t.instances BETWEEN 2 AND 4 THEN '2-4 times'
      WHEN t.instances BETWEEN 5 AND 10 THEN '5-10 times'
      ELSE '10+ times'
    END as frequency_bucket,
    COUNT(
      DISTINCT
      CASE
        WHEN p.sub_session_id IS NOT NULL
          THEN ds.sub_session_id
      END
    ) as did_thing,
    COUNT(
      DISTINCT
      CASE
        WHEN n.sub_session_id IS NOT NULL
          THEN ds.sub_session_id
      END
    ) as didnt_did_thing,
  FROM
    ${ref("agg_app_bucket_tags")} as t
  JOIN
    first_utm as fu
  ON
    fu.sub_session_id = t.sub_session_id
  JOIN
    thing_doer as ds
  ON
    t.sub_session_id = ds.sub_session_id
  LEFT JOIN 
    first_goal as fg
  ON
    t.sub_session_id = fg.sub_session_id
  LEFT JOIN 
    (
      SELECT
        DISTINCT
        _aabt.sub_session_id as sub_session_id 
      FROM
        ${ref("agg_app_bucket_tags")} as _aabt
      WHERE
        _aabt.tag = thing
    ) as p
  ON
    ds.sub_session_id = p.sub_session_id
  LEFT JOIN 
    (
      SELECT
        DISTINCT
        _aabt.sub_session_id as sub_session_id 
      FROM
        ${ref("agg_app_bucket_tags")} as _aabt
      WHERE
        _aabt.tag != thing
    ) as n
  ON
    ds.sub_session_id = n.sub_session_id
  WHERE
    t.tag NOT IN UNNEST(qualifier)
  AND
    t.tag != thing
  AND
    (
      fg.first_goal_dt IS NULL          -- user never converted
    OR
      t.min_dt < fg.first_goal_dt    -- behavior happened before goal
    )
  GROUP BY
    t.tag,
    t.app,
    fu.sub_session_id,
    fu.utm_campaign,
    fu.session_started,
    frequency_bucket
)
SELECT 
  -- per visit, per tag, per frequency, per etc...
  ts.sub_session_id as sub_session_id,
  ts.app as app,
  ts.utm_campaign as utm_campaign,
  ts.session_started as session_started,
  ts.tag as tag,
  ts.frequency_bucket as frequency_bucket,
  -- ts.feedback_givers_with_tag as feedback_givers_with_tag,
  -- ts.positive_count as positive_count,
  -- ts.negative_count as negative_count,
  ts.did_thing > 0 as has_positive,
  ts.didnt_did_thing > 0 as has_negative,
  
  -- TOTALS
  b.total_did_thing as total_did_thing,
  b.did_thing as did_thing,
FROM
  tag_sentiment as ts
CROSS JOIN 
  baseline_sentiment as b

ORDER BY
  has_positive DESC
)
```
