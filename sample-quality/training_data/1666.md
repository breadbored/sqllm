Show me a BigQuery query that identifies new sessions from event data, extracting the first app, URL path, and timestamp for each session that begins with a given session number pattern.

--- AGENT ---

This query identifies and tags new sessions by filtering for events where the `sub_session_num` starts with '0' (using a string split on the hyphen character), then groups by `sub_session_id`. For each group, it extracts the earliest `app` and `url_path` values—determined by whichever row has the minimum `created` timestamp via `MIN_BY`—and captures that earliest timestamp. The query assigns a hardcoded tag of "new_session" and a class label of "behavior" to each result, producing a structured record of session inception events.

```sql
SELECT
  eas.sub_session_id as sub_session_id,
  "new_session" as tag,
  'behavior' as class,
  MIN_BY(eas.app, eas.created) as app,
  MIN_BY(eas.url_path, eas.created) as url_path,
  MIN(eas.created) as dt

FROM
  ${ref("base_buckets_events_speedy")} as eas

WHERE
  SPLIT(eas.sub_session_num, '-')[OFFSET(0)] = '0'

GROUP BY 
  sub_session_id
```
