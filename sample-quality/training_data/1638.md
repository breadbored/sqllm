--- USER ---

Show me a query that segments users into new and returning cohorts based on comparing their signup dates against when they make transactions, broken down by month.

--- AGENT ---

This query calculates monthly user segmentation by distinguishing new users from returning users based on signup and transaction timing.

The first CTE (`total_daily_users`) captures all users with their transaction dates by joining the transactions table against the users table and grouping by user and date to establish a daily user footprint.

The second CTE (`returning_monthly_users`) identifies returning users—those whose transactions occur in a different calendar month than their signup month. The WHERE clause filters for records where `datetime_trunc(u.created, MONTH)` (signup month) differs from `datetime_trunc(dt.created, MONTH)` (transaction month), capturing users transacting after their initial signup period.

The final query left-joins these CTEs on the transaction date and aggregates by month. For each month, it calculates:
- **monthly_unique_users**: distinct count of all users with transactions that month
- **monthly_returning_users**: distinct count of users identified as returning (transacting in a month after signup)
- **monthly_new_users**: the difference, representing users transacting in their signup month

Results are ordered chronologically.

```sql
WITH
total_daily_users AS (
  SELECT
    dt.user_id,
    date(dt.created) AS created
  FROM
    ${ref("agg_transactions")} AS dt
  INNER JOIN
    ${ref("stg_users")} AS u
    ON
      dt.user_id = u.id
  GROUP BY
    dt.user_id,
    date(dt.created)
),

returning_monthly_users AS (
  SELECT
    dt.user_id,
    date(dt.created) AS created
  FROM
    ${ref("agg_transactions")} AS dt

  INNER JOIN
    ${ref("stg_users")} AS u
    ON
      dt.user_id = u.id
  WHERE
    datetime_trunc(u.created, MONTH) != datetime_trunc(dt.created, MONTH)
  GROUP BY
    dt.user_id,
    date(dt.created)
)

SELECT
  datetime_trunc(du.created, MONTH)                       AS date,
  count(DISTINCT du.user_id)                              AS monthly_unique_users,
  count(DISTINCT du.user_id) - count(DISTINCT ru.user_id) AS monthly_new_users,
  count(DISTINCT ru.user_id)                              AS monthly_returning_users
FROM
  total_daily_users AS du
LEFT JOIN
  returning_monthly_users AS ru
  ON
    du.created = ru.created
GROUP BY
  date
ORDER BY
  date
```
