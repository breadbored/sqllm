## Question
Produce a cleaned view of client users that lowercases the SAML ID and adds a single-element array of the client ID for compatibility with downstream consumers expecting a client_ids array.

--- AGENT ---

## 1. Identify Shapes

**Input shape:** `id, client_id, user_id, saml_id, created, updated` (from `client_users`)

**Target shape:** `id, client_id, user_id, saml_id (lowercased), created, updated, client_ids (array)`

## 2. Identify Sources and Relationships

Single source table: `client_users`. No joins required.

## 3. Identify Aggregation Depth

There is no aggregation in this query. It is a 1:1 row-preserving transformation. The only operations are:

- `LOWER(saml_id)` — a scalar function applied per row
- `ARRAY(SELECT client_id)` — a scalar subquery that wraps the row's own `client_id` into a single-element array

Arrow notation for completeness (no aggregation, passthrough):

```
id, client_id, user_id, LOWER(saml_id) as saml_id, created, updated, ARRAY(SELECT client_id) as client_ids
```

No `->` steps — single level, no GROUP BY.

## 4. Identify Dependencies

- All columns are sourced directly from `client_users` with no derived dependencies between them.
- `saml_id` is transformed in place via `LOWER()`.
- `client_ids` is derived from `client_id` on the same row — it is a scalar correlated subquery that produces a one-element array containing the current row's `client_id`.

## 5. Plan CTE Layers Bottom-Up

No CTEs are needed. This is a single SELECT with row-level transformations.

## 6. Write Each CTE with Justification

N/A — no CTEs.

## 7. Compose the Final Query

```sql
SELECT
  id,
  client_id,
  user_id,
  LOWER(saml_id) AS saml_id,
  created,
  updated,
  ARRAY(SELECT client_id) as client_ids,
FROM
  ${ref("client_users")}
```

## 8. Verify Output Shape

Target shape: `id, client_id, user_id, saml_id, created, updated, client_ids`

Final SELECT produces: `id, client_id, user_id, saml_id (lowercased), created, updated, client_ids (single-element array)`

✓ Output shape matches target. Row cardinality is 1:1 with the source table. The `ARRAY(SELECT client_id)` construct is BigQuery syntax for wrapping a scalar value into a single-element array — it references the outer row's `client_id` via a correlated scalar subquery.
