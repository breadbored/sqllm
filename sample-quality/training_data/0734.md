What happens if LIMIT is in the RETURN clause instead of immediately after ORDER BY?

--- AGENT ---

When `LIMIT` is placed in the `RETURN` clause instead of immediately after `ORDER BY`, the sort order is not preserved and results are returned unordered. This example shows that using `LIMIT` in `RETURN` does not preserve the `ORDER BY` sorting.

```googlesql
-- Warning: Using the LIMIT clause in the RETURN statement, but not immediately
-- after the ORDER BY statement, also returns the unordered transfer.amount
-- values.
GRAPH FinGraph
MATCH (src_account:Account)-[transfer:Transfers]->(dst_account:Account)
ORDER BY transfer.amount DESC
RETURN src_account.id AS account_id, transfer.amount AS transfer_amount
LIMIT 10
```
