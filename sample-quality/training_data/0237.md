Why does passing a table with collated columns to a table function result in an error?

--- AGENT ---

Table functions cannot accept table arguments containing collated columns, as this limitation prevents mixing collation specifications across function boundaries. This query demonstrates the restriction by attempting to call a user-defined table function with a SELECT statement that produces a collated column, which will fail with an error message indicating that collation specifications are not allowed on table function arguments.

```googlesql
CREATE TABLE FUNCTION my_dataset.my_tvf(x TABLE<col_str STRING>) AS (
  SELECT col_str FROM x
);

SELECT * FROM my_dataset.my_tvf(
  (SELECT COLLATE('abc', 'und:ci') AS col_str)
);
```
