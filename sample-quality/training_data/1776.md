## Question
Write a BigQuery table-valued function that calculates the average course watch time per user for a specific client and container, returning a single aggregated metric.

--- AGENT ---

This query creates a table-valued function that computes a single metric: average course progress minutes per distinct user. 

The function accepts two parameters: `clientId` (an INT64 identifier) and `containerId` (an INT64 identifier). It sources data from a table called `reporting_user_courses`, which appears to contain columns for `course_progress_minutes`, `user_id`, a `client_id` column (which appears to be an array type), and `container_id`.

The aggregation divides the total sum of `course_progress_minutes` by the count of distinct `user_id` values, effectively computing the mean minutes per unique user. The WHERE clause filters the source data to include only rows where the parameter `clientId` is found within the unnested values of the `client_id` array column, and where the `container_id` column exactly matches the `containerId` parameter.

The result is a single row with one column: `avg_watch_time_per_user`.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    (SUM(course_progress_minutes)/COUNT(DISTINCT user_id)) AS avg_watch_time_per_user
  FROM
    ${ref("reporting_user_courses")}
  WHERE
    clientId IN UNNEST(client_id)
    AND container_id = containerId)
```
