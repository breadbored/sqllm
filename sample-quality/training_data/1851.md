## Question
Provide me an example BigQuery SQL query that retrieves certification records with their associated client relationships and normalizes text fields.

--- AGENT ---

This query retrieves all certification records with their associated client relationships and applies text normalization.

The logic works in two stages:

**Stage 1 — CTE `cert_clients`**: Groups the `clients_certifications` table by `certification_id` and collects all distinct `client_id` values into an array for each certification, filtering out nulls.

**Stage 2 — Main query**: Selects all columns from the `certifications` table and applies the `LOWER()` function to normalize six text columns (name, description, headline, footer_description, for_me_description, seo). It left joins with the `cert_clients` CTE on the certification ID, so each certification record includes its array of associated client IDs. The `COALESCE()` function provides a default array `[1]` if a certification has no associated clients (i.e., the join returns null).

The result is a flattened, single-level certification record with lowercase text fields and an array of client IDs per certification.

```sql
WITH
cert_clients AS (
  SELECT
    cc.certification_id as certification_id,
    ARRAY_AGG(DISTINCT cc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_certifications")} as cc
  GROUP BY
    cc.certification_id
)
SELECT
  id,
  LOWER(name)               AS name,
  LOWER(description)        AS description,
  published,
  LOWER(headline)           AS headline,
  thumbnail,
  created,
  updated,
  has_been_published,
  is_public,
  lock_order,
  background_image,
  badge_image,
  LOWER(footer_description) AS footer_description,
  LOWER(for_me_description) AS for_me_description,
  LOWER(seo)                AS seo,
  COALESCE(cc.client_ids, [1]) AS client_ids
FROM
  ${ref("certifications")} as c
LEFT JOIN
  cert_clients as cc
ON
  c.id = cc.certification_id
```
