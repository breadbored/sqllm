How do I use GROUP BY ALL to infer grouping keys from non-aggregate expressions?

--- AGENT ---

The `GROUP BY ALL` clause automatically uses all non-aggregate expressions in the `RETURN` statement as grouping keys. This example groups by the implicit keys inferred from the return items.

```googlesql
GRAPH FinGraph
MATCH (:Account)-[:Transfers]->(account:Account)
RETURN account, COUNT(*) AS num_incoming_transfers
GROUP BY ALL
ORDER BY num_incoming_transfers DESC

NEXT

MATCH (account:Account)<-[:Owns]-(owner:Person)
RETURN owner.name AS owner_name, num_incoming_transfers
```
