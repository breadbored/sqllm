--- USER ---

How do I sort strings without treating numeric substrings as numbers?

--- AGENT ---

The 'kn-false' numeric ordering extension disables numeric-aware collation, treating all characters including digits as individual units in lexicographic order. This query sorts strings using the 'en-us-u-kn-false' collation, placing 'a12b' before 'a1b' in standard character-by-character comparison.

```googlesql
SELECT *
FROM UNNEST([
  'a12b',
  'a1b'
]) AS ids
ORDER BY ids COLLATE 'en-us-u-kn-false'
```
