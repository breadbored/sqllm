--- USER ---

Provide a BigQuery SQL query that enriches prefilled notes with the list of clients that should have access to each learning cue based on the course hierarchy.

--- AGENT ---

This query enriches prefilled notes records with the clients associated with their learning cues. The CTE `stg_prefilled_notes` traverses the hierarchy from clients_course_containers through courses, course_lessons, and into learning_cues to collect a distinct array of client_ids for each learning cue. The main query then left joins this aggregated client list onto the prefilled_notes table, using COALESCE to default to [1] (a single-element array) when no clients are found for a particular learning cue. This ensures every prefilled note has a client_ids field, either populated from the upstream join or defaulting to the fallback value.

```sql
WITH
stg_prefilled_notes AS (
    SELECT
        lc.id as learning_cue_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("learning_cues")} as lc
    ON
        lc.lesson_id = cl.lesson_id
    GROUP BY
        lc.id
)


SELECT
    pn.id,
    pn.learning_cue_id,
    pn.created,
    pn.updated,
    COALESCE(spn.client_ids, [1]) AS client_ids
FROM
    ${ref("prefilled_notes")} as pn
LEFT JOIN   
    stg_prefilled_notes as spn
ON
    spn.learning_cue_id = pn.learning_cue_id
```
