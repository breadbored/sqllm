How do I write a differentially private aggregation query with privacy parameters and contribution bounds?

--- AGENT ---

Use SELECT WITH DIFFERENTIAL_PRIVACY to specify the privacy parameters (epsilon and delta) and privacy_unit_column in the OPTIONS clause. Each aggregate function must include contribution_bounds_per_group to define the bounds on per-entity contributions, which determines the scale of noise added to the final aggregated results.

```googlesql
SELECT WITH DIFFERENTIAL_PRIVACY
  OPTIONS (epsilon=10, delta=.01, privacy_unit_column=id)
  item,
  COUNT(*, contribution_bounds_per_group=>(0, 100))
FROM students;
```
