## Question
Show me a BigQuery query that breaks down monthly active users into new and returning cohorts, identifying which users are signing up for the first time versus those returning from previous months.

--- AGENT ---

This query calculates monthly user acquisition and retention metrics by categorizing users as new or returning.

The query constructs two intermediate CTEs:

1. **`total_daily_users`** — captures all distinct users and their transaction dates from the transactions table, joined with the users table to ensure only valid users are included.

2. **`returning_monthly_users`** — identifies users whose transaction month differs from their signup month. This filter (comparing `datetime_trunc(u.created, MONTH)` to `datetime_trunc(dt.created, MONTH)`) isolates users transacting in months after they signed up, marking them as "returning."

The final SELECT aggregates by month and produces:
- **`monthly_unique_users`** — total distinct users active in each month
- **`monthly_returning_users`** — distinct users in that month who signed up in a different month
- **`monthly_new_users`** — derived as the difference (total users minus returning users), representing users whose first activity in the dataset occurs in that month

The LEFT JOIN on date ensures all transactions are counted, while the monthly grouping produces a time series showing cohort acquisition and retention patterns.

```sql
WITH
total_daily_users AS (
  SELECT
    dt.user_id,
    date(dt.created) AS created
  FROM
    ${ref("agg_transactions")} AS dt
  INNER JOIN
    ${ref("stg_users")} AS u
    ON
      dt.user_id = u.id
  GROUP BY
    dt.user_id,
    date(dt.created)
),

returning_monthly_users AS (
  SELECT
    dt.user_id,
    date(dt.created) AS created
  FROM
    ${ref("agg_transactions")} AS dt

  INNER JOIN
    ${ref("stg_users")} AS u
    ON
      dt.user_id = u.id
  WHERE
    datetime_trunc(u.created, MONTH) != datetime_trunc(dt.created, MONTH)
  GROUP BY
    dt.user_id,
    date(dt.created)
)

SELECT
  datetime_trunc(du.created, MONTH)                       AS date,
  count(DISTINCT du.user_id)                              AS monthly_unique_users,
  count(DISTINCT du.user_id) - count(DISTINCT ru.user_id) AS monthly_new_users,
  count(DISTINCT ru.user_id)                              AS monthly_returning_users
FROM
  total_daily_users AS du
LEFT JOIN
  returning_monthly_users AS ru
  ON
    du.created = ru.created
GROUP BY
  date
ORDER BY
  date
```
