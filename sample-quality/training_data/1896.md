## Question
How do I retrieve all user badges along with the client IDs associated with each user, ensuring every badge is tagged with at least client ID 1?

--- AGENT ---

This query retrieves all user badges and associates each with a set of client IDs. It uses a CTE to build an array of client identifiers for each user: the array always contains client ID 1, and additionally includes the user's `external_client_id` if one is present. The main query joins this CTE to the badges table on user ID, using a left join to ensure all badges are returned regardless of whether a matching user exists. The `COALESCE` function provides a fallback to `[1]` for any badges where no user client mapping is found, guaranteeing that every row has at least client ID 1 in its `client_ids` array.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  ub.id,
  ub.name,
  ub.image,
  ub.user_id,
  ub.learning_path_id,
  ub.created,
  ub.updated,
  ub.accredible_id,
  ub.credly_id,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_badges")} as ub
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = ub.user_id
```
