--- USER ---

Show me a BigQuery query that retrieves course lessons enriched with their associated client information as arrays.

--- AGENT ---

This query retrieves course lesson records and enriches them with client IDs. 

It works in two stages:

1. **CTE (`course_clients`)**: Builds a mapping from course IDs to client IDs by joining the `clients_course_containers` table with the `courses` table on a container relationship. It aggregates all distinct client IDs for each course into a single array using `ARRAY_AGG(DISTINCT ... IGNORE NULLS)`.

2. **Main query**: Selects the lesson records from `course_lessons` (columns: id, course_id, lesson_id, created, updated) and left-joins them with the `course_clients` CTE to attach the client arrays. If a lesson's course has no matching entry in `course_clients` (or the client_ids column is null), `COALESCE` provides a default array `[1]` instead.

The result is one row per course lesson, with a client IDs array that either contains the aggregated clients for that course or defaults to `[1]`.

```sql
WITH
course_clients AS (
  SELECT
    c.id as course_id,
    ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_course_containers")} as ccc
  JOIN
    ${ref("courses")} as c
  ON
    c.container_id = ccc.course_container_id
  GROUP BY
    c.id
)
SELECT
  cl.id,
  cl.course_id,
  cl.lesson_id,
  cl.created,
  cl.updated,
  COALESCE(cc.client_ids, [1]) AS client_ids
FROM
  ${ref("course_lessons")} as cl
LEFT JOIN
  course_clients as cc
ON
  cc.course_id = cl.course_id
```
