--- USER ---

How do I match paths that do not reuse the same edge twice, but may revisit nodes?

--- AGENT ---

Use the TRAIL path mode to filter out paths that traverse the same edge more than once, while still allowing nodes to be revisited.

```googlesql
GRAPH FinGraph
MATCH
  TRAIL (a1:Account)-[t1:Transfers]->(a2:Account)-[t2:Transfers]->
  (a3:Account)-[t3:Transfers]->(a4:Account)
WHERE a1.id < a4.id
RETURN
  t1.id as transfer1_id, t2.id as transfer2_id,
  t3.id as transfer3_id
```
