--- USER ---

How would you build a query that tags completed video events, both as generic behavior events and also enriched with course-specific metadata?

--- AGENT ---

This query builds a dual-tagged event feed from completed video state events. It produces two event streams stacked with UNION ALL:

**First branch** selects from the vaas events table where completion state is true, groups by sub_session_id and url_path, and emits each group as a generic 'vaas_completed' behavior tag with its earliest timestamp.

**Second branch** performs the same selection and grouping, but enriches each event by joining to the courses table on a video ID foreign key. For each matching course, it produces a compound tag in the form 'vaas_completed:coursename', tagged as a 'content' class rather than 'behavior'. The join expands the row set — a single session/url combination can produce multiple rows if the video appears in multiple courses.

Both branches preserve the session identifier, URL path, and application label ('vaas'), selecting the minimum timestamp per group as the event time.

The key structural difference is that the first branch produces one row per session-url pair (generic behavior), while the second produces one row per session-url-course triplet (content-specific).

```sql
SELECT
  e.sub_session_id as sub_session_id,
  'vaas_completed' as tag,
  'behavior' as class,
  'vaas' as app,
  e.vaas_url_url_path as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_vaas_speedy")} as e
WHERE
  e.vaas_state_completed = 'true'
GROUP BY
  sub_session_id,
  url_path

UNION ALL

SELECT
  e.sub_session_id as sub_session_id,
  CONCAT('vaas_completed:', c.name) as tag,
  'content' as class,
  'vaas' as app,
  e.vaas_url_url_path as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_vaas_speedy")} as e
JOIN
  ${ref("courses")} as c
ON
  c.vaas_id = e.vaas_url_video_id
WHERE
  e.vaas_state_completed = 'true'
GROUP BY
  sub_session_id,
  tag,
  url_path
```
