Can you extract the first select_option event within each user sub-session and capture the app and URL path where that event occurred?

--- AGENT ---

This query identifies the earliest occurrence of a `select_option` event within each `sub_session_id` and captures contextual information about that event.

The query filters the events table to include only rows where `name = 'select_option'`, then groups by `sub_session_id`. For each group, it extracts:
- The `app` and `url_path` values associated with the earliest event (using `MIN_BY` to retrieve these columns at the minimum timestamp)
- The minimum `created` timestamp for that sub-session
- Fixed literal values: `'select_option'` as a tag and `'behavior'` as a class label

The output is one row per sub-session showing when the first select_option event occurred and where (which app and URL path).

```sql
SELECT
    es.sub_session_id as sub_session_id,
    'select_option' as tag,
    'behavior' as class,
    MIN_BY(es.app, es.created) as app,
    MIN_BY(es.url_path, es.created) as url_path,
    MIN(es.created) as dt

FROM
    ${ref('base_buckets_events_docs_speedy')} as es 

WHERE
    es.name != 'page_view'
AND
    es.name != 'open_link'
AND
    es.name = 'select_option'

GROUP BY
    sub_session_id,
    tag
```
