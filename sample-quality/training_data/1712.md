--- USER ---

Show me a BigQuery query that tags user sessions with course topics based on their lesson progress during that session.

--- AGENT ---

This query correlates session-level events with user lesson progress to generate content tags for sessions. It joins events to a user's course lesson progress records—matching on both user ID and calendar date—then enriches that data by traversing through course metadata to retrieve associated topic tags via a course-container-tag mapping.

For each unique session, app, course, and topic tag combination, the query outputs a record containing the session ID, the tag name prefixed with "topic:", a content class indicator, the app name, the course URL path, and the minimum timestamp from the lesson progress records.

A temporal constraint is enforced via the HAVING clause: the earliest progress timestamp for that session-course-tag group must fall within the time range spanned by the events in that session. This ensures that recorded lesson progress is temporally valid relative to the observed events.

The result is a set of session-to-topic associations that label sessions with the course topics a user engaged with during that session, useful for behavioral tagging or correlating learning activities with application usage patterns.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  CONCAT("topic:", t.name) as tag,
  'content' as class,
  es.app AS app,
  CONCAT("/courses/", uclp.course_id) as url_path,
  MIN(uclp.created) AS dt,
FROM
  ${ref('base_buckets_events_app_speedy')} AS es
JOIN
  ${ref('user_course_lesson_progress')} AS uclp
ON
  uclp.user_id = es.user_id
AND
  DATE(uclp.created) = DATE(es.created)
JOIN
  ${ref("stg_courses")} as c
ON
  c.id = uclp.course_id
JOIN
  ${ref("course_container_tags")} as cct
ON
  cct.course_container_id = c.container_id
JOIN
  ${ref("tags")} as t
ON
  cct.tag_id = t.id
GROUP BY
  sub_session_id,
  es.app,
  url_path,
  tag
HAVING
  MIN(uclp.created) BETWEEN MIN(es.created) AND MAX(es.created)
```
