--- USER ---

How can I combine two queries on the same node type when the output columns are in a different order?

--- AGENT ---

This query uses UNION ALL to combine two separate MATCH queries on Person nodes, each assigning a different group identifier. Even though the RETURN clauses list columns in different orders (name then group_id, versus group_id then name), UNION ALL aligns them by position, allowing the result sets to merge successfully.

```googlesql
GRAPH FinGraph
MATCH (p:Person)
RETURN p.name, 1 AS group_id
UNION ALL
MATCH (p:Person)
RETURN 2 AS group_id, p.name
```
