## Question
Show me a BigQuery query that aggregates event data by session and document, capturing the earliest timestamp and associated metadata for each combination.

--- AGENT ---

This query aggregates event records into one row per sub_session_id and document pairing. It joins events from a document-focused event table against a documents reference table to look up document names.

For each sub_session_id and document combination, it:
- Tags the record with the document name (prefixed with `'document:'`)
- Marks the record class as `'content'`
- Captures the app and url_path values associated with the earliest event timestamp in that group using `MIN_BY()` — a function that returns the value of its first argument corresponding to the minimum value of its second argument
- Records the minimum (earliest) created timestamp for that group

The WHERE clause filters to exclude rows where doc_id is null, ensuring only events with valid document references are included. The GROUP BY collapses multiple events into aggregated rows keyed by sub_session_id and tag.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  CONCAT('document:', d.name) as tag,
  'content' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref('base_buckets_events_docs_speedy')} as es
JOIN
  ${ref('documents')} as d
ON
  d.id = es.doc_id

WHERE
  es.doc_id IS NOT NULL

GROUP BY
    sub_session_id,
    tag
```
