How do I split a pipeline into subpipelines and continue the main pipeline after the split?

--- AGENT ---

The `TEE` semi-terminal pipe operator splits the input into subpipelines for side effects or inspection, but continues the main pipeline after the split. Output tables from subpipelines are returned before the main pipeline output.

```googlesql
WITH
  Fruit AS (
    SELECT 'apple' AS item, 100 AS sales
    UNION ALL
    SELECT 'banana' AS item, 150 AS sales
  )
FROM Fruit
|> TEE
    (|> AGGREGATE COUNT(*) AS total_fruits),
    (|> AGGREGATE COUNT(*) AS count_by_item GROUP BY item)
|> SELECT *;
```
