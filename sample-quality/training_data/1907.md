--- USER ---

Enrich knowledge check answer records with each user's associated client identifiers, including a default client and any external client they may have.

--- AGENT ---

This query enriches knowledge check answer records with client affiliation data.

The CTE `user_clients` builds a mapping of users to their client identifiers. For each user with a non-null `external_client_id`, it creates an array containing the fixed value `1` plus that user's external client ID.

The main query then left-joins the knowledge check answers table against this mapping. For each answer record, it includes all original fields plus a `client_ids` column. This column is populated from the user's client array if a match exists in the CTE, or defaults to `[1]` if the user has no external client ID (via COALESCE). The result is that every user is associated with at least client `1`, and optionally a second external client.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  _utb.id,
  _utb.user_id,
  _utb.course_id,
  _utb.lesson_id,
  _utb.knowledge_check_id,
  _utb.question_id,
  _utb.question_set_id,
  _utb.answer_option_id,
  _utb.correct,
  _utb.created,
  _utb.updated,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_knowledge_check_answers")} as _utb
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = _utb.user_id
```
