--- USER ---

Show me a query that identifies sub-sessions where a feedback modal was viewed, and tags those sessions for feedback eligibility analysis.

--- AGENT ---

This query identifies sub-sessions that viewed a feedback modal and marks them as eligible to give feedback. It filters for events where `name = 'feedback_modal'` and `type = 'view'`, then adds two constant columns: a behavior tag `'eligible_to_give_feedback'` and a class label `'behavior'`. The query groups by `sub_session_id`, `app`, `url_path`, and `created` timestamp, returning one row per unique combination—effectively deduplicating modal view events while preserving the app and URL context where the modal appeared.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  'eligible_to_give_feedback' AS tag,
  'behavior' as class,
  es.app AS app,
  es.url_path AS url_path,
  es.created AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS es
WHERE
  es.name = 'feedback_modal'
AND
  es.type = 'view'
GROUP BY
  es.sub_session_id,
  es.app,
  es.url_path,
  es.created
```
