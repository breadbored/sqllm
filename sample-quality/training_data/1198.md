Why is this syntax for escaping extension path components with backticks incorrect?

--- AGENT ---

This query demonstrates incorrect backtick usage: wrapping multiple path components together in a single pair of backticks (`some.package`) is not valid. The documentation specifies that backticks must surround individual components, not groups or the entire path.

```googlesql {.bad}
SELECT
  foo_field.(`some.package`.`bar`).value = 5
FROM
  Test;
```
