## Question
Show me a query that enriches sentiment question records with the list of client IDs that have access to them through their associated course and lesson hierarchy.

--- AGENT ---

This query maps sentiment questions to their associated client audiences. It uses a staging CTE to traverse the hierarchy from clients → course containers → courses → lessons → learning cues, aggregating distinct client IDs for each learning cue. The main query then enriches each sentiment question record with its accumulated client_ids array, defaulting to [1] for questions without any associated clients via the hierarchy.

The aggregation is done at the learning_cue level (since sentiment questions are linked to learning cues, not directly to clients), and each client_id appears at most once per learning cue due to the DISTINCT qualifier.

```sql
WITH
stg_sentiment_question AS (
    SELECT
        lc.id as learning_cue_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("lessons")} as l
    ON 
        l.id = cl.lesson_id
    JOIN
        ${ref("learning_cues")} as lc
    ON
        lc.lesson_id = l.id
    GROUP BY
        lc.id
)


SELECT
    sq.id,
    sq.name,
    sq.learning_cue_id,
    sq.created,
    sq.updated,
    COALESCE(ssq.client_ids, [1]) AS client_ids
FROM
    ${ref("sentiment_questions")} as sq
LEFT JOIN
    stg_sentiment_question as ssq
ON
    ssq.learning_cue_id = sq.learning_cue_id
```
