Can you show me a BigQuery query that enriches course enrollment records with associated client IDs, handling cases where users may or may not have external client mappings?

--- AGENT ---

This query enriches user course enrollment records with client ID arrays. 

The CTE `user_clients` identifies users who have an external client mapping (where `external_client_id` is not null) and creates an array containing the value `1` plus their `external_client_id`.

The main query selects all columns from the `user_courses` table and left-joins it with `user_clients`. For users with no external client mapping (non-matching join), the `COALESCE` function defaults the `client_ids` array to `[1]`. All matched records receive the array `[1, external_client_id]`. The query also normalizes `payment_type` and `status` to lowercase.

The net effect: every course record gets a `client_ids` array that either contains a user's external client ID (if one exists) or defaults to a single-element array `[1]`.

```sql
WITH
user_clients AS (
  SELECT
    u.id as user_id,
    [1, u.external_client_id] as client_ids
  FROM
    ${ref("users")} as u
  WHERE
    u.external_client_id IS NOT NULL
)
SELECT
  _utb.id,
  _utb.user_id,
  _utb.course_id,
  _utb.created,
  _utb.updated,
  LOWER(_utb.payment_type) AS payment_type,
  _utb.payment_amount,
  _utb.payment_intent,
  _utb.coupon_id,
  LOWER(_utb.status)       AS status,
  _utb.access_code_id,
  _utb.completion_date,
  COALESCE(ucl.client_ids, [1]) as client_ids
FROM
  ${ref("user_courses")} as _utb
LEFT JOIN
  user_clients as ucl
ON
  ucl.user_id = _utb.user_id
```
