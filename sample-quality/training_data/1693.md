Provide me with an example BigQuery SQL query that identifies survey completion events and tags them for behavioral analysis, preserving session context and event details.

--- AGENT ---

This query identifies meta_horizon_survey events from the events table, tags each as a behavioral signal labeled 'answered_mh_survey_after_feedback', and returns the session identifier, application, URL path, and timestamp for each unique event. The GROUP BY deduplicates raw events, retaining one row per unique combination of sub_session_id, app, url_path, and created timestamp. The constant tag and class columns add classification labels for downstream processing.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  'answered_mh_survey_after_feedback' AS tag,
  'behavior' as class,
  es.app AS app,
  es.url_path AS url_path,
  es.created AS dt
FROM
  ${ref("base_buckets_events_app_speedy")} AS es
WHERE
  es.name = 'meta_horizon_survey'
GROUP BY
  sub_session_id,
  tag,
  app,
  url_path,
  created
```
