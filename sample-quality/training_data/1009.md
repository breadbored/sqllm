How do I extract the hostname, public suffix, and registered domain from a collection of URLs with varying formats and validity?

--- AGENT ---

This query demonstrates extracting three URL components—hostname, public suffix, and registered domain—from a variety of URL formats including standard URLs, URLs with ports and authentication, IPv6 hosts, internationalized domain names, and malformed inputs. NET.HOST extracts the hostname, NET.PUBLIC_SUFFIX extracts the ICANN-registered top-level domain, and NET.REG_DOMAIN extracts the registrable domain (public suffix plus one preceding label), returning NULL when components cannot be parsed.

```googlesql
SELECT
  FORMAT("%T", input) AS input,
  description,
  FORMAT("%T", NET.HOST(input)) AS host,
  FORMAT("%T", NET.PUBLIC_SUFFIX(input)) AS suffix,
  FORMAT("%T", NET.REG_DOMAIN(input)) AS domain
FROM (
  SELECT "" AS input, "invalid input" AS description
  UNION ALL SELECT "http://abc.xyz", "standard URL"
  UNION ALL SELECT "//user:password@a.b:80/path?query",
                   "standard URL with relative scheme, port, path and query, but no public suffix"
  UNION ALL SELECT "https://[::1]:80", "standard URL with IPv6 host"
  UNION ALL SELECT "http://例子.卷筒纸.中国", "standard URL with internationalized domain name"
  UNION ALL SELECT "    www.Example.Co.UK    ",
                   "non-standard URL with spaces, upper case letters, and without scheme"
  UNION ALL SELECT "mailto:?to=&subject=&body=", "URI rather than URL--unsupported"
);
```
