Generate a BigQuery query that extracts outbound link events from a session events table, grouping them by sub-session and destination URL, and returning the earliest app and path information for each group.

--- AGENT ---

This query extracts outbound hyperlink events from an events table, grouping by sub-session identifier and the action URL to identify unique link destinations accessed within each sub-session. For each group, it captures a behavior classification tag, the app and url_path values from the earliest event in that group (using MIN_BY), and the minimum timestamp across the group. The WHERE clause filters to only events named 'open_link' where the action field contains a valid HTTPS URL pattern.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  CONCAT('outbound_link:_', es.action) as tag,
  'behavior' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref("base_buckets_events_speedy")} as es

WHERE
  es.name = 'open_link'
AND
  es.action LIKE 'https://%'

GROUP BY 
  sub_session_id,
  tag
```
