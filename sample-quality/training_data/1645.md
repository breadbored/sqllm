## Question
Can you show me a BigQuery query that enriches user knowledge check answers with question details, course information, and answer validation? I want to see how to join multiple tables to create a denormalized answer record view.

--- AGENT ---

This query constructs a denormalized view of user knowledge check answers by joining the base `stg_user_knowledge_check_answers` table with five related tables to enrich each answer record with contextual information.

The query retrieves, for each answer:
- The user ID, course ID, and lesson ID that identify the context
- The question and its ID
- The user's selected answer option and its text (via `stg_question_options` aliased as `ua`)
- The correct answer option and its text (via `stg_question_options` aliased as `qo`)
- Course metadata (name and container_id)
- A `correct` flag indicating whether the user's answer matched the correct answer
- The timestamp when the answer was submitted

The joins are structured around two instances of `stg_question_options` — one for the user's answer and one for the correct answer — allowing the query to display both side-by-side. The LEFT JOIN to `stg_users` is used only for filtering, excluding test users (`test_user = 0`), and the WHERE clause also filters out records with NULL user IDs.

Here is the query:

```sql
SELECT
  ukca.user_id,
  ukca.course_id,
  c.name AS course_name,
  c.container_id,
  ukca.lesson_id,
  ukca.question_id,
  q.name AS question,
  ukca.answer_option_id,
  ua.name AS user_answer,
  ukca.correct,
  ukca.created AS date_answered,
  qs.correct_option_id,
  qo.name AS correct_answer
FROM
  ${ref("stg_user_knowledge_check_answers")} ukca
JOIN
  ${ref("stg_question_sets")} qs
ON
  qs.question_id = ukca.question_id
JOIN
  ${ref("stg_questions")} q
ON
  q.id = qs.question_id
JOIN
  ${ref("stg_question_options")} ua
ON
  ua.id = ukca.answer_option_id
JOIN
  ${ref("stg_question_options")} qo
ON
  qo.id = qs.correct_option_id
JOIN
  ${ref("stg_courses")} c
ON
  c.id = ukca.course_id
LEFT JOIN
  ${ref("stg_users")} u
ON
  u.id = ukca.user_id
WHERE
  user_id IS NOT NULL
  AND u.test_user = 0
```
