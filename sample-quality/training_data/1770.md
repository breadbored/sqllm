--- USER ---

For a given client ID, show me the monthly count of course assessments and the running cumulative total of assessments over time.

--- AGENT ---

This query creates a table function that, given a client ID, returns three columns: a month-year date, the count of assessments taken in that month, and a running cumulative sum of assessments from the earliest month through the current month.

The query joins assessment records through user and course tables to reach the clients table via a course container relationship, filtering to a single client. It groups assessments by month, formats the grouping key as a human-readable month-year string ("Jan 26"), and then applies a window function to compute the cumulative total. The window function orders by the parsed date to ensure the running sum accumulates in chronological order.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64) AS (
  SELECT
    date,
    monthly_assessments,
    SUM(monthly_assessments) OVER (ORDER BY PARSE_DATE('%b %y', date)) AS cumulative_assessments
  FROM (
    SELECT
      FORMAT_DATE('%b %y', uca.created) AS date,
      COUNT(uca.id) AS monthly_assessments
    FROM
      ${ref("stg_user_course_assessments")} uca
    JOIN
      ${ref("stg_users")} u
    ON
      u.id = uca.user_id
    JOIN
      ${ref("stg_courses")} c
    ON
      c.id = uca.course_id
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    WHERE
      ccc.client_id = clientId
    GROUP BY
      date) AS subquery
  ORDER BY
    PARSE_DATE('%b %y', date) )
```
