--- USER ---

Show me a query that fetches certification learning paths and enriches each one with a list of all distinct clients associated with its certification. If a certification has no clients, default to a placeholder client ID.

--- AGENT ---

This query performs a two-step enrichment:

**Step 1 (CTE):** The `cert_clients` CTE pre-aggregates the `clients_certifications` table, collecting all distinct `client_id` values (excluding nulls) for each `certification_id` into an array.

**Step 2 (Main query):** The main SELECT retrieves all certification learning paths from the `certification_learning_paths` table and performs a LEFT JOIN with the pre-aggregated client arrays. The COALESCE clause ensures that if a certification has no associated clients (NULL from the join), the result defaults to `[1]`.

The output is one row per certification learning path, preserving all its attributes (id, certification_id, learning_path_id, order, created, updated) and adding the aggregated array of client IDs.

```sql
WITH
cert_clients AS (
  SELECT
    cc.certification_id as certification_id,
    ARRAY_AGG(DISTINCT cc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_certifications")} as cc
  GROUP BY
    cc.certification_id
)
SELECT
  clp.id,
  clp.certification_id,
  clp.learning_path_id,
  clp.`order` as `order`,
  clp.created,
  clp.updated,
  COALESCE(cc.client_ids, [1]) AS client_ids
FROM
  ${ref("certification_learning_paths")} as clp
LEFT JOIN
  cert_clients as cc
ON
  cc.certification_id = clp.certification_id
```
