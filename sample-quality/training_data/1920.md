--- USER ---

Build a page-level engagement summary for documentation site sessions. For each page visited within a sub-session, show time on page, scroll depth, click counts, bounce detection, UTM attribution from the landing URL, geo data, and which clients are associated with the document — rolling up events to one row per sub-session per document per URL path.

**Input tables (observable from the query):**

**Target shape — one row per:** `session_id, sub_session_id, derived_doc_id, url_path` (the GROUP BY keys, with `doc_name`, `doc_type`, `lang`, and session-level totals carried as deterministic per-group values).

**Output columns:** `session_id`, `sub_session_id`, `user_id`, `current_user_id`, `doc_id`, `url_path`, `lang`, `doc_name`, `doc_type`, `seconds_on_page`, `engaged`, `max_scroll_depth`, `scroll_events_count`, `click_count`, `num_pages_visited`, `total_page_visits`, `total_session_click_count`, `total_session_duration`, `started_at_app`, `went_to_app`, `referrer`, `utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term`, `geo_country`, `geo_city`, `lat_long`, `date`, `client_id`, `client_name`.

## 2. Identify sources and relationships

- `stg_events_docs_speedy` is the fact table — all other tables join to it.
- `stg_documents` joins on `e.derived_doc_id = d.id` — document metadata lookup.
- `stg_documents_clients` joins on `dc.doc_id = d.id` — bridge table linking documents to clients.
- `clients` joins on `clients.id = dc.client_id` **OR** via a special URL-path-based condition for collection pages (Meta Horizon hardcoded matching). This OR-join means collection pages get client attribution even without a document-client row.
- `sub_session_data` (CTE) joins on `sub_session_id` — pre-aggregated session-level metrics from the same events table.
- `went_to_app` (CTE) joins on `sub_session_id` — pre-aggregated flag from `stg_events_speedy` (the broader events table, not docs-only).

## 3. Identify aggregation depth

There are three aggregation contexts happening in parallel, then composed:

**CTE 1 — sub_session_data (session-level pre-aggregation):**
```
sub_session_id, DATETIME_DIFF(MAX(created), MIN(created), SECOND) as total_session_duration, COUNTIF(click) as total_session_click_count, COUNT(DISTINCT url_path) as total_page_visits
```

**CTE 2 — went_to_app (session-level app-visit flags):**
```
sub_session_id, MIN_BY(app, created)='app' as started_at_app, MIN(app)='app' as went_to_app
```

**Main query (page-level aggregation after joining CTEs + dimension tables):**
```
[e joined to d on derived_doc_id=id, dc on doc_id=d.id, clients on client_id or URL pattern, ssd on sub_session_id, wta on sub_session_id]
session_id, sub_session_id, derived_doc_id, url_path, AGG(various) as page_metrics
```

The pre-aggregated CTE values (`total_session_duration`, `total_page_visits`, `total_session_click_count`) flow through as pass-through columns in the main GROUP BY since they are deterministic per `sub_session_id`.

There is also a window function layer on top of the grouped result:
- `COUNT(url_path) OVER (PARTITION BY sub_session_id ORDER BY MIN(created))` — running page visit count within a sub-session.
- `LEAD(MIN_BY(seconds_from_last_event, created)) OVER (...)` — borrows the next page's entry gap time and adds it to the current page's `seconds_on_page`.

## 4. Identify dependencies

**Grouping keys:** `session_id`, `sub_session_id`, `derived_doc_id`, `url_path` (core grain), plus `doc_name`, `doc_type`, `lang`, and the three `ssd.*` totals which are functionally dependent on `sub_session_id` but listed in GROUP BY for SQL correctness.

**Derived columns requiring aggregation:**
- `user_id`, `current_user_id` — `MAX()` (pick any, likely one per sub-session)
- `seconds_on_page` — `MAX(seconds_on_page)` plus a `LEAD()` window over the grouped result
- `engaged` — derived from CTE totals, not from aggregation itself
- `max_scroll_depth`, `scroll_events_count` — `MAX()` within page group
- `click_count` — `COUNTIF()` within page group
- `referrer`, UTM fields, geo fields — `MIN_BY(..., created)` (take value from earliest event)
- `client_id`, `client_name` — `ARRAY_AGG(DISTINCT ... IGNORE NULLS)`
- `date` — `MIN(created)`

**Dependencies requiring intermediate steps:**
- `total_session_duration`, `total_page_visits`, `total_session_click_count` must be computed at the `sub_session_id` grain before joining to the page-level grain — hence the `sub_session_data` CTE.
- `started_at_app`, `went_to_app` require the broader `stg_events_speedy` table (not just docs events) — hence the `went_to_app` CTE from a different source table.
- The `seconds_on_page` calculation uses `LEAD()` over the page-level groups, meaning the window function executes after the GROUP BY.

## 5. Plan CTE layers bottom-up

1. **`sub_session_data`** — Pre-aggregates session-level totals (duration, click count, distinct page visits) from `stg_events_docs_speedy` at the `sub_session_id` grain, so these can be joined back as scalar values per session without affecting the page-level GROUP BY.

2. **`went_to_app`** — Pre-aggregates app-visit signals from the broader `stg_events_speedy` table at the `sub_session_id` grain, determining whether the session started at the app and whether it ever visited the app.

3. **Main SELECT** — Joins the docs events table to document metadata, client bridge, clients (with OR-based collection matching), and both CTEs, then groups to page-level grain and applies window functions for running page count and cross-page time attribution.

## 6. Write each CTE with justification

**`sub_session_data`:** Exists because session-level totals (duration, total clicks, total pages) are needed per row but must be computed at a coarser grain than the main query's GROUP BY. Without pre-aggregation, these values would be inflated by the page-level and client-join row multiplication.

**`went_to_app`:** Exists because app-visit detection requires the broader events table (`stg_events_speedy`) which includes non-docs events, while the main query operates on `stg_events_docs_speedy`. Separate source table means separate pre-aggregation.

## 7. Compose the final query

```sql
WITH
  sub_session_data AS (
  SELECT
    sub_session_id,
    DATETIME_DIFF(MAX(e.created), MIN(e.created), SECOND) AS total_session_duration,
    COUNTIF(e.type = "click"
      AND e.name != "any_click") AS total_session_click_count,
    COUNT(DISTINCT url_path) AS total_page_visits,
  FROM
    ${ref("stg_events_docs_speedy")} e
  GROUP BY
    e.sub_session_id),
  went_to_app AS (
  SELECT
    e.sub_session_id,
    MIN_BY(e.app, e.created) = 'app' AS started_at_app,
    MIN(e.app) = 'app' AS went_to_app
  FROM
    ${ref("stg_events_speedy")} e
  GROUP BY
    e.sub_session_id)
SELECT
  e.session_id,
  e.sub_session_id,
  MAX(e.user_id) AS user_id,
  MAX(e.current_user_id) AS current_user_id,
  e.derived_doc_id AS doc_id,
  REGEXP_REPLACE(e.url_path, r'(/[a-z]{2}(-[a-z]{2})?)$', '') AS url_path,
  REGEXP_EXTRACT(e.url_path, r'/([a-z]{2}(?:-[a-z]{2})?)$') AS lang,
  CASE
    WHEN e.url_path = '/collections/meta-quest' OR e.url_path = '/collections/meta-horizon/' THEN 'Meta Horizon Collections'
    WHEN e.url_path LIKE '/collections/%' THEN INITCAP( REPLACE( REGEXP_EXTRACT(e.url_path, r'/collections/[^/]+/([^/]+)'), '-', ' ' ) )
    ELSE d.name
END
  AS doc_name,
  CASE
    WHEN e.url_path LIKE '/collections/%' THEN 'collection'
    ELSE d.type
END
  AS doc_type,
  MAX(seconds_on_page) + COALESCE(LEAD(MIN_BY(e.seconds_from_last_event, e.created)) OVER (PARTITION BY e.sub_session_id ORDER BY MIN(e.created)), 0) AS seconds_on_page,
  CASE
    WHEN total_session_duration < 30 AND total_page_visits = 1 AND total_session_click_count = 0 THEN FALSE
  -- Bounce criteria
    ELSE TRUE
  -- Engaged if not a bounce
END
  AS engaged,
  MAX(e.scroll_value) AS max_scroll_depth,
  MAX(e.scroll_num) AS scroll_events_count,
  COUNTIF(e.type = "click"
    AND e.name != "any_click") AS click_count,
  COUNT(e.url_path) OVER (PARTITION BY e.sub_session_id ORDER BY MIN(e.created)) AS num_pages_visited,
  ssd.total_page_visits,
  ssd.total_session_click_count,
  ssd.total_session_duration,
  COALESCE(MAX(wta.started_at_app), FALSE) AS started_at_app,
  COALESCE(MAX(wta.went_to_app), FALSE) AS went_to_app,
  MIN_BY(e.derived_referrer, e.created) AS referrer,
  LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, e.created), r'utm_source=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_source,
  LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, e.created), r'utm_medium=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_medium,
  LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, e.created), r'utm_campaign=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_campaign,
  LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, e.created), r'utm_content=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_content,
  LOWER(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE(REGEXP_REPLACE( REGEXP_EXTRACT(MIN_BY(url, e.created), r'utm_term=([^&]*)'), r'_', ' '), r'\+', ' '), r'%20', ' '), r'-', ' ')) AS utm_term,
  MIN_BY(e.geo_country, e.created) AS geo_country,
  MIN_BY(e.geo_city, e.created) AS geo_city,
  CONCAT(MIN_BY(e.geo_lat, e.created), ",", MIN_BY(e.geo_long, e.created)) AS lat_long,
  MIN(e.created) AS date,
  ARRAY_AGG(DISTINCT clients.id IGNORE NULLS) AS client_id,
  ARRAY_AGG(DISTINCT clients.name IGNORE NULLS) AS client_name
FROM
  ${ref("stg_events_docs_speedy")} e
LEFT JOIN
  ${ref("stg_documents")} d
ON
  e.derived_doc_id = d.id
LEFT JOIN
  ${ref("stg_documents_clients")} dc
ON
  dc.doc_id = d.id
LEFT JOIN
  ${ref("clients")} clients
ON
  clients.id = dc.client_id
  OR (REGEXP_REPLACE(e.url_path, r'(/[a-z]{2}(-[a-z]{2})?)$', '') LIKE '/collections/%'
    AND ((REGEXP_REPLACE(e.url_path, r'(/[a-z]{2}(-[a-z]{2})?)$', '') LIKE '/collections/meta-quest/%'
        AND clients.name = 'Meta Horizon')
      OR (REGEXP_REPLACE(e.url_path, r'(/[a-z]{2}(-[a-z]{2})?)$', '') LIKE '/collections/meta-horizon/%'
        AND clients.name = 'Meta Horizon')))
JOIN
  sub_session_data ssd
ON
  ssd.sub_session_id = e.sub_session_id
LEFT JOIN
  went_to_app AS wta
ON
  wta.sub_session_id = e.sub_session_id
GROUP BY
  session_id,
  sub_session_id,
  derived_doc_id,
  e.url_path,
  doc_name,
  doc_type,
  ssd.total_session_click_count,
  ssd.total_session_duration,
  lang,
  total_page_visits
```
