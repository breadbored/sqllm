## Question
Normalize user activity events from multiple learning platform tables into a single unified event stream with consistent columns for tracking user actions across courses, assessments, lessons, and account events.

--- AGENT ---

This query normalizes 17 disparate source tables from a learning management system into a single unified event table. Each UNION ALL branch represents a different type of user activity:

- **Engagement events** (course notes, lesson progress, course/lesson completion, watchlist additions) map source-specific data to a standard schema
- **Assessment events** (assessment answers, questions, and completions) capture user assessment interactions with completion status and scores
- **Knowledge check events** (answers and completions) track knowledge check responses with answer option IDs and correctness flags
- **Sentiment survey events** capture responses to satisfaction/feedback questions with answer options and associated learning cues
- **Badge and status events** (badge earned, course started) use badge names and course status values as action data
- **Account events** (user creation, client user records, lab server access, data downloads) capture system-level activities, with some branches having empty action_type and action values

All branches select the same 17 columns: user identifiers, timestamps, a data_source label identifying the originating table, an action_type describing the event category, an action value containing event-specific data, and optional foreign keys (course_id, lesson_id, assessment_id, question_id, knowledge_check_id, sentiment_question_id, etc.) that are populated only when relevant to that event type. This denormalization into a single table enables time-series analysis of user behavior across the entire learning platform.
