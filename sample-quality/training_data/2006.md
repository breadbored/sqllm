--- USER ---

Give me all prefilled notes enriched with the list of client IDs associated with each note's learning cue, defaulting to client ID 1 when no client association exists.

--- AGENT ---

## 1. Identify Shapes

**Input tables (observable from the query):**
- `prefilled_notes`: `id, learning_cue_id, created, updated`
- `learning_cues`: `id, lesson_id`
- `course_lessons`: `course_id, lesson_id`
- `courses`: `id, container_id`
- `clients_course_containers`: `client_id, course_container_id`

**Target shape:**
```
id, learning_cue_id, created, updated, client_ids
```
One row per prefilled note, with a `client_ids` array containing the distinct clients associated with that note's learning cue through the course container chain.

## 2. Identify Sources and Relationships

The core entity is `prefilled_notes`. Each note has a `learning_cue_id`. The goal is to find which clients are associated with each learning cue.

There is no direct relationship between `learning_cues` and `clients_course_containers`. The join chain is:

```
learning_cues.lesson_id -> course_lessons.lesson_id
course_lessons.course_id -> courses.id
courses.container_id -> clients_course_containers.course_container_id
```

This is a 3-hop chain from learning cue to client:
- `learning_cues` → `course_lessons` (on `lesson_id`)
- `course_lessons` → `courses` (on `course_id = id`)
- `courses` → `clients_course_containers` (on `container_id = course_container_id`)

Then the aggregated result joins back to `prefilled_notes` on `learning_cue_id`.

## 3. Identify Aggregation Depth

Single-level aggregation — we need to collapse the many clients per learning cue into one array per learning cue:

```
learning_cue_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

No further aggregation is needed. The final SELECT is a 1:1 left join from `prefilled_notes` to this pre-aggregated result.

Arrow notation:

```
learning_cue_id, ARRAY_AGG(DISTINCT client_id IGNORE NULLS) as client_ids
```

One level, one CTE.

## 4. Identify Dependencies

- **Grouping key:** `learning_cue_id` (aliased from `lc.id`)
- **Derived column:** `client_ids` — requires traversing the full 4-table join chain before aggregating
- **Intermediate step:** The join chain must be resolved first so that each `learning_cue_id` is paired with all its `client_id` values, then grouped
- **Final step:** `prefilled_notes` LEFT JOINs to the aggregated result. The LEFT JOIN ensures every prefilled note appears even if its learning cue has no client association. `COALESCE(..., [1])` provides a default array containing client ID 1 for unmatched rows.

## 5. Plan CTE Layers Bottom-Up

**`stg_prefilled_notes`** — Joins the 4-table chain from `clients_course_containers` through `courses` and `course_lessons` to `learning_cues`, then aggregates distinct client IDs into an array per learning cue.

**Final SELECT** — Left joins `prefilled_notes` to `stg_prefilled_notes` on `learning_cue_id`, applying a COALESCE default of `[1]` for notes with no client association.

## 6. Write Each CTE with Justification

**`stg_prefilled_notes`:** This CTE exists because the relationship between a learning cue and its clients is indirect (3 hops through `course_lessons`, `courses`, and `clients_course_containers`), and must be pre-aggregated to one row per learning cue before joining to `prefilled_notes` — otherwise the join would fan out prefilled notes across multiple client rows.

## 7. Compose the Final Query

```sql
WITH
stg_prefilled_notes AS (
    SELECT
        lc.id as learning_cue_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("learning_cues")} as lc
    ON
        lc.lesson_id = cl.lesson_id
    GROUP BY
        lc.id
)


SELECT
    pn.id,
    pn.learning_cue_id,
    pn.created,
    pn.updated,
    COALESCE(spn.client_ids, [1]) AS client_ids
FROM
    ${ref("prefilled_notes")} as pn
LEFT JOIN   
    stg_prefilled_notes as spn
ON
    spn.learning_cue_id = pn.learning_cue_id
```
