--- USER ---

Give me a detailed view of every user's course assessment answer, showing the question asked, the answer the user selected, and the correct answer, along with course details and the date the answer was submitted. Exclude test users.

--- AGENT ---

## 1. Identify Shapes

**Input tables and their observable columns:**

- `stg_user_course_assessment_answers` (ucaa): `user_id, course_id, question_id, answer_option_id, assessment_id, created`
- `stg_questions` (q): `id, name`
- `stg_question_options` (qo): `id, name`
- `stg_courses` (c): `id, container_id, name`
- `stg_question_sets` (qs): `question_id, correct_option_id`
- `stg_users` (u): `id, test_user`

**Target shape:**

`user_id, course_id, container_id, course_name, question_id, question, answer_option_id, user_answer, correct_option_id, correct_answer, answer_date, assessment_id`

This is a row-per-answer detail query — no aggregation, no reshaping. One output row per assessment answer.

## 2. Identify Sources and Relationships

The fact table is `ucaa` (user course assessment answers). All other tables are dimensional lookups joined via LEFT JOIN:

| Join | Key |
|---|---|
| `ucaa → q` | `ucaa.question_id = q.id` — resolve question name |
| `ucaa → qo` | `ucaa.answer_option_id = qo.id` — resolve the user's selected answer text |
| `ucaa → c` | `ucaa.course_id = c.id` — resolve course name and container_id |
| `ucaa → qs` | `ucaa.question_id = qs.question_id` — get `correct_option_id` for each question |
| `qs → con` (subquery) | `qs.correct_option_id = con.correct_option_id` — resolve the correct answer's text by joining `stg_question_options` to `stg_question_sets` inside a subquery, then matching on `correct_option_id` |
| `ucaa → u` | `ucaa.user_id = u.id` — used only for filtering (`test_user = 0`) |

The `con` subquery exists because the correct answer text requires a two-hop resolution: `qs.correct_option_id → qo.id → qo.name`. Since `qo` is already aliased for the user's answer, a subquery aliases a second instance of `stg_question_options` to get the correct answer's name.

## 3. Identify Aggregation Depth

There is no aggregation. This is a pure detail/lookup query. In arrow notation:

```
user_id, course_id, container_id, course_name, question_id, question, answer_option_id, user_answer, correct_option_id, correct_answer, answer_date, assessment_id
```

Single level, no `->`, no GROUP BY. The grain is one row per assessment answer.

## 4. Identify Dependencies

- All columns are either direct attributes of the fact table (`ucaa`) or looked up from dimension tables via foreign keys.
- `correct_answer` (the text of the correct option) depends on two lookups chained together: first `qs` to get `correct_option_id`, then the `con` subquery to resolve that ID to a name via `stg_question_options`.
- The `WHERE u.test_user = 0` filter depends on the join to `stg_users`, even though no columns from `u` appear in the SELECT.

## 5. Plan CTE Layers Bottom-Up

No CTEs are needed. The query is a single SELECT with LEFT JOINs and an inline subquery:

- **Inline subquery `con`**: joins `stg_question_sets` to `stg_question_options` to produce `(correct_option_id, name)` pairs, enabling the correct answer text to be resolved.

## 6. Write Each CTE with Justification

No CTEs. The inline subquery `con` exists because the correct answer's text requires joining `stg_question_options` on `correct_option_id` rather than on the user's `answer_option_id` — and since `stg_question_options` is already joined as `qo` for the user's answer, a separate subquery instance avoids ambiguity.

## 7. Compose the Final Query

```sql
SELECT
  ucaa.user_id,
  ucaa.course_id,
  c.container_id,
  c.name       AS course_name,
  ucaa.question_id,
  q.name       AS question,
  ucaa.answer_option_id,
  qo.name      AS user_answer,
  qs.correct_option_id,
  con.name     AS correct_answer,
  ucaa.created AS answer_date,
  ucaa.assessment_id
FROM
  ${ref("stg_user_course_assessment_answers")} AS ucaa
LEFT JOIN
  ${ref("stg_questions")} AS q
  ON
    ucaa.question_id = q.id
LEFT JOIN
  ${ref("stg_question_options")} AS qo
  ON
    ucaa.answer_option_id = qo.id
LEFT JOIN
  ${ref("stg_courses")} AS c
  ON
    ucaa.course_id = c.id
LEFT JOIN
  ${ref("stg_question_sets")} AS qs
  ON
    ucaa.question_id = qs.question_id
LEFT JOIN (
  SELECT
    qo.name,
    qs.correct_option_id
  FROM
    ${ref("stg_question_sets")} AS qs
  LEFT JOIN
    ${ref("stg_question_options")} AS qo
    ON
      qo.id = qs.correct_option_id
) AS con
  ON
    qs.correct_option_id = con.correct_option_id
LEFT JOIN
  ${ref("stg_users")} AS u
  ON
    ucaa.user_id = u.id
WHERE
  u.test_user = 0
```
