## Question
Show me all learning cues with their metadata, along with which clients have access to each cue through the course hierarchy. If a learning cue has no associated clients, default to client ID 1.

--- AGENT ---

This query retrieves all learning cues from the `learning_cues` table and enriches each one with a list of client IDs that have access to it.

The CTE `learning_cue_clients` performs a multi-level join to trace the relationship hierarchy: each learning_cue belongs to a lesson, lessons belong to courses via course_lessons, courses belong to course containers, and course containers belong to clients. For each learning_cue, it aggregates all distinct client IDs that reach it through this path.

The main query then selects all columns from `learning_cues` and left-joins with the CTE to attach the client list. The `COALESCE` ensures that if a learning_cue has no associated clients through the join chain (resulting in NULL), it defaults to a single-element array containing client ID 1.

The output is a denormalized row per learning cue with all its attributes plus its client access list.

```sql
WITH
learning_cue_clients AS (
    SELECT
        lc.id as learning_cue_id,
        ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
    FROM
        ${ref("clients_course_containers")} as ccc
    JOIN
        ${ref("courses")} as c
    ON
        c.container_id = ccc.course_container_id
    JOIN
        ${ref("course_lessons")} as cl
    ON
        cl.course_id = c.id
    JOIN
        ${ref("lessons")} as l
    ON 
        l.id = cl.lesson_id
    JOIN
        ${ref("learning_cues")} as lc
    ON
        lc.lesson_id = l.id
    GROUP BY
        lc.id
)
SELECT
  id,
  start_seconds,
  lesson_id,
  fullscreen,
  force_pause,
  created,
  updated,
  fullscreen_seconds,
  COALESCE(lcc.client_ids, [1]) AS client_ids
FROM
  ${ref("learning_cues")} as lc
LEFT JOIN
    learning_cue_clients as lcc
ON
    lcc.learning_cue_id = lc.id
```
