--- USER ---

Show me a summary of knowledge check questions in a specific course container for a given client, including how many course versions each question appears in, whether the course is live, and the total number of user answers received.

--- AGENT ---

This query is a BigQuery table function that returns a summary of knowledge check questions for a given `clientId` and `containerId`.

**What it does:**

The query joins user knowledge check answers through their associated questions to course data, filtering to a specific client and course container. For each question (grouped by question name), it calculates:

- The maximum question ID in the group
- The question name
- The earliest publication date of the knowledge check question (formatted as MMM-DD-YYYY)
- The count of distinct course versions this question appears in
- Whether the course associated with this question is the live version (determined by comparing the course ID to the maximum course ID in the container)
- The total count of user answers to this question

The subquery on the `stg_courses` table filters to the specified container and client, excludes unpublished courses, and uses a window function to identify the latest course ID per container (the live version).

The outer query filters to exclude test users and applies the container and client parameters in the WHERE clause. Results are ordered by question ID.

---

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    MAX(q.id) AS knowledge_check_question_id,
    q.name AS knowledge_check_question,
    FORMAT_DATE('%b-%d-%Y', MIN(kcq.created)) AS publish_date,
    COUNT(DISTINCT ukca.course_id) AS versions,
    (MAX(c.id) = MAX(c.course_max_id)) AS live,
    COUNT(ukca.id) AS answers,
  FROM
    ${ref("stg_user_knowledge_check_answers")} ukca
  JOIN
    ${ref("stg_knowledge_check_questions")} kcq
  ON
    kcq.question_id = ukca.question_id
  JOIN
    ${ref("stg_questions")} q
  ON
    q.id = kcq.question_id
  JOIN (
    SELECT
      c.id,
      c.container_id,
      MAX(c.id) OVER (PARTITION BY container_id) AS course_max_id
    FROM
      ${ref("stg_courses")} c
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    WHERE
      container_id = containerId
      AND client_id = clientId
      AND has_been_published = 1) c
  ON
    c.id = ukca.course_id
  JOIN
    ${ref("stg_users")} u
  ON
    u.id = ukca.user_id
  WHERE
    c.container_id = containerId
    AND u.test_user = 0
  GROUP BY
    q.name
  ORDER BY
    knowledge_check_question_id)
```
