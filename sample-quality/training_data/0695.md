What happens if I don't declare an outer-scoped variable in a subquery's variable scope list?

--- AGENT ---

When you omit a variable from the scope list in `CALL ()`, that variable becomes independent of the outer scope and is treated as a new variable in the subquery. This example shows that when `p` is not declared in `CALL ()`, it matches independently, returning different results than if it were properly scoped.

```googlesql
GRAPH FinGraph
MATCH (p:Person)  -- Outer-scoped variable `p`
CALL () {  -- `p` not declared
  MATCH (p)-[:Owns]->(a:Account)  -- Inner-scoped variable `p`, independent of outer-scoped `p`
  RETURN a.Id AS account_Id
  ORDER BY account_Id
  LIMIT 2
}
RETURN p.name AS person_name, account_Id
ORDER BY person_name, account_Id;
```
