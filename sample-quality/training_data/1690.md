--- USER ---

Show me a BigQuery query that extracts the first event from each user sub-session and marks it as a logged-in behavior tag.

--- AGENT ---

This query groups events by `sub_session_id` and extracts the earliest event metadata from each group, but only for sub-sessions where a user is logged in (i.e., where `current_user_id` IS NOT NULL). For each sub-session, it returns:

- The `sub_session_id` as the grouping key
- A literal `'logged_in'` tag and `'behavior'` class to mark the event category
- The `app` and `url_path` values from whichever event occurred first in the session (using `MIN_BY` to select the value associated with the minimum `created` timestamp)
- The earliest `created` timestamp in the session

This effectively produces one row per logged-in sub-session, capturing the session identifier, start time, and the initial app/URL context for that session.

```sql
SELECT
  es.sub_session_id as sub_session_id,
  'logged_in' as tag,
  'behavior' as class,
  MIN_BY(es.app, es.created) as app,
  MIN_BY(es.url_path, es.created) as url_path,
  MIN(es.created) as dt

FROM
  ${ref('base_buckets_events_app_speedy')} as es

WHERE
  es.current_user_id IS NOT NULL

GROUP BY
  sub_session_id
```
