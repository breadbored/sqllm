--- USER ---

How would I enrich course resources with the list of clients that have access to each course, defaulting to client 1 if no clients are explicitly associated?

--- AGENT ---

This query enriches course resources with their associated client lists. 

The first CTE (`course_clients`) builds a lookup table by joining the `clients_course_containers` table with the `courses` table on a container relationship, then groups by course ID and aggregates all distinct client IDs into an array for each course.

The main query then selects all columns from `course_resources`, performs a left join with this CTE on `course_id`, and applies two transformations:
1. Lowercases the course name
2. Uses `COALESCE` to default the client_ids array to `[1]` if the join finds no matching course (i.e., if a resource's course has no associated clients)

The result is one row per course resource with all its original columns plus an array of client IDs (or `[1]` if none exist).

```sql
WITH
course_clients AS (
  SELECT
    c.id as course_id,
    ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_course_containers")} as ccc
  JOIN
    ${ref("courses")} as c
  ON
    c.container_id = ccc.course_container_id
  GROUP BY
    c.id
)
SELECT
  cr.id,
  LOWER(cr.name) AS name,
  cr.url,
  cr.lock_until_over,
  cr.course_id,
  cr.created,
  cr.updated,
  cr.`order`,
  cr.file,
  COALESCE(cc.client_ids, [1]) AS client_ids
FROM
  ${ref("course_resources")} as cr
LEFT JOIN
  course_clients as cc
ON
  cc.course_id = cr.course_id
```
