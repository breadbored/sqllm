Show me a BigQuery table function that returns user engagement metrics grouped by course container for a specific client, including distinct user count and total questions answered.

This query is a parameterized table function that accepts a `clientId` and returns engagement metrics aggregated by course container. It joins user question-answer records through three intermediate tables (courses, client-course container mappings, and users) to filter for a single client, then counts the distinct users and total number of question answers per container. The result is grouped by `container_id`, producing one row per container with the user count and question count columns.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64) AS (
  SELECT
    container_id,
    COUNT(DISTINCT ucsqa.user_id) AS total_users,
    COUNT(ucsqa.id) AS questions_answered
  FROM
    ${ref("stg_user_course_sentiment_question_answers")} AS ucsqa
  JOIN
    ${ref("stg_courses")} c
  ON
    c.id = ucsqa.course_id
  JOIN
    ${ref("stg_clients_course_containers")} ccc
  ON
    ccc.course_container_id = c.container_id
  JOIN
    ${ref("stg_users")} u
  ON
    u.id = ucsqa.user_id
  WHERE
    ccc.client_id = clientId
  GROUP BY
    c.container_id)
```
