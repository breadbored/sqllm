Perform multiple different nested DML operations (DELETE, UPDATE, INSERT) on proto fields in the correct order.

--- AGENT ---

This query performs three nested DML operations on an album: it deletes residences in a city, updates song titles, and inserts a new song, executing them in the required order of DELETE, UPDATE, then INSERT.

```googlesql
UPDATE Singers s
SET
    (DELETE FROM s.SingerInfo.Residence r WHERE r.City = 'Seattle'),
    (UPDATE s.Albums.Song song SET song.songtitle = 'No, This Is Rubbish' WHERE song.songtitle = 'This Is Pretty Good'),
    (INSERT s.Albums.Song VALUES ("songtitle: 'The Second Best Song'"))
WHERE SingerId = 3 AND s.Albums.title = 'Go! Go! Go!';
```
