How do I filter items and calculate aggregate statistics in one query using the pipe syntax?

--- AGENT ---

The pipe syntax makes it simpler to filter rows, aggregate, and sort in a single flow by chaining operators. Each operator takes the output of the previous operator as input, so the filtering happens before aggregation. This query filters the Produce table to fruit items excluding bananas, then groups by item to count and sum sales, and finally sorts by item descending.

```googlesql
FROM Produce
|> WHERE
    item != 'bananas'
    AND category IN ('fruit', 'nut')
|> AGGREGATE COUNT(*) AS num_items, SUM(sales) AS total_sales
   GROUP BY item
|> ORDER BY item DESC;
```
