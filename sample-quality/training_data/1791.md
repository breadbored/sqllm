Create a table function that counts user sentiment question answers by month for a specific client and container, and includes a cumulative total column showing the running sum of answers over time.

--- AGENT ---

This query defines a parameterized BigQuery table function that calculates monthly sentiment question answer metrics for a given client and container, with a cumulative running total.

**Structure:**

The function accepts two parameters: `clientId` and `containerId`.

**Inner subquery logic:**
- Joins sentiment question answer records with user, course, and client-container relationship tables
- Filters to a specific client and container combination
- Formats the answer creation date as "Mon YY" (e.g., "Jan 26")
- Counts answer records per month (`COUNT(ucsqa.id)`)

**Outer query logic:**
- Takes the monthly answer counts and applies a window function
- `SUM(monthly_answers) OVER (ORDER BY PARSE_DATE(...) ASC)` creates a cumulative sum ordered chronologically
- Returns three columns: the formatted month, the count of answers for that month, and the cumulative total through that month
- Results are ordered by date ascending

The function essentially tracks sentiment question activity over time, showing both monthly volume and cumulative volume for a specific client-container combination.

**SQL:**

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64,
    containerId INT64) AS (
  SELECT
    date,
    monthly_answers,
    SUM(monthly_answers) OVER (ORDER BY PARSE_DATE('%b %y', date) ASC) AS cumulative_answers
  FROM (
    SELECT
      FORMAT_DATE('%b %y', ucsqa.created) AS date,
      COUNT(ucsqa.id) AS monthly_answers
    FROM
      ${ref("stg_user_course_sentiment_question_answers")} ucsqa
    JOIN
      ${ref("stg_users")} u
    ON
      u.id = ucsqa.user_id
    JOIN
      ${ref("stg_courses")} c
    ON
      c.id = ucsqa.course_id
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    WHERE
      ccc.client_id = clientId
      AND container_id = containerId
    GROUP BY
      date) AS subquery
  ORDER BY
    PARSE_DATE('%b %y', date))
```
