--- USER ---

Show me a BigQuery query that identifies when a user opens something from the menu within a sub-session and captures which app and URL path were active at that moment.

--- AGENT ---

This query extracts the first occurrence of an 'open_from_menu' event for each sub_session_id and captures the app and url_path values from that earliest event by timestamp.

The query filters the events table to include only rows where the event name is 'open_from_menu' (the first two WHERE conditions are structural redundancy given the third). For each distinct sub_session_id, it uses MIN_BY() to pull the app and url_path values from whichever row has the earliest created timestamp—effectively capturing the app context and path location at the moment the menu action occurred. It also captures that earliest timestamp as dt. The result is tagged with the literal 'open_from_menu' and classified as a 'behavior' event, structurally flagging this as a tagged event for downstream analysis or segmentation.

```sql
SELECT
    es.sub_session_id as sub_session_id,
    'open_from_menu' as tag,
    'behavior' as class,
    MIN_BY(es.app, es.created) as app,
    MIN_BY(es.url_path, es.created) as url_path,
    MIN(es.created) as dt

FROM
    ${ref('base_buckets_events_docs_speedy')} as es 

WHERE
    es.name != 'page_view'
AND
    es.name != 'open_link'
AND
    es.name = 'open_from_menu'

GROUP BY
    sub_session_id,
    tag
```
