How do I compute a 128-bit hash of a string with a secret key to prevent hash-flooding attacks?

--- AGENT ---

The HIGHWAY_FINGERPRINT128 function accepts an optional 32-byte key argument that adds cryptographic unpredictability to the hash output, protecting against denial-of-service attacks. The key must be exactly 32 bytes long.

```googlesql
SELECT HIGHWAY_FINGERPRINT128(
    'Hello World',
    -- 32-byte key
    b'abcdefghijklmnopqrstuvwxyzabcdef'
)
```
