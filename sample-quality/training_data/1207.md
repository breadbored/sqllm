What value does GoogleSQL return when a nested protocol buffer is set but its inner field is not explicitly defined?

--- AGENT ---

This query casts a protocol buffer where the nested message exists but the inner field has no explicit value. When you access the field in this case, GoogleSQL returns the default value for that field type (0 for int64).

```googlesql
SELECT
  proto_field.nested.value
FROM
  (SELECT
     CAST("nested { }" AS examples.package.OuterMessage) AS proto_field);
```
