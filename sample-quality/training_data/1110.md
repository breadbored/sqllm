How are columns matched when using UNION without the BY NAME modifier?

--- AGENT ---

Without the `BY NAME` modifier, `UNION` matches columns by position in the query. Column names are ignored, so the same column names in different order will be treated as different columns.

```googlesql
SELECT 1 AS one_digit, 10 AS two_digit
|> UNION ALL
    (SELECT 20 AS two_digit, 2 AS one_digit);
```
