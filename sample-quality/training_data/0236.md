How does Unicode collation handle invisible format control characters when sorting strings?

--- AGENT ---

Unicode collation ignores certain format control code points during string comparison, so strings that differ only in these invisible characters sort as if they were identical. This query demonstrates that three strings differing only in the presence of the U+2060 WORD JOINER character sort to the same logical position when using case-insensitive collation.

```googlesql
SELECT *
FROM UNNEST([
  COLLATE('oran\u2060ge1', 'und:ci'),
  COLLATE('\u2060orange2', 'und:ci'),
  COLLATE('orange3', 'und:ci')
]) AS fruit
ORDER BY fruit
```
