--- USER ---

Show me a BigQuery query that breaks down daily active users into new and returning cohorts, where returning users are defined as those who registered on a different date than their transaction date.

--- AGENT ---

This query produces a daily user cohort analysis with three metrics:

1. **active_users**: Total distinct users with transactions on each date
2. **returning_users**: Count of distinct users whose account creation date differs from their transaction date
3. **new_users**: The difference (active_users - returning_users), representing users whose transaction date matches their creation date

The logic works by:
- Joining transactions with a user dimension table to ensure referential integrity
- Using a LEFT JOIN to a subquery that identifies "returning" users — those with transactions on dates different from when they created their accounts
- Computing the new user count as a derived metric (total active minus returning)
- The LEFT JOIN with COALESCE(0) gracefully handles dates with no returning users

The result is grouped and ordered by date, showing the daily split between new and returning user activity.

```sql
SELECT
  date(t.created)                                                 AS date,
  count(DISTINCT t.user_id)                                       AS active_users,
  coalesce(returning.active_users, 0)                             AS returning_users,
  count(DISTINCT t.user_id) - coalesce(returning.active_users, 0) AS new_users
FROM ${ref("agg_transactions")} AS t
INNER JOIN
  ${ref("stg_users")} AS u
  ON cast(u.id AS string) = cast(t.user_id AS string)
LEFT JOIN
  (
    SELECT
      date(t.created)           AS date,
      count(DISTINCT t.user_id) AS active_users
    FROM ${ref("agg_transactions")} AS t
    INNER JOIN
      ${ref("stg_users")} AS u
      ON cast(u.id AS string) = cast(t.user_id AS string)
    WHERE date(u.created) != date(t.created)
    GROUP BY date(t.created)
  ) AS returning
  ON returning.date = date(t.created)
GROUP BY
  date(t.created), returning.active_users
ORDER BY date
```
