## Question
Show me an example BigQuery query that enriches session-level event data with course and client information, aggregating related clients for each session.

--- AGENT ---

This query produces a session-level dataset by joining RUM (Real User Monitoring) events against course, course container, and client tables. It filters for production traffic from a specific service, excluding bot traffic, and groups events by session and associated attributes.

The join chain flows: RUM events → courses → course containers → clients. This allows the query to associate multiple clients with each course container and session.

For each grouped session, the query captures:
- Core session identifiers: `session_id`, `user_id`, `course_id`
- First timestamp in the session (via `MIN(timestamp)`)
- Course container reference and name
- User context: location (city, country, country_code), device type, user agent, referrer
- Marketing attribution: UTM parameters (source, medium, campaign, id, term)
- Aggregated client data: distinct client names and client IDs as ordered arrays

The aggregation uses `ARRAY_AGG(DISTINCT ... IGNORE NULLS ORDER BY ...)` to deduplicate and sort both client names and client IDs across all events within a session-level grouping.

The output is a denormalized session-level table suitable for downstream analysis of user journeys within courses, enriched with associated client and geo-referencing information.

```sql
SELECT
  dd.session_id,
  dd.user_id,
  dd.course_id,
  cc.container_id,
  MIN(dd.timestamp) AS timestamp,
  dd.url_path,
  cc.name,
  dd.city,
  dd.country,
  dd.country_code,
  dd.user_agent,
  dd.device_type,
  dd.referrer,
  dd.utm_source,
  dd.utm_medium,
  dd.utm_campaign,
  dd.utm_id,
  dd.utm_term,
  ARRAY_AGG(DISTINCT cl.name IGNORE NULLS
  ORDER BY
    cl.name) AS client_name,
  ARRAY_AGG(DISTINCT CAST(client_id AS INT64) IGNORE NULLS
  ORDER BY
    client_id) AS client_id
FROM
  ${ref("stg_dd_rum_events")} dd
JOIN
  ${ref("stg_courses")} cc
ON
  CAST(dd.course_id AS INT64) = cc.id
JOIN
  ${ref("stg_clients_course_containers")} ccc
ON
  cc.container_id = ccc.course_container_id
JOIN
  ${ref("stg_clients")} cl
ON
  ccc.client_id = cl.id
WHERE
  dd.resource_url_path = dd.url_path
  AND dd.environment = "production"
  AND service = 'app.dataprotocol.com'
  AND device_type != "Bot"
GROUP BY
  dd.session_id,
  dd.utm_source,
  dd.utm_medium,
  dd.utm_campaign,
  dd.utm_id,
  dd.utm_term,
  dd.user_id,
  dd.service,
  dd.user_agent,
  dd.url_path,
  dd.city,
  dd.country,
  dd.country_code,
  dd.device_type,
  dd.referrer,
  dd.course_id,
  cc.container_id,
  cc.name
```
