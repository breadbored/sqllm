## Question
Show me a BigQuery SQL function that computes monthly knowledge check answer counts for a specific client, including a running cumulative total across months.

--- AGENT ---

This query creates a table function that accepts a client ID and returns a monthly time series of knowledge check activity for that client.

The function works in two layers:

**Inner subquery**: Joins four tables (user knowledge check answers, users, courses, and client course containers) and filters to the specified client. It groups by month-year (formatted as "Mon YY") and counts the number of answer records per month.

**Outer query**: Takes the monthly answer counts and adds a cumulative sum window function that runs across the ordered months. This produces three columns: the month label, the count of answers submitted that month, and the running total of all answers up to and including that month. Results are ordered chronologically.

The query structure allows it to be called as a parameterized function, making it reusable for different clients without needing to modify the SQL.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64) AS (
  SELECT
    date,
    monthly_answers,
    SUM(monthly_answers) OVER (ORDER BY PARSE_DATE('%b %y', date)) AS cumulative_answers
  FROM (
    SELECT
      FORMAT_DATE('%b %y', ukca.created) AS date,
      COUNT(ukca.id) AS monthly_answers
    FROM
      ${ref("stg_user_knowledge_check_answers")} ukca
    JOIN
      ${ref("stg_users")} u
    ON
      u.id = ukca.user_id
    JOIN
      ${ref("stg_courses")} c
    ON
      c.id = ukca.course_id
    JOIN
      ${ref("stg_clients_course_containers")} ccc
    ON
      ccc.course_container_id = c.container_id
    WHERE
      ccc.client_id = clientId
    GROUP BY
      date) AS subquery
  ORDER BY
    PARSE_DATE('%b %y', date) )
```
