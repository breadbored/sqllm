## Question
Write a BigQuery table function that returns engagement metrics for a specific client, counting the number of distinct users and total knowledge check answers associated with their courses.

--- AGENT ---

This query creates a parameterized table function that accepts a single client ID and returns two engagement metrics for that client's course portfolio. It performs a four-way join chain: starting from knowledge check answer records, connecting through users and courses, then through a client-course container mapping table to filter by the specified client. The output consists of two columns: the count of distinct users who answered questions (total_users) and the count of individual answer records (questions_answered). The function effectively aggregates participation and answer volume across all courses belonging to a single client.

```sql
CREATE OR REPLACE TABLE
  FUNCTION ${self()} (clientId INT64) AS (
  SELECT
    COUNT(DISTINCT user_id) AS total_users,
    COUNT(ukca.id) AS questions_answered
  FROM
    ${ref("stg_user_knowledge_check_answers")} ukca
  JOIN
    ${ref("stg_users")} u
  ON
    u.id = ukca.user_id
  JOIN
    ${ref("stg_courses")} c
  ON
    c.id = ukca.course_id
  JOIN
    ${ref("stg_clients_course_containers")} ccc
  ON
    ccc.course_container_id = c.container_id
  WHERE
    ccc.client_id = clientId)
```
