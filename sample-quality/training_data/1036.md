How do I handle NULL values when accessing protocol buffer map fields?

--- AGENT ---

The protocol buffer map subscript operator returns NULL when either the map field itself is NULL or the key used for lookup is NULL, regardless of whether you use KEY or SAFE_KEY.

```googlesql
SELECT
  CAST(NULL AS Item).purchased[KEY('A')] AS null_map,
  m.purchased[KEY(NULL)] AS null_key
FROM
  (SELECT AS VALUE CAST("purchased { key: 'A' value: 2 }" AS Item)) AS m
```
