--- USER ---

Generate a BigQuery query that tags user sessions by their course progress level, segmented into percentage buckets, and validates that the course start date falls within their event activity window.

--- AGENT ---

This query creates behavioral tags that segment user sessions by course progress percentages. It has two complementary branches:

**Branch 1** uses a CROSS JOIN UNNEST to generate eighteen predefined progress thresholds (0.95, 0.90, 0.85, ... 0.10). For each user session and course, it tags the session with the highest threshold the user meets or exceeds (e.g., "progress_>95%" for users at 95%+ completion, "progress_>90%" for users at 90%+ completion, etc.). The GROUP BY + HAVING validates that the course start date falls within the window of event activity for that session.

**Branch 2** creates a single "progress_<10%" tag for users whose course progress is below 10%, using the same validation logic.

The HAVING clause in both branches enforces that `uc.course_started` must lie between the earliest and latest event timestamps (`MIN(es.created)` and `MAX(es.created)`) in that session, ensuring the tag is only emitted when the course start date is temporally consistent with the events.

The output is one row per valid (sub_session_id, progress_tag) combination, suitable for downstream behavioral segmentation or cohort analysis.

```sql
SELECT
  es.sub_session_id AS sub_session_id,
  CONCAT('progress_>', ROUND(perc * 100, 0), '%') as tag,
  'behavior' as class,
  es.app AS app,
  CONCAT("/courses/", uc.course_id) AS url_path,
  uc.course_started AS dt,
FROM
  ${ref('base_buckets_events_app_speedy')} AS es
JOIN
  ${ref('reporting_user_courses')} AS uc
ON
  uc.user_id = es.user_id
CROSS JOIN
  UNNEST([
    0.95,
    0.90,
    0.85,
    0.80,
    0.75,
    0.70,
    0.65,
    0.60,
    0.55,
    0.50,
    0.45,
    0.40,
    0.35,
    0.30,
    0.25,
    0.20,
    0.15,
    0.10
  ]) as perc
WHERE
  uc.course_progress_percentage >= perc
GROUP BY
  sub_session_id,
  uc.course_started,
  es.app,
  url_path,
  tag
HAVING
  uc.course_started BETWEEN MIN(es.created) AND MAX(es.created)

UNION ALL

SELECT
  es.sub_session_id AS sub_session_id,
  'progress_<10%' as tag,
  'behavior' as class,
  es.app AS app,
  CONCAT("/courses/", uc.course_id) AS url_path,
  uc.course_started AS dt,
FROM
  ${ref('base_buckets_events_app_speedy')} AS es
JOIN
    ${ref('reporting_user_courses')} AS uc
ON
    uc.user_id = es.user_id
WHERE
    uc.course_progress_percentage < 0.10
GROUP BY
  sub_session_id,
  tag,
  app,
  url_path,
  dt
HAVING
  uc.course_started BETWEEN MIN(es.created) AND MAX(es.created)
```
