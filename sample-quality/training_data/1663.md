Show me a BigQuery query that extracts behavioral event tags from bucket assignment events, capturing the earliest app and URL path for each event occurrence.

--- AGENT ---

This query extracts bucket assignment behavioral events from an event stream. It filters the `base_buckets_events_speedy` table for events where the event name is `'bucket_assignment'`, the action is `'sso'`, and a success field in the JSON `action_meta` equals `'true'`.

For each combination of sub_session_id, generated tag, and timestamp, the query captures the earliest app and url_path values (by creation time) using `MIN_BY`. The tag is constructed by concatenating `'_bas_'` with the `full_bucket_name` field extracted from the JSON metadata, with semicolons replaced by underscores.

Each result row is marked with `class = 'behavior'` and includes the event timestamp as `dt`. The query returns one row per unique (sub_session_id, tag, created_timestamp) combination.

Note: There is a commented-out UNION ALL branch above the active query that would filter for unsuccessful assignments (where success ≠ 'true') using the simpler bucket name and '_ba_' tag prefix, but this branch is currently disabled.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  CONCAT(
    '_bas_',
    REPLACE(JSON_EXTRACT_SCALAR(e.action_meta, '$.full_bucket_name'), ';', '_')
  ) as tag,
  'behavior' as class,
  MIN_BY(e.app, e.created) as app,
  MIN_BY(e.url_path, e.created) as url_path,
  e.created as dt
FROM
  ${ref("base_buckets_events_speedy")} as e
WHERE
  e.name = 'bucket_assignment'
AND
  e.action = 'sso'
AND
  JSON_EXTRACT_SCALAR(e.action_meta, '$.success') = 'true'
GROUP BY
  sub_session_id,
  tag,
  dt
```
