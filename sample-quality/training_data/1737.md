## Question
Generate a BigQuery SQL query that creates behavior and content tags for video viewing sessions by capturing visibility state changes and associated course metadata, with the earliest timestamp for each unique combination per sub-session.

--- AGENT ---

This query produces a union of two tag types for a video platform, both tracking visibility state events within user sub-sessions:

**First branch (behavior tags):**
Queries the VAAs events table, filters to rows where visibility state is not null, and groups by sub_session_id, url_path, and a constructed tag combining the visibility state. Each group's earliest timestamp is captured. These rows are labeled with class 'behavior' and app 'vaas'.

**Second branch (content tags):**
Performs the same grouping logic but joins the events table to a courses table on video ID, enriching each group's tag with the course name. This produces similar output but with class 'content' instead of 'behavior', and the tag includes both visibility state and course metadata.

The result is a single output with both tag types, each row representing the first occurrence of a unique visibility state (and optionally course) within a sub-session and URL path combination.

```sql
SELECT
  e.sub_session_id as sub_session_id,
  CONCAT('vaas_visible:', e.vaas_state_visible) as tag,
  'behavior' as class,
  'vaas' as app,
  e.vaas_url_url_path as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_vaas_speedy")} as e
WHERE
  e.vaas_state_visible IS NOT NULL
GROUP BY
  sub_session_id,
  tag,
  url_path

UNION ALL

SELECT
  e.sub_session_id as sub_session_id,
  CONCAT('vaas_visible:', e.vaas_state_visible, ':', c.name) as tag,
  'content' as class,
  'vaas' as app,
  e.vaas_url_url_path as url_path,
  MIN(e.created) as dt
FROM
  ${ref("base_buckets_events_vaas_speedy")} as e
JOIN
  ${ref("courses")} as c
ON
  c.vaas_id = e.vaas_url_video_id
WHERE
  e.vaas_state_visible IS NOT NULL
GROUP BY
  sub_session_id,
  tag,
  url_path
```
