How do I update a column while still being able to reference its original value through a table alias?

--- AGENT ---

The `SET` operator updates top-level columns, but table aliases preserve the original row values. After a `SET` operation, you can still reference the original value using the alias (e.g., `t.x`), while the bare column name `x` refers to the new value.

```googlesql
FROM (SELECT 2 AS x, 3 AS y) AS t
|> SET x = x * x, y = 8
|> SELECT t.x AS original_x, x, y;
```
