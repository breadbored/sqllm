This query enriches a documents-courses relationship table with client information aggregated from two separate sources.

The query builds two CTEs to pre-aggregate client relationships at different levels:
- `doc_clients` collects all distinct clients linked to each document
- `course_clients` collects all distinct clients linked to each course container

The main SELECT joins both CTEs to the `documents_courses` table via left joins on document_id and course_container_id respectively. The CASE statement in the final output determines which client array to return:
- If the document has no associated clients, use the course-level clients (or [1] as a default if neither source has data)
- If the course has no associated clients, use the document-level clients (or [1] as a default if neither source has data)
- If both sources have client data, concatenate the arrays and deduplicate

This pattern ensures every row receives a client_ids array, combining associations from document and course container levels when both exist.

```sql
WITH
doc_clients AS (
  SELECT
    dc.doc_id as doc_id,
    ARRAY_AGG(DISTINCT dc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("documents_clients")} as dc
  GROUP BY
    dc.doc_id
),
course_clients AS (
  SELECT
    ccc.course_container_id as container_id,
    ARRAY_AGG(DISTINCT ccc.client_id IGNORE NULLS) as client_ids
  FROM
    ${ref("clients_course_containers")} as ccc
  GROUP BY
    ccc.course_container_id
)
SELECT
  dco.id,
  dco.document_id,
  dco.created,
  dco.updated,
  dco.`order`,
  dco.course_container_id,
  CASE
    WHEN dc.client_ids IS NULL THEN COALESCE(cc.client_ids, [1])
    WHEN cc.client_ids IS NULL THEN COALESCE(dc.client_ids, [1])
    ELSE ARRAY(SELECT DISTINCT * FROM UNNEST(ARRAY_CONCAT(dc.client_ids, cc.client_ids)))
  END as client_ids
FROM
  ${ref("documents_courses")} as dco
LEFT JOIN
  doc_clients as dc
ON
  dc.doc_id = dco.document_id
LEFT JOIN
  course_clients as cc
ON
  cc.container_id = dco.course_container_id
```
