You are given a GoogleSQL documentation page. It may contain one or many distinct SQL examples. Your task is to produce one training record per SQL example found in the document.

For each SQL example, output a USER PROMPT / ASSISTANT RESPONSE pair using exactly the format described below. If the document contains five examples, output five pairs. If it contains one, output one.

---

OUTPUT FORMAT (repeat once per SQL example):

--- USER PROMPT ---
A short, specific natural-language question (one to two sentences) that a developer would ask to arrive at this particular query. The question must be specific to this example — not a general question about the whole document. Do not mention implementation details like function signatures or clause names in the question. Write it as if a developer is asking for help with a concrete task.

--- ASSISTANT RESPONSE ---
A concise explanation (one to three sentences) of what this specific query does and why it is structured this way, followed by the SQL query reproduced verbatim. The explanation must describe only what is directly observable from the query and the documentation. End the response with the SQL inside a googlesql code fence, exactly as written in the documentation — do not alter it in any way.

---

RULES:

1. Produce exactly one USER PROMPT / ASSISTANT RESPONSE block per SQL example. Do not merge multiple examples into one block.
2. Each block must be self-contained. Do not refer to other examples or use phrases like "as shown above" or "similarly".
3. The SQL in every ASSISTANT RESPONSE must be reproduced verbatim — no reformatting, no reordering, no paraphrasing.
4. The USER PROMPT must be specific to that single example. Do not write a generic question about the topic of the whole document.
5. Do not explain examples you cannot see. If a section references behavior without a query, skip it.
6. Do not add commentary after the final block. Output ends after the last ASSISTANT RESPONSE.
7. Use exactly the delimiter text shown above: `--- USER PROMPT ---` and `--- ASSISTANT RESPONSE ---` on their own lines, with no extra characters.

The documentation is provided below in <doc> tags.
